# Exercises — OpenAI Agents SDK: Handoffs, Guardrails, Tracing

## Exercises

1. **Implement a two-agent handoff.** Create a `triage_agent` that greets a user and hands off to a `company_lookup_agent` when the user asks about a specific company. Run the pipeline with the input `"Tell me about Acme Corp"` and print which agent handled the final response. Confirm the `company_lookup_agent` received the conversation history by printing the full message list it sees inside its own instructions.

2. **Build an input guardrail that blocks competitor domains.** Write an `InputGuardrail` that trips when the user message contains a domain from a blocklist (`["competitor1.com", "rival.io", "enemy.co"]`). Run the agent three times—once with a safe input, once with a blocked domain, once with a blocked domain embedded in a longer sentence—and print the guardrail result (`tripwire_triggered` boolean) for each run. Verify the safe input proceeds and the other two halt.

3. **Trace a three-agent run and reconstruct the execution path.** Build a pipeline where a `router_agent` hands off to either a `research_agent` or a `qualification_agent` based on user intent. Run it twice with different inputs, then export both traces as JSON. From the exported trace files, write a script that parses the spans and prints the ordered list of agent names that executed, plus any guardrail evaluations that occurred. Your terminal output should show two different execution paths.

4. **Compare single-prompt vs. multi-agent for lead enrichment.** Implement the same lead enrichment task two ways: (a) one agent with a single long system prompt that researches a domain, evaluates fit, and writes a summary; (b) three agents chained via handoffs, each handling one stage. Run both against the same three test domains. Print wall-clock time, token usage, and the output for each approach side-by-side. Identify which approach produces more consistent structured output and which is faster.

5. **Build a GTM enrichment pipeline with guarded handoffs.** Create a pipeline where a `prospector_agent` (fed a CSV of company domains loaded from disk) enriches each lead and hands off to a `scoring_agent` that assigns a priority tier. Add an output guardrail on the `scoring_agent` that trips if the priority is assigned without a rationale field. Run it against a sample of 10 Apollo-exported company records (or a CSV with `company_name,domain,employee_count` columns). Emit a trace for the full batch run. Save the pipeline as `handlers/enrichment_pipeline.py` and the final scored lead list as `outputs/skill-enrichment-scores.md` in a markdown table sorted by priority tier.

6. **Diagnose a broken multi-agent run from its trace.** You are given a trace JSON (generate one by deliberately introducing a bug: an input guardrail that trips on any message containing the word "free," a handoff target name that doesn't match any registered agent, or an output guardrail with a thrown exception). Export the trace, then write a diagnostic script that loads the trace JSON, walks the span tree, and prints: which guardrail tripped (or which handoff failed), the exact span ID where execution halted, and the conversation state at the point of failure. Save the diagnostic script as `handlers/trace_diagnostic.py` and run it against your buggy trace. The terminal output should identify the failure point and root cause.
