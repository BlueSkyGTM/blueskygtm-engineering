# Exercises — ReWOO and Plan-and-Execute: Decoupled Planning

## Exercises

1. **Implement a minimal Plan-and-Execute loop that prints its plan and execution trace.** Using the agent scaffold from the lesson, add a `print_plan` call after the planner generates its step list and a status line (`SUCCESS` / `FAILED`) after each step executes. Run the agent against the query "Find the CEO and CTO of the company at domain stripe.com." Confirm your terminal shows the full plan before any tool calls, followed by per-step status lines.

2. **Implement a ReWOO planner that outputs dependency-ordered slot variables.** Write a function `build_rewoo_plan(query: str) -> list[dict]` that prompts an LLM to return JSON steps where each step's `args` field may reference prior steps using `#E1`, `#E2`, etc. Run it against the query "What is the tech stack of snowflake.com and who is their VP of Engineering?" and print the parsed plan. Verify that at least one step references a prior `#E` slot — if none do, adjust your prompt and re-run.

3. **Compute and compare token cost and wall-clock time for a ReAct agent versus a Plan-and-Execute agent on the same task.** Write a script that runs both agents against the query "Enrich the company at domain notion.so: find employee count, funding stage, and a technical contact." Use `tiktoken` or your provider's usage metadata to capture total tokens consumed. Print a comparison table to stdout with columns: pattern, total_tokens, wall_clock_seconds, num_llm_calls, final_answer. No scaffold is provided — structure the harness yourself.

4. **Apply the Plan-and-Execute pattern to a multi-source enrichment waterfall.** Given a list of domains (`airbnb.com`, `figma.com`, `notion.so`), build a planner that creates enrichment steps using two data sources (e.g., Apollo API for company data and Hunter.io for email lookup). Configure the replanner to detect when Apollo returns zero employees and insert a fallback step querying Clearbit or a WHOIS lookup. Run the agent and print a per-domain summary showing which sources were consulted and what data was returned.

5. **Build a complete ReWOO agent for account research that produces a structured brief.** Implement the planner (generates a static plan with `#E` slot variables), worker (executes each slot by resolving dependencies and calling the appropriate tool), and solver (combines all slot outputs into a final answer). Test it against the query "Research the company at domain datadog.com: industry, funding history, leadership team, and competitors." Your solver should output a markdown brief. Write the full implementation to `handlers/rewoo_agent.py` and save the generated brief for datadog.com to `outputs/skill-rewoo-account-brief.md`.

6. **Design and validate a decision rubric for choosing between ReWOO and Plan-and-Execute for GTM workflows.** Write a Python function `select_pattern(task_description: str) -> tuple[str, str]` that returns `("rewoo" | "plan_and_execute", rationale)`. Encode at least four decision criteria (e.g., need for replanning, tool failure likelihood, latency budget, plan complexity). Test your selector against six scenario descriptions covering outbound prospecting, enrichment waterfalls, churn investigation, and competitive analysis. Write the function and test scenarios to `signals/examples/select_pattern.py` and print each scenario with its selected pattern and rationale.
