# Exercises — Skill Libraries and Lifelong Learning (Voyager)

## Exercises

1. Implement a `SkillLibrary` class with `add(description, code)` and `search(query, top_k=3)` methods. Store each skill as a dict with `description`, `code`, `embedding`, and `created_at` fields. For embeddings, use `sentence-transformers` (`all-MiniLM-L6-v2`) or a mock function that returns a deterministic 384-dim vector if the model is unavailable. Compute cosine similarity manually with numpy — do not call a library similarity function. Add three skills with descriptions like `"craft wooden pickaxe"`, `"mine stone blocks"`, and `"smelt iron ore in furnace"`. Query `"how do I get stone tools"` and print the top result with its similarity score. Verify the semantic match surfaces the pickaxe or stone-block skill despite zero word overlap.

2. Trace Voyager's three-component data flow by implementing a `voyager_step(state, skill_lib)` function that prints each stage. Given a `state` dict (e.g., `{"inventory": {"wood": 4, "stick": 2}, "biome": "plains", "nearby": ["tree", "cow"]}`), call a `propose_task(state, skill_lib)` function that returns a task string, then `retrieve_skills(task, skill_lib)`, then `generate_code(task, retrieved)` that returns a code string. Print each stage with a clear label: `[CURRICULUM]`, `[RETRIEVAL]`, `[GENERATION]`. Run three sequential steps where each step's output state feeds the next. Confirm from terminal output that each stage executes in order and that the skill library grows after each successful step.

3. Build two retrieval functions over the same set of at least eight skill descriptions: `exact_match(query, skills)` that returns skills whose description contains any query token, and `semantic_match(query, skills)` that returns skills by cosine similarity over embeddings. Run both functions against five queries that use synonyms or paraphrases rather than exact keywords (e.g., query `"defeat hostile monster"` against a skill described as `"attack zombie with sword"`). Print a comparison table showing which skills each method returns. Verify that exact-match returns zero results on at least two queries where semantic match returns a correct top result.

4. Implement an execution loop that attempts a task, catches failures, and retries with the error message appended to the prompt. Define a `tasks` list of three operations (e.g., `"place block"`, `"craft table"`, `"dig down"`). Define a `execute(code_str)` function that fails on the first attempt for each task by raising an exception (simulate with a regex check or a hardcoded failure counter), then succeeds on retry. After each success, commit the code to the skill library. Print each attempt number, the error message received, and the revised code. Verify from terminal output that every task required at least one retry and that the skill library ends with three committed skills.

5. Build a GTM enrichment skill library that stores reusable enrichment playbooks as code snippets with natural-language descriptions. Seed the library with at least four playbooks: a Clay webhook call that enriches a domain with firmographics, an Apollo person-search API call, a Hunter.io email-verification call, and a Clearbit company-lookup call. Each playbook stores its HTTP method, endpoint, required parameters, and a description like `"enrich company by domain — returns employee count, industry, funding"`. Implement `search_playbook(goal)` using embedding-based retrieval. Query with `"find decision makers at a company"` and `"validate that an email address is deliverable"` — verify the correct playbook surfaces despite no keyword overlap. Then implement a composition function that chains two playbooks: first enrich a company by domain, then search for people at that company. Execute the composition against a real or mock API endpoint and print the intermediate and final results. Save the library implementation to `handlers/enrichment_skill_library.py`.

6. Design a lifelong-learning enrichment agent that processes a queue of ten GTM tasks (e.g., `"enrich lead: acme.com"`, `"find VP Engineering at stripe.com"`, `"verify email for contact at notion.so"`) using the three-component Voyager loop. The curriculum proposes the next task based on which skills already exist and which tasks depend on prior capabilities. The execution loop attempts each task, retries on failure, and commits successful playbooks back to the library. Run the agent to completion and print: the skill library size after each task, the retrieval results for each query, and any task that required composition of two or more existing skills. Write a final summary to `outputs/skill-enrichment-agent.md` documenting which skills were reused, which were learned from scratch, and where the library compounded (i.e., a later task succeeded only because an earlier skill was committed).
