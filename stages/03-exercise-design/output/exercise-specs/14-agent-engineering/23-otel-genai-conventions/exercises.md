# Exercises — OpenTelemetry GenAI Semantic Conventions

## Exercises

1. **Implement** a manual OTel span wrapping a raw LLM call using GenAI semantic convention attributes. Start a console exporter so spans print to the terminal. Include at minimum: `gen_ai.system` set to `"openai"`, `gen_ai.request.model` set to `"gpt-4o"`, `gen_ai.usage.input_tokens`, and `gen_ai.usage.output_tokens` with hardcoded integer values (you do not need a real API key — use a mock response function). Run the script and confirm the printed span JSON contains all four attributes with correct key names.

2. **Configure** content-capture by running your Exercise 1 script twice: once with `OTEL_SEMCONV_STABILITY_OPT_IN` unset, once with it set to `gen_ai` plus the appropriate opt-in flag for prompt body capture. Print the span attributes both times and identify which run includes `gen_ai.prompt` content versus which suppresses it. Write a one-line comment in your script explaining why prompt capture defaults to off.

3. **Compare** span attributes emitted by two different providers under the same convention schema. Build a script that emits two GenAI spans — one with `gen_ai.system` set to `"openai"` and one set to `"anthropic"` — using the same manual-span approach from the lesson. Export both spans to the console as JSON. Parse both JSON objects in-process and assert that the key sets for usage attributes (`gen_ai.usage.input_tokens`, `gen_ai.usage.output_tokens`) are identical across both provider spans. Print `"KEY MATCH: True"` or `"KEY MATCH: False"` as the final line.

4. **Compute** cost-per-enrichment-run from a set of exported spans. Write a script that generates 10 GenAI spans representing enrichment calls: 6 using `gpt-4o` ($2.50 / 1M input, $10.00 / 1M output) and 4 using `gpt-4o-mini` ($0.15 / 1M input, $0.60 / 1M output). Tag each span with a custom `pipeline.correlation_id` attribute — use two distinct IDs, splitting the calls evenly. Export the spans to a JSON file, then write a second script that reads that file, groups by `(model, correlation_id)`, and prints a table showing total input tokens, output tokens, and dollar cost per group.

5. **Build** a multi-step enrichment pipeline that traces token spend back to its originating input record. Create `handlers/enrichment_tracer.py` that takes a list of 5 company names (seeded from a static list — do not hardcode results), looks up each company using the Apollo People Search API or a mock enrichment function if no API key is available, then calls an LLM (or mock) to summarize the enriched profile. Emit a parent span per input record with a custom `enrichment.input_record` attribute set to the company name. Nest child spans for the enrichment lookup and the LLM summary call, each carrying the correct GenAI semantic attributes. Export all spans to a JSON file at `outputs/enrichment-traces.json`. Run the script and confirm the parent-child relationship is visible in the exported JSON — every LLM child span's `parent_span_id` must resolve to a record-level parent span.

6. **Design** a cost-attribution report generator that reads `outputs/enrichment-traces.json` from Exercise 5 and produces `outputs/skill-genai-cost-attribution.md`. The markdown must contain: a table mapping each input company name to its total token spend (walked up from child spans to the parent's `enrichment.input_record` attribute), a summary section showing aggregate cost by model, and a section identifying which input record consumed the most tokens. Run the generator against your Exercise 5 output and verify the markdown renders correctly.
