# Exercises — Self-Refine and CRITIC: Iterative Output Improvement

## Exercises

1. **Implement** a three-prompt Self-Refine loop that generates a 3-sentence cold outreach email, requests feedback in JSON `{score: 1-10, issues: [...]}`, and produces a revised version. Run the loop for exactly 3 iterations and print the full history at each step: iteration number, current score, feedback issues, and the refined output. Your script must accept a `--prospect` CLI argument (company name + role) and terminate with a final summary line showing the score progression from iteration 1 to 3.

2. **Compare** internal LLM self-critique against external verification on the same generated email. Run the generate step once, then run two independent evaluation passes: (a) a feedback prompt that asks the LLM to score its own output for factual accuracy, and (b) a Python function that checks the email for hallucination red flags — any statistic, metric, or company-specific claim that isn't grounded in the input prospect data. Print both scores side by side and a diff of the issues each method caught. The terminal output must make visible at least one case where the two methods disagree.

3. **Build** a convergence-aware refinement loop for generating LinkedIn connection-request messages. Implement both stopping conditions: a hard score threshold (default 8.0, configurable via CLI flag) and a delta-based early stop (if the score improves by less than 0.5 across two consecutive iterations, halt). The loop must accept a `--max-iterations` argument and must print, at exit, which stopping condition triggered and how many iterations actually ran. Include at least one test prospect where the delta condition fires before the threshold is met, and one where the threshold is reached.

4. **Apply** the evaluator-optimizer pattern to a GTM enrichment workflow. Given a CSV of 10 prospects (columns: `name`, `company`, `title`, `industry`), generate a personalized opening line for each, refine it through feedback, and compute the total token cost across all iterations using real `tiktoken` counts for your prompts and completions. Output a summary table to the terminal: prospect name, iterations to convergence, total tokens consumed, and estimated cost at $5/M input + $15/M output. The final line must show aggregate cost for the full batch.

5. **Extend** the Self-Refine loop into a CRITIC-validated cold email generator that cross-checks every generated claim against a live enrichment source. Pull 5 prospects from Apollo (or a provided `signals/examples/apollo_export.json` fallback), generate a personalized email for each, then verify company facts, headcount, and recent news by querying the Apollo company enrichment endpoint or Hunter.io API. If any claim fails verification, feed the discrepancy back into the refine prompt. Produce a handler artifact at `handlers/critic_email_refiner.py` that exposes a `refine_prospect_email(prospect: dict) -> dict` function returning `{email, verified_claims, unverified_claims, iterations, final_score}`. Running the file as `__main__` must print one JSON block per prospect to stdout.

6. **Design** and run an experiment that determines where internal self-critique breaks down versus where external CRITIC verification adds the most value in GTM personalization. Construct 6 test prospects — 3 in well-documented industries (SaaS, fintech) and 3 in underrepresented ones (manufacturing, local services). Run both evaluation modes on identical generated emails, measure false-positive rates (unverified claims scored as credible) and false-negative rates (real claims flagged as hallucinations). Write your findings, methodology, token-cost comparison, and a recommended decision rule for when to use each method to `outputs/skill-critic-vs-self-refine.md`. The document must include a results table and a concrete threshold rule (e.g., "use CRITIC when industry data coverage is below X").
