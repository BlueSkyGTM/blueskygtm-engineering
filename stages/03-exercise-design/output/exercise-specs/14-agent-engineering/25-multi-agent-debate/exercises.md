# Exercises — Multi-Agent Debate and Collaboration

## Exercises

1. **Implement a 3-agent, 2-round debate loop.** Each agent independently answers the question "Should outbound prioritization weight company headcount or recent funding events more heavily?" In round 1, all three agents produce independent proposals. In round 2, each agent reads the other two proposals, writes a one-sentence critique, and submits a revised answer. Print each agent's round-1 proposal, round-2 critique, round-2 revision, then print the final converged answer with a confidence score (0–1) computed as the fraction of agents whose final answer matches the majority.

2. **Configure the same debate loop for 5 agents across 3 rounds.** After each round, compute and print the percentage of agents whose current answer matches the eventual final answer. Trace total token usage per round (prompt + completion) and print a summary table with columns: round, tokens consumed, agreement percentage, delta from previous round. Confirm that agreement increases (or tokens plateau) before the final round.

3. **Compare full-mesh and star communication topologies on the same question.** In full-mesh, every agent reads all other agents' proposals each round. In star topology, agents only read a central aggregator's summary, never peer proposals directly. Run both configurations with 4 agents across 3 rounds on the question "What is the optimal re-engagement cadence for dormant SaaS trial users?" Print a side-by-side comparison: total tokens, rounds to convergence (first round where ≥75% of agents agree), and whether the final answers match. State which topology won on each metric.

4. **Build a task-router agent squad for an inbound lead event.** Given a mock Apollo lead payload (company name, domain, lead name, title), implement three specialist agents: an enrichment agent that calls a people-data API (use the Clearbit, Hunter, or People Data Labs endpoint from the lesson), a research agent that summarizes the company's recent news, and a drafting agent that writes a personalized cold email. Add a critic gate that scores the draft on a 0–1 scale; if the score is below 0.6, suppress the output and print "DRAFT SUPPRESSED — score: {score}"; if above, escalate by printing the full enrichment payload, research summary, draft, and critic score.

5. **Build a multi-agent ROI evaluator that produces a decision artifact.** For each of 5 GTM prompts (e.g., "Draft a sequence of 3 follow-up emails for a champion who went silent after the demo," "Score this ICP fit: {company details}"), run both a single-agent call and a 3-agent debate over 2 rounds. Score each output 0–1 on specificity and actionability (you may use a separate LLM-as-judge call or a rubric-based heuristic). Print a table comparing single-agent vs multi-agent scores and token costs, then write the full comparison, including a recommendation for which prompt types justify multi-agent overhead, to `outputs/skill-multi-agent-roi.md`.

6. **Design a hybrid debate-plus-suppression pipeline for a Clay enrichment workflow.** When a new enriched row arrives (company, contacts, technographics), route it through a 3-agent debate that proposes the best outreach angle, then pass the winning proposal through a critic gate that checks for ICP alignment using firmographic rules. If the critic approves, emit a finalized personalization brief; if suppressed, log the rejection reason. Run the pipeline across 5 sample Clay rows (mock or pulled via the Clay API) and write the handler module to `handlers/clay_agent_squad.py`. Print a run summary showing how many proposals were approved vs suppressed and the average debate rounds consumed.
