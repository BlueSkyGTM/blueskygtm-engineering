## Exercises

1. **Implement first-visit MC prediction.** Write a function `first_visit_mc_prediction(episodes, gamma=1.0)` that takes a list of episodes (each episode is a list of `(state, reward)` tuples) and returns a dictionary mapping each state to its estimated value. Run it on at least 500 episodes generated from a simple 4-state chain MDP where state 0 always starts, each state transitions right with probability 0.8 and left with probability 0.2, and only the terminal state yields a reward of +10. Print the resulting value function sorted by state name. Verify that states closer to the terminal have higher estimated values.

2. **Trace return computation for a single episode.** Write a function `trace_episode(episode, gamma)` that prints, for each time step in the episode: the time index, the state, the reward received at that step, the full list of remaining rewards, the computed discounted return `G_t`, and a boolean indicating whether this is the first occurrence of that state in the episode. Feed it the episode `[("A", 0), ("B", 0), ("C", 1), ("B", 0), ("D", 5)]` with `gamma=0.9` and confirm that the two visits to `"B"` produce different returns and that only the first is flagged as a first visit.

3. **Compare first-visit and every-visit MC estimators.** Implement both `first_visit_mc_prediction` and `every_visit_mc_prediction` (the latter averages returns from *all* visits to a state, not just the first). Generate 1000 episodes from the same chain MDP as Exercise 1. For each estimator, compute the value function. Then repeat the entire experiment 50 times and report, for each state, the mean estimate and the standard deviation across runs. Print a side-by-side table showing state, first-visit mean, first-visit std, every-visit mean, every-visit std. Comment in a print statement which estimator shows lower variance and whether the means differ.
