# Exercises — Deep Q-Networks (DQN)

## Exercises

1. **Implement epsilon-greedy action selection with linear decay.** Write a standalone script that defines an epsilon schedule starting at 1.0 and decaying linearly to 0.05 over 10,000 steps. Run a 10,000-step loop (no environment needed — just iterate the step counter), print epsilon every 1,000 steps, and at each step select an action from a 4-action space using the epsilon-greedy rule. Track how many times each action was selected when exploration (random) was chosen vs. when exploitation (argmax of a dummy Q-vector) was chosen. Print the final per-action selection counts. Verify that early steps show roughly uniform action distribution and late steps are dominated by the argmax action.

2. **Compute TD loss by hand-tracing a single Bellman update.** Given a transition with `r = 1.0`, `γ = 0.99`, and the following network outputs — `Q_online(s, a) = [2.5, 1.8, 3.1, 0.4]`, `Q_target(s', a') = [1.2, 0.9, 1.5, 2.0]` — write a script that computes the TD target `y = r + γ · max_a' Q_target(s', a')`, computes the TD error for the action taken (assume `a = 2`, zero-indexed), and prints both the scalar loss `(y − Q_online(s, a))²` and the raw TD error. Then recompute the loss using `Q_online` in place of `Q_target` for the bootstrap term (i.e., no target network) and print the difference. Confirm the two loss values differ.

3. **Build a complete DQN agent on CartPole-v1.** Using PyTorch and Gymnasium, implement a DQN with: (a) a replay buffer storing 10,000 transitions, (b) an online Q-network and a target Q-network, (c) target network synchronization every 500 steps, (d) epsilon-greedy with linear decay from 1.0 to 0.05 over 5,000 steps, and (e) Adam optimizer with learning rate 1e-3. Train for 300 episodes. Print episode number, total reward, current epsilon, and average loss every 20 episodes. Confirm that rewards trend upward and the average loss stabilizes or decreases over training.

4. **Compare DQN training stability with and without the target network.** Modify your solution from Exercise 3 to run two variants: one with target network synchronization (every 500 steps) and one where the target network is replaced by the online network on every gradient step (no frozen target). Run both for 300 episodes on CartPole-v1 with the same hyperparameters and random seed. After training completes, compute and print the mean and standard deviation of episode rewards over the final 50 episodes for each variant. Additionally, write the per-episode reward and average loss for both variants to `outputs/dqn-comparison.csv` with columns `episode, variant, reward, avg_loss`. Report which variant achieves higher mean reward and lower reward variance.

5. **Build a next-best-action recommender for a GTM outreach sequence using DQN.** Define a discrete state representation from prospect attributes (e.g., company size bucket, industry, engagement score, days since last contact) and a 5-action space (`email`, `call`, `linkedin_touch`, `break`, `escalate_to_sales`). Generate a synthetic dataset of 2,000 prospect transitions with reward defined as engagement signal (reply = +1, meeting booked = +5, no response = 0, unsubscribe = −2). Pull real prospect firmographics from Apollo's People Search API (or a local CSV export from Clay) for at least 50 companies to seed the state space. Train a DQN agent for 1,000 steps and evaluate it on 20 held-out prospects. For each held-out prospect, print the recommended next-best-action and its predicted Q-value for all 5 actions. Save the full evaluation table to `outputs/skill-nba-recommender.md` with a markdown table of prospect, state, top action, and Q-values.

6. **Design and verify a stabilizer ablation on the GTM DQN.** Extend the recommender from Exercise 5 to test three configurations: (1) no experience replay (train on consecutive transitions only), (2) no target network, (3) full DQN (both stabilizers). Run each for 1,000 training steps on the same data and random seed. Print the final TD loss, the number of divergent episodes (reward below −10), and the mean predicted Q-value across all held-out prospects for each configuration. Write a brief analysis to `outputs/skill-stabilizer-ablation.md` stating which stabilizer contributes more to stability in the GTM domain and why, supported by the three numeric metrics. Save the ablation script to `handlers/dqn_ablation.py`.
