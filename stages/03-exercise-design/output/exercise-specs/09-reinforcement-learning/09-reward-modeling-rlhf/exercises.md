# Exercises — Reward Modeling & RLHF

## Exercises

1. **Build a pairwise preference dataset and compute inter-annotator agreement.** Create a JSONL file with at least 20 pairwise comparisons across 5 prompts. Each record must contain `prompt`, `chosen`, `rejected`, and a list of annotator labels (3 annotators per pair, each voting "A" or "B"). Write a script that computes pairwise agreement using Cohen's kappa for each annotator pair and prints the mean kappa to the terminal. Flag any pair where all three annotators disagree (i.e., one annotator flipped relative to the other two) and print those record IDs.

2. **Implement the reward model's scoring head and pairwise ranking loss.** Load a frozen `distilbert-base-uncased` backbone from Hugging Face. Attach a linear scalar head that outputs a single float. Implement the Bradley-Terry pairwise loss: `loss = -log(sigmoid(r_chosen - r_rejected))`. Feed 5 synthetic (prompt, chosen, rejected) triples through the model, compute the loss for each, and print the loss values. Confirm that swapping chosen and rejected inverts the sign of the reward difference.

3. **Trace one step of the PPO loop by hand.** Given a fixed prompt and two policy outputs (one from the current policy, one from a reference/SFT policy), compute: (a) the reward-model score for each output, (b) the per-token log-probability ratio between current and reference policy, (c) the KL divergence penalty term `beta * KL(policy || ref)`, and (d) the final adjusted reward `r_adjusted = r_rm - beta * KL`. Print each intermediate value. Then increase `beta` from 0.01 to 0.5 and show how the adjusted reward changes — identify the `beta` value at which the higher-scoring output stops being preferred after the penalty.

4. **Compare reward-model score distributions before and after a simulated alignment step.** You are given two directories: `outputs/pre_alignment/` and `outputs/post_alignment/`, each containing 50 generated completions for the same set of prompts. Load a pre-trained reward model checkpoint (or use a sentiment/scoring heuristic model such as `distilbert-base-uncased-finetuned-sst-2-english` as a proxy scorer). Compute the scalar score for every completion in both directories. Print the mean, median, and standard deviation for each group, then run a two-sample Kolmogorov-Smirnov test and print the statistic and p-value. State whether the alignment step produced a statistically significant reward shift.

5. **Build a reward-model critic gate in a GTM content pipeline.** Implement a handler at `handlers/gtm_reward_gate.py` that takes a list of prospect-company pairs (fetched from Clay or a CSV export containing company name, ICP fit score, and decision-maker title), generates two cold-email variants per prospect using any LLM API, scores both variants with a reward model (use an OpenAI moderation or quality-scoring endpoint, or a local proxy scorer), and writes the higher-scoring variant plus its score to `outputs/skill-reward-gated-emails.jsonl`. The handler must enforce a minimum reward threshold (configurable via a constant at the top of the file) — prospects whose best variant falls below the threshold get a `"status": "blocked"` record instead of an email. Run the handler against a 10-row sample and print a summary: how many emails passed the gate, how many were blocked, and the mean score of accepted emails.

6. **Design and prove a reward-hacking failure scenario.** Construct a synthetic reward model (a simple function you define) that scores text completions. Demonstrate a case where a PPO-style optimizer would increase the reward score monotonically while the output quality degrades — for example, the reward model rewards length or keyword repetition, and the policy exploits this. Write a script at `signals/examples/reward_hacking_proof.py` that: (a) defines the flawed reward function, (b) generates 5 "optimized" completions that greedily maximize the reward via simple token insertion, (c) scores them with both the flawed reward and a ground-truth quality metric you define separately, and (d) prints a table showing reward increasing while quality decreases. Conclude with a one-paragraph analysis (printed to terminal) of which KL penalty configuration would have prevented the collapse.
