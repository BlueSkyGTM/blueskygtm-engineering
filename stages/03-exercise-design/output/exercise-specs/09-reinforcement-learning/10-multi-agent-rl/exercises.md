# Exercises — Multi-Agent RL

## Exercises

1. **Implement independent Q-learning for two cooperative agents on a 2-row gridworld.** Each agent maintains its own Q-table over `(row, col)` states and `up/down/left/right/stay` actions. Both agents receive the shared team reward (+10 when both occupy the goal cell, −1 per step otherwise). Run 500 training episodes with epsilon-greedy exploration (ε = 0.3 decay) and log the per-episode return. After training, run 50 evaluation episodes with ε = 0 and print the mean evaluation return alongside the standard deviation of the last-100-episode training returns. You should observe higher variance than a single-agent run on the same environment — quantify it.

2. **Trace the three MARL paradigms by instrumenting your Exercise 1 code.** Add a `--paradigm` flag that switches between `independent`, `ctde`, and `communicate`. For `independent`, agents select actions from local Q-tables (already done in Exercise 1). For `ctde`, add a joint-state Q-critic used only during training updates but still select actions from local tables. For `communicate`, broadcast each agent's last action to the other before its update step. Print the training-reward curve (mean over 10 seeds) for each paradigm and report which paradigm converges fastest in wall-clock time.

3. **Compute credit assignment for a 3-agent cooperative task.** Build a small matrix-game environment where three agents each choose one of 3 actions, and the team reward depends on the joint action tuple. Define a reward function where the optimal joint action yields reward +5 but several near-optimal tuples yield +3 or +4. Implement random Q-learning agents, log every `(joint_action, reward)` pair across 1000 episodes, then implement a monotonicity check: verify whether reward strictly increases as more agents choose the "correct" action. Print the percentage of episodes where the monotonicity constraint holds. If it holds, compute per-agent marginal contribution by holding two agents fixed and varying the third.

4. **Apply the CTDE pattern to a multi-agent predator-prey gridworld with partial observability.** Two predators cooperate to catch one prey; each predator observes only a 5×5 window around itself but the critic receives the full board state plus both agents' positions. Train the centralized critic with a Q-learning update on the joint state-action tuple, while each predator's individual policy selects actions from its local observation. After 2000 episodes, evaluate the catch-rate with and without the critic active during execution (critic should be dropped at inference — verify that performance degrades by less than 15% relative to a full-observation oracle policy).

5. **Design a GTM agent squad that mirrors CTDE and build a traceability artifact.** Implement three agents in `signals/examples/gtm_ctde_squad.py`: (a) an enrichment agent that takes a company domain and calls the Apollo API to fetch firmographics, (b) a scoring agent that takes enriched data and outputs a 0–100 fit score, and (c) a sequencing agent that takes the score plus enrichment data and recommends an outreach channel and timing. All three agents receive a team reward: +1 if a simulated prospect replies within 14 days, 0 otherwise. Implement a centralized critic that observes all three agents' inputs and outputs and assigns credit by computing each agent's marginal contribution to the reward via leave-one-out ablation (replace each agent's output with a baseline and measure reward delta). Run the pipeline on 20 sample companies, log the per-agent credit scores, and write a markdown report to `outputs/skill-gtm-credit-assignment.md` showing which agent contributed most to positive outcomes and whether the credit signal is consistent across runs.

6. **Extend your Exercise 5 squad with a communication channel and prove it helps under partial observability.** Modify the enrichment agent so it only receives the company name (not the domain — it must infer the domain), and modify the scoring agent so it only receives industry and headcount (not the full firmographics). Add a message-passing step where agents can broadcast a 4-dimensional vector to each other before acting. Train the squad with and without communication enabled across 200 simulated prospects, then write a comparison table to `outputs/skill-communication-benefit.md` showing reply-rate lift, per-agent credit redistribution, and whether communication specifically helps the enrichment agent (which now operates under stricter partial observability). State your hypothesis, the measured result, and whether it confirms or rejects.
