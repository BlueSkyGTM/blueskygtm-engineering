## Exercises

1. **Compute** both Monte Carlo and TD(0) advantages for the following 5-step trajectory: rewards `[1, 0, 1, 0, 2]`, with a critic that predicts `V(s) = [0.8, 0.5, 0.6, 0.3, 0.4]` for each state, and `γ = 0.99`. Print both advantage vectors side by side along with their sample variance. Then re-run the computation across 100 random 5-step trajectories (rewards sampled uniformly from `[0, 1)`, critic outputs sampled from `N(0.5, 0.2)`) and print the mean absolute advantage and variance for each method. The terminal output should make the variance difference visible.

2. **Trace** the gradient flow through a two-headed network by building a shared-trunk model in PyTorch: two linear trunk layers (64→64), one actor head outputting softmax over 2 actions, one critic head outputting a scalar. Feed a dummy state tensor, compute `loss = -log_prob * advantage + 0.5 * (return - value)^2`, call `.backward()`, and print the gradient norms for every layer. Label which layers received gradients from the actor loss term only, the critic loss term only, or both. Verify that the shared trunk receives gradients from both heads by zeroing one loss term at a time and printing the resulting gradient norms.

3. **Implement** a complete A2C agent from scratch for CartPole-v1. Use a shared-trunk network (two hidden layers of 128 units, ReLU), separate actor and critic heads, TD(0) advantage with `γ = 0.99`, a single Adam optimizer (`lr = 1e-3`), and collect one full episode per update. Train for 300 episodes and print the moving average reward (window of 20) every 25 episodes. Your agent must achieve a 20-episode average reward of 195 or higher. Print the final average and whether it passed the threshold. Do not use any RL library — write the network, the environment loop, the advantage computation, and the loss/backward calls yourself.

4. **Configure** four variants of your A2C agent from Exercise 3 and run each for 200 episodes across 3 random seeds. The variants are: (a) baseline with no stabilization, (b) entropy regularization with `β = 0.01` added to the loss, (c) gradient clipping at `max_norm = 0.5` via `torch.nn.utils.clip_grad_norm_`, and (d) reward normalization by dividing the episode's advantages by `(adv_std + 1e-8)`. For each variant and seed, print the final 20-episode average reward and the number of episodes where the loss spiked by more than 3× the running mean. Produce a summary table in the terminal showing which technique had the largest impact on stability versus final performance.

5. **Build** an A3C trainer using Python `multiprocessing` with 4 worker processes, each running its own CartPole-v1 environment and its own copy of the shared network. Workers push gradients to a shared `SharedAdam` optimizer via a lock and pull updated parameters back before each episode. Run it for 300 total episodes across all workers and print: (a) wall-clock training time, (b) final average reward, (c) the variance of gradient norms across workers at each synchronization point (averaged over all syncs), and (d) the standard deviation of final rewards across 3 independent training runs. Re-run your synchronous A2C from Exercise 3 with the same total episode budget and print the same four metrics side by side. Save the comparison results as a markdown table to `outputs/skill-a2c-vs-a3c.md`.
