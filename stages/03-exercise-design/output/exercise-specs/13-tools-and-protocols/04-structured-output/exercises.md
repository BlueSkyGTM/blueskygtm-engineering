## Exercises

1. Write a JSON Schema 2020-12 document for a firmographic extraction target. The schema must represent an Apollo-style company enrichment response with these fields: `company_name` (string, minLength 1), `employee_count` (integer, minimum 1, maximum 10000000), `industry` (string, enum of at least five values), `domain` (string, pattern matching a URL), `founded_year` (integer, minimum 1600, maximum 2025), and `linkedin_url` (string, nullable). Mark `company_name`, `domain`, and `industry` as required. Save the schema to `outputs/firmographics-schema.json` and validate it loads as well-formed JSON by printing the loaded dict to the terminal. Confirm every constraint keyword is present by asserting each one exists in the parsed file.

2. Trace the constrained decoding loop for a toy vocabulary of exactly eight tokens: `{ "`, `name`, `:`, `" `, `bot`, `"`, `}`, and `<eos>`. Assume the grammar expects the exact string `{"name":"bot"}`. Build a table (as a Python dict of `step → list of allowed token strings`) that lists, for each of the eight decoding steps, which tokens are *not masked* and why. Print the table to stdout. For each masked token, include a one-line rationale (e.g., "expected opening brace, `:` cannot start an object").

3. Implement a Pydantic model called `EnrichmentResult` that mirrors the schema you wrote in Exercise 1 — including the enum, the numeric bounds, and the URL pattern (use a ` constr` with regex for the domain). Instantiate it from a dict of raw enrichment data you hardcode (pull realistic values from an Apollo company profile or fabricate plausible ones for a real company). Print the model instance, then call `.model_json_schema()` and print the generated schema. Visually compare the generated schema against your hand-written one from Exercise 1 and note three differences (e.g., Pydantic adding `title`, reformatting the enum, adding `type` wrappers). Print each difference as a line to the terminal.

4. Using the same Pydantic model from Exercise 3, implement a script that runs the same prompt against a local or hosted LLM twice: once with raw JSON output and manual `EnrichmentResult.model_validate_json()`, and once with constrained decoding (via Outlines, llama-cpp-python's grammar mode, or your instructor's provided schema-enforced endpoint). Run each approach 10 times with the same prompt. Count and print: how many raw runs raised a `ValidationError`, how many constrained runs raised one, and the total wall-clock time for each approach. Print a summary table to the terminal.

5. Design three deliberately adversarial prompts that each trigger a different failure mode: (a) a prompt whose raw output is not valid JSON at all (parse error), (b) a prompt whose output is valid JSON but violates a schema constraint such as an out-of-range integer or a wrong enum value (schema violation), and (c) a prompt whose output passes both JSON parsing and schema validation but is semantically wrong — for example, the model fills `employee_count` with a plausible-sounding but hallucinated number for a real company. Write a script in `handlers/failure_modes.py` that runs each prompt through raw generation and through constrained decoding, prints the raw and constrained outputs side by side, and prints a verdict for each prompt confirming which failure mode constrained decoding eliminated and which it did not. Save the three prompts and all six outputs to `outputs/failure-modes-report.md`.

6. Build a mini enrichment pipeline that takes a CSV of 20 company domains (you may scrape them from a public company list or hardcode real domains), sends each to an LLM with your `EnrichmentResult` schema enforced via constrained decoding, and writes results to `outputs/enriched-companies.jsonl`. After all enrichments complete, compute and print three diagnostics: how many results have an `industry` value from the allowed enum, how many `founded_year` values fall within the schema bounds, and how many `employee_count` values you can spot-check against a public source and confirm are semantically correct. Explain in a printed summary which of the three failure classes from Exercise 5 each diagnostic targets.
