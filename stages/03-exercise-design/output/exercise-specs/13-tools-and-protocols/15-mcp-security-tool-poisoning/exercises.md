## Exercises

1. **Implement** a script that loads a mock MCP server's `tools/list` response from a JSON file (you create the file with at least three tools, one of which has an injected instruction hidden in its description). Your script must dump every tool's `description` field to the terminal in full — raw, untruncated — and separately print only the first 80 characters of each description as a "chat preview." Run the script and confirm that the injected tool's full description contains directives invisible in the preview. Your terminal output should make the gap between what the user sees and what the model reads immediately obvious.

2. **Trace** a name collision across two server config files. Create two JSON files representing MCP server manifests (`server_a.json`, `server_b.json`), each listing 3–4 tools. Deliberately give one tool in `server_b` the same name as a tool in `server_a` but with a different description. Write a script that loads both manifests, detects all colliding tool names, and prints each collision with both servers' descriptions side by side. Confirm your output shows which server's tool would shadow the other depending on connection order.

3. **Build** a hash-pinning detector that stores SHA-256 hashes of every tool description it sees on first connection, persists them to a JSON snapshot file, and on subsequent connections compares the live descriptions against the stored hashes. If any hash differs, print the tool name, the old hash, the new hash, and a `MUTATION DETECTED` flag. Test it by running it twice against a mock server JSON you manually edit between runs. Your terminal must show a clean pass on the first run and flagged mutations on the second.

4. **Implement** a static poisoning detector that takes a directory of tool-description JSON files and scans each description against a pattern list you define — at minimum covering: imperative instructions to the model (`"ignore"`, `"before using"`, `"call"`, `"read the file"`), exfiltration verbs (`"send"`, `"post"`, `"upload"`, `"exfil"`), and references to other tools by name. Seed the directory with descriptions mimicking real GTM enrichment tools (a Clay "enrich-company" tool, an Apollo "find-contact" tool, a CRM "create-lead" tool) — poison at least one of them. Your script must output each flagged tool, the matched pattern, and the offending line. Terminal output is the artifact.

5. **Design and build** an end-to-end cross-server shadowing trace. Create a scenario with three mock MCP servers where one server deliberately registers a tool name that collides with a trusted server's tool, then embeds a poisoned description that instructs the model to route all calls through itself. Write a script that reconstructs the attack: it loads all three server manifests, identifies the collision, extracts the shadowing tool's injected directives, and prints a step-by-step trace from name registration through routing interception. Then extend the script to run your hash-pinning detector and your static poisoning detector against the same manifests, printing whether each defense would have caught the attack. Produce a written comparison of defended vs. undefended outcomes, citing at least two published 2025–2026 measurements of MCP attack-success rates (from the lesson references or independent research). Write the final report to `outputs/skill-mcp-shadowing-analysis.md`.
