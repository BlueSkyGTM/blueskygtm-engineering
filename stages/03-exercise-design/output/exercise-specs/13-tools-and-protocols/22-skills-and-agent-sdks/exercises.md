## Exercises

1. **Implement a single-tool calling loop with the Anthropic Messages API.** Define one tool — `lookup_company_domain(company_name: str) -> str` — that returns a hardcoded domain for a given company. Send a user prompt asking Claude to find the domain for "Stripe," then implement the full loop: detect the `tool_use` block, execute your local function, feed the result back as a `tool_result`, and print Claude's final response. Add a `print` statement at each stage of the loop (tool requested → tool executed → result returned → final answer) so the terminal shows a numbered trace of the control flow.

2. **Write an `AGENTS.md` file and validate it programmatically.** Create an `AGENTS.md` with YAML frontmatter declaring at minimum: `project_name`, `description`, `capabilities` (list), `permissions` (object with `allowed_tools` and `denied_tools`), and `context_files` (list). Then write a Python script that reads the file, parses the frontmatter, and asserts that every required field exists and has the correct type. Run the script — it should print `VALID` or raise an assertion error with the missing field name. Intentionally delete one field and re-run to confirm the validator catches it.

3. **Re-implement the same tool-calling loop from Exercise 1 using the OpenAI function-calling API.** Define the equivalent `lookup_company_domain` function in OpenAI's `tools` schema format, run the loop against `gpt-4o` or `gpt-4o-mini`, and print a numbered trace identical in structure to Exercise 1. After both traces are visible in your terminal, write a brief comment block at the top of your script documenting three concrete differences in how the two APIs structure the tool-call exchange (e.g., where the tool name lives, how results are passed back, what signals the loop to terminate).

4. **Write a `SKILL.md` for a GTM ICP-scoring workflow.** The YAML frontmatter must include `name`, `description`, and `version`. The body should instruct the agent to collect firmographic data, score fit against an ICP, and reference two subdirectory paths for deeper context (e.g., `context/icp_definition.md` and `context/scoring_rubric.md`). Create those subdirectory files with minimal content. Then write a validation script that parses the `SKILL.md` frontmatter, confirms all three required fields are present, and checks that every path referenced in the body actually exists on disk. Terminal output should list each referenced path with `FOUND` or `MISSING`.

5. **Build an enrichment agent that chains multiple data-provider tools and halts autonomously.** Implement at least three tools: `apollo_company_search(domain)` returning industry and headcount, `scrape_website(domain)` returning a homepage summary, and `clearbit_technographics(domain)` returning detected tech stack. Use either the Anthropic or OpenAI SDK tool loop. The agent receives a target company name, calls tools in whatever order it chooses, and decides on its own when it has collected enough data to produce an enrichment profile — your loop must not hardcode a tool sequence or a fixed iteration count. The agent must print its final structured profile (company, industry, headcount, tech stack, ICP fit score) to the terminal. Persist the full agent implementation — including tool definitions, the loop, and any helper functions — as a runnable script at `handlers/enrichment_agent.py`.

6. **Design a three-layer specification map for a Clay-to-CRM sync workflow and prove the boundaries hold.** Write three artifacts: (a) a snippet of the SDK execution loop (the `while` loop that handles tool calls) in `handlers/clay_sync_loop.py`, (b) a `SKILL.md` whose body describes the know-how for mapping Clay enrichment fields to CRM custom fields, and (c) an `AGENTS.md` whose permissions block restricts the agent to read-only CRM access and write access only to a staging table. Then write a short proof script that loads all three files and demonstrates that each layer answers a different question — "how does the loop execute," "what does the agent know," and "what is the agent allowed to do" — by extracting and printing the relevant section from each file. Save the proof output to `outputs/skill-agent-architecture.md`.
