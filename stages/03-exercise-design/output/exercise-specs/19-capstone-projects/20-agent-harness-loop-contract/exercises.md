# Exercises — Agent Harness Loop Contract

## Exercises

1. Implement the six harness states (`IDLE`, `PLANNING`, `EXECUTING`, `AWAITING_TOOL`, `REFLECTING`, `DONE`) as a Python `Enum`. Define a transition adjacency map as a dictionary mapping each state to its list of legal successor states. Write a function `get_legal_transitions(state: HarnessState) -> list[HarnessState]` that returns the successors. Call it for all six states and print each result. Verify three invariants in your output: `IDLE` transitions only to `PLANNING`, `REFLECTING` never appears as a successor of `EXECUTING` unless going through the no-tool path, and `DONE` has zero outgoing transitions.

2. Define two Pydantic models: `HarnessInput` (fields: `goal: str`, `session_id: str`, `budget_tool_calls: int`) and `ModelResponse` (fields: `action: Literal["plan", "tool_call", "reflect", "done"]`, `tool_name: str | None`, `reasoning: str`, `confidence: float`). Write a `validate_response(raw: dict) -> ModelResponse` function that returns the validated model or raises `ValidationError`. Run it against three test payloads — one well-formed, one missing `reasoning`, one with `confidence` set to a string — and print the validated object or exception message for each.

3. Build a `ToolRegistry` class with `register(name: str, handler: Callable)` and `invoke(name: str, **kwargs) -> dict` methods. Add a `Budget` class that tracks remaining tool calls and exposes a `termination_predicate() -> Callable[[], bool]` callback returning `True` when the budget is exhausted. Simulate a session that registers three tools, invokes them in sequence, checks the termination predicate after each call, and attempts a fourth invocation after budget depletion. Print the predicate result and invocation status after each step. Confirm that the registry holds no state between sessions by instantiating a second `ToolRegistry` and asserting it has zero registered tools.

4. Implement the Clay enrichment waterfall as a concrete harness loop instance. Register three enrichment providers as tools — `hunter_io`, `apollo`, and `zerobounce` — each returning a mocked response dict containing an `email` field and a `confidence` float between 0.0 and 1.0. Write the loop body that starts in `PLANNING`, transitions to `EXECUTING`, invokes each provider in sequence via `AWAITING_TOOL`, evaluates confidence in `REFLECTING`, and terminates to `DONE` as soon as confidence exceeds 0.85 or all providers are exhausted. Feed three test company records through the loop and print the state transition trace, provider attempted, confidence score, and termination reason for each record.

5. Implement a minimal hand-rolled harness loop that satisfies the six-state contract with typed Pydantic I/O, a tool registry, and budget-based termination. Then adapt the same enrichment task from Exercise 4 to run through the OpenAI Agents SDK loop and through a LangGraph `StateGraph` with a cyclic edge. Write a comparison documenting which contract guarantees each approach enforces natively (state transition validation, schema enforcement, budget limits, termination predicates) versus what you had to add manually. Save the comparison as a Markdown file at `outputs/skill-harness-comparison.md` with a summary table and per-framework notes.
