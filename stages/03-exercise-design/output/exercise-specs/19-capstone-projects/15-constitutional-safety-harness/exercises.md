## Exercises

1. Implement a `Principle` class that holds a `name`, a natural-language `rule`, a `score(text) -> float` method, a `threshold`, and a `severity` level. Write three principles targeting common GTM safety violations (e.g., "do not fabricate metrics," "do not impersonate a human," "do not claim medical benefits"). Score five sample outputs (mix of safe and unsafe text) against each principle and print a table showing principle name, score, threshold, and whether the principle was violated.

2. Configure an action policy as a function that takes a list of principle-score results and returns one of `pass`, `rewrite`, `block`, or `escalate` based on the worst-case severity and whether any score fell below threshold. Feed ten LLM-generated outputs (provided as a JSON file or inline list) through a constitution of at least four principles and print each output's ID, the triggering principle (if any), and the final action.

3. Implement a self-correction loop that accepts a prompt, a constitution (list of principles), a rewrite function, and `max_retries`. The loop generates an output, scores it, rewrites if any principle falls below threshold, re-scores, and repeats until all principles pass or retries are exhausted. Run the loop on five prompts where the initial output is likely to violate at least one principle. Print each iteration's scores and the final verdict (`accepted` or `exhausted retries`).

4. Build a red-team evaluation pipeline. Create an adversarial prompt suite with at least 15 prompts organized into four attack families: `prompt_injection`, `social_engineering`, `hallucination_bait`, and `compliance_trap`. Run each prompt through your constitutional harness and print a summary table showing each attack family's total prompts, passes, failures, and pass rate percentage. Include the worst-performing family in a final line of output.

5. Compare classifier-based safety against your constitution-based harness. Implement a keyword-regex classifier with a denylist of unsafe phrases. Run the same adversarial suite from Exercise 4 through both systems. Print a side-by-side table of per-family pass rates for each approach. Identify two prompts where the classifier passes but the constitution fails (or vice versa) and print each one with a one-line explanation of the discrepancy. Write your full comparison — including per-family numbers, the discrepancy examples, and a recommendation for which approach to use in a production GTM system — to `outputs/skill-constitutional-comparison.md`.

6. Design a domain-specific safety constitution for GTM copy generation covering three channels: cold email sequences, Apollo/Clay enrichment summaries, and LinkedIn comments. Define at least four principles per channel (e.g., "enrichment summaries must not fabricate data points not present in the source record," "LinkedIn comments must not make product claims without disclaimers," "cold emails must comply with CAN-SPAM unsubscribe requirements"). Load a set of ten sample GTM outputs (or generate them with an LLM call using realistic enrichment data from a sample CSV), route each through the appropriate channel's constitution, and run the self-correction loop. Write the full constitution, all sample outputs with their scores and final actions, and any rewrites the loop produced to `handlers/gtm_safety_harness.py` as an importable, executable module that prints a launch-readiness report when run directly.
