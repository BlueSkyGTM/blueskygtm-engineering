# Exercises — Cross-Attention Fusion

## Exercises

1. **Implement single-head cross-attention in NumPy.** Create a function `cross_attention(A, B, W_Q, W_K, W_V)` that takes query sequence `A` of shape `(L_A, d)`, memory sequence `B` of shape `(L_B, d)`, and three projection matrices. Compute Q from A, K and V from B, apply scaled dot-product attention with the standard `1/sqrt(d_k)` scaling factor, and return both the output and the attention weight matrix. Feed in `A = np.random.randn(4, 8)` and `B = np.random.randn(6, 8)` with appropriately sized projection matrices of your choosing. Print the output shape and confirm it equals `(4, d_v)` — the query sequence's length, not the memory stream's.

2. **Trace every intermediate shape.** Modify your function from Exercise 1 to print the shape of Q, K, V, the raw score matrix `Q @ K.T`, the softmax output, and the final result — each with a descriptive label. Use `A` of shape `(3, 5)` and `B` of shape `(7, 5)` with `d_k = 4` and `d_v = 6`. Before running the code, write down the shape you expect at each stage on paper. Then execute and verify your predictions match the terminal output. If any shape disagrees with your prediction, identify which step you misreasoned about.

3. **Compare three fusion strategies on the same pair of sequences.** Given a query stream `A` of shape `(L_A, d)` and a memory stream `B` of shape `(L_B, d)`, implement: (a) concatenation along the feature dimension after zero-padding the shorter sequence to match lengths, (b) element-wise averaging after truncating both to `min(L_A, L_B)`, and (c) your cross-attention function from Exercise 1. For each strategy, print the output shape and the L2 norm of the output. Then write a brief comment in your script answering: which strategy destroys positional correspondence between A and B? Which preserves A's length but not B's? Which can selectively weight different positions in B for each position in A?

4. **Apply cross-attention to fuse two real enrichment signals for a list of companies.** Create a synthetic dataset: a query stream where each row represents a target company encoded as a 16-dimensional vector (firmographics like employee count, revenue band, industry code embeddings), and a memory stream where each row represents a signal from a different source — say, technographic stack data or intent data — also 16-dimensional. Use 10 target companies and 15 enrichment signals. Run cross-attention and print, for the first company in the list, the attention weights across all 15 enrichment signals sorted by magnitude. Identify which enrichment signal the model attended to most heavily for that company and print its index and raw weight value.

5. **Build a cross-attention enrichment fusion layer that logs attention weights for every query-memory pair.** Implement a class `CrossAttentionFusionLayer` that accepts a list of company records (each with a name and a feature vector) and a list of enrichment source records (each with a source name like `"Apollo"`, `"Clay"`, `"Clearbit"` and a feature vector). The class projects both streams, runs cross-attention, and writes a Markdown report to `outputs/skill-cross-attention-fusion.md` containing a table of each company's top-3 attended enrichment sources with their weights. Use at least 5 companies and 3 distinct enrichment sources with 2-3 records each. Run the script and open the generated file to verify the table is populated and every company has three rows with decreasing weight values.

6. **Design an experiment that proves cross-attention outperforms averaging fusion when the memory stream contains conflicting signals.** Construct a memory stream `B` where the first half of rows are "positive enrichment" vectors and the second half are noise vectors that are roughly orthogonal to the query stream. Run both averaging fusion and cross-attention (with hand-tuned or randomly initialized weights) across 50 random query vectors, and compute the average cosine similarity between each query and its fused output for both strategies. Write the per-strategy averages to `signals/examples/cross_attention_vs_avg.py` as a runnable script that prints the comparison. State in a docstring why cross-attention should win in this setting and confirm the empirical results match your reasoning.
