# Exercises — Perplexity and Calibration

## Exercises

1. **Compute** perplexity from a list of token negative log-probabilities. Write a Python script that takes a hardcoded list of NLL values (e.g., `[2.3, 0.4, 1.1, 3.7, 0.9, 1.5]`), computes the average NLL, exponentiates it, and prints the perplexity along with the average per-token probability. Then re-run with a second list of your own choosing where all NLLs are below 0.5 and confirm perplexity drops below 2.0. Print both results to the terminal.

2. **Build** a calibration curve from a list of (predicted_probability, actual_outcome) pairs. Write a script that defines at least 50 pairs of data, bins them into 5 probability buckets (0.0–0.2, 0.2–0.4, 0.4–0.6, 0.6–0.8, 0.8–1.0), computes the average predicted probability and actual frequency of positive outcomes per bin, and prints a table showing bucket, mean prediction, actual frequency, and the gap between them. State in a printed summary whether the system is overconfident, underconfident, or well-calibrated.

3. **Implement** temperature scaling on a set of raw logits. Write a function that accepts a list of logit vectors and a temperature parameter, divides each logit by the temperature, applies softmax, and returns the calibrated probabilities. Then write a search loop that tries temperatures from 0.5 to 3.0 in increments of 0.1, computes expected calibration error (ECE) against binary outcomes for each, and prints the optimal temperature and its ECE. Use at least 100 simulated logit/outcome pairs.

4. **Compare** perplexity and calibration on two synthetic model outputs you construct so that one has low perplexity but poor calibration and the other has high perplexity but good calibration. Write a script that prints both metrics for both models alongside a one-line interpretation of what each metric captures and why it fails to capture the other. The goal is to demonstrate that neither number subsumes the other.

5. **Build** a calibration audit for a Clay-style lead enrichment table. Export or simulate a CSV of 200 scored leads with columns: `company_name`, `icp_fit_score` (0.0–1.0 from your enrichment workflow), `industry`, `employee_count`, and `converted` (0 or 1). Write a script at `signals/examples/lead_calibration_audit.py` that loads the CSV, bins ICP fit scores into deciles, computes actual conversion rate per decile, prints a calibration table, and computes ECE. The script must write a markdown report to `outputs/skill-lead-calibration.md` containing the calibration table, the ECE value, and a diagnostic paragraph stating whether the lead scorer is overconfident or underconfident and which score deciles are most dangerous to act on.

6. **Design** and implement a calibration monitoring handler for an ICP matching system that ingests batch predictions and outcomes over time. Write a script at `handlers/icp_calibration_monitor.py` that accepts a JSON file of prediction-outcome records, maintains a rolling window of the last 500 predictions, computes per-bin calibration drift between the current window and the prior window, and prints an alert if any bin's calibration gap has shifted by more than 0.15. The script must produce a terminal summary table and write the full drift report to `outputs/skill-icp-drift.md`. Include a synthetic dataset of at least 300 records with a deliberate miscalibration injection and demonstrate that the monitor catches it.
