# Exercises — End-to-End RAG System

## Exercises

1. **Run the end-to-end RAG pipeline on the provided fixture documents and trace the output through each stage.** Execute the pipeline from the lesson against `data/fixtures/products/` with the query "What is the refund window for enterprise plans?" Print the chunker's output count, the hybrid retriever's top-5 chunk IDs, the reranker's reordered top-3, and the final grounded answer with citations. Confirm the answer cites at least one source file and passage index.

2. **Modify the refuse-on-low-confidence threshold and observe the fallback behavior.** Change the confidence threshold from `0.45` to `0.85`, re-run the same query from Exercise 1, and confirm the pipeline now emits a refusal instead of a grounded answer. Then run three additional queries from `data/fixtures/queries.jsonl` at the elevated threshold and log which queries succeed vs. refuse. Print the pass/refuse counts to the terminal.

3. **Implement a standalone citation parser that extracts structured citations from a generated answer.** Given a list of answer strings produced by the pipeline (use the five queries in `data/fixtures/queries.jsonl`), write a function that returns a list of `(query, source_file, passage_index)` tuples for every citation found. Run it against the pipeline's outputs and print any query whose answer contains zero citations — those are your ungrounded responses.

4. **Apply the pipeline to a different document domain — sales playbooks.** Point the chunker at `data/fixtures/sales_playbooks/` (MEDDIC guides, competitor battle cards, objection handling docs). Run the six fixture queries from `data/fixtures/sales_queries.jsonl` through the full pipeline. Measure recall@3 for the hybrid retriever stage and compare it to the recall@3 the same queries achieve with keyword-only retrieval. Print both numbers and the delta.

5. **Build a RAG-powered GTM research assistant that grounds answers in enriched account data.** Write a handler that loads a sample of Clay-enriched company profiles from `data/fixtures/clay_exports/companies.jsonl`, chunks the enrichment notes (firmographics, tech stack, recent triggers), runs the full RAG pipeline, and answers queries like "Which accounts show intent for a data warehouse migration?" with citations pointing to the specific company record and field. The assistant must refuse when no enrichment fields exceed the confidence threshold. Save the handler to `handlers/gtm_rag_assistant.py` and produce a markdown summary of three sample queries, their answers, and citations at `outputs/skill-gtm-rag.md`.

6. **Design and run a failure-mode audit on the full pipeline.** Create a fixture set of 15 queries split across three categories: (a) answerable from the documents, (b) unanswerable but plausible, (c) adversarial — queries designed to trigger hallucination from partial matches. Run the pipeline against all 15 and classify each output as `correct`, `refused-correctly`, or `hallucinated`. Print a 3×3 confusion matrix (query category × output classification) to the terminal. Write a one-paragraph diagnosis of the dominant failure mode and the threshold change you would make to fix it — save both to `outputs/skill-rag-audit.md`.
