## Exercises

1. **Implement AST-aware function-level chunking for Python files.** Write a script that walks a directory of `.py` files, parses each with the `ast` module, and emits one chunk per top-level function and method. Print each chunk's function name, source file, start line, and end line to the terminal. Run it against at least two files in this repo and verify that no chunk starts or ends mid-function.

2. **Embed your chunks and run a single cosine-similarity query.** Using the chunks from Exercise 1, generate embeddings with the model of your choice and store them alongside provenance metadata (repo, path, language, start_line, end_line). Accept a free-text query from a command-line argument and print the top 5 matching chunks with their similarity scores and full provenance. Confirm that identical function names in different files produce separate, distinguishable results.

3. **Compare function-level, class-level, and file-level chunking strategies.** Implement all three chunkers over the same set of source files. For each strategy, run a fixed set of 5 queries (e.g., "How does retry logic work?", "Where is the embedding model configured?", "What does the CLI entrypoint do?") and print the top-3 results per query per strategy. Compute a simple relevance score (1 = relevant, 0 = not) for each result by manual inspection and print the aggregate per strategy. State which strategy wins for each query type.

4. **Implement incremental indexing that skips unchanged files.** Store a content hash (e.g., SHA-256 of file contents) for each indexed file in a JSON sidecar. On subsequent runs, re-chunk and re-embed only files whose hash has changed; reuse cached embeddings for unchanged files. Print a run summary showing how many files were skipped, how many were (re-)indexed, and total wall-clock time. Verify that a second run with no file changes completes in under 20% of the first run's time.

5. **Build a persistent, incrementally-indexed CLI search tool.** Package the chunker, embedder, provenance tracker, incremental indexer, and cosine-similarity retriever into a single CLI with two subcommands: `index <repo-path>` and `search "<query>"`. The index must persist to disk between invocations. Demonstrate it by indexing two separate repositories (e.g., this curriculum repo and a cloned open-source repo), then running a query that returns results from both. Print each result with repo name, file path, line range, language, similarity score, and the first 5 lines of the matched chunk. Artifact lands at `handlers/code_search.py`.

6. **Design and run an evaluation harness for solutions engineering query patterns.** Define 10 realistic queries a solutions engineer might ask when onboarding a customer (e.g., "How do we map CRM account_id to our internal tenant_id?", "Where is the rate-limit header parsed?", "What enrichment fields does the Apollo integration populate?"). Run each query through your tool from Exercise 5, capture the top-5 results, and rank them by manual relevance (Exact / Partial / Miss). Compute precision@5 and mean reciprocal rank across all 10 queries. Write up your findings — which query types the AST-based pipeline excels at, where it breaks down, and what you would change. Artifact lands at `outputs/skill-code-search-eval.md`.
