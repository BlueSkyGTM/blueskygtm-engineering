# Exercises — Experiment Runner

## Exercises

1. **Implement a minimal experiment runner** that takes a list of 60 prospect emails and two variant names (`"A"` and `"B"`), assigns each email to a variant using `random.choice`, executes a simulated `send_email(variant, email)` call that returns a dict `{"email": ..., "variant": ..., "opened": bool}`, and prints a summary table of open rates per variant to the terminal. Seed `random` with `42` so your output is reproducible and verifiable.

2. **Implement deterministic sample splitting** by writing a function `assign_variant(unit_id, variants, seed=42)` that hashes the `unit_id` (use `hashlib.md5`) and maps the hash modulo `len(variants)` to produce a stable assignment. Demonstrate it by assigning 200 generated account IDs (`"acct_000"` through `"acct_199"`) across three variants `"control"`, `"treatment_v1"`, and `"treatment_v2"`, then print each variant's count and the percentage split. Confirm the splits are approximately equal and that re-running produces identical assignments.

3. **Build a permutation test function** `permutation_p_value(outcomes, assignments, n_permutations=1000)` that takes a list of binary outcomes (0 or 1) and a corresponding list of variant labels, computes the observed difference between the two variant means, then shuffles the labels `n_permutations` times and counts how often the shuffled difference equals or exceeds the observed difference. Print the observed difference, the p-value, and whether the result is significant at alpha = 0.05. Test it on two synthetic datasets: one where variant B clearly outperforms A (e.g., B open rate 0.35 vs A open rate 0.15, 100 units each) and one where both are near 0.22.

4. **Configure an experiment with stopping conditions.** Write an `ExperimentConfig` dataclass with fields for `sample_size`, `confidence_threshold` (default 0.95), `min_sample_per_variant`, and `early_stop_margin` (the minimum effect size required to stop early). Then write a function `should_stop(results_by_variant, config)` that returns `True` if all variants have at least `min_sample_per_variant` observations AND either (a) the permutation p-value between the top two variants meets the confidence threshold, or (b) the total sample reaches `sample_size`. Feed it incremental batches of simulated outcome data (50 units at a time, up to 500 total) and print, after each batch, whether the experiment should continue or stop — including which condition triggered the stop.

5. **Build a complete GTM experiment pipeline** that reads a list of 300 prospect records from `outputs/prospects.json` (each record containing `email`, `company`, `title`), assigns each prospect to one of three email subject-line variants using your hash-based splitter, simulates outcomes with variant-specific true open rates (e.g., control: 0.20, v1: 0.28, v2: 0.17), runs the permutation test between all variant pairs, selects a winner, and writes a full experiment report to `outputs/skill-experiment-report.md`. The report must include: per-variant sample sizes and open rates, pairwise p-values in a table, the selected winning variant, the confidence level, and a one-paragraph recommendation stating whether to ship the winner or run a follow-up experiment.

6. **Design an experiment for real enrichment data.** Using Apollo's people search API (or a cached response file at `outputs/apollo_people.json` if the API is unavailable), pull 200 prospects matching a target ICP. Write a script `handlers/experiment_apollo.py` that assigns each prospect to one of two outreach variants (e.g., `direct_cta` vs `value_first`), simulates reply outcomes using a configurable rate dictionary, runs the permutation test, and outputs a JSON artifact to `outputs/apollo_experiment_result.json` containing the assignment table, outcome table, p-value, confidence interval (computed via bootstrap with 2000 resamples), and a `decision` field set to `"ship"`, `"iterate"`, or `"inconclusive"` based on the evidence. Print a human-readable summary to the terminal. Include a `--dry-run` flag that performs assignment and prints the split without executing outcomes.
