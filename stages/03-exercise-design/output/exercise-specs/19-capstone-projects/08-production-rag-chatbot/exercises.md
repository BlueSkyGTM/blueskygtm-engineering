## Exercises

1. **Verify role-based post-filtering produces different retrieval sets.** Run the lesson's retrieval pipeline against the mock HIPAA corpus with the query `"patient data breach notification timeline"` using two roles: `compliance_officer` and `external_auditor`. Print the chunk IDs and similarity scores returned for each role to the terminal. Confirm the two result sets differ — at least one chunk present for `compliance_officer` must be absent for `external_auditor`.

2. **Implement the "not available" fallback for citation enforcement.** Modify the generation step so that when post-filtering removes all chunks (i.e., the permitted retrieval set is empty), the pipeline returns the literal string `"not available in permitted sources"` instead of attempting to generate an answer. Test with the query `"internal legal strategy for breach response"` run under the `public` role and print the output. Then run the same query under `compliance_officer` and confirm a sourced answer with bracketed source IDs is returned instead.

3. **Implement audit logging that writes a JSONL record per query.** Add a logging step to the pipeline that appends a JSON object to `outputs/audit-log.jsonl` after every query. Each record must contain: `timestamp` (ISO 8601), `user_role`, `query`, `retrieved_chunk_ids` (the post-filtered set), and `answer`. Run five queries across two roles, then read the file back and print a summary table showing query count per role and average retrieved chunks per role.

4. **Apply role-based post-filtering to a GTM sales-enablement corpus.** Build a mini RAG pipeline for a sales team where the corpus has two document classes: `battle_cards` tagged `roles: ["sales_internal"]` and `case_studies` tagged `roles: ["sales_internal", "partner_portal"]`. Ingest five mock documents, run the query `"objection handling for competitor X"` under both `sales_internal` and `partner_portal` roles, and print the returned chunk IDs for each. Confirm `partner_portal` only sees case-study chunks, never battle-card chunks.

5. **Build a compliance comparison harness that audits a naive baseline against the regulated pipeline.** Create `handlers/audit_comparison.py` that runs the same five queries through (a) a naive retrieve-and-generate baseline with no role filtering and no citation enforcement, and (b) the full regulated pipeline. For each query, print a side-by-side table with columns: `query_id`, `naive_chunks_returned`, `naive_has_citations`, `regulated_chunks_returned`, `regulated_has_citations`, `access_violation_in_naive` (true if any returned chunk's `roles` list excludes the querying user's role). At the bottom, print the total count of access violations across all five queries for the naive pipeline — this number must be greater than zero.

6. **Compute citation precision and recall against a ground-truth retrieval set.** Create `handlers/citation_eval.py` that defines five queries, each with a manually labeled ground-truth set of expected source IDs (the chunk IDs a `compliance_officer` *should* retrieve). Run the regulated pipeline under the `compliance_officer` role for all five queries, extract the bracketed source IDs from each generated answer, and compute per-query precision (`cited ∩ ground_truth / cited`) and recall (`cited ∩ ground_truth / ground_truth`). Print a results table with columns: `query_id`, `ground_truth_ids`, `cited_ids`, `precision`, `recall`. At the bottom, print mean precision and mean recall across all five queries.
