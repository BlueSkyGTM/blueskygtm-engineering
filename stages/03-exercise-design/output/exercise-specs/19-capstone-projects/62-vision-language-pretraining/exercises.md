# Exercises — Vision-Language Pretraining

## Exercises

1. **Implement InfoNCE loss from scratch and train it on synthetic image-text pairs.** Create two randomly initialized projection heads (one for image embeddings, one for text embeddings), each mapping 512-dimensional synthetic inputs into a shared 256-dimensional space. Generate a batch of 32 paired embeddings (use `torch.randn` with a shared seed so pairs have correlated structure). Implement the symmetric InfoNCE loss—both image-to-text and text-to-image directions—without using any library function. Train for 200 steps with a learning rate of 1e-3 and print the loss every 20 steps. Your terminal output should show the loss decreasing from approximately `ln(32) ≈ 3.47` toward values below 1.0.

2. **Extract and inspect the CLIP projection layer to locate where cross-modal alignment physically occurs.** Load `openai/clip-vit-base-patch32` using the `transformers` library. Print the model architecture and identify the exact linear layers that project the vision encoder output and the text encoder output into the shared embedding space. Extract the weight matrices of both projection layers and print their shapes. Compute the cosine similarity between the first 5 rows of each projection weight matrix to demonstrate that the two projections do not share weights but map into a shared dimensionality. Your terminal output should display layer names, weight shapes, and a 5×5 similarity matrix.

3. **Run a temperature parameter sweep and measure the effect on embedding sharpness.** Using your InfoNCE implementation from Exercise 1, train five separate models for 200 steps each using temperature values `[0.01, 0.05, 0.1, 0.5, 1.0]`. After training each model, compute the average pairwise cosine similarity across all positive pairs and across all negative pairs in a held-out batch of 64 pairs. Print a table to the terminal showing, for each temperature: final loss, average positive similarity, average negative similarity, and the positive-minus-negative gap. Report which temperature produces the largest separation and which produces the most collapsed embeddings.

4. **Compare contrastive, generative, and masked pretraining objectives by their inference-time behavior.** Write a script that loads three different pretrained models or model configurations (e.g., CLIP for contrastive, a small captioning model for generative, and a masked-language-model checkpoint for reconstruction). For each model, perform two tasks: (a) compute image-text similarity scores on 5 image-caption pairs of your choosing, and (b) attempt to generate a caption from an image input. Print a comparison table to the terminal showing which model supports which capability. Include a one-line summary per objective family explaining why it can or cannot perform each task based on the training signal it was given.

5. **Build a text-text similarity ranker for GTM lead enrichment using InfoNCE structure.** Implement a retrieval system that takes a list of 20 company descriptions (seed it from the Apollo API or a static CSV of real company boilerplate text if no API key is available) and a list of 5 search queries (e.g., "companies selling developer tools," "B2B SaaS in healthcare"). Encode all texts using `sentence-transformers` (e.g., `all-MiniLM-L6-v2`). Compute the similarity matrix using the same symmetric structure as InfoNCE—the dot product of L2-normalized embeddings scaled by a learned or fixed temperature. For each query, print the top 5 matching companies ranked by score. Save the full ranked results and the reusable ranking function to `signals/examples/infonce_ranker.py`. Your terminal output should show each query followed by its top 5 matches with similarity scores.

6. **Design a proof-of-concept that demonstrates why contrastive embeddings enable zero-shot retrieval in a GTM pipeline.** Fetch 50 company records from the Apollo People Search API (or use a provided static dataset at `data/companies.json`). Build a small index of company embeddings using `sentence-transformers`. Implement an enrichment signal: given a free-text ICP description (e.g., "Series A startups hiring their first VP of Sales"), rank all 50 companies and tag the top 10 with a `fit_score`. Write the ranked companies with scores to `outputs/skill-contrastive-icp-ranking.md` as a formatted table. In the same file, include a 3-paragraph analysis of how the temperature parameter and embedding dimensionality affected your ranking quality, referencing concrete numbers from a sweep you run across at least two configurations.
