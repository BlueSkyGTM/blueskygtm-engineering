# Exercises — Projection Layer for Modality Alignment

## Exercises

1. **Implement a linear projection layer** that maps 768-dimensional embeddings into a 256-dimensional shared space using `nn.Linear`. Generate two batches of 8 random embeddings (one from a 768-dim modality, one from a 512-dim modality), apply a separate projection to each batch, and print both output shapes. Verify both projected batches share the same dimensionality by computing and printing the 8×8 pairwise cosine similarity matrix between them.

2. **Build an MLP projection head** (`Linear → GELU → Linear`) for the same two modalities. Feed the same random batches from Exercise 1 through the MLP projections and print the cosine similarity matrix. Compare the distribution of off-diagonal similarities (print mean and std) between the linear projections from Exercise 1 and the MLP projections — the MLP should produce a tighter spread around zero on random data since GELU introduces non-linear scaling.

3. **Implement symmetric contrastive loss from scratch** using only `torch.nn.functional` primitives — no pre-built loss functions. Your implementation must: compute the A→B and B→A cross-entropy terms, average them, and expose temperature `τ` as a parameter. Run it on a batch of 16 projected pairs and print the loss value for `τ ∈ {0.01, 0.07, 0.5}`. Additionally, call `.backward()` on the `τ=0.07` loss and print the L2 norm of the gradients flowing into projection A versus projection B to confirm the symmetric formulation produces balanced gradients.

4. **Compare linear versus MLP projections on a non-linear alignment task.** Generate synthetic paired embeddings where the mapping between modalities is non-linear (e.g., apply a random quadratic transformation to produce modality B from modality A). Train two alignment setups — one with linear projections, one with MLP projections — for 100 epochs using your contrastive loss. Print mean positive-pair cosine similarity at epochs 10, 50, and 100 for both architectures. State which architecture converges faster and by how much.

5. **Build a projection-dimension diagnostic tool** that trains a dual-encoder system at projection dimensions `{64, 128, 256, 512, 768}` on a fixed synthetic dataset, logs alignment metrics (mean positive-pair cosine sim, mean negative-pair cosine sim, loss curve) for each, and writes a markdown report to `outputs/skill-projection-diagnostics.md`. The report must include a table mapping projection dimension to final alignment quality and must flag any dimension where the model exhibits overfitting (training loss decreasing while positive-pair similarity plateaus or degrades).

6. **Design a cross-modal GTM retrieval pipeline** that aligns prospect signals with internal knowledge assets in a shared vector space. Fetch 20 company records from Apollo's API (or use a CSV export with fields like company description, technographics, employee count) and pair each with the most relevant internal asset (case study, one-pager, battle card) from a local `knowledge_base/` directory. Build a dual-encoder projection that maps Apollo-enriched embeddings and knowledge-base embeddings into a shared space, train it for 50 epochs, then run a retrieval query: given a new prospect, return the top-3 matching knowledge assets by cosine similarity. Save the trained retrieval handler to `handlers/cross_modal_retrieval.py` and print the top-3 matches for at least two held-out prospects.
