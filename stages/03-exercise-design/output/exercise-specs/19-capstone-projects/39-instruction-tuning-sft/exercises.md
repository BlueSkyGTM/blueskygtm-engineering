# Exercises — Capstone Lesson 39: Instruction Tuning by Supervised Fine-Tuning

## Exercises

1. **Implement** the Alpaca template formatter. Given a list of `(instruction, response)` tuples, concatenate each pair with `### Instruction:\n` and `### Response:\n` delimiters, tokenize the full string, then build a label tensor that sets every token position belonging to the instruction region (including the delimiter tokens themselves) to `-100` while leaving response tokens at their original IDs. Print, for the first example only, the decoded instruction tokens (where `labels == -100`) and the decoded response tokens (where `labels != -100`) on separate lines so you can visually confirm the boundary lands exactly on the `### Response:` delimiter.

2. **Build** a custom `collate_fn` that accepts a list of pre-formatted `(input_ids, labels)` tuples and returns padded batch tensors. Left-pad or right-pad input IDs with the tokenizer's pad token; extend `labels` with `-100` at every padded position so the loss ignores padding. After collating a batch of 4 examples of different lengths, print `input_ids.shape`, `labels.shape`, the count of `-100` entries in each row, and decode each row's `attention_mask == 1` region to verify no response token was masked out and no padding token leaked into the loss.

3. **Train** a small causal LM (GPT-2 or DistilGPT2) under the SFT objective using a manual PyTorch training loop — no `Trainer`, no `transformers` training utilities. Use any small instruction dataset (Alpaca subset, Dolly subset, or your own 200–500 pairs). Loop over batches, compute per-token cross-entropy with the label mask, backprop, step the optimizer. After each epoch, print epoch number, average loss, and learning rate. Run a minimum of 3 epochs and confirm the loss curve is monotonically decreasing before declaring success.

4. **Compare** generations from the base model and the SFT model on three held-out instructions the model never saw during training. Load the base checkpoint and your fine-tuned checkpoint, feed each instruction through the same template, generate with greedy decoding (temperature 0) for a fixed `max_new_tokens` of 128, and print both outputs side by side. Write one sentence per instruction identifying the specific behavioral shift (e.g., "base model continues the instruction text; SFT model produces a complete answer and stops").

5. **Trace** which token positions contribute gradients under SFT versus continued pretraining on the same batch. For one batch of instruction-response pairs, compute two loss tensors: (a) standard SFT loss where instruction tokens are masked to `-100`, and (b) continued-pretraining loss where every token carries its label. For each regime, sum the per-token loss across the sequence and print the ratio. Then produce a side-by-side bar chart (matplotlib) showing per-token loss for the first 32 positions under each regime. Save the figure and the analysis script to `handlers/sft_vs_pretrain_loss.py` — the script must be re-runnable from the repo root and must write the chart to `outputs/sft_vs_pretrain_loss.png`.

6. **Design** and execute an end-to-end SFT pipeline on a real GTM dataset. Pull 100–300 prospect records from Apollo or Clay (or use a CRM export containing company name, industry, headcount, and a known-good personalized opener), format each row as an instruction like `### Instruction:\nWrite a 2-sentence cold email opener for {company} ({industry}, {headcount} employees).\n### Response:\n{opener}`, fine-tune a small LM, and generate openers for 5 held-out prospects. Print the 5 generated openers alongside their ground-truth counterparts. Persist the full pipeline — data formatting, training loop, generation — as a single reproducible script at `signals/examples/sft_gtm_outreach.py` that can be executed from the repo root with `python signals/examples/sft_gtm_outreach.py`.
