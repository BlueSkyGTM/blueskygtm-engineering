## Exercises

1. **Build a single-page vision extractor.** Render page 3 of the sample PDF to a PNG using `pdf2image` (or `fitz`), base64-encode it, and send it to a vision-language model with a prompt asking for the page's content as structured markdown (headings, tables, lists). Print the model's output to the terminal. Confirm you can see table rows rendered as markdown tables in stdout.

2. **Extract the same page with `pdfplumber`.** Open the same PDF, pull page 3, call `extract_tables()`, and print each detected table row-by-row. Compare the terminal output side-by-side with Exercise 1's markdown. Note which extraction preserved merged header cells and which dropped them.

3. **Compute structural fidelity between the two extractions.** Without a scaffold, parse both outputs into comparable table-cell sets and compute: (a) row-count agreement, (b) cell-level Jaccard similarity after normalizing whitespace, (c) a boolean for whether numeric values survived in both. Print a JSON report like `{"rows_text": 12, "rows_vision": 14, "cell_jaccard": 0.71, "numbers_preserved": true}`. Run it on a table-heavy page and again on a chart-heavy page; print both reports.

4. **Implement a hybrid extractor with confidence-gated fallback.** Write a function `extract_page(pdf_path, page_num)` that tries `pdfplumber` first, computes a confidence signal (e.g., table cell fill rate, non-empty text density, presence of `None` cells indicating merge failures), and falls back to the vision pipeline from Exercise 1 when confidence drops below a threshold you define. Run it across all pages of the sample PDF and print a per-page log: `page 1: text (conf 0.92)`, `page 4: vision-fallback (conf 0.31)`, etc.

5. **Build a GTM-flavored document QA system with page citations.** Implement a CLI tool that takes a multi-page PDF (use a vendor case study, security questionnaire, or competitive battle card) and answers natural-language sales-qualifying questions — e.g., "What integrations does this vendor support?", "What is their stated SOC 2 status?", "Which Fortune 500 logos are referenced?". The system must use the hybrid extractor from Exercise 4, build a per-page index, retrieve the most relevant pages, and answer with a citation like `(p. 4)`. Output a JSON array of `{question, answer, page_citation, confidence}` objects. Persist the handler at `handlers/doc_qa.py` so it can be invoked as `python -m handlers.doc_qa --pdf vendor.pdf --questions questions.json`.

6. **Compare late-interaction multi-vector retrieval against an OCR-then-text baseline.** Build two retrieval indexes over a chart-heavy PDF (one where visual content carries information text can't recover): (a) a ColBERT-style late-interaction index over vision-extracted page representations, and (b) a dense-text index over Tesseract OCR output. Construct 10 queries whose answers depend on chart or diagram content (e.g., "Which quarter showed the largest revenue jump?"). Run both retrievers, compute recall@3 and MRR, and print a comparison table. Write up findings — including two failure cases for each approach — to `outputs/skill-multimodal-retrieval-eval.md`.
