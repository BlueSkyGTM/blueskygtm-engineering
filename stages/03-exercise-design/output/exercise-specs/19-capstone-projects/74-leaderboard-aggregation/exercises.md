# Exercises — Leaderboard Aggregation

## Exercises

1. **Implement min-max, z-score, and percentile normalization.** Create a script that takes three leaderboards with different raw scales — accuracy percentages (0–100), F1 decimals (0.0–1.0), and Elo ratings (1200–2800), each as a list of `(model_name, score)` tuples — and prints all three normalized versions side-by-side in a formatted table. Verify that each method maps every leaderboard to the [0, 1] range (for min-max and percentile) or centered at 0 (for z-score).

2. **Implement Borda count aggregation.** Given 4 full ranked lists (no missing items) over 6 models, compute the Borda count consensus ranking and print the final ordering with point totals. Confirm the output is a single total order with no ties, or explicitly report ties if they occur.

3. **Build a Copeland's method aggregator on enrichment vendor rankings.** You have export data from three enrichment sources (e.g., Apollo, ZoomInfo, and a custom Clay enrichment table), each ranking 8 target accounts by "fit score." Implement Copeland's method: for every pair of accounts, count how many sources prefer one over the other, assign wins/losses, and rank by net win count. Print the pairwise win matrix and the final ranking.

4. **Handle missing items with coverage-aware aggregation.** Four leaderboards each rank 10 models, but each leaderboard is missing 2–3 different models. Implement a strategy that penalizes models appearing on fewer leaderboards (e.g., discount by coverage factor) and produces a consensus ranking that does not over-reward a model that only appears once but happens to rank #1. Compare the naive Borda result (treating missing as unranked with zero points) to your coverage-adjusted result and print both rankings side by side so you can see the difference.

5. **Build a full consensus-ranking pipeline with a Kemeny-Young approximation.** Combine everything: load 5 heterogeneous leaderboards from a JSON file (mixed scales, partial coverage), normalize scores using percentile rank, convert to rankings, and compute a consensus using (a) Borda count and (b) a Kemeny-Young approximation via a local search heuristic (start from Borda, try pairwise swaps that reduce total Kendall tau distance, stop when no swap improves). Print both consensus rankings and the total Kendall tau distance each achieves. Save the final consensus ranking and diagnostics (which method won, the distance gap) to `outputs/skill-consensus-ranking.md`.

6. **Diagnose and fix the three failure modes on a live aggregation.** Design an experiment that demonstrates all three failure modes on a single dataset: (1) **Scale conflation** — show what happens when you sum raw scores across leaderboards with different scales and how normalization fixes it. (2) **Coverage trap** — show what happens when a model appearing on only 1 of 5 leaderboards gets inflated or deflated, and apply your coverage adjustment. (3) **Dictatorship problem** — identify which single leaderboard has the highest correlation with the final consensus and quantify its dominance (e.g., Spearman correlation). Write a script that prints a diagnostic report for each failure mode, the before/after fix, and a final recommendation on whether rank-based or score-based aggregation is more defensible for this data. Save the full diagnostic report to `outputs/skill-aggregation-diagnostics.md`.
