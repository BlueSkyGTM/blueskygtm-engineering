# Exercises — Vision Transformer Encoder

## Exercises

1. **Implement patch extraction in pure NumPy and verify against `nn.Conv2d`.** Write a function `extract_patches(image, patch_size=16)` that takes a `(224, 224, 3)` NumPy array and returns a `(196, 768)` matrix using reshape and transpose only — no loops, no convolutions. Then build the equivalent `nn.Conv2d(3, 768, kernel_size=16, stride=16)` with the same weight matrix reshaped to `(768, 3, 16, 16)`, feed the same image, and print `np.allclose(numpy_output, conv_output, atol=1e-5)`. Your terminal should print `True`.

2. **Trace shape transformations at every stage and print them.** Starting from a `(3, 224, 224)` tensor, manually log the shape after patch extraction, after linear projection, after `[CLS]` prepend, after positional embedding addition, and after one LayerNorm. Print a table like `Stage 0 | shape=(3, 224, 224)` through `Stage 4 | shape=(197, 768)`. Modify the patch size to 32 and re-run — your printed table must show `(49, 3072)` at the projection stage and `(50, D)` after CLS prepend.

3. **Implement one pre-LayerNorm transformer block from scratch using only `torch.nn.MultiheadAttention` and `torch.nn.Linear`.** The block must: (a) apply LayerNorm before attention, (b) residual-add the attention output, (c) apply LayerNorm before MLP, (d) residual-add the MLP output. Feed a `(197, 768)` tensor of random values and assert the output shape is `(197, 768)`. Then verify the residual property: replace attention weights with an identity initialization and confirm the block output equals the input within `1e-4`. Print the max deviation.

4. **Compare ViT patch-embedding locality against a CNN first layer.** Initialize a ViT patch projection (random weights) and a `nn.Conv2d(3, 64, kernel_size=3, padding=1)` (random weights). Feed the same image through both. For the CNN, compute the receptive field of each output position — it should be 3×3 pixels. For the ViT projection, compute the receptive field of each token — it should be 16×16 pixels. Print both receptive fields and write a one-paragraph interpretation to `outputs/skill-vit-vs-cnn-locality.md` explaining which architecture enforces locality as a hard inductive bias and which learns it from data.

5. **Build a complete ViT encoder and export it as a reusable artifact.** Implement `ViTEncoder(image_size=224, patch_size=16, dim=768, depth=12, heads=12, mlp_dim=3072)` with patch embedding, learned positional embeddings, `[CLS]` token, and a stack of 12 pre-LN blocks. Load `torchvision.models.vit_b_16` pretrained weights, copy them into your implementation, and feed a real image through both. Assert that the `[CLS]` output vectors match within `atol=1e-3`. Save your encoder to `handlers/vit_encoder.py` with a `encode_image(path) -> np.ndarray` entrypoint that returns a 768-dim vector.

6. **Connect the ViT encoder to a GTM enrichment pipeline.** Using the encoder from Exercise 5, build a script that: (a) encodes 10 company logo images into 768-dim vectors, (b) computes cosine similarity between a query logo and all 10, (c) retrieves the top-3 most similar companies, and (d) uses Apollo's People Search API or a Clay-enriched CSV (`signals/examples/companies.csv` with `name`, `logo_url`, `industry`, `website` columns) to pull decision-maker contacts for those companies. Print the top-3 matches with similarity scores and contact names. Save the full pipeline to `signals/examples/vit_logo_retrieval.py` and write a short design doc to `outputs/skill-vit-rag-outreach.md` explaining how visual similarity of logos could seed a knowledge-augmented outreach campaign.
