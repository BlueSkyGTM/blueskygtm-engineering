# Exercises — Critic Loop

## Exercises

1. **Implement a generate–critic–refine loop for a single cold email.** Your generator takes a prospect name and company name, produces a draft email. Your critic returns a JSON score vector with three dimensions (`clarity`, `personalization`, `cta_strength`), each scored 1–5. Loop until all three dimensions score ≥ 4 or you hit 3 iterations. Print each iteration's scores, which dimension(s) failed, and the final draft.

2. **Implement a `check_convergence` function** that accepts a list of score vectors (one per iteration), a threshold, a plateau delta, and a max iteration count. It returns one of three states: `"target_met"`, `"plateau_detected"`, or `"budget_exhausted"`. A plateau is flagged when no dimension improves by more than the plateau delta between two consecutive iterations. Feed it three hardcoded test cases (one per outcome) and print the returned state for each.

3. **Build a critic loop for landing page hero copy.** The generator receives a product description and target persona, then outputs a headline, subheadline, and CTA button text. The critic scores four dimensions: `specificity`, `persona_fit`, `urgency`, and `length_compliance` (headline ≤ 10 words). Run with max 4 iterations, threshold 4/5 on all dimensions. Print the final hero copy, the iteration count, and a score history table showing all dimensions across every iteration.

4. **Compare internal critique against external critique on the same set of cold email drafts.** Write 5 cold email drafts by hand (or generate them once, then freeze them). Strategy A (internal): use the same LLM with a different system prompt that scores `clarity`, `personalization`, `cta_strength`. Strategy B (external): a deterministic validator using regex and string checks for: prospect first name present, company name present, a question or CTA sentence present, and word count under 150. Run both strategies on all 5 drafts. Print a comparison table showing pass rate and average iteration count per strategy.

5. **Build a production-grade critic loop module** that accepts a generation function, a critique function, a convergence config (thresholds per dimension, max iterations, plateau delta), and a token tracking wrapper. Log per iteration: iteration number, full score vector, prompt tokens, completion tokens, and cumulative cost using a configurable per-1K-token rate. Run the loop on a task that generates personalized outreach from enriched Apollo lead data — use a sample JSON array of 5 leads with fields like `first_name`, `company`, `title`, `linkedin_bio`. The loop must accept any lead and produce a scored, refined email. Save the reusable module to `handlers/critic_loop.py` and print a cost-per-iteration summary table for the batch of 5 leads.
