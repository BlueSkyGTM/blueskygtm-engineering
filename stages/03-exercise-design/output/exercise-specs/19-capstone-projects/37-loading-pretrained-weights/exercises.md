# Exercises — Loading Pretrained Weights

## Exercises

1. **Inspect checkpoint internals in both formats.** Save the `state_dict` of a small `nn.Sequential` model (two linear layers) to disk using both `torch.save` and `safetensors.torch.save_file`. Then load each file back, print every key alongside its tensor shape, and print the file size of each artifact. Confirm in your terminal that the safetensors file is smaller and that neither format drops or renames any parameter.

2. **Compare key sets across two checkpoints.** Load two different checkpoints of the same architecture (create them by training a tiny model for one step with two different random seeds, then saving each). Write a script that prints the symmetric difference of their `state_dict` key sets — keys only in checkpoint A, keys only in checkpoint B, and keys shared. The terminal output should make obvious whether the two checkpoints are structurally compatible.

3. **Implement key remapping for a renamed architecture.** Given a checkpoint whose keys are prefixed `backbone.` (e.g., `backbone.layers.0.weight`) and a model definition whose keys are prefixed `model.` (e.g., `model.layers.0.weight`), write a function `remap_keys(state_dict, src_prefix, dst_prefix)` that transforms every matching key and returns a new `state_dict`. Load the remapped dict into the live model with `strict=True` and confirm zero missing or unexpected keys. Print the count of keys remapped.

4. **Diagnose shape mismatches before assignment.** Build a checkpoint with `layer1.weight` of shape `[64, 32]` and a model definition where `layer1.weight` expects `[64, 16]`. Write a `check_shapes(model, state_dict)` function that iterates over the intersection of keys, compares shapes, and returns a list of mismatches as `(key, checkpoint_shape, model_shape)` tuples. Call `load_state_dict` only when the list is empty; otherwise print every mismatched key and abort the load. Verify your terminal shows the abort message before any weight is copied.

5. **Load CRM lead-scoring weights partially and validate fidelity.** Download a small pretrained model checkpoint representing a lead-scoring model trained on Apollo-enriched CRM data (you may synthesize this: a 3-layer MLP taking features like company size, funding, and engagement score). Load the checkpoint into a model where you have intentionally renamed the final classification head (`fc` → `classifier`). Use `load_state_dict(..., strict=False)`, then print the `missing_keys` and `unexpected_keys` lists. Implement a validation script that runs ten random input tensors through both the checkpoint's original model (reconstructed from the saved config) and your partially loaded model, asserts `torch.allclose` on every output pair, and writes a markdown report to `outputs/skill-weight-loading-validation.md` documenting which keys loaded, which were skipped, and the max numerical delta observed.

6. **Design a safe-by-default loader.** Write a `SafeLoader` class in `handlers/safe_loader.py` that accepts a path, auto-detects `.pt` vs `.safetensors`, performs pre-load shape validation against a provided model, remaps keys via a configurable prefix map, falls back to `strict=False` with a logged warning when keys don't perfectly align, and raises a `WeightCorruptionError` if any shape mismatch is detected. Include a `validate_fidelity(model, reference_outputs)` method that compares outputs and returns a pass/fail boolean. Demonstrate the class on a real model checkpoint and print a structured load summary to the terminal.
