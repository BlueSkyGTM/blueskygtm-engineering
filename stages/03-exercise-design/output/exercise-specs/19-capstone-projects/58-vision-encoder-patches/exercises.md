# Exercises — Vision Encoder Patches

## Exercises

1. **Compute token counts for a grid of configurations.** Write a Python script that prints a table of token counts for image sizes `[128, 224, 384]` and patch sizes `[8, 16, 32]`. For each combination, print the image size, patch size, token count, flattened patch dimension, and a flag indicating whether the patch size evenly divides the image. Run the script and verify that a `224×224` image with patch size `16` yields exactly `196` tokens with a flattened dimension of `768`.

2. **Implement patch extraction via `unfold` and verify shape transformations.** Create a synthetic image tensor of shape `(1, 3, 224, 224)` filled with random values. Use `torch.Tensor.unfold` to extract non-overlapping `16×16` patches, then flatten each patch into a vector. Print the shape at each stage: raw image, after unfolding rows, after unfolding columns, and after final flatten. Confirm the output shape is `(1, 196, 768)` before any projection is applied.

3. **Compare `unfold` + `Linear` against `Conv2d`-based patch projection.** Implement both approaches in a single script: (a) unfold patches, flatten, then apply `nn.Linear(768, 768)`, and (b) apply `nn.Conv2d(3, 768, kernel_size=16, stride=16)` then flatten the spatial dimensions. Initialize both layers so their weights are identical (copy the Conv2d weight tensor into the Linear weight tensor reshaped appropriately). Feed the same `224×224×3` input through both paths and print the maximum absolute difference between the two output token sequences. Verify the difference is within floating-point tolerance (`< 1e-5`).

4. **Apply patch projection to a different image domain.** Logo images scraped from company websites are often `64×64×3`. Compute the token count for patch sizes `8` and `16` on this resolution. Then implement a patch projection pipeline that takes a `(4, 3, 64, 64)` batch of logo tensors, projects to a hidden dimension of `384`, prepends a learnable CLS token, and adds sinusoidal positional embeddings. Print the final tensor shape and confirm it matches `(4, tokens + 1, 384)` for each patch size. Identify which patch size fails to divide the image evenly and report the token count discrepancy.

5. **Build a reusable patch embedding module and export a skill document.** Implement a `PatchEmbedding` class that accepts configurable `image_size`, `patch_size`, `in_channels`, and `hidden_dim`. Include input validation that raises a clear error when the patch size does not evenly divide the image. Support both `unfold` and `conv` backends selectable via a constructor flag. Add a CLS token and standard positional embeddings. Instantiate it for a `384×384` screenshot with patch size `16` and hidden dimension `768`, run a forward pass on random input, and print every intermediate shape. Write a summary of the module's interface, supported configurations, and measured tensor shapes to `outputs/skill-patch-embedding.md`.

6. **Design a vision-based enrichment pipeline for processing company website screenshots.** Suppose a GTM team wants to classify landing page screenshots (resized to `224×224`) into layout archetypes (e.g., hero-centered, grid, sidebar) using a ViT encoder. The pipeline must process `10,000` screenshots from an Apollo export. Write a script that computes and prints: total token count per image, estimated attention FLOPs per image (assume `O(tokens² × hidden_dim)`), and total projected inference cost across the full dataset for patch sizes `8`, `16`, and `32`. Identify the patch size that keeps per-image attention cost under `500M` multiply-adds while maximizing spatial resolution. Write the recommendation with supporting numbers to `outputs/skill-patch-sizing-analysis.md`.
