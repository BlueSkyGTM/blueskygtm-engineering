# Exercises — Gradient Accumulation

## Exercises

1. **Compute the effective batch identity for multiple training configurations.** Write a script that takes three `(micro_batch, accum_steps)` pairs — `(8, 4)`, `(16, 2)`, `(32, 1)` — and prints each pair alongside the computed `effective_batch`. Then loop over `accum_steps` values from 1 to 16 with `micro_batch = 8` and print a table showing `effective_batch` for each. Confirm by inspection that every configuration yielding `effective_batch = 128` prints the same value.

2. **Implement loss-per-micro-batch scaling and verify gradient accumulation matches a single backward pass.** Create a tiny MLP (2 inputs → 16 hidden → 1 output) with a fixed random seed. Generate 32 synthetic samples. Run two code paths: (a) one forward-backward on the full batch of 32 with `loss.backward()`, and (b) four forward-backward passes on micro-batches of 8 with `loss / 4` per step, accumulating gradients without calling `zero_grad()` between micro-batches. Print the `.grad` norm of the first linear layer for both paths. They should match to within floating-point tolerance (~1e-6).

3. **Compare per-parameter-group gradient norms between true-batch and accumulated-batch training.** Build a 3-layer MLP on a synthetic classification dataset (512 samples, 20 features, 3 classes). Train one model with `batch_size = 128` for one optimizer step. Train a second model with `micro_batch = 16, accum_steps = 8` for one effective step. After stepping both optimizers, compute and print the L2 norm of the gradient difference for each parameter group (`layer1`, `layer2`, `layer3`, `output`). Report whether the maximum difference across groups stays below `1e-5`.

4. **Diagnose BatchNorm running-statistic divergence under accumulation.** Build a small CNN with two `Conv2d → BatchNorm2d → ReLU` blocks followed by a linear head (MNIST or synthetic 28×28 data). Run one effective step using `micro_batch = 8, accum_steps = 16` and print `running_mean` and `running_var` for each BatchNorm layer after each of the 16 micro-batch forward passes. Repeat with a single true batch of 128. Print the final `running_mean` for both configurations side-by-side and compute the L1 difference. Document whether the statistics converge or diverge and explain in a printed summary why BatchNorm updates per micro-batch rather than per effective step.

5. **Build a complete accumulated training loop with gradient clipping, mixed precision, and logging.** Implement a `train_with_accumulation(model, dataloader, optimizer, accum_steps, max_grad_norm, use_amp)` function that performs: AMP autocast on each forward pass, `GradScaler` for backward, `max_norm` clipping applied only on the final micro-batch before `optimizer.step()`, and per-micro-batch loss logging to stdout. Train a ResNet-18 or comparable model for 3 effective steps on CIFAR-10 (or synthetic image tensors). After each effective step, print the accumulated gradient norm (pre-clip and post-clip) and the mean micro-batch loss. Save the implementation to `handlers/accumulated_trainer.py`. Verify by running `python handlers/accumulated_trainer.py` and confirming that pre-clip norms exceed the `max_grad_norm` threshold on at least one step (proving clipping fired).
