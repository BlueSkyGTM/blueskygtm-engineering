# Exercises — Classical Metrics

## Exercises

1. **Build** a `confusion_matrix(y_true, y_pred)` function that takes two lists of 0/1 labels and returns the tuple `(TP, FP, FN, TN)`. **Compute** precision, recall, F1, specificity, and accuracy from those four cells in a single `metrics_from_cells(tp, fp, fn, tn)` function. Test with `y_true = [1,0,1,1,0,0,1,0]` and `y_pred = [1,0,0,1,0,1,1,0]`. Print every metric rounded to 4 decimals so you can visually verify precision = 0.75 and recall = 0.6.

2. **Apply** the same two functions to a balanced dataset where predictions are mostly correct (e.g., 50 samples, 25 positives, 23 predicted correctly on each side) and to a degenerate case where the model predicts all negatives (`y_pred = [0]*len(y_true)`). Print the confusion matrix and all five metrics for both cases. Confirm that recall and F1 collapse to 0 in the all-negative case while accuracy and specificity remain high — this is the failure mode that motivates multi-metric reporting.

3. **Implement** BLEU-4 from scratch using only `math`, `collections.Counter`, and (optionally) `numpy`. Write `modified_ngram_precision(candidate, reference, n)` for n=1..4, a `brevity_penalty(candidate, reference)` function, and combine them as the geometric mean of the four precisions weighted by the brevity penalty. Test on `candidate = "the cat sat on the mat"` and `reference = "the cat is on the mat"`. Print the four individual n-gram precisions and the final BLEU-4 score rounded to 4 decimals. Verify your output matches `0.5543` (or document the rounding your implementation produces).

4. **Implement** ROUGE-L using longest common subsequence. Write `lcs_length(a, b)` using dynamic programming, then `rouge_l(candidate, reference, beta=1.2)` that computes LCS-based recall and precision, combines them via F-beta, and returns all three values. Test on `candidate = "police killed the gunman"` / `reference = "police kill gunman"` (tokenize on whitespace). Print LCS length, precision, recall, and F-beta. Then re-run with `beta=1.0` and `beta=3.0` on the same pair and print a comparison table showing how F-beta shifts toward recall as beta increases.

5. **Build** a `metrics_report(y_true, y_pred, name="")` function that returns a dictionary containing all five classification scores, the raw confusion matrix cells, and the model/eval name. **Extend** it to accept a list of (name, y_true, y_pred) evaluation tuples and print a side-by-side comparison table to the terminal. Construct an imbalanced dataset (5% positive rate, 200 samples) with two synthetic models: Model A predicts everything as the majority class; Model B predicts the positive class with 70% recall but high false-positive cost. Run the comparison and observe which metrics flag each model's failure mode. Write the full report (the comparison table plus a short interpretation paragraph) to `outputs/skill-classical-metrics.md`.

6. **Design** an end-to-end evaluation pipeline for a lead-scoring classifier trained on a Clay-enriched export. Simulate or load a CSV (`data/clay_leads_sample.csv`) with columns `icp_fit` (ground-truth 0/1) and `model_score` (a probability). Threshold the scores at 0.5 to produce predictions, compute the full metrics report, then sweep thresholds from 0.3 to 0.7 in steps of 0.05 and print precision/recall/F1 at each threshold. Identify the threshold that maximizes F1 and write the final configuration (threshold choice, metric values, confusion matrix, and a one-paragraph justification) to `handlers/lead_scoring_eval.py` as a self-contained, re-runnable script. The script must be executable via `python handlers/lead_scoring_eval.py` and print the full report to stdout.
