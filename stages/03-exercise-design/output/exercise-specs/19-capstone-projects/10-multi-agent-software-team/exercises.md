## Exercises

1. **Implement** a typed message-passing system. Define a `Message` dataclass with a `type` field (`task`, `result`, `critique`, `summary`), a `sender` field, a `recipient` field, and a `payload` dict. Define an `Agent` class with a `name`, a `role` string, and a `process(message) -> Message` method that returns a response. Build a `Router` that reads `message.recipient` and delivers accordingly. Create three agents (orchestrator, coder, reviewer), wire them through the router, and run a three-message exchange where the orchestrator sends a task to the coder, the coder sends a result to the reviewer, and the reviewer sends a critique back to the orchestrator. Print every message to the terminal as it routes, including sender, recipient, type, and payload.

2. **Configure** retry limits and a termination condition on your message bus. Extend the `Router` with a `max_messages` ceiling (default 10) and a `max_retries` limit per task (default 3). Create a scenario where the reviewer keeps rejecting the coder's output and routing it back. Run the system and print the full message trace, then print the termination reason (`max_messages_exceeded`, `max_retries_exceeded`, or `converged`). Verify that a converged scenario terminates early and that a stuck scenario hits a ceiling instead of looping forever.

3. **Compare** orchestrator-worker and debate patterns on the same task: review a Python function that contains a subtle bug (e.g., an off-by-one error in a loop, or an unhandled `None` return). For orchestrator-worker: one orchestrator dispatches the function to a single coder agent who writes a fix, then routes it to a single reviewer agent who approves or rejects — no cross-talk. For debate: two coder agents independently produce fixes, then each reviews the other's fix across two rounds, then a final reviewer picks the better solution. Run both patterns, print each pattern's final output, and print a comparison: which pattern caught the bug, how many messages each consumed, and wall-clock time for each.

4. **Trace** and **Diagnose** handoff failures from message logs. Write a function `diagnose(trace: list[Message]) -> str` that inspects a message trace and returns one of three diagnoses: `infinite_loop` (same payload routed back and forth more than 3 times without modification), `context_overflow` (cumulative payload size exceeds a configurable token threshold — approximate tokens as `len(payload_str) // 4`), or `role_drift` (an agent sends a message type outside its allowed set — e.g., a coder sending a `critique`). Create three trace fixtures as Python lists, one per failure mode, run your diagnostic on each, and print the trace length, payload sizes, and diagnosis for all three.

5. **Build** a multi-agent code generation pipeline with message history summarization and evaluate it against a single-agent baseline. Your pipeline should include an orchestrator, a coder, and a reviewer, with a summarizer step that compresses the message history when it exceeds 200
