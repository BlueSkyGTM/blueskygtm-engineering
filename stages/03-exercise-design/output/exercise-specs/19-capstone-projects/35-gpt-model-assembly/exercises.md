# Exercises — GPT Model Assembly

## Exercises

1. **Implement a complete GPT model** in PyTorch using the five stages from the lesson: token embedding lookup, positional embedding lookup (added element-wise), a `ModuleList` of transformer blocks, a final `LayerNorm`, and a language modeling head (linear projection to vocabulary size). Initialize with the GPT-2 small configuration (`vocab_size=50257`, `hidden_dim=768`, `num_layers=12`, `num_heads=12`, `max_seq_len=1024`). Pass a dummy input tensor of shape `(2, 32)` containing random token IDs through the model and print the output logits shape. Then print the total parameter count using `sum(p.numel() for p in model.parameters())`.

2. **Compute the parameter count by hand** for a smaller GPT configuration: `vocab_size=1000`, `hidden_dim=256`, `num_layers=4`, `num_heads=4`, `max_seq_len=512`. Break down each component (token embeddings, positional embeddings, per-block parameters, final LayerNorm, LM head) and write the arithmetic in a comment block or printed string. Then instantiate that exact model in PyTorch and print the framework's parameter count. Both numbers must match — if they don't, identify which component you miscounted and correct it.

3. **Implement weight tying** between the token embedding matrix and the output language modeling head. Build two versions of the same GPT configuration (`vocab_size=5000`, `hidden_dim=512`, `num_layers=6`, `num_heads=8`): one where the LM head is a separate `nn.Linear` layer, and one where `lm_head.weight` is set to `token_embedding.weight` (tied). Print the parameter count of each version, compute the absolute savings, and print the percentage reduction. Verify that both models accept the same input shape and produce the same output logits shape.

4. **Compare three GPT configurations** that vary one hyperparameter at a time: (a) `num_layers` ∈ {2, 6, 12}, (b) `hidden_dim` ∈ {128, 512, 768}, (c) `num_heads` ∈ {4, 8, 16}. Keep all other settings at a fixed baseline (`vocab_size=1000`, `max_seq_len=256`). For each configuration, print the parameter count, run a forward pass with a dummy batch of shape `(8, 128)`, and measure wall-clock time using `time.perf_counter()`. Print a formatted summary table showing config, params, and latency. State which hyperparameter has the largest impact on parameter count and which has the largest impact on latency.

5. **Build a GPT-based encoder for CRM account enrichment data** and trace a single token through the entire forward pass. Load a sample of company descriptions (use synthetic data mimicking Apollo enrichment exports — fields like `company_name`, `industry`, `description`, `employee_count`). Tokenize the text field using `tiktoken` with the `r50k_base` encoding. Construct a small GPT model (`vocab_size=50257`, `hidden_dim=384`, `num_layers=4`, `num_heads=6`, `max_seq_len=256`) and pass one tokenized account description through the model. At every stage — embedding lookup, positional addition, each transformer block, final LayerNorm, LM head — print the tensor name, shape, `dtype`, and mean activation value. Write the complete trace output (all shapes + a one-paragraph analysis of how the representation changes depth-wise) to `outputs/skill-gpt-gtm-trace.md`.
