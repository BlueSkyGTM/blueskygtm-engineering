# Exercises — Image Retrieval & Metric Learning

## Exercises

1. **Implement contrastive loss from scratch.** Write a function `contrastive_loss(emb_a, emb_b, label, margin=1.0)` where `label=1` means the pair is similar and `label=0` means dissimilar. Use L2 distance between embeddings. Generate 100 random 128-dim embedding pairs with known labels (50 positive, 50 negative) using a fixed random seed, compute the mean loss, and print it. Then shift all negative-pair embeddings closer together by 50% and reprint the loss — the value should increase. Print both numbers to the terminal.

2. **Build a FAISS L2 index from pre-computed embeddings and query it.** Using torchvision's ResNet50, extract pooled feature vectors for every image in a small dataset (e.g., CIFAR-10 test set, first 1000 images, or any local image folder with ≥50 images). Build a `faiss.IndexFlatL2`, add all embeddings, and query with 5 held-out images from the same classes. Print the top-5 nearest neighbors for each query with their L2 distances and class labels. Verify that same-class results appear closer than cross-class results.

3. **Implement triplet loss and trace how it reshapes embedding geometry compared to contrastive loss.** Create a synthetic 2D embedding space with 3 clusters of 20 points each (centers at predefined locations so clusters partially overlap). Run 200 steps of gradient descent (embeddings are the only parameters) using your `triplet_loss(anchor, positive, negative, margin=0.5)` with random triplet sampling. Repeat the same optimization using contrastive loss. After each run, compute and print the mean intra-cluster distance, mean inter-cluster distance, and their ratio. The triplet-optimized space should show a higher ratio.

4. **Compare ResNet50, CLIP, and DINOv2 as retrieval backbones using recall@k.** For the same set of images (≥200 images across ≥5 classes), extract embeddings from all three models. Build three FAISS indexes. For each model, query every image against the index, compute recall@1, recall@5, and recall@10 (fraction of queries where at least one same-class image appears in the top-k). Print a comparison table to the terminal showing all nine metrics across the three backbones.

5. **Build a visual-asset lookalike pipeline for prospect identification.** Given a CSV of 20 prospect company website URLs and a folder of 50 existing customer logo screenshots, embed all logos using CLIP, build a FAISS index of customer embeddings, and query each prospect logo against it. For each prospect, output the top-3 most visually similar customers with cosine similarity scores. Write results to `outputs/prospect-visual-matches.csv` with columns `prospect_domain, matched_customer, similarity_score, rank`. Include a summary line printing how many prospects had at least one match above 0.75 similarity.

6. **Design a metric learning training loop that fine-tunes an embedding backbone and evaluate the retrieval improvement.** Start with a pretrained ResNet18 (frozen backbone, trainable projection head). Train the projection head on a labeled image dataset (e.g., Fashion-MNIST or a local folder with ≥5 classes) using triplet loss with semi-hard mining for 10 epochs. Evaluate recall@5 on a held-out query set before training and after training. Write a comparison report to `outputs/skill-metric-learning-eval.md` containing the before/after recall@5, the triplet loss curve (epoch-by-epoch), and a 3-sentence analysis of which classes improved most and why.
