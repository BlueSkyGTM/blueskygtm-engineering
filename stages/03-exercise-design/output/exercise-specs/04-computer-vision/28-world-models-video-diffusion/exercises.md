# Exercises — World Models & Video Diffusion

## Exercises

1. **Compare world model and video diffusion architectures.** Build a Python script that defines two dictionaries—one for DreamerV3-style world models and one for Sora-style video diffusion—where each key is a component name (`"encoder"`, `"rssm_transition"`, `"reward_head"`, `"action_conditioning"`, `"3d_patch_embedding"`, `"temporal_attention"`, `"spatial_attention"`, `"reverse_diffusion"`, `"decoder"`) and each value is `True` or `False`. Print both dictionaries side by side, then print the set of components present in one but absent in the other. Run the script and confirm the terminal shows the asymmetric components.

2. **Trace the forward pass of a spatiotemporal denoising block.** Implement a `SpatiotemporalDenoiseStep` class with three methods: `patch_embed(x)` that reshapes a `(B, C, T, H, W)` tensor into `(B, num_patches, embed_dim)`, `temporal_attention(h)` that applies attention across the time dimension, and `spatial_attention(h)` that applies attention across the spatial dimension. Use `torch.nn.MultiheadAttention` or a pure NumPy softmax-weighted sum. Feed a random tensor of shape `(1, 3, 4, 16, 16)` through all three steps and print the shape after each stage. Confirm the output shape matches the input patch count.

3. **Implement a minimal latent dynamics model and roll it forward.** Write a `LatentDynamicsModel` with an `encode(obs)` method (linear layer mapping a flattened observation to a latent vector), a `transition(latent, action)` method (one MLP layer producing next latent, plus a stochastic noise term), and a `decode(latent)` method (linear layer mapping latent back to flattened observation). Initialize it, feed a random starting observation, and roll forward 20 steps with random actions. Print the L2 distance between consecutive latents at each step to confirm the trajectory is evolving rather than collapsing to a fixed point.

4. **Classify GTM pipeline tasks by model type.** You are given a JSON file at `inputs/gtm_tasks.json` containing six task descriptions (e.g., "Generate a synthetic walkthrough video of a SaaS onboarding flow for personalized outreach," "Simulate how a lead's engagement state changes after 5 email touches," "Produce a product demo video conditioned on a Clay-enriched persona profile," "Forecast next-quarter pipeline from historical CRM transition data"). For each task, write a function that prints the task name, your recommended model (`world_model` or `video_diffusion`), and a one-line justification referencing a specific architectural property. Read the JSON, iterate, and print the classification table to the terminal.

5. **Design a text-conditioned video generation config and persist a skill spec.** Write a Python script that constructs a pipeline configuration dictionary specifying: the denoiser architecture (3D U-Net or transformer), the conditioning mechanism (cross-attention on CLIP text embeddings), the number of diffusion steps, the temporal frame count, and the scheduler (DDIM or PNDM). Export this config to `outputs/skill-video-gen-config.md` as a human-readable skill spec with sections for Architecture, Conditioning, Inference Parameters, and Coherence Checks. The script must print the full spec to the terminal and write the file. Open the file and confirm it contains all four sections.

6. **Build a world-model decision framework for GTM enrichment.** Create a Python module at `handlers/world_model_selector.py` that exports a function `select_model(task_dict)` which inspects an input task dictionary (keys: `requires_action`, `requires_visual_output`, `temporal_horizon`, `personalization_source`) and returns a recommendation: `"world_model"`, `"video_diffusion"`, `"hybrid"`, or `"neither"`. Encode at least four decision rules grounded in architectural differences. Include a `__main__` block that runs five test cases (pull at least one case's input fields from a Clay or Apollo-style enrichment payload in `inputs/enrichment_sample.json`) and prints each recommendation with the matched rule. Run the module directly and verify the terminal shows five labeled outputs.
