## Exercises

1. **Generate a synthetic point cloud and compute geometric properties.** Write a Python script that samples N=2000 points on the surface of a torus using parametric equations (R=2, r=0.7). Compute and print the centroid, the axis-aligned bounding box (min/max per axis), and the mean distance from each point to the centroid. Verify the bounding box extents match the expected torus dimensions.

2. **Estimate surface normals via local PCA.** Extend your torus point cloud script: for each point, find its 16 nearest neighbors using `scipy.spatial.cKDTree`, compute the local covariance matrix, and extract the smallest eigenvector as the surface normal. Print the mean normal vector across all points and the standard deviation of normal components. For a torus centered at the origin, the mean normal should be close to zero in x and y — verify this is the case and explain any deviation in a printed summary.

3. **Implement a minimal volume rendering loop.** Write a function that casts a bundle of 64×64 rays through a synthetic 3D Gaussian density field centered at the origin (σ=1.0). March each ray in 128 uniform steps between near=2.0 and far=6.0 planes. At each sample point, evaluate the density, convert to alpha via `1 - exp(-density * step_size)`, and composite front-to-back using the standard alpha-blending accumulation formula. Output the rendered image as a 64×64 numpy array and print the min, max, and mean pixel values — you should see a bright Gaussian blob centered in the frame.

4. **Build and downsample a point cloud from Apollo-enriched company data.** Pull company records from Apollo (or load a provided CSV with `latitude`, `longitude`, `employee_count` columns for 500+ companies). Treat each company as a 3D point `(lon, lat, log10(employees))`. Apply voxel downsampling at a resolution of 0.1 degrees in lat/lon and 0.5 in log-employees. Print the original point count, the downsampled point count, and the top-5 densest voxels by company count. This mirrors how GTM teams spatially segment account distributions before territory assignment.

5. **Build a complete ray-marching renderer with positional encoding.** Implement a script at `handlers/volume_renderer.py` that combines ray casting, positional encoding (sinusoidal encoding up to L=6 frequencies), a density query against a hardcoded analytically defined density field (sum of three Gaussians at different positions), and volumetric alpha compositing. Render a 128×128 image, save it as `outputs/rendered_scene.npy`, and print the rendering time, peak density value encountered, and fraction of rays whose final alpha exceeded 0.5.

6. **Design a representation-selection decision framework.** Write a structured Markdown document to `outputs/skill-3d-representation-selection.md` that evaluates point cloud processing versus NeRF reconstruction across five concrete 3D vision scenarios (e.g., autonomous vehicle LiDAR perception, e-commerce product 3D capture, indoor scene reconstruction for real estate, industrial defect detection on manufactured parts, augmented reality product placement). For each scenario, specify the input modality, required output, memory budget, latency constraint, and your recommended representation with a two-sentence justification rooted in the explicit-vs-implicit tradeoffs covered in this lesson. Include a decision tree (as ASCII or a numbered checklist) that a practitioner could follow given a new task.
