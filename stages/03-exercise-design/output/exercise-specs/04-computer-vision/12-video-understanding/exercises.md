## Exercises

1. **Implement uniform frame sampling and temporal mean-pooling.** Create a synthetic video tensor of shape `(64, 3, 224, 224)` representing a 64-frame clip. Write a function `sample_frames(video, num_frames)` that uniformly samples 8 frames and returns a tensor of shape `(8, 3, 224, 224)`. Then write `temporal_pool(frames)` that applies mean-pooling across the temporal dimension and returns a single `(3, 224, 224)` frame embedding. Print the shape at each stage — original, sampled, pooled — to confirm dimensions.

2. **Compute the temporal receptive field for a stack of 3D convolution layers.** Implement a function `temporal_receptive_field(kernel_sizes, strides)` that walks a list of `(k, s)` pairs and returns the cumulative temporal receptive field in frames. Run it on two configurations: (a) four layers of `kernel_size=3, stride=1` and (b) four layers of `kernel_size=3, stride=2`. Print the receptive field for each and state which configuration can cover a 16-frame action at the first layer's output resolution.

3. **Implement and compare three minimal video classifier heads.** Build three PyTorch `nn.Module` classes: `Conv2DPoolClassifier` (2D conv on each frame, global average pool across time, linear head), `Conv3DClassifier` (single 3D conv layer, global pool, linear head), and `TwoStreamClassifier` (RGB stream + temporal-difference stream, each a small 2D conv, outputs concatenated through a linear head). Run a forward pass on a dummy clip of shape `(1, 3, 16, 64, 64)` through each model. Print the parameter count, output logit shape, and a one-line note on whether each architecture learns motion explicitly.

4. **Build a conversation analysis pipeline with temporal segmentation.** You are given a Gong-style sales-call transcript at `data/call_transcript.json` containing a list of turn objects with fields `speaker`, `start_sec`, `end_sec`, and `text`. Implement temporal segmentation: split the call into 30-second windows, flag any window containing objection phrases ("too expensive", "not a priority", "need to think", "budget", "competitor"), and compute the talk-to-listen ratio for the rep versus prospect across the full call. Print a per-window summary table showing window index, dominant speaker, talk ratio, and an `OBJECTION` flag. If no transcript file exists, generate a synthetic one with at least 40 turns and two objection moments so your pipeline has signal to detect.

5. **Design and implement a temporal-axis diagnostic for a GTM dataset.** Write a script that takes a path to a CSV of enrichment or CRM records (e.g., Apollo export, HubSpot activity log, or Clay table) and evaluates whether the data has a temporal axis that warrants sequence modeling. Implement checks for: (a) presence of timestamp or sequence-order columns, (b) whether multiple events exist per entity (account, contact, deal), (c) variance in inter-event timing that a static classifier would collapse. Run the diagnostic on `data/apollo_export.csv` (or a synthetic enrichment dataset you generate with at least 200 rows). Print a structured report to the terminal with a final verdict: `SEQUENCE_MODELING_RECOMMENDED` or `STATIC_CLASSIFICATION_SUFFICES`, along with the evidence scores. Write the full report to `outputs/skill-temporal-evaluation.md`.
