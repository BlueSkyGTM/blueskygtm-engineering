# Exercises — Transfer Learning & Fine-Tuning

## Exercises

1. **Compute** the trainable parameter counts for a frozen vs. unfrozen backbone. Load `distilbert-base-uncased` for sequence classification with 5 labels. Print the total parameter count and the trainable parameter count. Then iterate over the model's parameters and set `requires_grad=False` on every layer whose name does not contain `"classifier"`. Print both counts again. Your terminal must show the trainable ratio dropping below 5% of total parameters after freezing.

2. **Replace** the classifier head of the frozen model from Exercise 1 with a new `nn.Linear` layer sized for a 3-class problem (e.g., "not-a-fit", "watch", "priority"). Generate 200 synthetic training examples (simple string inputs with integer labels), wrap them in a `DataLoader`, and train only the new head for 5 epochs using cross-entropy loss. Print per-epoch loss and accuracy to the terminal. Confirm that the backbone weights never change by printing the hash of `model.distilbert.embeddings.word_embeddings.weight` before and after training — both must be identical.

3. **Implement** LoRA from scratch. Write a `LoRALinear` wrapper class that takes a frozen `nn.Linear` layer, injects two trainable low-rank matrices `A` (rank × in_features) and `B` (out_features × rank) initialized to zero-mean and zero respectively, and computes `frozen_output + B(A(x))`. Wrap the query and value projection layers of a small transformer encoder (2 layers, hidden size 128). Print the trainable parameter count and confirm it is less than 10% of the full model's parameter count. Then train for 3 epochs on a 100-example toy dataset and print the final loss, confirming the LoRA path is actually learning.

4. **Diagnose** three failure modes from logs. Write a script that loads a frozen DistilBERT with a new head and runs three short training loops on the same dataset, each with a deliberately broken configuration: (a) learning rate `1e-2` on an unfrozen backbone, (b) only the final 1-layer classifier trainable with a frozen embedding layer producing near-constant outputs, and (c) a model pretrained on sentiment classification fine-tuned on a dataset of raw HTTP request logs. For each run, print the per-epoch loss curve and final accuracy. Label each run in the terminal output with the failure mode name — catastrophic forgetting, feature starvation, or distribution mismatch — and a one-sentence fix for each.

5. **Build** a custom ICP tier classifier that outputs tier labels consumable by Clay waterfall enrichment. Train a frozen-backbone classifier on a labeled dataset of company descriptions (use the dataset from Exercise 1 or generate 300 synthetic rows with fields: company_name, industry, employee_count, description, and a tier label of "tier_1" / "tier_2" / "reject"). Serialize the trained model and write a Python script at `handlers/icp_tier_classifier.py` that loads the model, accepts a JSON array of company records on stdin, and prints a JSON array where each record is annotated with `"icp_tier"` and `"confidence"`. The tier values must match the string format Clay expects for conditional branching (`tier_1`, `tier_2`, `reject`). Verify by piping 5 sample companies through the script and confirming the JSON output parses and contains tier annotations.

6. **Design** and run a comparison experiment across all three adaptation strategies on a real CRM export or a dataset of 500+ Apollo lead records. For each strategy — feature extraction, full fine-tuning, and LoRA — train on an 80/20 split, report final accuracy, trainable parameter count, wall-clock training time, and inference latency per 1000 rows. Write the full results table, your strategy recommendation for a team with a $50/month compute budget and 200 labeled examples, and the rationale to `outputs/skill-transfer-strategy.md`. The markdown file must contain a comparison table with all four metrics and a recommendation section that references the specific dataset-size and domain-distance thresholds from the lesson.
