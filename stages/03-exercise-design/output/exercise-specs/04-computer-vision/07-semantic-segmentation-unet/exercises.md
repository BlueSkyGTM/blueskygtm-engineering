## Exercises

1. **Implement a single encoder block and verify its shape contract.** Write a PyTorch `nn.Module` called `EncoderBlock` that takes an `in_channels`, `out_channels`, and applies two 3×3 convolutions (each followed by ReLU), then a 2×2 max-pool with stride 2. Instantiate it with `in_channels=1, out_channels=64`, pass a dummy tensor of shape `(1, 1, 572, 572)` through it, and `print` both the input and output tensor shapes. Then instantiate a second block with `in_channels=64, out_channels=128`, feed the first block's output into it, and print the resulting shape. Confirm the spatial dimensions are halved at each stage and channels double.

2. **Implement Dice loss and compare it against cross-entropy on a boundary case.** Write a `dice_loss(pred, target, num_classes, smooth=1e-6)` function that accepts logits of shape `(B, C, H, W)` and a target of shape `(B, H, W)` with integer class labels. Apply softmax, compute per-class intersection and union, and return `1 - mean_dice`. Create a synthetic batch where the prediction is perfect (copy the target as one-hot logits) and a second batch where one small object (a 5×5 region) is entirely missed. Print the Dice loss and `nn.CrossEntropyLoss` for both cases. Observe which loss penalizes the missed small object more heavily.

3. **Build a complete U-Net and trace every intermediate tensor shape.** Implement a `UNet` class with four encoder blocks (channels 64→128→256→512), a bottleneck (1024 channels), and a decoder with transposed-convolution upsampling blocks that concatenate skip-connection feature maps at matching resolutions. Use `torch.no_grad()` and place `print` statements inside `forward()` at each stage showing the tensor shape after every convolution, pooling, transpose-conv, and concatenation. Run a `(1, 3, 256, 256)` input through the network and verify that the decoder restores the input spatial resolution by the final layer.

4. **Compute per-class IoU and Dice metrics on model predictions and diagnose failure modes.** Generate a synthetic segmentation dataset: a 128×128 image with three classes (background=0, circle=1, thin rectangle=2) where the rectangle occupies fewer than 50 pixels. Train your U-Net from Exercise 3 for 20 epochs using a combined loss (`0.5 * CE + 0.5 * Dice`). After training, compute and print per-class IoU and Dice scores. Identify which class has the lowest score, then visualize (by printing a text-based confusion count per class) whether the failure is due to small-object recall, boundary precision, or systematic class confusion. Print a one-line diagnosis string stating the root cause.

5. **Build a document-region segmentation handler for automated invoice processing.** Implement a pipeline that takes a folder of scanned invoice images (`data/invoices/*.png`, 512×512 grayscale), runs them through a U-Net trained to segment four regions (background, header, line_items, totals), and writes a CSV to `outputs/segmented_regions.csv` with columns `filename, header_pixels, line_item_pixels, totals_pixels, mean_confidence`. If no trained checkpoint exists, initialize the model with random weights and log a warning. Place the runnable entry point in `handlers/invoice_segmenter.py` so that `python handlers/invoice_segmenter.py --input data/invoices --output outputs/segmented_regions.csv` executes end-to-end and prints per-file region summaries to the terminal.

6. **Design a segmentation task selection document for a GTM analytics team.** Write a technical spec to `outputs/skill-segmentation-task-selection.md` that defines three pixel-level prediction problems a revenue operations team might face: (a) detecting and segmenting competitor logos in screenshots scraped via Apollo's company-search API, (b) separating individual text fields on scanned business cards for CRM ingestion, and (c) classifying every pixel in a product-shelf photo as shelf-edge, product, or empty-space for retail intelligence. For each problem, state whether semantic, instance, or panoptic segmentation is the correct task, justify the choice in 2–3 sentences referencing what the downstream consumer needs, and name the evaluation metric you would optimize. Include a final table comparing all three problems across task type, metric, and data source.
