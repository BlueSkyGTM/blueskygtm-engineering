# Exercises — Vision Transformers (ViT)

## Exercises

1. **Compute token counts for multiple resolutions and patch sizes.** Write a Python script that, given a list of `(image_size, patch_size)` pairs — e.g. `(224, 16), (224, 32), (384, 16), (512, 16)` — computes the number of patches, the sequence length after prepending `[CLS]`, and the flattened patch dimension. Print a formatted table to the terminal showing each configuration and whether the resulting sequence length exceeds 1024.

2. **Run pretrained ViT inference and extract the `[CLS]` embedding.** Using `transformers.AutoModel` with `google/vit-base-patch16-224`, load a sample image from `PIL` or `torchvision.datasets`, run a forward pass, and extract the last hidden state for the `[CLS]` token (index 0). Print the embedding shape and the first 10 values. Also print the top-3 predicted classes using `AutoModelForImageClassification` on the same image.

3. **Implement patch embedding from scratch using `torch.nn.Conv2d`.** Build a `PatchEmbedding` module that takes a `(B, 3, H, W)` tensor, applies a convolution with the given patch size and stride, and flattens the output to `(B, N, D)`. Instantiate it with `patch_size=16`, `embed_dim=768`, pass a dummy `(1, 3, 224, 224)` tensor through it, and assert that the output shape is `(1, 196, 768)`. Print the output shape and the number of learnable parameters in the module.

4. **Build the full token sequence with `CLS` and positional embeddings.** Extend your `PatchEmbedding` into a `ViTStem` module that prepends a learnable `[CLS]` token and adds learnable positional embeddings. Initialize it for 224×224 images with patch size 16 and `embed_dim=384`. Pass a random tensor through and verify the output is `(1, 197, 384)`. Print every tensor shape in the pipeline: raw image → conv output → flattened tokens → CLS-prepended → positionally-embedded. Also print the total parameter count.

5. **Build a landing-page screenshot classifier for lead enrichment triage.** Download 8–10 website screenshots (or generate synthetic ones) and use a pretrained ViT's `[CLS]` embeddings as features for a lightweight `sklearn` classifier that predicts a category label such as `SaaS`, `Ecommerce`, `Agency`, or `Blog`. Split into train/test, fit a `LogisticRegression`, and report accuracy. Save your trained pipeline and inference code as an artifact at `handlers/vit_landing_classifier.py` that exposes a `predict_category(image_path: str) -> str` function. Print the classification report and a per-image prediction table.

6. **Write a ViT vs. CNN tradeoff analysis for a GTM engineering team.** Choose a concrete scenario — classifying B2B email screenshots into `invoice / receipt / newsletter / personal` for CRM auto-routing, where only 200 labeled examples are available and inference must run under 50ms on CPU. Write a one-page analysis to `outputs/skill-vit-vs-cnn.md` comparing ViT, a small CNN (ResNet-18), and Swin Transformer across: data efficiency at small scale, latency at edge deployment, inductive bias implications, and enrichment quality. Recommend one architecture with justification and include a table comparing all three on each axis.
