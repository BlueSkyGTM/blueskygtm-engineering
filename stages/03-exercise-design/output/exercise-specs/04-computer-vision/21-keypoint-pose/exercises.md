# Exercises — Keypoint Detection & Pose Estimation

## Exercises

1. **Implement Gaussian heatmap generation and argmax coordinate extraction.** Write a function that takes a keypoint center `(cx, cy)`, standard deviation `sigma`, and image dimensions `(H, W)`, then produces a 2D Gaussian heatmap as a NumPy array. Flatten the heatmap, apply `np.argmax`, and convert the flat index back to `(x, y)` coordinates. Run your function for three keypoints at `(50, 80)`, `(120, 200)`, and `(310, 60)` with `sigma=5.0` on a `480×640` canvas. Print each ground-truth center alongside its argmax-recovered coordinate and the pixel error between them.

2. **Implement confidence thresholding across multiple cutoff values.** Create a list of 17 COCO-format keypoints, each as `(x, y, confidence)`, where confidence values range from 0.05 to 0.98 (simulate realistic variation—some keypoints near 0.1, others above 0.8). Write a function that filters keypoints below a given threshold and returns both the passing keypoints and a count of rejected ones. Run the filter at thresholds `0.2`, `0.5`, and `0.7`. Print a table showing threshold, surviving keypoint count, and the names of the filtered-out keypoints at each level.

3. **Implement simplified Part Affinity Field (PAF) grouping logic.** You are given two lists: (a) detected keypoints for a 2-person scene where each keypoint has `(x, y, person_id_unknown)`, and (b) PAF affinity scores for each candidate connection between joint pairs (e.g., `[(nose_idx, neck_idx, affinity_score), ...]`). Write a greedy assignment algorithm that uses affinity scores to connect keypoints into person instances: for each connection type, sort candidate pairs by affinity score descending, assign greedily while ensuring each keypoint is used at most once per person. Output the final person grouping as a JSON structure mapping `person_id → {keypoint_name: (x, y)}`. Print the grouping and verify that each person has a connected skeleton with no cross-person limb assignments.

4. **Compare top-down versus bottom-up pipeline characteristics on a multi-person scenario.** Write a script that simulates detection results for a crowded frame containing 8 people. For the top-down path, model the cost as: person detection (50ms) + per-person pose regression (30ms × 8 people). For the bottom-up path, model the cost as: shared keypoint detection (120ms) + PAF grouping (40ms). Print a latency breakdown for each approach at person counts of 1, 2, 4, 8, and 16. Then write a function `recommend_pipeline(person_count, latency_budget_ms)` that returns `"top_down"` or `"bottom_up"` based on which approach meets the budget. Print the recommendation for budgets of `[100, 200, 400]` ms at each person count.
