## Exercises

1. **Export a PyTorch vision model to ONNX and benchmark inference latency.** Load `torchvision.models.mobilenet_v2(pretrained=True)` (or any small classifier from the lesson), export it to ONNX with a 1×3×224×224 input, then run 100 warm-free iterations through `onnxruntime.InferenceSession`. Print the mean latency (ms), standard deviation (ms), and achieved FPS. Your terminal should output three labeled numbers.

2. **Apply dynamic quantization to the exported ONNX graph and compare against the FP32 baseline.** Use `onnxruntime.quantization.quantize_dynamic` to produce an INT8-quantized ONNX file, re-run the 100-iteration benchmark from Exercise 1 on the quantized model, and print a side-by-side comparison: FP32 mean, INT8 mean, absolute delta (ms), and speedup ratio (×).
3. **Trace the full edge inference pipeline and report per-stage budget share.** Build a script that times each stage independently—preprocess (resize + normalize a 1920×1080 raw frame to 224×224 tensor), forward pass (ONNX Runtime `run`), and postprocess (softmax + argmax over the logits)—across 100 frames. Print a table with each stage's mean latency, its percentage of total frame budget, and the stage name that consumes the largest share.
4. **Compute hardware feasibility for a target deployment.** Given a model with 3.5 M parameters at INT8 (1 byte each), 150 MACs per inference, and a target of 20 FPS on a Jetson Nano (memory bandwidth: 25.6 GB/s, compute throughput: 0.5 TFLOPS INT8), write a script that calculates memory bytes transferred per frame, memory bandwidth required, compute FLOPs required per second, and prints whether the platform passes both constraints or which constraint it violates.
5. **Implement a configurable edge frame-processing loop with thermal throttle and dropped-frame reporting.** Build a loop that accepts a `target_fps` argument (e.g., 15, 30), runs simulated inference using the ONNX model from Exercise 1 for 300 frames, introduces a thermal-throttle factor that increases per-frame latency by 15% every 100 frames (simulating junction-temperature rise), and logs any frame that misses its deadline (1/target_fps seconds) as "DROPPED." At the end, print total frames processed, total dropped, effective FPS, and a per-100-frame throttle summary. Save the implementation to `handlers/edge_frame_loop.py` so it can be imported as a module and called with different FPS targets.
6. **Design an edge deployment plan for a GTM enrichment pipeline that classifies prospect company website screenshots at the edge.** You are processing screenshots pulled from a Clay enrichment table at a rate of 50 companies/minute. Each screenshot is a 1920×1080 PNG. Choose a model (MobileNetV2, EfficientNet-Lite, or ResNet-18), choose a quantization strategy, calculate the required FPS and whether a single Jetson Orin Nano (memory bandwidth: 102.4 GB/s, compute: 40 TOPS INT8) can sustain the workload, determine how many devices you need if thermal throttling reduces throughput by 20% after 10 minutes, and write the full plan—specifying model, quantization level, measured or projected latency per frame, device count, and bottleneck stage—to `outputs/skill-edge-deployment-plan.md`. Include a table of the throughput math.
