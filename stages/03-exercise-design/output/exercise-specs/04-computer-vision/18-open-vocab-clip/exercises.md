## Exercises

1. **Implement zero-shot classification on a single image.** Load the CLIP model (`ViT-B/32`) from `open_clip` or OpenAI's `clip` package. Download a product screenshot or logo image of your choice. Construct a label set of at least 6 categories relevant to a GTM context (e.g., `"a screenshot of a SaaS dashboard"`, `"a photo of a business card"`, `"a logo of a technology company"`, `"a photo of an office building"`, `"a screenshot of an email client"`, `"a photo of a product packaging"`). Encode the image and all text labels through CLIP's dual encoders, compute cosine similarity between the image embedding and each text embedding, and print the predicted label along with the raw similarity scores for every class sorted descending. Verify the top result is reasonable by eye.

2. **Compute InfoNCE contrastive loss on a toy batch.** Create a toy batch of 4 image embeddings and 4 text embeddings as random tensors of shape `[4, 512]` (simulating CLIP's output dimensionality). Designate indices 0 and 1 as matched pairs and indices 2 and 3 as deliberately mismatched (swap their text counterparts). Implement the symmetric InfoNCE loss from scratch using only `torch.nn.functional` operations (no pre-built loss functions). Print the loss value for the correctly matched batch configuration and for the swapped configuration. Confirm that the matched-pair loss is lower and print both values with at least 4 decimal places so the difference is observable.

3. **Compare prompt template effects on classification accuracy.** Select a custom taxonomy of 8–10 visual concepts that a GTM team might need to classify (e.g., advertisement types, document types, UI component screenshots). Use a small set of 8–12 images — either from a public dataset like CIFAR-10 subsets or images you collect yourself. Implement three different prompt templates: `"a photo of a {label}"`, `"a screenshot of a {label}"`, and `"a {label}"`. Run zero-shot classification with each template across all images, compute accuracy against your ground-truth labels, and print a comparison table showing per-template accuracy. Report which template performs best and by what margin.

4. **Build a batch inference pipeline for a directory of images.** Write a Python script that accepts a directory path and a JSON file containing a label list. The script loads every `.jpg`, `.png`, and `.webp` image in the directory, encodes them in batches of 16 through CLIP's vision encoder, encodes the label set once through the text encoder, computes cosine similarity for all images, and writes results to `outputs/clip_predictions.csv` with columns: `filename`, `predicted_label`, `confidence` (top similarity score), and `top_3_labels` (semicolon-separated). Run the pipeline on a directory of at least 10 images and print a summary showing the distribution of predicted labels.

5. **Evaluate CLIP's zero-shot limitations on domain-specific GTM imagery.** Build a script that tests CLIP on two categories of images: (a) 5 in-distribution images — standard product screenshots or website headers that CLIP was likely trained on, and (b) 5 out-of-distribution images — niche B2B-specific visuals such as CRM pipeline views, Clay enrichment tables, Salesforce dashboards, or specialized schematics. For each image, compute the maximum similarity score across your label set and the entropy of the full similarity distribution (after softmax with temperature 0.01). Write a markdown report to `outputs/clip-domain-eval.md` containing a table of all 10 images with their category (in-dist vs out-of-dist), max similarity, entropy, predicted label, and ground-truth label. Conclude the report with a paragraph summarizing where CLIP breaks down on domain-specific GTM content and what score/entropy thresholds could flag low-confidence predictions.

6. **Design and implement a CLIP-powered logo-to-industry classifier for enrichment workflows.** Build a script in `handlers/logo_classifier.py` that takes a CSV of company website URLs (with a `website` column), downloads each company's favicon or homepage screenshot using a headless browser or favicon API, runs CLIP zero-shot classification against an industry taxonomy of at least 12 labels (e.g., `"fintech company logo"`, `"healthcare company logo"`, `"logistics company logo"`), and writes enriched results to `outputs/logo_industries.csv` with columns `website`, `predicted_industry`, `confidence`, and `alternative_industry`. Process at least 8 real company websites. Print a summary of industry distribution and flag any predictions below a confidence threshold of 0.20 as `"low_confidence"` for human review.
