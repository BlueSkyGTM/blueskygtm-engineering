# Exercises — 3D Gaussian Splatting from Scratch

## Exercises

1. **Implement a 2D Gaussian alpha compositor.** Write a Python script that defines 5–10 Gaussians in 2D, each with a center `(x, y)`, a 2×2 covariance matrix, an opacity `α`, and an RGB color. Evaluate each Gaussian on a 64×64 pixel grid, sort by a manually assigned depth value, and alpha-composite front-to-back using the standard over-operator: `C_out += α_i · (1 − α_accum) · c_i`. Print the composited image as a 64×64 grid of ASCII brightness characters to the terminal so you can visually confirm overlaps, ordering, and opacity behavior.

2. **Compute per-Gaussian storage and verify the parameter budget.** Write a script that enumerates the five parameter groups (position: 3 floats, rotation quaternion: 4 floats, scale: 3 floats, opacity: 1 float, SH color degree-3: 48 floats), prints a table with each group's name, float count, and byte count (float32). Then compute and print total storage in MB for scenes of 100K, 500K, and 1M Gaussians. Add a second mode that computes storage under SH-band reduction (degree 1 = 12 floats/channel-set instead of 48) and opacity pruning (drop all Gaussians below α = 0.01 from a provided count). Print both tables side by side.

3. **Build the 3D-to-2D projection and depth-sorted splat pipeline.** Generate 500 random 3D Gaussians with positions sampled in a unit cube, random scale vectors, identity rotations, and random colors. Define a perspective camera with specified intrinsics and extrinsics. For each Gaussian, project its center to screen space, approximate its 2D footprint by projecting the covariance (or use a fixed screen-space radius as a simplification), then sort all Gaussians by their camera-space z-depth and alpha-composite onto a 128×128 canvas. Save the result as `outputs/projected-splat.png` and print the min/max/median depth values used for sorting. Verify visually that nearer Gaussians occlude farther ones.

4. **Apply Gaussian splatting to GTM territory density visualization.** Load a Clay or Apollo export CSV containing company records with `latitude`, `longitude`, and `employee_count` columns (use `signals/examples/` sample data if no live export is available). Treat each company as a 2D Gaussian on an equirectangular map projection: center = projected `(lon, lat)`, covariance scale proportional to `log(employee_count)`, opacity = 0.3, color mapped from industry or revenue tier. Render the composited density field to a 256×128 grid and save as `outputs/territory-density.png`. Print the top-5 highest-density pixel regions with their approximate geographic coordinates and the companies contributing to each. This tests whether you can transfer the splatting rasterizer to an unconventional spatial dataset.
