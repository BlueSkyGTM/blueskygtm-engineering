# Exercises — Diffusion Transformers & Rectified Flow

## Exercises

1. **Trace the architectural substitutions** from U-Net DDPM → DiT → rectified flow. Write a Python script (`trace.py`) that prints, in order, every component replaced (e.g., "conv encoder-decoder → patch-embed + transformer blocks", "convolutional downsampling → token sequence at fixed resolution", "DDPM noise schedule + curved SDE → rectified-flow straight-line ODE"). For each substitution, print what was removed, what replaced it, and one sentence on why. Run the script and confirm the table prints with at least five substitutions covered.

2. **Implement a minimal DiT block** with patch embedding, sinusoidal timestep embeddings, adaptive layer-norm (adaLN-Zero) conditioning, and self-attention. Constraints: ≤ 100 lines of PyTorch, single file (`dit_block.py`), no external model libraries beyond `torch` and `torch.nn`. Instantiate the block with `hidden_dim=384`, `num_heads=6`, `patch_size=4`, and a 32×32 input. Run a forward pass on random input plus a timestep tensor, print the output shape, and confirm it matches `(1, 64, 384)`.

3. **Implement a rectified-flow training loop** that samples straight-line trajectories between data and noise. Use a 2D toy dataset — two Gaussian blobs centered at `(-3, -3)` and `(3, 3)` with 2000 samples each. For each training step, draw `t ~ Uniform(0, 1)`, interpolate `x_t = (1 - t) * x_data + t * noise`, and train a small MLP to predict the velocity `(noise - x_data)`. After 3000 steps, plot 10 sampled straight-line trajectories from `t=0` to `t=1` overlaid on the data — they should visibly connect blob centers to noise.

4. **Compare curved (DDPM) vs. straight (rectified-flow) sampling** on the same 2D dataset from Exercise 3. Implement a DDPM sampler that follows the standard discrete reverse process with cosine schedule and an Euler-step rectified-flow sampler using the model from Exercise 3. For each method, sweep Euler step counts `[1, 2, 5, 10, 50]` and measure the KL divergence between 1000 generated samples and the true data distribution. Print a comparison table showing steps vs. KL for both methods. Confirm rectified flow reaches convergence (KL < 0.05) in far fewer steps.

5. **Evaluate inference optimizations** on the DiT block from Exercise 2 operating at batch size 16, hidden dim 768, 12 layers, input resolution 64×64. Profile three configurations: (a) eager baseline, (b) `torch.compile(mode="reduce-overhead")`, (c) FlashAttention-2 via `F.scaled_dot_product_attention` (or `xformers` if available). For each, measure peak VRAM (`torch.cuda.max_memory_allocated`) and median latency over 20 forward passes. Run the same sweep at rectified-flow step counts `[1, 4, 10]` to quantify step-count reduction. Write results to `outputs/skill-dit-inference-benchmarks.md` as a markdown table with one row per (config, step-count) combination.

6. **Build a synthetic-enrichment data generator** that uses rectified flow to produce realistic company enrichment records for GTM pipeline testing. Train a small flow-matching model (MLP or mini-DiT) on a dataset of real company profiles pulled from Apollo's API (or the Apollo CSV export from `data/apollo_companies.csv`) with fields like employee_count, funding_total, founded_year, industry_code. Generate 1000 synthetic company records by integrating the learned velocity field over 4 Euler steps from noise. Verify output realism by computing column-wise Wasserstein distance between real and synthetic distributions (should be < 15% of each column's range). Persist the generator as a callable module in `handlers/synthetic_enrichment_flow.py` with a `generate(n: int) -> list[dict]` entrypoint.
