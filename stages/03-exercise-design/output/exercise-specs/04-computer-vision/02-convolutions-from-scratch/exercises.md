# Exercises — Convolutions from Scratch

## Exercises

1. **Implement pure-Python 2D convolution.** Write a function `conv2d_python(signal, kernel, stride=1)` that slides the kernel across the signal, computes element-wise products, sums them, and returns the output map as a list of lists. Test it on a 5×5 signal of zeros with a 3×3 kernel of ones and a single center value of 9. Print the output map to the terminal and confirm the center of the output is 9 and all other entries are 0.

2. **Reimplement with NumPy and verify equivalence.** Rewrite the convolution using NumPy array slicing (no Python loops over individual elements — vectorize each window placement). Run both implementations on a random 8×8 signal with a 3×3 random kernel. Print both outputs and print `np.allclose(python_output, numpy_output)` to confirm they are identical.

3. **Compute output dimensions for a sweep of architectures.** Write a function `output_dim(W, K, P, S)` that returns `(W - K + 2*P) // S + 1` or raises a `ValueError` if the kernel does not fit evenly. Call it for the following layer configs and print a table to the terminal: `(W=224, K=3, P=0, S=1)`, `(W=224, K=3, P=1, S=1)`, `(W=224, K=5, P=2, S=1)`, `(W=224, K=3, P=0, S=2)`, `(W=32, K=3, P=1, S=2)`. Verify the second config preserves the input dimension.

4. **Design and apply edge-detection and sharpening kernels.** Using your NumPy convolution function, define three 3×3 kernels: (a) a Sobel horizontal edge detector, (b) a Laplacian sharpening kernel with a center value of 5 and surrounding values summing to −4, (c) a first-derivative estimator `[[-1, 0, 1]]` (1×3). Apply each to a synthetic 10×10 image that has a sharp vertical edge (left half = 0, right half = 10). Print each output map. For each kernel, state in a comment whether the output matches your prediction for where the activation peaks.

5. **Build a 1D rising-signal detector for GTM engagement data.** Implement 1D convolution over a list of weekly engagement scores per account (e.g., `[0, 0, 2, 3, 5, 8, 12, 20]`). Design a kernel that produces a high positive activation when the signal is accelerating upward (e.g., `[-1, -1, 1, 2]` or similar — experiment). Write a script that loads engagement data from `outputs/engagement_timeseries.csv` (columns: `account_id`, `week_1` … `week_8`), applies your kernel to each account's time series, and prints the top 5 accounts by peak activation score. Save the ranked output to `signals/examples/rising_signal_detector.py` as a self-contained, runnable script.

6. **Compare convolution vs. cross-correlation empirically.** Implement both true convolution (flip the kernel horizontally and vertically before sliding) and cross-correlation (slide without flipping). Apply both to an asymmetric 5×5 signal using an asymmetric 3×3 kernel. Print both output maps side by side. Then check what `scipy.signal.correlate2d` and `scipy.signal.convolve2d` produce, and print `np.allclose` comparisons against your implementations. State in a printed summary which operation deep learning libraries (PyTorch `nn.Conv2d`, TensorFlow `tf.nn.conv2d`) actually perform under the name "convolution." Save your script and the printed comparison output to `outputs/skill-conv-vs-correlation.md`.
