## Exercises

1. **Implement the NT-Xent loss from scratch and verify it on a synthetic batch.** Create a 4×128 tensor of L2-normalized embeddings where rows 0–1 are a positive pair (cosine similarity ~0.9) and rows 2–3 are a second positive pair (cosine similarity ~0.9), with cross-pair similarities near 0. Write the NT-Xent computation by hand — cosine similarity matrix, temperature-scaled logits, symmetric cross-entropy — and print the loss value, the per-pair gradient direction (toward convergence or divergence), and confirm positives are pulled together while negatives are pushed apart. Vary the temperature across `[0.07, 0.1, 0.5, 1.0]` and print the loss at each setting to observe how temperature changes the sharpness of the contrastive signal.

2. **Trace the positive-pair convergence mechanism on a 2D embedding space.** Generate eight 2D points: two positive pairs (each pair starts close) and four negatives scattered far away. Implement a single gradient-descent step using your NT-Xent loss from Exercise 1 with `temperature=0.1` and `lr=0.5`. Print the cosine-similarity matrix before and after the step. Verify that positive-pair similarities increased and negative-pair similarities decreased. Then repeat with only 2 points total (one positive pair, zero negatives) and print the loss — confirm that without negatives, the contrastive signal collapses and the loss is near zero regardless of representation quality.

3. **Build a patch masking and reconstruction pipeline and measure reconstruction error across mask ratios.** Load a grayscale image (any 224×224 image from `torchvision.datasets` or a local file), split it into 16×16 patches (196 total), randomly mask patch ratios of `[0.3, 0.5, 0.6, 0.75, 0.9]`, and reconstruct the masked patches using pixel-mean fill (a naive baseline reconstruction). Compute and print the mean squared error between original and reconstructed patches for each mask ratio. Plot MSE vs. mask ratio and save the figure to `outputs/mask_ratio_mse.png`. Comment in a printed summary on which mask ratio gives the steepest error increase and why high mask ratios (75%) are preferred by MAE despite higher reconstruction difficulty.

4. **Compare all three collapse-avoidance mechanisms on the same embedding tensor.** Create a 256×64 tensor of embeddings drawn from a normal distribution. Implement three collapse checks: (a) SimCLR-style — compute the fraction of negative pairs with cosine similarity below a threshold of 0.2; (b) DINO-style — apply EMA centering (subtract a running mean updated with momentum 0.996) and temperature sharpening (`softmax(z / 0.04`)), then measure the entropy of the output distribution (collapse produces near-zero entropy); (c) MAE-style — randomly mask 75% of embedding dimensions and measure reconstruction MSE using a linear decoder. Print all three metrics. Then artificially collapse the embeddings by setting every row equal to the same vector and re-run all three checks. Confirm each mechanism detects collapse and print which metric moved the most.

5. **Evaluate a pretrained self-supervised backbone with linear probing on CIFAR-10.** Load DINO or SimCLR pretrained weights from `torch.hub` (e.g., `facebookresearch/dino:dino_vits16`), extract embeddings for the CIFAR-10 train and test splits, train a single linear layer on the frozen train embeddings using cross-entropy loss, and report test accuracy. Repeat with an ImageNet-pretrained supervised ResNet-18 backbone from `torchvision.models` for comparison. Print a table with backbone name, embedding dimensionality, linear-probe accuracy, and the delta between the two backbones. Write a short analysis (3–5 bullet points) covering which backbone wins, by how much, and what the gap implies about self-supervised representation quality. Save the analysis and results table to `outputs/skill-ssl-linear-probe.md`.

6. **Deploy a visual similarity-scoring pipeline that enriches GTM prospect records using logo images.** Build a handler that accepts a directory of company logo images (use any public logo dataset or scrape from Clearbit Logo API `https://logo.clearbit.com/{domain}`), embeds each logo using a DINO or SimCLR backbone from `torch.hub`, and computes pairwise cosine similarity. For each company, output the top-3 most visually similar companies (useful for detecting brand siblings, competitor clusters, or design-agency overlap). Integrate with a Clay or Apollo prospect export CSV (`company_name,domain,industry,logo_url`) and append a `visual_cluster_id` and `top_similar_companies` column. Run the pipeline on a sample of 20 prospects and write the enriched CSV to `outputs/logo_similarity_enriched.csv`. Save the handler code to `handlers/visual_logo_similarity.py` and verify the script runs end-to-end, printing a summary of detected visual clusters and any cross-industry brand-similarity surprises.
