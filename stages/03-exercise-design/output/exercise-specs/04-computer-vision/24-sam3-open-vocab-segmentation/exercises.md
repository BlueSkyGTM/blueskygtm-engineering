# Exercises — SAM 3 & Open-Vocabulary Segmentation

## Exercises

1. **Run the lesson's SAM+CLIP pipeline on the provided sample image located at `data/samples/landing_page.png`.** Print each detected mask's ID, bounding box coordinates, and top-3 CLIP label predictions with cosine similarity scores to the terminal. Verify that at least two masks receive distinct top labels.

2. **Trace how the frozen CLIP encoder grounds text to regions by varying the prompt set.** Replace the default label list with `["navigation bar", "pricing table", "hero headline", "footer", "product screenshot"]` and re-run the pipeline on the same image. Print a before-and-after comparison showing how each mask's top label changes when the candidate vocabulary shifts. Confirm at least one mask flips its top prediction.

3. **Build a standalone open-vocabulary segmentation script that accepts an image path and a comma-separated list of labels as CLI arguments.** The script must load the image, run SAM mask generation, crop each masked region, encode it through CLIP, compute cosine similarity against all provided labels, and write a JSON file to `outputs/segmentation_results.json` containing one entry per mask with ranked label scores. Run it against `data/samples/dashboard.png` with labels `["chart", "sidebar", "data table", "button", "user avatar"]` and print a summary table to stdout.

4. **Compare class-constrained detection against the open-vocabulary pipeline on the same input image.** Run a pretrained Mask R-CNN or YOLOv8 segmentation model on `data/samples/landing_page.png` and capture detected classes, mask count, and total inference time. Then run the SAM+CLIP pipeline on the same image with an equivalent label vocabulary. Print a side-by-side table to the terminal covering: number of masks produced, unique labels detected, label flexibility (can you add a new class without retraining?), and average latency per image. State which approach captures more semantically meaningful regions for this particular image and justify in one paragraph written to stdout.

5. **Build a screenshot decomposition enrichment handler for a Zone 4 GTM pipeline.** The handler reads a CSV of competitor company data exported from Clay or Apollo (columns: `company_name`, `website`, `screenshot_url`). For each row, download the landing page screenshot, run the SAM+CLIP pipeline with GTM-relevant prompts (`["pricing card", "free trial CTA", "logo", "testimonial quote", "feature grid", "contact form"]`), and append columns `detected_elements` (comma-separated top label per mask) and `has_pricing_card` (boolean). Write the enriched CSV to `outputs/enriched_competitors.csv` and save the handler code to `handlers/screenshot_enrichment.py`. Print a summary showing how many companies had a detectable pricing card.
