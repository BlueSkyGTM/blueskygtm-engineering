## Exercises

1. **Trace the five pipeline components and print their inference role.** Load `runwayml/stable-diffusion-v1-5` with `StableDiffusionPipeline.from_pretrained`. For each of the five components (`vae`, `text_encoder`, `unet`, `scheduler`, `safety_checker`), print the component class name, its input tensor shape, its output tensor shape, and a one-line description of what it modifies during inference. Run a single generation call with `torch.inference_mode()` and confirm all five components executed by checking their `requires_grad` status (all should be `False`).

2. **Compute the 48× latent-space compression from first principles.** Without loading any model, write a pure-Python script that computes the element count of a pixel-space diffusion tensor (`3 × 512 × 512`) versus the latent-space tensor (`4 × 64 × 64`). Print both counts and the integer ratio. Then extend the calculation to show what the ratio would be at `1024 × 1024` resolution assuming the same VAE compression factor. Verify both ratios match the expected `48×`.

3. **Implement a forward diffusion loop and verify SNR degradation.** Using only `torch` and `torch.nn.functional`, implement a linear-beta-schedule forward diffusion process for 1,000 timesteps starting from a `1 × 3 × 64 × 64` random image tensor. At timesteps `[0, 100, 250, 500, 750, 999]`, compute and print the signal-to-noise ratio (`log(mean(image²) / mean(noise²)`) and confirm it monotonically decreases toward zero. No `diffusers` imports allowed — build the beta schedule and the `q_sample` equation from the closed-form formula.

4. **Compare forward diffusion schedulers on a real product image.** Download a product photo (use any publicly available shoe, bottle, or electronics image — at least `512 × 512`). Implement the forward diffusion loop from Exercise 3 three times using three different beta schedules: linear, scaled-linear (SD default, `betas = torch.linspace(0.85, 12.0, 1000) ** 2 / 1000`), and cosine. At timestep `t=500`, save the noised image from each schedule as `outputs/noise-comparison-{schedule}.png` using `torchvision.utils.save_image`. Print which schedule retains the most recognizable product structure at `t=500` (measure via pixel-space MSE against the original).

5. **Fine-tune LoRA on a product dataset and export a deployable pipeline.** Collect 15–20 images of a single product category (screenshots from a brand website, your own photos, or a public dataset). Run `train_text_to_image_lora.py` from the `diffusers` examples with rank `4`, learning rate `1e-4`, and 500 steps. After training, merge the LoRA weights into the base U-Net and export the full pipeline in `safetensors` format to `outputs/product-lora-pipeline/`. Generate two images — one with the base model and one with the fine-tuned model — using the same prompt and seed. Compute the brand-color histogram distance (use OpenCV `compareHist` with `HISTCMP_BHATTACHARYYA`) between each output and the nearest training image. Write a one-paragraph evaluation to `outputs/skill-lora-brand-eval.md` stating whether the fine-tuned model improved color consistency.

6. **Build a GTM creative-generation handler with inference optimizations.** Using Apollo or a CSV of 10 company names and their primary brand hex color, construct a handler script at `handlers/product-image-gen.py` that reads the company list, generates a branded product hero image for each company using your fine-tuned pipeline from Exercise 5, and applies at least two inference optimizations: `enable_xformers_memory_efficient_attention()` and `enable_model_cpu_offload()`. For each company, sample the prompt with the brand color name injected (e.g., `"product hero shot, [company] style, dominant color [color_name], studio lighting"`), generate at `512 × 512` with 30 steps, and save to `outputs/branded-heroes/{company_slug}.png`. Print a summary table (company, generation time in seconds, peak GPU memory in MB, dominant-color match percentage vs. declared brand hex) and confirm that xFormers and CPU offload are both active in the pipeline config at runtime.
