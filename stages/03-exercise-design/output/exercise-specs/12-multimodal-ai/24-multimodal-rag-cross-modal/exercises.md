# Exercises — Multimodal RAG and Cross-Modal Retrieval

## Exercises

1. **Implement a basic cross-modal retrieval system.** Use a contrastive vision-language model (CLIP via `sentence-transformers` or `open_clip`) to encode 5 text queries and 10 images (use any public image dataset or download sample product/marketing images) into the shared vector space. Compute cosine similarity between each text query and every image, then print the top-3 image matches per query with their similarity scores. Your terminal output should show query text, matched image filename, and score for each ranking position.

2. **Implement and compare three fusion strategies.** Given two separate top-10 ranked lists — one from text retrieval and one from image retrieval over the same document corpus — implement (a) score fusion as a normalized weighted sum, (b) attention-based fusion using softmax-weighted combination of per-modality similarity scores, and (c) mixture-of-experts fusion with a learnable gating function (you may initialize gate weights randomly and update via a single gradient step on a small relevance signal). Print the final top-5 ranking from each strategy side by side in a table so the differences are visible.

3. **Build a modality-aware vector index with filtered retrieval and recall tracking.** Construct an index over a mixed corpus of at least 30 documents (text + image, you may synthesize these) where each entry carries a `modality` tag. Implement filtered retrieval that accepts a `modalities=` parameter and returns only results from the requested modalities. Compute and print recall@5 separately for text queries and image queries against a held-out ground-truth mapping. Then introduce an artificial embedding scale mismatch (multiply all image embeddings by 15×) and re-run recall — print the before/after comparison so the modality bias failure is observable as a recall collapse.

4. **Diagnose all three cross-modal failure modes on a provided retrieval log.** Build a diagnostic script that ingests a JSON file of retrieval results (you will construct this yourself with at least 20 queries and their top-5 cross-modal retrievals) and flags: (a) modality bias — detected by checking if one modality dominates top-k beyond a threshold ratio, (b) missing chunk boundaries — detected by finding consecutive image-split retrievals that should have been merged but were returned as separate low-overlap fragments, and (c) hallucinated relevance — detected by flagging retrievals with similarity score above 0.8 but with no token-level or semantic overlap with the query terms. Print one concrete example of each failure mode with evidence (scores, modality breakdown, overlap metrics). For the hallucinated-relevance check, integrate enrichment data from Apollo or Hunter API — retrieve company descriptions for any entity-named queries and verify whether the retrieved image or document actually references that company. Print a verification line per flagged retrieval.

5. **Build a complete multimodal RAG pipeline that traces from ingestion to grounded generation.** Implement end-to-end: (1) document ingestion that splits PDFs or HTML pages into text chunks and image regions, (2) cross-modal encoding into a shared index, (3) retrieval that fuses text and image results using the strategy from Exercise 2, and (4) generation using a local or API LLM that produces mixed-modality citations — each cited claim tagged as `[text:doc_id]` or `[image:doc_id:region]`. Use a real GTM dataset: pull 10–20 company profiles from Apollo or Clay, enrich with their website screenshots (use a screenshot API or `playwright`), and run the pipeline on queries like "which of these companies has a security-focused landing page?" Save the runnable pipeline as a persistent artifact at `handlers/multimodal_rag_pipeline.py` and write a sample query-and-answer trace (showing retrieved chunks, fused scores, and the cited generation) to `outputs/skill-multimodal-rag.md`.
