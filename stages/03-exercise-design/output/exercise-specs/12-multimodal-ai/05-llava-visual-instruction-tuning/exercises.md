# Exercises — LLaVA and Visual Instruction Tuning

## Exercises

1. **Implement a two-layer MLP projector.** Create a PyTorch `nn.Module` named `MLPProjector` that accepts a tensor of shape `(batch, 576, 1024)` and outputs `(batch, 576, 4096)`. Use a hidden dimension of 1024 with GELU activation between the two linear layers. Instantiate it, pass a dummy tensor of shape `(1, 576, 1024)`, and print the output shape and total trainable parameter count.

2. **Construct a LLaVA-format multimodal prompt.** Write a Python function `build_llava_prompt(image_token_count: int, user_instruction: str) -> str` that returns a string using the Vicuna chat template (`USER: ... ASSISTANT:`) with `<image>` placeholders inserted before the instruction text — one placeholder per token. Call the function with `image_token_count=576` and an instruction asking the model to describe a product screenshot. Print the first 500 characters of the result.

3. **Compare linear versus MLP projection.** Implement both a `LinearProjector` (single `nn.Linear(1024, 4096)`) and the `MLPProjector` from Exercise 1. Compute and print the parameter count for each. Then time 100 forward passes of each on a random `(1, 576, 1024)` tensor (use `torch.no_grad()` and report median time in milliseconds). Finally, compute the Frobenius norm of the output for each and print both values so you can compare representational capacity empirically.

4. **Trace the two-stage training recipe.** Write a script that loads the LLaVA stage-1 dataset metadata (558K image-caption pairs) and stage-2 dataset metadata (158K GPT-4-synthesized conversations) from their respective JSON files hosted at the LLaVA GitHub release URLs. Compute and print: (a) total unique images across both stages, (b) average conversation turns per sample in stage 2, (c) the ratio of stage-1 to stage-2 samples. If the full JSON files are too large to download quickly, write a streaming JSON parser that processes the first 5,000 records of each file and extrapolates.

5. **Build a quantized inference pipeline with VRAM monitoring.** Implement a script at `handlers/llava_quantized_inference.py` that loads a LLaVA model with 4-bit quantization via `bitsandbytes`, runs inference on a single image (use any image from `signals/examples/` or download a sample), and logs peak VRAM usage before and after model load, and during generation. The script should accept a `--image` path and `--prompt` string as CLI arguments. Print the model output text, peak VRAM in GB, and inference latency in seconds. Save the VRAM report to `outputs/skill-llava-quant-vram.md`.

6. **Design a GTM visual enrichment agent.** Build a script at `handlers/screenshot_analyzer.py` that takes a company website URL, captures a screenshot using a headless browser (Playwright or Selenium), and feeds it through a LLaVA-style multimodal pipeline to extract structured GTM signals: company tagline, product category, tech stack indicators visible on the page, and presence of pricing information. Output the extracted signals as JSON to stdout and write a formatted summary to `outputs/skill-gtm-screenshot-analysis.md`. Include error handling for pages that fail to load and a `--batch` mode that processes a CSV of URLs (simulating an Apollo or Clay export) and writes results for each company.
