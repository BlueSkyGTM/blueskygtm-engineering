# Exercises — Long-Video Understanding at Million-Token Context

## Exercises

1. **Compute visual-token counts across encoding configurations.** Write a Python function that takes video duration (minutes), FPS, frame resolution (height × width), patch size, and pooling factor as inputs. Compute the total visual-token count for each combination in this grid: FPS ∈ {0.5, 1, 2}, patch size ∈ {14, 28}, pooling factor ∈ {1, 2, 4}, for a 45-minute 1080p video. Print a formatted table showing tokens per configuration and flag any combination that exceeds a 1M-token context window.

2. **Build a uniform frame budget calculator.** Implement a script that accepts a video file path (use any `.mp4` you have locally, or download a 5-minute sample with `yt-dlp`) and a model context limit (default 128K tokens). Extract the video's actual duration and resolution with OpenCV, assume a 14×14 patch size with no pooling, and print: total frames sampled at 1 FPS, resulting token count, percentage of context window consumed, and whether the video fits. Output a single summary line like `VIDEO: demo.mp4 | tokens: 812064 | context_fit: NO | headroom: -684064`.

3. **Implement scene-boundary detection with histogram differencing.** Write a script that reads a video frame-by-frame, computes a color histogram for each frame, calculates the chi-square or correlation distance between consecutive histograms, and detects scene cuts where the distance exceeds a threshold. Print each detected cut as `CUT at frame 1247 (00:41:33) | distance: 0.87`. Then print the number of scenes found and the average scene length in seconds. Tune the threshold so it produces between 5 and 30 scene cuts for your test video.

4. **Compare ingestion strategies on recorded sales demos.** Download two Zoom or Loom recording URLs from a public GTM demo (e.g., a product walkthrough from a vendor's YouTube channel). For each video, compute the token cost under three strategies: (a) brute-context at 1 FPS with no pooling, (b) token-compression via keyframe-only extraction (1 frame per detected scene), and (c) ring-attention with 4-second windowed segments at 2 FPS. Print a comparison table with columns: strategy, frames_sampled, token_count, estimated_inference_cost (assume $0.01 per 1K tokens). Identify which strategy is feasible under a 200K-token context limit.

5. **Build a complete frame budget calculator with temporal degradation scoring.** Create a Python module at `signals/examples/frame_budget.py` that accepts a video path and a JSON config specifying model context limit, patch size, pooling factor, and a list of sampling strategies (uniform, scene_boundary, keyframe). The module should: (a) evaluate each strategy and report token counts and context fit, (b) simulate temporal retrieval degradation by placing a "needle" fact at 5 equidistant positions in the resulting token sequence (0%, 25%, 50%, 75%, 100%) and scoring retrieval accuracy at each position using a decay model `accuracy = base * exp(-position * decay_rate)` with `base=0.95` and `decay_rate=0.5`, and (c) write a markdown report to `outputs/skill-frame-budget.md` with a strategy comparison table and a degradation curve table. Run the module on a video and verify the markdown file renders correctly.
