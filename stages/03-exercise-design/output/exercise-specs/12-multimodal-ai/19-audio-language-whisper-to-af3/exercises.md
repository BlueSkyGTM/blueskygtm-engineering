# Exercises — Audio-Language Models: the Whisper to Audio Flamingo 3 Arc

## Exercises

1. **Compute and inspect a log-mel spectrogram from a WAV file.** Load any short WAV recording (use `librosa.load` to get the waveform at 16 kHz), then compute a log-mel spectrogram with 80 Mel bands, a 25 ms FFT window, and a 10 ms hop length. Print the spectrogram array's shape, the number of time frames, the duration of audio in seconds, and verify that `num_frames ≈ duration_seconds / 0.01`. This confirms you can trace the tokenization pipeline from raw waveform through STFT, Mel filterbank, and log transform to the 2D representation every model in this arc consumes.

2. **Implement a Whisper transcription pipeline that prints structured output.** Using the `whisper` Python package, load the `base` model and transcribe a WAV file of your choice (download a sample from the Common Voice dataset or record 30 seconds yourself). Print three things to the terminal: the detected language with its confidence probability, the full transcript text, and a segment-by-segment table showing start time, end time, and text for each segment. Confirm that segment timings are monotonic and that each segment text is non-empty.

3. **Compare Whisper's encoder-decoder architecture against an audio-language model by inspection.** Load the Whisper `base` model and programmatically print its encoder layer count, decoder layer count, and the output dimensionality of the decoder's final vocabulary projection. Then write a short function that constructs a hypothetical audio-language model pairing the same Whisper audio encoder with a small causal LLM decoder (e.g., 6 transformer layers, vocabulary size 50257). Print both architectures side by side and state in a printed comment which model can perform open-ended text generation and which cannot, referencing the specific decoder type (seq2seq vs. causal LM) as the determining factor.

4. **Build a GTM signal extraction pipeline from a sales-call transcript.** Using the Whisper transcription output from Exercise 2 (or any call transcript with timestamps), write a Python script that calls an LLM API to extract three structured signal categories from the transcript: (a) objections raised by the prospect, (b) buying-intent indicators on a scale of low/medium/high, and (c) competitor mentions with the competitor name. Output the results as a JSON object printed to the terminal. Add a logging hook that records processing time per signal category and logs a warning if any category returns zero results (which may indicate a transcription or prompt failure). Use a real enrichment tool — pull company metadata for any mentioned competitor via the Apollo API or a mock CRM CSV at `data/competitors.csv` — to attach firmographic data to each competitor mention.

5. **Implement a persistent sales-call audio handler that produces a structured artifact.** Build a handler at `handlers/call_analyzer.py` that accepts a path to a sales-call WAV file, runs Whisper transcription, extracts GTM signals (objections, buying intent, competitor mentions with firmographic enrichment), and writes the combined output as a Markdown report to `outputs/call-analysis-<timestamp>.md`. The report must include: call duration, detected language, full transcript with segment timestamps, a table of extracted signals with confidence levels, and a pipeline-health section showing per-stage timing and any warnings raised. Configure Python `logging` to write structured logs to `outputs/call-analyzer.log` with timestamps so pipeline failures are traceable. Run the handler on at least two different audio files and confirm both Markdown reports and log entries are generated.

6. **Evaluate the transcription-then-LLM cascade against scenarios where acoustic information matters.** Write a script at `signals/examples/acoustic_loss_eval.py` that processes three audio samples through the pipeline from Exercise 5: one clear studio recording, one noisy recording with background conversation, and one where the speaker's tone (frustrated vs. enthusiastic) is the primary signal. For each sample, print the Whisper transcript confidence, the extracted buying-intent score, and your assessment of whether the cascade captured or missed information that a human listener would catch. Conclude with a printed decision rule: under what acoustic conditions should a team invest in a direct audio-language model (e.g., Audio Flamingo 3) rather than relying on transcription followed by text-based reasoning? Justify with evidence from the three samples.
