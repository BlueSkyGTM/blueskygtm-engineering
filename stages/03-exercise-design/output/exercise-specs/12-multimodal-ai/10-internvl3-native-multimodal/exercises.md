# Exercises — InternVL3: Native Multimodal Pretraining

## Exercises

1. **Compute visual token counts before and after pixel-shuffle downsampling.** Implement a Python function that takes an image resolution (width, height) and a pixel-shuffle downsample factor, then prints the raw patch count (14×14 patches), the post-shuffle token count, and the percentage reduction. Run it for three resolutions: 448×448, 896×896, and 1344×576 (a widescreen screenshot). Print a formatted table to the terminal showing all six numbers.

2. **Trace InternVL3's training pipeline stages and print the divergence from InternVL2.** Write a script that defines each stage of InternVL3's progressive pipeline as a list of dictionaries with `stage_name`, `vision_frozen`, `llm_frozen`, and `data_type` keys. Do the same for InternVL2. Iterate both lists and print only the stages where the two pipelines disagree, with a one-line explanation of what changed. The terminal output should show exactly which stages diverge and why.

3. **Build a token budget estimator for a multimodal enrichment batch.** You are processing 50 company website screenshots at mixed resolutions (some 1024×768, some 1920×1080, some mobile 750×1334). Write a script that loads a CSV of screenshot filenames and dimensions, computes the total visual token count after pixel-shuffle downsampling (factor of 4), and reports whether the batch fits within a 128K context window alongside 2,000 text tokens. Print a per-image breakdown and a final PASS/FAIL verdict.

4. **Quantify alignment debt symptoms using a public VLM benchmark.** Download or create a small JSONL file of 20 image-description pairs (use any open screenshot dataset or generate synthetic pairs where some descriptions are deliberately wrong). Write a scoring script that computes three metrics across the dataset: (a) visual-text consistency rate, (b) hallucinated entity count, and (c) format compliance rate. Print all three metrics. Then annotate which metric corresponds to which symptom of alignment debt from the lesson.

5. **Build a multimodal enrichment handler that extracts structured fields from website screenshots.** Create `handlers/screenshot_enrichment.py` that accepts a directory of company homepage screenshots, sends each to a vision-language model API (OpenAI, Anthropic, or a local model), and extracts: company name, primary value proposition, CTA text, pricing model (if visible), and tech stack hints. Output results as JSON to `outputs/screenshot_enrichment_results.json`. Include pixel-shuffle token estimation in the logs so you can verify the architecture pattern from the lesson is reflected. Integrate with Clay or Apollo by accepting a CSV of company domains as input, correlating extracted fields to the company records, and writing enriched rows back to the CSV.

6. **Design and implement a tracing-based extraction quality drift monitor.** Create `signals/examples/extraction_drift_monitor.py` that processes a time-series of enrichment outputs (simulate 5 batches of 20 extractions each, with batch 3 introducing degraded screenshots). Implement tracing signals that flag when visual-text inconsistency exceeds a threshold — using metrics like field coverage rate, entity overlap between visual OCR text and model output, and confidence-score variance across batches. The script should print a batch-by-batch signal report to the terminal and write a markdown summary to `outputs/skill-extraction-drift-report.md` documenting which batch drifted, which signal caught it first, and recommended remediation.
