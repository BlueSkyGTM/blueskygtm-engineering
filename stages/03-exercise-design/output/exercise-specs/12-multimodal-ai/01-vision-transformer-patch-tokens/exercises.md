# Exercises — Vision Transformers and the Patch-Token Primitive

## Exercises

1. **Implement patch partitioning and flattening.** Create a synthetic RGB image tensor of shape `(3, 224, 224)` filled with random values. Partition it into a grid of `16×16` patches without using any library's built-in `PatchEmbed` module — use `reshape`, `permute`, or `unfold` only. After partitioning and flattening each patch, print: the total number of patches `N`, the flattened dimension of each patch vector, and the final tensor shape. Verify that `N` matches `(224 × 224) / 16²`.

2. **Compute a sequence-length table across resolutions and patch sizes.** Write a script that iterates over image resolutions `[128, 224, 256, 384, 512]` and patch sizes `[8, 16, 32]`. For every valid `(resolution, patch_size)` pair (skip pairs where the patch does not evenly divide the resolution), print a row with the resolution, patch size, sequence length `N`, and flattened patch dimension `C × P²` for `C = 3`. Identify which configuration produces the longest sequence and estimate its self-attention cost as `O(N²)` relative to the `224/16` baseline.

3. **Implement a full `PatchEmbedding` module.** Build a `nn.Module` that accepts `(B, C, H, W)` image tensors and produces `(B, N, D)` token sequences. The module must: partition the image into patches of configurable size `P`, flatten each patch, apply a learnable linear projection from `C × P²` to `D`, and prepend a learnable CLS token. Instantiate it with `P=16, D=768` for `224×224` RGB input. Pass a dummy batch of shape `(4, 3, 224, 224)` through it and print the output shape and total parameter count.

4. **Compute parameter counts and compare ViT against CNN inductive biases.** Write a function `patch_embed_params(P, D, C=3)` that returns the number of learnable parameters in the patch projection layer. Call it for `(P=16, D=768)` and `(P=8, D=768)` and print the difference. Then write a second function `attention_flops(N, D)` that estimates the matrix-multiply cost of one self-attention layer as `2 × N² × D`. Compute and print the attention cost ratio between `P=8` and `P=16` at `224×224`. In a `print()` statement of no more than three sentences, state why this cost structure means a ViT needs more pre-training data than a CNN with the same parameter count to reach competitive accuracy on small datasets.

5. **Build a logo-similarity pipeline using patch embeddings.** Download 5–10 company logo images (you may use a directory of locally saved image files or fetch via a URL list). Run each logo through your `PatchEmbedding` module from Exercise 3, apply mean pooling over the patch dimension to produce one `D`-dimensional vector per logo, and compute pairwise cosine similarity across all logos. Print a similarity matrix. Then, for each logo, print the name of its nearest neighbor (excluding itself). Write the full pipeline as a script that produces output to `handlers/logo_similarity.py`. The script must be runnable as `python handlers/logo_similarity.py` and must print the similarity matrix and nearest-neighbor pairs to stdout.

6. **Compare pooling strategies for landing-page classification.** Implement three pooling heads — CLS token, mean-pool over patch tokens, and a single register token (an extra learnable token appended after the CLS token, not attended to by the image) — on top of your `PatchEmbedding` module. For each strategy, attach a `nn.Linear(D, 3)` classification head with classes `["SaaS", "E-commerce", "Agency"]`. Generate 20 synthetic image tensors of shape `(3, 224, 224)`, assign random labels, run one forward pass through each strategy, and print: the output logits shape, the total parameter count of head + pooling, and the predicted class distribution across the 20 samples. Write your analysis as a markdown file to `outputs/skill-pooling-strategies.md` containing: a table comparing parameter counts, a recommendation for which pooling strategy to use when you have fewer than 1,000 labeled landing-page screenshots, and one paragraph explaining your reasoning.
