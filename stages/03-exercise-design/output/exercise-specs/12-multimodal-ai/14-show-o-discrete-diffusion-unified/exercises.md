# Exercises — Show-o and Discrete-Diffusion Unified Models

## Exercises

1. **Implement a cosine masking schedule and print intermediate states.** Write a function `cosine_mask_probs(timesteps, num_steps)` that returns the probability of each token being masked at each discrete timestep. Print a table showing timestep, mask probability, and a 20-token sample sequence rendered with `█` for kept tokens and `░` for masked tokens. Verify that early steps show few masks and later steps approach full masking.

2. **Compare discrete diffusion forward corruption against autoregressive teacher-forcing on the same sequence.** Take the sentence `"GTM teams use Clay to enrich lead data"` tokenized into a flat list of token IDs. Produce two outputs side by side: (a) the discrete-diffusion corrupted sequence at t=0, t=num_steps//2, and t=num_steps (using your cosine schedule from Exercise 1), and (b) the autoregressive shifted view where each position predicts the next token. Print both views with aligned columns so the difference in how information is destroyed is visible.

3. **Implement a minimal mask–unmask discrete diffusion loop with a random predictor.** Build a `discrete_diffusion_denoise(masked_tokens, vocab_size, num_steps)` function that, starting from a fully masked sequence, iteratively unmaskes positions by sampling a random token from the vocabulary at each step. Log the sequence state at every step. Run it on a 16-token sequence with vocab_size=100 over 10 steps and print the progressive unmasking. This establishes the reverse-process scaffold before a learned model replaces the random predictor.

4. **Apply discrete diffusion to a different domain: CRM event sequences.** Tokenize a list of 5 CRM event types (`email_sent`, `meeting_booked`, `demo_completed`, `proposal_sent`, `deal_closed`) into integer IDs. Run your forward masking process to corrupt a 10-event timeline at three corruption levels (low, medium, high). Print the original timeline and each corrupted version. Then implement a frequency-based denoiser that predicts the most common token for each masked position given the unmasked neighbors, and print the reconstructed timeline alongside the original for comparison.

5. **Instrument and log a joint training step for a Show-o-style dual-branch model on GTM data.** Create a minimal setup with two dummy loss functions: `understanding_loss` (autoregressive cross-entropy on a tokenized Apollo-style company description) and `generation_loss` (masked-diffusion cross-entropy on a masked version of the same description). Write a `joint_step()` function that computes both losses, combines them with a configurable weight `alpha`, and logs the two component losses plus the joint loss to a structured JSONL file. Run 20 simulated training steps with varying alpha values (0.1, 0.5, 0.9) and write the loss curves to `outputs/skill-show-o-joint-loss.jsonl`. Print a summary table showing mean understanding loss and mean generation loss for each alpha setting.

6. **Design and document a unified token vocabulary for a multi-task GTM model.** Write a specification for a shared vocabulary that must encode: natural-language company descriptions (from Apollo enrichment data), structured ICP firmographic fields (employee count, revenue, industry), and discrete diffusion mask tokens. Your spec must address: (a) how each data type maps to token IDs, (b) whether a single vocabulary enables or blocks joint understanding + generation training, (c) at least two concrete failure modes of a poorly unified vocabulary. Write the spec to `outputs/skill-unified-vocab-design.md` and include a small worked example showing 3 sample inputs (one text description, one firmographic record, one masked sequence) encoded under your vocabulary, printed as aligned token-ID tables.
