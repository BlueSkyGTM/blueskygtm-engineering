# Exercises — Omni Models: Qwen2.5-Omni and the Thinker-Talker Split

## Exercises

1. **Compute the TTFA advantage of a Thinker-Talker split.** Implement a function that takes a reasoning-token count, an audio-token count, per-token Thinker decode time, per-token Talker decode time, and a monolithic per-token decode time. Compute time-to-first-audio-byte for (a) a monolithic pipeline that must emit all reasoning tokens before any audio tokens and (b) a Thinker-Talker pipeline where the Talker begins streaming after the first Thinker token is committed. Print both values in milliseconds and the speedup ratio. Vary the reasoning-token count across three values (10, 50, 200) and print a comparison table to the terminal.

2. **Build a latency budget breakdown for a round-trip voice interaction.** Write a script that accepts a dictionary of named stage timings — audio capture, network uplink, STT, prefill, Thinker first-token latency, Talker first-frame latency, network downlink — and outputs each stage's contribution as a percentage of total round-trip time. Highlight which single stage dominates. Run your script with two configurations: a datacenter deployment (low network, high compute) and an edge deployment (high network, low compute). Print both breakdowns side by side and state which stage flips between them.

3. **Implement M-RoPE position assignment for a mixed-modality sequence.** Given a token list where each token is tagged as `text`, `image`, or `audio` (with audio tokens carrying a timestamp and image tokens carrying spatial coordinates), compute the three-component position vector (temporal, spatial, textual) for each token using the M-RoPE partitioning scheme described in the lesson: the head dimension splits into three segments, each tracking its respective modality stream independently. Print the full position-embedding assignment table. Then demonstrate that two tokens from different modalities that share the same sequential index receive distinct position vectors — confirm no collapse by printing the cosine similarity between each cross-modal pair.

4. **Simulate interruption handling across three conversational architectures.** Implement a turn-state machine that processes a scripted dialogue containing a user barge-in event (the user starts speaking while the agent is mid-response). Run the same script through three configurations: half-duplex (agent finishes, then listens), turn-taking (agent yields at the next phrase boundary), and full-duplex (agent aborts within one audio frame). For each configuration, print: whether the interruption was handled, the latency between the barge-in and the agent's response change, and the number of wasted audio frames generated after the barge-in.

5. **Build a Thinker-Talker pipeline tracer.** Implement a class that wraps a simulated Thinker decode loop and a Talker synthesis loop, instrumenting each stage with `time.perf_counter()` stamps on every token or audio frame emitted. Your tracer must record per-token timings, compute rolling tokens-per-second for each loop independently, and flag degradation when any rolling window drops below 80% of the first-window rate. Feed the tracer a mock multimodal input (text + a 3-second audio clip represented as a numpy array of random floats). Output a trace report
