# Exercises — CLIP and Contrastive Vision-Language Pretraining

## Exercises

1. **Implement** the symmetric InfoNCE contrastive loss from scratch in PyTorch. Create two random tensors of shape `(8, 512)` representing image and text embeddings, L2-normalize them, compute the 8×8 cosine similarity matrix, scale by a temperature of 0.07, and compute the symmetric loss (average of image-to-text and text-to-image cross-entropy with diagonal targets). Print the loss value. Then change τ to 0.01 and again to 1.0, recompute, and print all three loss values. Verify that lower temperature produces a higher, more peaked loss.

2. **Compute** the cosine similarity matrix between two batches of four L2-normalized embedding vectors of your choice (you may use `torch.randn`). Print the full 4×4 matrix with row and column labels. Identify the diagonal entries and at least one off-diagonal pair with high similarity. Write a one-line print statement that reports which pair (row, col) has the highest off-diagonal similarity score. Run the script and confirm the output is a single tuple.

3. **Evaluate** zero-shot classification accuracy using the pretrained `openai/clip-vit-base-patch32` model from Hugging Face's `transformers` library. Load a small labeled image dataset (e.g., 10–20 images from `torchvision.datasets.CIFAR10` or any local folder of labeled images). For each image, compute CLIP similarity against text prompts for each class label (e.g., "a photo of a cat", "a photo of a dog"), predict the argmax class, and compare to the ground truth label. Print per-image predictions and the final accuracy percentage. Use `CLIPModel` and `CLIPProcessor` — do not call a high-level `zero_shot_classification` pipeline.

4. **Compare** softmax-based InfoNCE loss against sigmoid pairwise loss (SigLIP) on the same batch of eight image-text embeddings. Implement both loss functions: InfoNCE as cross-entropy over the similarity matrix scaled by temperature, and SigLIP as element-wise `-log(sigmoid(s_ij * label))` where labels are +1 for matched diagonal pairs and −1 for all off-diagonal pairs. Print both loss values. Then vary the batch size from 4 to 16 to 32 (using random embeddings) and print a table showing how each loss changes with batch size. State in a print statement which loss is more sensitive to batch size and why.

5. **Build** a logo-detection enrichment script that takes a list of company website URLs, screenshots their homepages using Playwright or Selenium, and runs CLIP zero-shot classification to tag each page with a label from the set `["company logo visible", "no logo detected", "login form", "pricing page", "blog post"]`. Write the script to `handlers/logo_detector.py`. It should accept a JSON file of URLs as input (e.g., `signals/examples/company_urls.json` with at least 5 entries) and output a JSON file to `outputs/logo_tags.json` containing each URL, its predicted tag, and the top similarity score. Run the script on at least 5 real company websites and print the results table. Use `requests` or `playwright` for screenshots and `CLIPModel` for inference.

6. **Design** and **implement** a CLIP-based page-type classifier for lead enrichment that categorizes screenshots into `["product page", "about page", "contact page", "pricing page", "blog article", "careers page"]`. Write it as a reusable module at `handlers/page_classifier.py` with a function `classify_page(image_path: str) -> dict` returning the predicted label, confidence score, and top-3 candidate scores. Create a markdown report at `outputs/skill-page-classifier.md` documenting your prompt engineering choices (why those exact text prompts, what temperature or preprocessing you applied), the results on 10 sample screenshots, and where this fits in a GTM enrichment pipeline (e.g., routing contacts based on page type, scoring intent based on pricing-page visits). Run the classifier on 10 real or sample screenshots and include the confusion matrix or per-class accuracy in the report.
