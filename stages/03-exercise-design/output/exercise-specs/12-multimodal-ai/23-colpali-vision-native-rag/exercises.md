# Exercises — ColPali and Vision-Native Document RAG

## Exercises

1. **Implement MaxSim scoring from scratch.** Given two numpy arrays — `query_embeddings` of shape `(Q, 128)` and `doc_embeddings` of shape `(D, 128)` — implement the late-interaction MaxSim operator without using any RAG library. Compute the per-query-token maximum similarity across all document tokens, then sum across query tokens. Run your function on the toy arrays provided in the lesson and print the scalar score. Then compare against a single-vector bi-encoder baseline: pool both sides via mean and compute a single cosine similarity. Print both scores side by side for at least three query-document pairs so you can observe where MaxSim and bi-encoder ranking diverge.

2. **Trace patch tokenization dimensions.** Load a single page image using `Pillow` from a PDF rendered at 224×224. Pass it through the ViT patch embedding step from the lesson (the `patch_embed` module only — no full forward pass). Print the input tensor shape, the patch grid dimensions, and the output tensor shape. Then re-render the same page at 448×448 and repeat. Print a table to the terminal showing how the patch count and embedding tensor shape change with resolution. Identify which spatial dimensions are preserved and which are collapsed at each stage.

3. **Build a minimal ColPali-style page ranker.** Implement a script that takes a directory of page images (you may use the lesson's sample PDF rendered with `pdf2image`), encodes each page into patch embeddings using the ViT backbone, and stores them as a flat list of numpy arrays. For a given text query, encode the query tokens, compute MaxSim against every stored page, and print a ranked list of `(page_id, score)` sorted descending. Use a query like *"which accounts have churn risk above 40%?"* against a dashboard-style page. Your script should be invocable as `python rank_pages.py --pages ./data/pages --query "..."` and print the top-5 results.

4. **Compare vision-native retrieval vs OCR-based text RAG on a table-heavy document.** Take a PDF containing a multi-column table (e.g., a pricing sheet or an ICP fit matrix). Run two pipelines: (a) OCR the page with `pytesseract`, chunk the text, and rank using TF-IDF or a lightweight embedding model; (b) render the page image and rank using your MaxSim pipeline from Exercise 3. Issue three queries that reference specific cell values visible only in the table structure (e.g., *"what is the Enterprise plan overage rate?"*). Print both pipelines' top-3 results for each query and note which approach retrieves the correct page. Write a one-paragraph summary to stdout explaining where OCR loses information that the vision pipeline retains.

5. **Build a GTM document search artifact over enriched account data.** Using Clay or Apollo, export a list of 10 target accounts with enrichment fields (firmographics, tech stack, revenue band). Format this data as a visually dense one-pager per account (table + sidebar chart) — you may use `matplotlib` or an HTML-to-PDF library. Render each one-pager to an image, encode with your ColPali pipeline, and build a queryable index saved to `signals/examples/colpali_index.pkl`. Implement a CLI entry point in `signals/examples/gtm_doc_search.py` that accepts a natural-language question (e.g., *"which accounts use Salesforce and have revenue over $50M?"*) and prints the top-matching account pages with MaxSim scores. The artifact must persist the index file and the script so a teammate could re-run it against a different export.

6. **Prove where spatial information is destroyed.** Design and run an experiment that isolates the contribution of spatial layout to retrieval accuracy. Construct two synthetic page images: one with a logically structured two-column table and one with the same cells randomly shuffled in position. Encode both with your ColPali pipeline and issue five queries that depend on row-column relationships (e.g., *"what is the value in the row labeled Retention for the Q3 column?"*). Print MaxSim scores for both layouts across all queries. Then take the OCR text from both pages (identical text content, different positions) and run the same queries through the text RAG baseline. Write your findings to `outputs/skill-colpali-spatial-proof.md` as a structured report: include the score table, a 2–3 sentence conclusion on whether spatial ordering affects MaxSim, and a recommendation on document types where vision-native retrieval is justified versus where OCR is sufficient.
