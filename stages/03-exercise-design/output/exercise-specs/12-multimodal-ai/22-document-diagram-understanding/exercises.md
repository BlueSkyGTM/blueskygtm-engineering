# Exercises — Document and Diagram Understanding

## Exercises

1. **Run both extraction paths on the same document image.** Obtain a document image that contains a table and a diagram (e.g., a one-page product spec sheet). Run it through Tesseract OCR and print the raw text stream. Then run the same image through a VLM multimodal extraction call from the lesson, requesting structured JSON. Print both outputs side by side. In your terminal, print a summary listing at least three structural signals that OCR destroyed but the VLM preserved (e.g., table cell boundaries, reading order, diagram labels).

2. **Modify the VLM prompt for multi-column layout preservation.** Take the VLM extraction code from the lesson and change the system prompt so that the model explicitly respects column-based reading order before cross-column synthesis. Feed it a two-column document image (any newsletter, research abstract, or side-by-side comparison sheet). Print the extracted JSON and verify that field values reflect correct column reading order — not a flattened left-to-right sweep.

3. **Implement a three-tier fallback chain.** Write a Python script that accepts a document image path and attempts, in order: (1) VLM-based structured extraction, (2) OCR-plus-heuristics extraction using regex patterns for dates, monetary amounts, and proper nouns, (3) human-review queue logging. Define a confidence threshold or error condition that triggers each degradation step. Process a directory of at least five test images and print a routing summary showing which tier each document landed in and why.

4. **Extract structured data from a GTM intelligence dashboard.** Capture a screenshot of an Apollo company profile or a Clay enrichment results view — one that mixes a data table, a chart or logo, and descriptive prose. Write a VLM prompt that produces grounded JSON with per-field bounding-box coordinates for company name, employee count, industry, revenue band, and any technographic signals visible. Print the JSON and overlay-verify at least three bounding boxes by printing their pixel coordinates against the image dimensions.

5. **Build a per-field evaluation harness.** Assemble a test set of at least ten document images across three document types (e.g., invoices, org charts, product specification sheets). For each document, write a ground-truth JSON file with the expected field values. Run your VLM extraction pipeline over the entire set and compute precision and recall per field. Output a markdown report containing a results table (field name, document type, precision, recall, F1), a list of the top three error patterns, and a short analysis of which document type degrades fastest. Save the report to `outputs/skill-doc-eval-report.md`.

6. **Design a Clay webhook handler for document-based enrichment.** Build a handler that receives a Clay webhook containing a company URL, fetches the company's public page as a screenshot, runs it through your multimodal extraction pipeline with the fallback chain from Exercise 3, and returns a JSON payload mapping extracted fields to Clay columns (company name, employee count, industry, technographics, recent news headline). Include structured logging, a confidence score per field, and a human-review fallback payload for low-confidence extractions. Save the handler to `handlers/clay-doc-enrichment.py` and test it end-to-end by sending a mock webhook payload from the terminal.
