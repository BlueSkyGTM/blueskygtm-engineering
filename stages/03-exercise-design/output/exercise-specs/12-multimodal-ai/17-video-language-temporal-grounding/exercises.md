# Exercises — Video-Language Models: Temporal Tokens and Grounding

## Exercises

1. **Build a temporal token sequence and span prediction head.** Implement a `TemporalSpanHead` module that takes a tensor of frame-level embeddings of shape `(batch, num_frames, embed_dim)` and predicts `(start_idx, end_idx)` for a given natural-language query. Use a linear projection over the concatenated mean-pooled and max-pooled frame tokens. Feed it a synthetic embedding tensor of shape `(1, 32, 768)` and print the predicted span indices and their confidence (softmax over the two output logits). Confirm the output indices fall within `[0, 31]`.

2. **Compute temporal Intersection-over-Union (tIoU).** Write a function `compute_tiou(pred_span, gt_span, fps)` that accepts spans in seconds, converts them to frame indices using a configurable FPS, and returns the tIoU score. Test it against three prediction/ground-truth pairs: a near-perfect match, a partial overlap, and a completely disjoint pair. Print all three scores to the terminal.

3. **Compare three spatiotemporal tokenization strategies on the same video.** Given a synthetic video tensor of shape `(1, 3, 16, 224, 224)` (batch, channels, frames, height, width), implement and run three tokenization front-ends: (a) factorized attention that applies spatial self-attention per frame then temporal self-attention per spatial position, (b) 3D tubelet embedding with tubelet size `2×16×16`, and (c) per-frame CLS-token pooling that produces one token per frame. Print the resulting token sequence length and the total FLOPs (or multiply count) for each strategy in a comparison table. State which strategy produces the fewest tokens per second of video at 2 FPS.

4. **Implement and benchmark three frame samplers.** Download a short public video (e.g., a 60-second clip from any open dataset or a sample screen-recording). Implement (a) uniform sampling at a fixed FPS, (b) dynamic-FPS sampling that increases frame rate during segments with high inter-frame pixel difference, and (c) event-driven sampling triggered by scene-change detection (frame-diff threshold). For each sampler, print the number of frames extracted, the effective tokens-per-second (assuming 196 spatial tokens per frame from a 224×224 ViT patch grid), and a short textual summary of which segments were densely sampled.

5. **Build a sliding-window temporal grounding pipeline.** Implement a pipeline that processes a long-form video (90+ seconds — use a Zoom sales-call recording, product demo, or webinar replay) by dividing it into overlapping 30-second windows with 10-second stride. For each window, call a frame-level embedding model (CLIP, SigLIP, or any open vision-language model) on uniformly sampled frames, run your span prediction head against a query string (e.g., "pricing discussion" or "product demo"), and then merge overlapping predicted spans whose tIoU exceeds 0.3. Apply a confidence threshold of 0.5 and discard sub-threshold predictions. Output a JSON list of merged spans with start time, end time, and confidence. Save the pipeline as `handlers/sliding_window_grounding.py` and print the final merged spans to the terminal.

6. **Design and evaluate a full temporal grounding benchmark on GTM video data.** Using at least three recorded sales calls or product demos (from your CRM, Gong, Zoom, or any call-recording platform), create ground-truth timestamp annotations for three event types per video (e.g., "objection raised," "pricing mentioned," "next steps discussed"). Run your sliding-window pipeline from Exercise 5 against each annotation and compute mean tIoU and recall@0.5 across all queries. Compare results across two sampler configurations (uniform 1 FPS vs. event-driven). Write a findings document that includes the tIoU table, the tokens-per-second cost for each configuration, and a recommendation for which sampler to use in a sales-intelligence enrichment pipeline. Save the document to `outputs/skill-temporal-grounding-benchmark.md`.
