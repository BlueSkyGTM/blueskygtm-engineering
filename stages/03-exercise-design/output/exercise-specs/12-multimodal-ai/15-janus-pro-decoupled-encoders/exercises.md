# Exercises — Janus-Pro: Decoupled Encoders for Unified Multimodal Models

## Exercises

1. **Run Janus-Pro-7B multimodal understanding inference on a company logo image.** Load the model from Hugging Face (`deepseek-ai/Janus-Pro-7B`), feed it an image of a company logo alongside the prompt "What company is this and what industry are they in?", and print the generated text response to your terminal. Capture the full response, the number of visual tokens extracted by the SigLIP encoder, and the wall-clock inference time. Verify by checking that the output names the correct company and includes an industry classification.

2. **Generate an outbound email banner image with Janus-Pro's generation pathway.** Using the same model checkpoint, submit the text prompt "professional SaaS product banner, blue gradient, minimal style, for a B2B sales email" and run image generation inference. Save the output image to `outputs/banner-test.png` and print the generation seed, number of VQ tokens decoded, and total inference time to your terminal. Verify by opening the saved image and confirming it matches the prompt description.

3. **Implement dual-encoder routing logic that dispatches input to the correct pathway.** Write a Python module that accepts either an image file path or a text prompt, inspects the input type, and routes accordingly — SigLIP encoding for image inputs, direct text tokenization for text-only inputs. Your module must print which encoder pathway was activated, the number of tokens produced, and the embedding tensor shape after projection into the LLM's space. Test with both an image input and a text-only input, and print both routing traces to the terminal. No scaffold — design the interface yourself.

4. **Compute representation distance between SigLIP and VQ tokenizer pathways on identical input.** Pass the same image through both the SigLIP understanding encoder and the VQ generation tokenizer, project both outputs into a common dimensionality, and compute cosine similarity and L2 distance between the two representation sets. Print a table showing per-token distances and the mean distance across all tokens. Then write a 3–5 sentence analysis of what the distance metric reveals about representational conflict. Save your analysis as `outputs/encoder-distance-analysis.md`.

5. **Build a GTM enrichment handler that uses Janus-Pro's understanding pathway to classify prospect screenshots and the generation pathway to produce personalized outreach visuals.** Your handler must: (a) accept a directory of prospect company website screenshots, (b) run understanding inference to extract each company's value proposition and industry, (c) run generation inference to produce a tailored outreach image for each prospect based on the extracted value proposition, (d) log which encoder pathway was used at each step with timing and token counts, and (e) write a structured JSONL file mapping each prospect to their enrichment data and generated image path. Save the handler to `handlers/janus-gtm-enrichment.py` and write a sample run output to `outputs/janus-gtm-results.jsonl`. Include at least 3 prospects in your test run and print a summary table to the terminal showing prospect name, detected industry, image path, and total processing time.

6. **Design and justify an architecture comparison benchmark for Janus-Pro, Transfusion-style coupled-continuous, and Show-o-style coupled-discrete approaches.** Define a fair evaluation protocol across three axes — parameter efficiency, representational conflict (measured via encoder pathway divergence on a shared image set), and task quality (BLEU/ROUGE on understanding, FID on generation). Write a Python script that runs Janus-Pro through your protocol and produces a results table, then write a design document explaining how you would adapt the protocol for the two coupled baselines if checkpoints were available. Save the benchmark script to `signals/examples/architecture-comparison.py` and the design document to `outputs/skill-multimodal-arch-comparison.md`. Print the Janus-Pro results table to the terminal for verification.
