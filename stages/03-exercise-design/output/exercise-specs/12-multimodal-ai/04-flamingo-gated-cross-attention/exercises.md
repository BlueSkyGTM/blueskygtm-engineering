# Exercises — Flamingo and Gated Cross-Attention for Few-Shot VLMs

## Exercises

1. **Implement** a `GatedCrossAttention` module with a learnable scalar parameter `α` gated through `tanh(α)` and initialized to exactly zero. Feed a batch of text hidden states (shape `(2, 8, 64)`) and visual latents (shape `(2, 4, 64)`) through it. Print the max absolute difference between the input text states and the module's output — confirm it is below `1e-6`, proving identity behavior at initialization.

2. **Build** a `PerceiverResampler` that holds `M = 16` learned latent queries of dimension `D = 64` and cross-attends to variable-length visual features. Pass three different synthetic visual feature tensors with `N = 32`, `N = 128`, and `N = 512` tokens respectively. Print the output shape for each — confirm all three produce `(1, 16, 64)` regardless of input length.

3. **Trace** the gate-opening behavior over simulated training steps. Initialize `α = 0`, then manually update it in increments of `0.1` up to `α = 2.0` (simulating learned drift). At each step, pass the same text and visual batch through your gated layer and compute the cosine similarity between the output and the original text states. Print a table showing `α`, `tanh(α)`, and cosine similarity for each step — confirm similarity degrades smoothly from `1.0` as the gate opens, never discontinuously.

4. **Compare** output perturbation magnitude across four `α` initializations: `0.0`, `0.5`, `1.0`, and `2.0`. For each, run the same text batch through the gated layer with fixed random visual latents and compute the L2 norm of `output - input_text`. Print a formatted comparison table. Explain in a comment block why `α = 0` is the only initialization that guarantees the frozen LM's distribution is preserved at step 0.

5. **Construct** a 4-layer frozen language model (use `nn.TransformerEncoderLayer` or build simple self-attention blocks) with `GatedCrossAttention` inserted after layers 1 and 3. Freeze all original LM parameters and leave only the inserted modules trainable. Run a forward pass, call `backward()` on the output sum, then iterate over all named parameters and print each parameter's name, requires_grad status, and gradient norm. Confirm that only parameters in the inserted gated layers and resampler have non-zero gradients — all frozen LM parameters show `grad_norm = 0.0` or `None`. Save the module definition to `handlers/gated_vlm.py` so it can be imported and tested independently.

6. **Apply** the Flamingo architecture to a GTM enrichment workflow. Fetch 5 company records from the Apollo API (or load them from `data/apollo_companies.json` if the API key is unavailable), download each company's homepage screenshot via a free screenshot API (e.g., `https://api.microlink.io/?url=<company_url>&screenshot=true`), pass each screenshot through a pretrained vision encoder (`torchvision.models.vit_b_16`), compress with your Perceiver Resampler, and feed the visual latents through your gated VLM alongside a prompt like `"This company's product category is:"`. Collect the top-5 predicted tokens for each company and write a structured report comparing predicted categories against the `industry` field from Apollo. Save the report to `outputs/skill-vlm-enrichment.md`.
