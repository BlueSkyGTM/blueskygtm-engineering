# Exercises — Linear Systems

## Exercises

1. **Solve two systems and justify your method choice.** Construct a 4×4 matrix `A_sq` with full rank and a matching vector `b_sq`. Solve it with `numpy.linalg.solve`. Then construct a 10×4 matrix `A_over` (10 accounts, 4 signals) and a 10-element `b_over`, and solve it with `numpy.linalg.lstsq`. Print the shape of each `A`, the method you used, and the solution vector. Verify your square-system solution by printing `A_sq @ x - b_sq` (should be near-zero). Verify your least-squares solution by printing the residual norm `||A_over @ x - b_over||`.

2. **Compute and interpret condition numbers.** Build three matrices: (a) a well-conditioned 4×4 with random integers in `[1, 10]`, (b) a near-singular 4×4 where two columns are almost identical (differ by `0.001`), and (c) a Hilbert matrix of size 5 (`scipy.linalg.hilbert(5)`). For each, compute `numpy.linalg.cond` and print the condition number alongside a verdict: `TRUSTWORTHY` if `cond < 100`, `SUSPECT` if `100 ≤ cond < 10⁶`, `UNRELIABLE` otherwise. For matrix (b), solve `Ax = b` with an arbitrary `b`, then perturb `b` by adding `0.01` to one entry, re-solve, and print the change in `x` to demonstrate sensitivity.

3. **Implement Gaussian elimination with partial pivoting from scratch.** Write a function `gaussian_solve(A, b)` that performs forward elimination with partial pivoting (selecting the largest-magnitude pivot in each column at or below the current row) followed by back-substitution. Do not use `numpy.linalg.solve` or `numpy.linalg.lstsq` inside the function. Test it on a 4×4 system of your choice and print your solver's solution alongside `numpy.linalg.solve`'s solution for the same system. Print the maximum absolute difference between the two solution vectors — it should be below `1e-10`.

4. **Derive lead-scoring weights from historical conversion data.** Create a synthetic dataset of 50 accounts, each with 5 signal columns: `website_visits`, `email_opens`, `demo_requests`, `pricing_page_views`, and `employee_count`. Generate these from reasonable ranges (e.g., website visits 0–20, email opens 0–50, demo requests 0–3, etc.). Define a "true" weight vector and compute conversion scores as `A @ w_true + noise`, then binarize scores at a threshold to produce a conversion label vector `b`. Solve for weights using `numpy.linalg.lstsq` and print each signal name paired with its derived weight, sorted by absolute weight descending. Print which two signals contribute most to conversion.

5. **Build a production lead-score stabilizer that diagnoses and regularizes ill-conditioned signal matrices.** Write a script that loads or constructs a GTM signal matrix (columns: Apollo firmographic scores, Clay enrichment fields, email engagement counts, page visit counts, technographic indicators — at least 6 columns and 80 rows). The script must: (a) compute the rank and condition number of the matrix and print both, (b) detect whether any columns are linearly dependent or near-dependent (print which column pairs have correlation above 0.98), (c) if the condition number exceeds `1e4`, apply ridge regularization by solving `(AᵀA + λI)w = Aᵀb` with `λ = 1.0`, (d) print the regularized weights alongside the unregularized `lstsq` weights, and (e) print the difference in residual norm between the two solutions. Save the script to `handlers/lead_score_stabilizer.py` so it can be imported and called on any signal matrix.

6. **Design and prove a comparison between direct solvers under increasing collinearity.** Starting from the 50-account lead-scoring dataset in Exercise 4, progressively corrupt the matrix by making one signal column increasingly collinear with another (blend it from 100% original to 100% copy of a neighbor in 10 steps). At each step, solve with both `numpy.linalg.lstsq` and your `gaussian_solve` (applied to the normal equations `AᵀA x = Aᵀb`), compute the condition number of `A`, and record the L2 distance between the two solvers' weight vectors. Write the results table (step, correlation between the two columns, condition number, solver divergence) to `outputs/skill-collinearity-analysis.md`. State in the markdown which condition-number threshold corresponds to solver divergence exceeding `0.1` and justify why ridge regularization would help at that point.
