# Exercises — Complex Numbers for AI

## Exercises

1. **Implement complex arithmetic from primitives.** Build a module that defines functions `cadd`, `cmul`, `cdiv`, `cconj`, `to_polar`, and `from_polar` — each operating on tuples `(a, b)` representing `a + bi`. No use of Python's built-in `complex` type or `cmath`. For the inputs `z1 = (3, 4)` and `z2 = (1, -2)`, compute and print the sum, product, quotient, conjugate of each, and the polar form `(r, theta)` of each. Verify that `from_polar(to_polar(z))` round-trips back to the original tuple within floating-point tolerance.

2. **Verify Euler's formula numerically.** Write a script that computes `e^(iθ)` for `θ = 0, π/4, π/2, π, 3π/2, 2π` using both the Taylor series expansion of `e^(iθ)` (truncated at 20 terms) and the closed-form `(cos θ + i sin θ)`. Print a table showing θ, real-part error, and imaginary-part error for each angle. Confirm the maximum absolute error is below `1e-10`.

3. **Implement and test the Discrete Fourier Transform.** Write a function `dft(signal)` that takes a list or array of `N` real numbers and returns `N` complex frequency coefficients using the formula `X[k] = Σ x[n] · e^(-2πi·kn/N)` for `k = 0..N-1`. Do not use `numpy.fft`. Test it on the signal `x = [1, 2, 3, 4, 3, 2, 1, 0]` and print each coefficient's magnitude and phase. Then compare every coefficient against `numpy.fft.fft(x)` and print the maximum absolute difference — confirm it is below `1e-10`.

4. **Apply the DFT to a GTM enrichment signal.** Generate a synthetic weekly signal vector representing lead-conversion counts over 16 weeks: embed a weekly pattern (high Monday/Friday, low weekends repeated across weeks) plus a monthly cycle. Run your `dft` function on the flattened vector. Print the top-3 dominant frequency bins by magnitude, and state in a comment which real-world cadence each bin corresponds to (daily, weekly, monthly). Save the magnitude spectrum to `outputs/skill-dft-gtm-spectrum.txt` as a two-column `bin_index, magnitude` table.

5. **Build a rotation-scaling visualizer for complex multiplication.** Write a script `signals/examples/rotation_demo.py` that takes a list of complex points (representing signal vectors on the complex plane), multiplies each by a fixed complex scalar `w = r·e^(iθ)`, and prints the before-and-after coordinates alongside the expected geometric transformation (`scale by r, rotate by θ`). Use `r = 2.0` and `θ = π/3`. Include at least five input points spread across all four quadrants. Confirm programmatically that every output point's magnitude equals `r` times the input magnitude and every output angle equals the input angle plus `θ`.

6. **Design a DFT-based filter for noisy GTM time-series data.** You have a 32-point signal of daily outbound-email reply rates stored in a CSV at `inputs/email_reply_rates.csv` (two columns: `day, rate`). Load the data, apply your `dft` function, zero out the coefficients whose magnitude falls below the 70th percentile, apply the inverse DFT (which you must also implement), and write the denoised signal to `outputs/skill-dft-denoised.csv`. Print the original mean reply rate, the denoised mean, and the number of frequency components retained. Explain in a header comment why zeroing small-magnitude coefficients acts as a low-pass filter and what GTM insight the retained dominant frequencies reveal.
