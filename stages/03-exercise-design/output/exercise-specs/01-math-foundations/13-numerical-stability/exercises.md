# Exercises — Numerical Stability

## Exercises

1. **Implement** both a naive softmax and a stable softmax (using the max-subtraction trick) as standalone Python functions. Feed each function the array `[1000.0, 1001.0, 1002.0]`. Print the output of both side by side, along with the boolean `np.allclose` comparison. Then run both on `[1.0, 2.0, 3.0]` and confirm they agree. The naive version should produce `nan` on the first input; the stable version should produce three finite probabilities summing to 1.0.

2. **Trace** catastrophic cancellation by computing `exp(1e-9) - 1` two ways: (a) directly with `numpy.exp`, and (b) using `numpy.expm1`. Print both results with 20 decimal places using `format(value, '.20f')`. Then compute the relative error of each against `mpmath.expm1(1e-9)` (or `decimal` if you prefer) as a high-precision reference. Print the relative errors. The direct subtraction should lose roughly 9 significant digits; `expm1` should retain near-full precision.

3. **Compare** three summation strategies — naive `sum()`, `math.fsum()`, and a hand-implemented Kahan summation — on the array `[1e16] + [1.0]*10000 + [-1e16]`. Print each result on a separate line. The exact answer is `10000.0`. Report which methods recover the correct answer and which lose precision. Then add a fourth strategy: a Welford-style online mean computed incrementally over the same array, and print the resulting mean. Confirm whether the Welford mean equals `5000.0` (the true mean of those 10001 values).

4. **Build** a log-space probability accumulator that takes a list of raw log-probabilities (some as extreme as `±800`) and computes `log(sum(exp(lp_i)))` for the entire list. Generate 10,000 log-probability values by drawing from `numpy.random.normal(loc=-400, scale=150, size=10000)`. Implement the computation two ways: (a) naive `np.log(np.sum(np.exp(log_probs)))`, and (b) the stable log-sum-exp with max-subtraction. Print both results, or print the exception/`nan`/`inf` if the naive version fails. The stable version must produce a single finite float.

5. **Design and implement** a production ICP scoring normalizer for an enrichment pipeline. Write a script at `handlers/normalize_scores.py` that simulates a Clay-style enrichment output: generate 500 company fit scores drawn from `numpy.random.lognormal(mean=8, sigma=2.5, size=500)` (these represent raw engagement signals — some will be enormous). The script must (a) apply naive z-score normalization `np.exp((x - mean) / std)` to produce softmax-like probabilities, (b) apply a stable version using max-subtraction and log-sum-exp, and (c) print the rank correlation (Spearman, via `scipy.stats.spearmanr`) between the two normalized rankings, along with the count of `inf` or `nan` values in the naive output. Run it with `python handlers/normalize_scores.py`. If the rank correlation is below 1.0 or the naive version produces any non-finite values, print `UNSTABLE: rank inversions detected`. Otherwise print `STABLE`. The script must be self-contained and executable from the project root.
