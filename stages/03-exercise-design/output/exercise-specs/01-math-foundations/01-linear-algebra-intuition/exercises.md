# Exercises — Linear Algebra Intuition

## Exercises

1. **Implement dot product and magnitude from scratch.** Write a Python script that defines two 4-dimensional vectors, computes their dot product manually (element-wise multiply, then sum), and computes each vector's magnitude using the Pythagorean formula. Print every intermediate value: the element-wise products, the running sum, the squared components, and the final magnitudes. Verify your dot product matches NumPy's `np.dot`. Your script should run with `python dot_product.py` and print all intermediate values to the terminal.

2. **Compute cosine similarity between two lead-description embeddings.** Load the pre-computed embedding vectors from `signals/data/sample_embeddings.json` (two keys: `"target_icp"` and `"incoming_lead"`). Implement the cosine similarity formula manually using only `math.sqrt` and a list comprehension — no NumPy, no `sklearn`. Print both vectors' magnitudes, their dot product, and the final cosine similarity score (should be between -1 and 1). Run `python cosine_check.py` and confirm the output.

3. **Apply Euclidean distance and cosine similarity to a failure case.** Construct three 5-dimensional vectors by hand: `a = [1, 1, 1, 1, 1]`, `b = [10, 10, 10, 10, 10]`, and `c = [1, 1, 1, 1, 0]`. Compute Euclidean distance and cosine similarity for both pairs `(a, b)` and `(a, c)`. Print all four values with clear labels. Write a comment in your script explaining which metric says `a` is closer to `b` and which says `a` is closer to `c`, and why a GTM team scoring accounts by semantic fit should prefer cosine similarity in this scenario. Run the script and confirm output.

4. **Diagnose rank deficiency via singular values.** Construct a 4×4 matrix where the third row is a linear combination of the first two rows (e.g., `row3 = 2 * row1 + 3 * row2`). Compute its singular values using `numpy.linalg.svd`. Print all singular values and identify which one is near-zero. Then construct a second 4×4 matrix with full rank (all rows linearly independent) and print its singular values for comparison. Label both outputs clearly so a reader can see the difference at a glance.

5. **Build a lead-scoring comparison tool.** Write a script at `handlers/lead_scorer.py` that loads a target ICP embedding and a JSON list of 5 account embeddings from `signals/data/accounts.json` (you create this file with dummy data — use 8-dimensional vectors). For each account, compute cosine similarity to the ICP vector, then print a ranked list of accounts from highest to lowest similarity. Include the rank, account name, and similarity score (4 decimal places). Run `python handlers/lead_scorer.py` and confirm the output shows all five accounts sorted correctly.

6. **Trace a vector through a rotation matrix and explain the geometry.** Write a script that defines a 2D vector `v = [3, 1]`, constructs a 45-degree rotation matrix using `cos(π/4)` and `sin(π/4)`, computes the matrix-vector product `R @ v`, and prints the original vector, the rotation matrix, and the transformed vector. Compute and print the magnitude of both the original and transformed vector — they should be equal. In a comment block at the top of the script, write 3–4 sentences explaining why the magnitude is unchanged and what this tells us about orthogonal matrices as geometric transformations. Save the script to `outputs/skill-rotation-trace.md` as a formatted writeup (embed the code in a fenced block and your explanation as prose).
