# Exercises — Norms and Distances

## Exercises

1. **Compute all three norms on a set of vectors.** Given the vectors `v1 = [3, 4]`, `v2 = [1, 2, 3, 4]`, and `v3 = [0, 0, 0, 5]`, implement functions for L1, L2, and L∞ norms using only `math`. Print a formatted table showing each vector and all three norm values. Verify by hand that L2 of `v1` equals 5.0 and L∞ of `v3` equals 5.

2. **Implement Euclidean, Manhattan, and cosine distance from first principles.** Using only `math.sqrt`, `math.fabs`, `math.sum` or plain loops, write three distance functions that each take two equal-length numeric lists. Test them on `a = [1, 0, 0]` and `b = [0, 1, 0]`: confirm Euclidean ≈ 1.4142, Manhattan = 2.0, and cosine distance = 0.5. Print all three results with 4 decimal places.

3. **Diagnose when cosine and Euclidean disagree.** You are given three 2D points representing lead embeddings: `P = [1, 1]`, `Q = [100, 100]`, and `R = [1, 0]`. Compute both Euclidean and cosine distance from P to Q and from P to R. Print which neighbor each metric ranks as closest to P. In a comment, explain why the metrics disagree and state which ranking is more meaningful if these vectors represent topic-mixture embeddings where magnitude reflects volume, not topic.

4. **Apply distance metrics to Clay-enriched account data.** A Clay enrichment table exports five ICP accounts as feature vectors with three dimensions: `[employee_count, annual_revenue_k, tech_stack_count]`. The accounts are: `Acme = [50, 5000, 12]`, `Globex = [120, 15000, 20]`, `Initech = [45, 4200, 10]`, `Umbrella = [2000, 800000, 45]`, `Stark = [80, 12000, 18]`. Your target account is `[60, 6000, 14]`. Compute Manhattan, Euclidean, and cosine distances from the target to each account. Print a ranked list (closest to farthest) for each metric. Identify which metric is most appropriate for this raw, mixed-scale GTM data and explain why in a printed summary line. No normalization libraries allowed — use only `math`.

5. **Build a configurable nearest-neighbor ranking engine.** Implement a `nearest_neighbors(query, dataset, metric="cosine", k=5)` function where `dataset` is a list of `(label, vector)` tuples and `metric` accepts `"euclidean"`, `"manhattan"`, or `"cosine"`. The function must return a sorted list of `(label, score)` tuples ranked by distance (ascending), limited to `k` results. Use your own distance implementations from Exercise 2 — no `scipy`, `sklearn`, or `numpy`. Test it on the provided 8-dimensional embedding vectors in `signals/examples/embeddings_sample.py` (create this file with 10 labeled vectors of your choice representing company descriptions). Run the function with each of the three metrics on the same query vector and print all three rankings side by side. Save your implementation as `handlers/nearest_neighbors.py`.
