# Exercises — Calculus for Machine Learning

## Exercises

1. **Compute numerical derivatives for a composite function and compare against the analytical derivative.** Write a function `f(x)` that computes `sin(x² + 1)` and a function `f_prime(x)` that computes the analytical derivative using the chain rule. For `x` values `[0.0, 0.5, 1.0, 1.5, 2.0, 3.0]`, print a table showing `x`, the numerical derivative (central difference, `h=1e-5`), the analytical derivative, and the absolute difference. Confirm all differences are below `1e-6`.

2. **Implement 1D gradient descent from scratch and observe the effect of learning rate.** Write a script that minimizes `f(x) = x⁴ - 3x² + 2x + 1` starting from `x = 2.0`. Compute `f'(x)` analytically (hardcode the derivative). Run gradient descent for 100 iterations at three learning rates: `0.01`, `0.1`, and `0.3`. Print the iteration number, current `x`, and loss every 10 iterations for each learning rate. Report which learning rates converge and which diverge.

3. **Trace the chain rule through a two-layer composed function and verify backpropagation by hand.** Implement `g(x) = exp(x)` and `f(u) = 3u² + u`. Compose them as `y = f(g(x))`. Compute `dy/dx` numerically using central differences. Then compute it analytically as `f'(g(x)) * g'(x)` and print both side by side for `x = [0.0, 0.5, 1.0]`. Now generalize: write a function `chain_rule_grad(x, funcs)` that takes a list of `(function, derivative)` tuples and composes them, computing the total gradient via the chain rule. Verify it matches the numerical derivative for the two-layer composition above.

4. **Derive and code the MSE loss gradients for a linear regression model and run one step of gradient descent.** Given weight `w`, bias `b`, input features `X` (a list of floats), and targets `y`, implement `mse(w, b, X, y)` and functions `grad_w(w, b, X, y)` and `grad_b(w, b, X, y)` using the closed-form gradients you derived on paper. Start with `w=0.0, b=0.0`, `X = [1, 2, 3, 4, 5]`, `y = [2.1, 3.9, 6.2, 7.8, 10.1]`. Print the initial loss, the gradients, and the loss after one gradient descent step with `lr=0.01`. Verify the loss decreased.

5. **Train a single-neuron linear model on GTM account data and save the training handler.** Load account data from a CSV (you can generate synthetic data: `employee_count` as input, `annual_revenue` as target, ~50 accounts with realistic noise). Train a linear model `revenue = w * employee_count + b` using gradient descent on MSE loss. Print the loss trajectory every 50 epochs for 500 epochs. Tune the learning rate to achieve convergence. Save the reusable training function as `handlers/train_linear_model.py`, which reads a CSV path from `sys.argv[1]`, trains the model, and prints the final `w`, `b`, and MSE. Run it on your synthetic dataset and confirm the learned weight approximates the true relationship.

6. **Extend the linear trainer to two features and evaluate convergence behavior with and without feature normalization.** Modify the handler to accept two input columns (e.g., `employee_count` and `prior_year_revenue`) and predict `current_year_revenue`. Compute partial derivatives for both weights and the bias. Run training on raw features, then run again with z-score normalized features. Print the loss trajectory for both runs and compare convergence speed. Document your findings — including whether normalization changed the learning rate you could use — in `outputs/skill-linear-model-convergence.md`.
