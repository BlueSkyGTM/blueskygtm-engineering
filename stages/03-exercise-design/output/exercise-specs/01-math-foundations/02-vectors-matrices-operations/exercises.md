# Exercises — Vectors, Matrices & Operations

## Exercises

1. **Compute** the dot product of vectors `a = [2, 5, -1]` and `b = [3, -2, 4]` by hand on paper. Then write a Python script that computes the same dot product three ways: `np.dot(a, b)`, `a @ b`, and `np.sum(a * b)`. Print all three results and assert they are identical. In a comment, show your hand-computed value.

2. **Normalize** the vectors `[3, 4]`, `[1, 2, 2]`, and `[0, -3, 0, 4]` using L2 normalization in NumPy. Print each original vector, its L2 norm (magnitude), and the normalized vector. Verify that every normalized vector has magnitude `1.0` (within floating point tolerance). Then compute the dot product of each normalized vector with itself and confirm it equals `1.0`. In a printed statement, explain in one sentence why this property matters before comparing two vectors for similarity.

3. **Trace** the dimension changes through the following chain and print every intermediate shape without computing values. Create random matrices: `X` of shape `(4, 8)`, `W1` of shape `(8, 16)`, `W2` of shape `(16, 6)`, `b1` of shape `(16,)`, `b2` of shape `(6,)`. Compute `H = np.maximum(W1.T @ X + b1[:, None], 0)` followed by `Y = W2.T @ H + b2[:, None]`. Before running, write down your predicted shape for `H` and `Y`. Then run and print all shapes. Assert your predictions match.

4. **Implement** a single-layer neural network forward pass `relu(W @ X + b)` entirely with NumPy matrix operations — no loops, no Python-level iteration over samples. Create `X` as a `(3, 50)` matrix (3 features, 50 samples drawn from a standard normal), `W` as a `(5, 3)` weight matrix (5 output neurons), and `b` as a `(5,)` bias vector. Compute the output, print its shape, and confirm it is `(5, 50)`. Then count how many output entries are exactly zero (indicating ReLU deactivated that neuron-sample pair) and print the count alongside the percentage of the output that is zero.

5. **Build** a pairwise cosine similarity tool for GTM account data. Fetch 10 company descriptions using the Apollo People Search API (or use a provided CSV of company descriptions in `inputs/companies.csv` if no API key is available). Embed each description using the `all-MiniLM-L6-v2` sentence-transformers model to produce a `(10, 384)` matrix. L2-normalize all rows. Compute the full `10×10` cosine similarity matrix. Extract the top-5 most similar company pairs (excluding self-pairs and duplicates). Write a script at `handlers/similarity_pairs.py` that prints each pair with both company names, their cosine similarity score, and a one-line interpretation of why they are similar. The script must be runnable as `python handlers/similarity_pairs.py` and produce formatted terminal output.

6. **Design** a scoring layer that combines embeddings with structured CRM features. Given 8 prospect companies with both a 384-dimensional embedding vector and a 3-feature vector `[employee_count, revenue_estimate, tech_stack_score]` (normalize structured features independently), concatenate the two into 387-dimensional vectors and run a single forward pass through a learned weight matrix `W` of shape `(1, 387)` with a bias scalar. Apply a sigmoid to produce a fit-score between 0 and 1 for each prospect. Write this as `handlers/fit_scorer.py` that loads data from `inputs/prospects.json` (you create this file with realistic sample data), computes all scores, ranks prospects from highest to lowest, and prints a ranked table with company name, embedding-only score, combined score, and the delta between them. Produce a markdown summary of the ranking and your observations at `outputs/skill-fit-scorer.md`.
