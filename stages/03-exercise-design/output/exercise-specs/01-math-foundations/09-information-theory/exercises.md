# Exercises — Information Theory

## Exercises

1. **Implement entropy from scratch.** Write a Python function `entropy(p)` that takes a list of probabilities (not necessarily normalized) and returns Shannon entropy in bits using `math.log2`. Compute `entropy([0.5, 0.5])` and `entropy([0.99, 0.01])` and print both values with 6 decimal places. Verify by hand that the first equals 1.0. Then test `entropy([0.25, 0.25, 0.25, 0.25])` — confirm it equals 2.0. Your function should handle zero-probability events without raising (define `0 * log2(0) = 0`).

2. **Compute cross-entropy and KL divergence between two distributions.** Given `p = [0.7, 0.2, 0.1]` (true distribution) and `q = [0.4, 0.4, 0.2]` (model's predicted distribution), implement `cross_entropy(p, q)` and `kl_divergence(p, q)` without imports beyond `math`. Print both values. Then verify the identity `H(p, q) = H(p) + KL(p || q)` numerically — print the left side and right side side by side and confirm they match to at least 4 decimal places.

3. **Implement mutual information and rank categorical features by predictive signal.** Write a function `mutual_information(feature_values, target_values)` that accepts two parallel lists of strings, computes the joint and marginal distributions, and returns MI in bits. Create a small synthetic dataset: a list of 20 rows with fields like `industry` (SaaS, Fintech, Healthcare), `region` (NA, EMEA, APAC), and `target` (converted / not_converted). Design the data so that `industry` is strongly predictive of the target and `region` is not. Print a ranked table of feature name → MI score, highest first.

4. **Audit a GTM enrichment export for degenerate columns.** Export 50–100 rows of company or contact data from Clay or Apollo (or generate a realistic synthetic CSV with columns like `company_name`, `website`, `employee_count`, `industry`, `linkedin_url`, `country`, `is_customer`). Write a script that loads the CSV, computes entropy for each categorical column, and flags any column whose entropy falls below 0.1 bits as "degenerate — drop or skip enriching." Print a table of column name, entropy value, and a KEEP/DROP recommendation. Identify at least one column that should be dropped.

5. **Build a reusable CLI tool that audits any CSV for information-theoretic signal.** Create `handlers/column_audit.py` that accepts a CSV filepath and a target column name via command-line arguments (`--csv`, `--target`). The tool computes per-column entropy, mutual information with the target, and a combined "signal score" (e.g., MI normalized by target entropy). It writes a Markdown report to `outputs/column-audit-report.md` with three sections: a ranked entropy table, a ranked MI table, and a "degenerate columns" warning list. Run it against the CSV from Exercise 4 and confirm the report file is created. Print the top 3 most predictive columns to stdout as a summary.

6. **Design and justify an enrichment budget policy using information-theoretic evidence.** Using the report from Exercise 5, write a short policy document to `outputs/skill-enrichment-budget-policy.md` that specifies: (a) which columns to stop enriching (entropy or MI justification), (b) which columns to prioritize enriching given limited budget, and (c) a threshold rule — stated in bits of mutual information — below which a field is excluded from enrichment workflows. Include the numeric evidence from your audit output. This artifact should be readable by a GTM operations teammate who has not taken this lesson.
