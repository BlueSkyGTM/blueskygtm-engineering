# Exercises — Conditional GANs & Pix2Pix

## Exercises

1. Implement a U-Net generator in PyTorch with at least 3 encoder blocks, a bottleneck, 3 decoder blocks, and skip connections that concatenate matching-resolution activations. Instantiate it, pass a `(1, 1, 256, 256)` tensor representing a grayscale sketch, and print the output shape. Verify it matches `(1, 3, 256, 256)` for an RGB image output. Then pass a `(4, 1, 256, 256)` batch and confirm the output is `(4, 3, 256, 256)`.

2. Implement a PatchGAN discriminator that accepts a concatenated pair `(condition_image, target_image)` along the channel dimension. Instantiate it with a 70×70 receptive field configuration. Feed two inputs through it: a real pair `(sketch, photo)` and a fake pair `(sketch, generator_output)`. Print the discriminator's output tensor shape for both and confirm it produces a spatial map (e.g., `(1, 1, 30, 30)`), not a single scalar. Then add a print statement at each convolutional layer to show the tensor shape flowing through, confirming where the conditioning image enters the network.

3. Build a minimal paired-image dataset loader from any two aligned image folders (e.g., edges and photos, or day and night). Train two small generators on it for 50 epochs each: one optimized with L1 loss only and one optimized with adversarial loss (BCEWithLogits) plus L1 loss with a lambda weight of 100. Save three sample outputs from each model. Compute and print the mean L1 pixel error across the test set for both models and report which produces lower reconstruction error versus which produces visually sharper edges.

4. Replace your PatchGAN discriminator from Exercise 2 with a full-image discriminator — one that outputs a single scalar probability for the entire image. Reuse your U-Net generator and training loop. Retrain for the same number of epochs as Exercise 3's adversarial model. Save three sample outputs. Compare them against the PatchGAN-trained outputs from Exercise 3 and report: (a) whether high-frequency texture detail is preserved or lost, (b) whether the model produces more or less mode collapse, and (c) the L1 error difference. Print a summary table of both discriminators' results.

5. Build a complete Pix2Pix training pipeline that combines your U-Net generator, PatchGAN discriminator, and a composite loss function (adversarial + L1 with configurable lambda). Train on the CMP Facades dataset or any paired dataset with at least 200 image pairs. Log the generator and discriminator losses at every epoch to a CSV file. After training, generate outputs for 10 held-out test images and save them to a results directory. Write a markdown summary documenting your architecture choices (number of filters, number of layers, lambda value), the final loss curves, and 3 observations about where the model succeeds versus fails structurally. Save the summary to `outputs/skill-pix2pix-training.md`.

6. Design and implement a conditional enrichment pipeline for a CRM workflow that mirrors how conditioning works in a cGAN. Pull 50 leads from the Apollo or Clay API (or a local CSV export). Define two enrichment conditions (e.g., company size > 200 AND industry = "SaaS") that act as the conditioning variable `y`. For each condition value, route leads through a different enrichment path (e.g., different enrichment fields, different scoring weights) and write the enriched results to separate output files. Print a comparison showing how the same lead would be enriched differently under each condition. Save the pipeline script to `handlers/crm_conditional_enrichment.py`. Include a docstring that explicitly maps each pipeline component to its cGAN analog (leads = data distribution `x`, condition = `y`, enrichment routing = generator conditioned on `y`, validation step = discriminator checking pair consistency).
