# Exercises — ControlNet, LoRA & Conditioning

## Exercises

1. **Compute** the LoRA parameter reduction ratio for a given weight matrix. Write a Python script that takes a matrix dimension `d` and rank `r`, then prints the full parameter count (`d × d`), the LoRA parameter count (`2 × d × r`), the reduction ratio as a percentage, and the resulting merged weight shape for three stacked LoRAs. Run it for SDXL attention layers (`d=2048`) at ranks `r=8`, `r=32`, and `r=64`. Verify that each rank produces a monotonically increasing parameter count but all three remain under 5% of the original.

2. **Implement** classifier-free guidance extrapolation in numpy. Given two arrays — `eps_cond` (conditioned prediction) and `eps_uncond` (unconditioned prediction), each of shape `(B, C, H, W)` — write a function `cfg_extrapolate(eps_cond, eps_uncond, guidance_scale)` that returns `eps_uncond + guidance_scale * (eps_cond - eps_uncond)`. Test it on random tensors with `guidance_scale` values of `1.0`, `7.5`, and `15.0`, printing the mean and standard deviation of the output at each scale. Confirm that `guidance_scale=1.0` returns the conditioned prediction exactly and that higher scales amplify directional difference.

3. **Trace** the ControlNet data flow by implementing a minimal mock in numpy. Create a function `controlnet_forward(conditioning_input, cloned_encoder, residual_blocks)` where `conditioning_input` is a `(1, 3, 64, 64)` array, `cloned_encoder` is a callable that produces a list of 4 feature maps, and `residual_blocks` is a list of 4 zero-initialized convolution layers. The function should: pass the conditioning input through the encoder, apply each zero-conv layer to its corresponding feature map, and return the list of residual outputs. Print the output shapes at each stage and verify that on the first forward pass all residuals are near-zero (confirming the zero-conv initialization property that keeps ControlNet dormant at init).

4. **Configure** a multi-adapter generation pipeline for a brand-asset scenario using `diffusers`. Load `stabilityai/stable-diffusion-xl-base-1.0`, then attach a product-style LoRA (weight 0.8) and a quality-enhancement LoRA (weight 0.4) from HuggingFace paths of your choice. Add a Canny ControlNet adapter and feed it a product photo edge map. Generate a 1024×1024 branded hero image at guidance scale 7.5 with 30 steps. Save the output to `outputs/brand-asset-sdxl.png`. Print the pipeline's component list before generation so you can confirm all three adapters are registered.

5. **Compare** additive LoRA merging against full fine-tuning empirically. Write a script that: (a) loads an SDXL UNet and reports its total parameter count and on-disk size, (b) simulates three rank-16 LoRA adapters by instantiating random `(2048, 16)` and `(16, 2048)` numpy matrices and computing their combined storage cost, (c) estimates training time scaling by printing `lora_trainable / full_trainable` as a ratio, and (d) writes a markdown summary table with columns for storage (MB), trainable params, estimated relative training time, and a qualitative quality note. Save the comparison to `outputs/skill-lora-vs-finetune.md`. The table must contain rows for: full fine-tune, single LoRA at r=8, single LoRA at r=64, and three stacked LoRAs at r=16.
