# Exercises — Inpainting, Outpainting & Image Editing

## Exercises

1. **Implement** mask-based inpainting on a sample image. Load a 512×512 image, draw a filled rectangle over the center 25% of the canvas to create a binary mask (1 = regenerate, 0 = preserve), and run a diffusion inpainting pipeline conditioned on the prompt "a serene mountain lake." Save the result to `outputs/inpaint-rectangle.png` and print the mean pixel difference between the original and output within the masked region versus the unmasked region. Verify that the unmasked-region difference is near zero and the masked-region difference is substantial.

2. **Compare** naive compositing against latent-injection inpainting. For the same mask and prompt from Exercise 1, generate a standalone text-to-image result, crop it to the mask bounding box, paste it directly onto the original image at the mask location, and save to `outputs/naive-composite.png`. Then load your Exercise 1 output. Print the per-channel mean absolute difference along a 10-pixel-wide band straddling the mask boundary for both approaches. Confirm the naive composite shows a sharp discontinuity at the boundary while the inpainted result does not.

3. **Generate** an outpainting by extending a square 512×512 image to 768×512. Create a new 768×512 black canvas, paste the original image centered horizontally, construct a binary mask that covers only the left- and right-most 128-pixel columns, and run inpainting with a prompt describing a plausible scene extension. Save the result to `outputs/outpaint-wide.png` and print the mask coverage percentage to confirm it matches the expected ratio of new pixels.

4. **Build** an automated mask-generation function that takes an image and a target hex color range, applies color thresholding in HSV space to produce a binary mask, runs `cv2.findContours` to keep only the largest connected component, and returns the mask. Test it on a product screenshot where a solid-color logo sits on a contrasting background — mask out the logo region and inpaint a replacement using the prompt "clean white background." Save the mask to `outputs/auto-mask.png` and the inpainted result to `outputs/logo-removed.png`. Print the number of contours detected before and after filtering by area.

5. **Extend** your mask pipeline into a landing-page hero-image localization tool. Pull a screenshot of any SaaS landing page (use Playwright or a saved HTML file), detect the primary CTA button via color thresholding on its accent color, generate a mask around the detected button bounding box, and inpaint a replacement CTA with different button text rendered via a text-to-image prompt. Write the full pipeline as `handlers/cta-swap.py` that accepts a URL, a target hex color, and a replacement prompt as CLI arguments, then outputs the edited screenshot to `outputs/cta-swapped.png`. Print the original button's bounding-box coordinates and the mask pixel count.

6. **Design** and implement a decision-matrix evaluator that takes an editing task description (e.g., "replace background of product photo," "extend banner to 16:9," "change photo style to watercolor") and outputs a recommendation among inpainting, outpainting, img2img, and instruction-based editing based on the ratio of preserved to generated pixels, spatial precision requirements, and compute budget. Encode the rubric as a scored function in `signals/examples/editing-paradigm-selector.py` that reads task specs from a JSON file and prints a ranked recommendation table to stdout. Include at least five diverse task specs covering all four paradigms and verify each maps to the expected recommendation.
