## Exercises

1. **Compute FID between two image distributions.** Implement a function that loads all images from two directories (e.g., `data/real_images/` and `data/generated_images/`), resizes each to 299×299, extracts 2048-dimensional feature vectors from the penultimate layer of a pretrained Inception-v3 model, fits a multivariate Gaussian (mean μ and covariance Σ) to each set, and computes the Fréchet distance using the formula `||μ_r − μ_g||² + Tr(Σ_r + Σ_g − 2·(Σ_r·Σ_g)^½)`. Print the final FID score to the terminal. Verify that swapping the two directories produces the same score (distance is symmetric).

2. **Calculate CLIP Score for prompt-image pairs.** Build a script that loads a JSON file of `{"image_path": ..., "prompt": ...}` entries, encodes each image and prompt using a pretrained CLIP model, computes the cosine similarity between the image and text embeddings for each pair, and prints a per-pair score table followed by the mean CLIP Score across all pairs. Run it on a provided set of 10 prompt-image pairs and report which pair scores highest and which scores lowest.

3. **Implement a pairwise human preference protocol with forced-choice aggregation.** Build a CLI tool that loads a directory of images, presents randomized pairs to the user one at a time, forces the user to pick A or B (no ties allowed), records every vote, and after all pairwise comparisons are exhausted computes a win-rate matrix and prints a ranked leaderboard with each image's overall win rate. Include at least 6 images so the pairwise count is non-trivial. No scaffold — design the comparison scheduling, input handling, and aggregation from scratch.

4. **Construct a failure-mode demonstration where FID and CLIP Score disagree.** Create two small image sets: Set A containing 20 images that are photorealistic but semantically unrelated to a target prompt, and Set B containing 20 images that are noisy/blurry but semantically on-topic. Compute FID of each set against a photorealistic reference set and compute CLIP Score of each set against the target prompt. Print both metrics side-by-side and write a short terminal-printed analysis (3–5 sentences) explaining which metric captures which failure mode and why they disagree.

5. **Build a GTM creative evaluation pipeline using Apollo-enriched data.** Export a list of 20 companies from Apollo (or use a provided `data/apollo_companies.json` containing company names and descriptions). Generate or collect one marketing image per company. Compute CLIP Score between each image and its company description, compute FID of the full image set against a `data/brand_reference/` directory of on-brand reference images, and run a simulated pairwise preference vote (using a heuristic or human input) across all 20 images. Produce a ranked evaluation report at `outputs/skill-creative-eval.md` containing all three metric results per image, a final ranking, and a recommendation for which creatives to ship.
