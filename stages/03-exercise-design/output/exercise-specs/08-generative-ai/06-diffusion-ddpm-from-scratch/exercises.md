## Exercises

1. **Implement** the closed-form forward diffusion sampler. Given a 1D tensor of 500 samples drawn from `N(0.5, 0.3)`, a linear beta schedule with `β_start=0.0001`, `β_end=0.02`, and `T=1000` steps, write a function that computes `x_t` directly from `x_0` at any timestep using `ᾱ_t`. Apply it at timesteps `t ∈ {10, 100, 500, 999}` and print the mean and standard deviation of the corrupted samples at each timestep. Confirm that mean drifts toward 0 and standard deviation drifts toward 1 as `t` approaches `T`.

2. **Configure** two linear noise schedules — one with `β_end=0.02` and another with `β_end=0.2`, both with `T=1000`. Compute `ᾱ_t` for both schedules and print a table (timestep, `ᾱ_t` slow, `ᾱ_t` fast) at every 100th timestep. Identify and print the timestep at which each schedule crosses `ᾱ_t < 0.5`. Explain in a comment what this crossing point means for signal retention.

3. **Implement** the full reverse sampling loop given a pre-trained noise prediction model (use a provided checkpoint or a simple identity stub that returns the input as the predicted noise). Starting from `x_T ~ N(0, I)` with 1000 samples of dimension 2, run 1000 denoising steps using the DDPM update rule `x_{t-1} = (1/√α_t)(x_t - (β_t/√(1-ᾱ_t)) · ε_θ(x_t, t)) + σ_t · z`. Print the mean and covariance matrix of the final samples. Vary the number of sampling steps used (50, 100, 500, 1000) and print how the output statistics change.

4. **Build** a minimal noise-prediction network in PyTorch that accepts a noisy input `x_t` and an integer timestep `t`. Use sinusoidal positional embeddings for the timestep (same formulation as the Transformer paper), project it to the same dimension as `x_t`, concatenate, and pass through 3 fully-connected layers with ReLU activations. Instantiate the network for 2D inputs and 1000 timesteps. Pass a batch of 16 random samples at random timesteps through it, print the output shape, and verify it matches `(16, 2)`. Print a single forward pass's output values for `x_t = [0.5, -0.3]` at `t = 1` and `t = 999` to confirm the timestep embedding changes the output.

5. **Train** a complete DDPM from scratch on a 2D Gaussian mixture (three modes at `[2, 2]`, `[-2, 2]`, and `[0, -2]`, each with `σ=0.3`). Use the forward closed-form sampler, the network from Exercise 4, and Adam optimizer with learning rate `1e-3`. Train for at least 5000 steps with batch size 256. After training, generate 1000 samples via the reverse loop and compute the mean, standard deviation, and a coarse 2D histogram (10×10 bins) of generated versus training samples. Write the full training curve, final statistics comparison, and histogram to `outputs/skill-ddpm-evaluation.md`.

6. **Apply** the diffusion framework to a GTM scenario: train a DDPM on a real customer feature dataset. Pull 500+ company records from Apollo's API or a CSV export (use features: employee count, founding year, and a normalized tech stack score from 0 to 1). Standardize all features to zero mean and unit variance. Train the DDPM for 3000 steps, then generate 200 synthetic look-alike company profiles. Compute the Wasserstein distance between real and generated feature distributions for each dimension. Print the 5 generated samples that are furthest from any real training point (potential edge-case leads). Save the trained model, generation function, and evaluation metrics to `handlers/synthetic_lead_generator.py`.
