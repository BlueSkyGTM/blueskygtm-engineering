# Exercises — Visual Autoregressive Modeling (VAR): Next-Scale Prediction

## Exercises

1. **Compute token counts for a multi-scale VAR pyramid versus raster-scan ordering.** Implement a function that, given a target resolution (e.g., 256×256) and a starting scale of 1×1, enumerates each scale map size (1×1, 2×2, 4×4, …, 256×256), computes the per-scale token count, and prints a formatted table showing scale index, resolution, tokens at that scale, and cumulative tokens. Then compute the raster-scan total (256 × 256 = 65,536) and print the ratio of VAR cumulative tokens to raster-scan tokens. Run the script and verify the ratio is well below 1.0.

2. **Construct and print the block-causal attention mask for a 3-scale VAR configuration (1×1, 2×2, 4×4).** The flattened token sequence has 1 + 4 + 16 = 21 positions. Build a 21×21 binary matrix where entry (i, j) = 1 if token i may attend to token j (i.e., j is in the same scale as i or in an earlier scale), and 0 otherwise. Print the matrix as a grid of `#` and `.` characters to the terminal. Verify that the diagonal blocks are all-ones (bidirectional within scale) and the upper-right blocks are all-zeros (no future-scale leakage).

3. **Implement a multi-scale VQ-VAE tokenizer that converts an image into K discrete token maps.** Write a script that loads an image (use any 256×256 RGB image — provide a URL or local path), downsamples it to each scale in {1, 2, 4, 8, 16, 32}, quantizes each downscaled feature map against a randomly initialized codebook of 512 entries (dimension 256), and outputs the integer codebook indices per scale. Print the shape and unique-token-count of each scale's index map. No scaffold; use any framework (PyTorch, NumPy, or JAX).

4. **Compute and plot the context-growth curves for VAR versus raster-scan AR across resolutions from 32×32 to 1024×1024.** For each resolution, compute the VAR total context length (sum of $S_k^2$ across all scales up to that resolution) and the raster-scan context length ($H \times W$). Print a table of resolution, VAR tokens, raster tokens, and ratio. Then answer: at what resolution does the VAR advantage exceed 10×? Apply this to a GTM scenario — pull 5 company logos from the Clearbit Logo API (`https://logo.clearbit.com/{domain}`) and report which generation approach (VAR vs. raster AR) would be more efficient for batch-generating 1024×1024 ad creative variants from those logos.

5. **Build a minimal end-to-end VAR inference pipeline and run it on a small image.** Implement: (a) a multi-scale tokenizer that encodes an image into K token maps, (b) a block-causal transformer that autoregressively predicts each scale conditioned on previous scales (use a small model — 4 layers, 256-dim, 4 heads), and (c) a decoder that reconstructs the image from predicted tokens. Run the pipeline on a 64×64 image (6 scales: 1×1 through 64×64), print the per-scale prediction latency, and save the reconstructed image to disk. **Artifact: `handlers/var_pipeline.py`** — must be runnable as `python handlers/var_pipeline.py --image <path>` and produce a decoded output image plus a latency report printed to stdout.
