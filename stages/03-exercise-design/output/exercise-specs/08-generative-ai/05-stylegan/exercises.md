# Exercises — StyleGAN

## Exercises

1. **Implement** the mapping network `f: Z → W` as an 8-layer MLP using PyTorch. Generate three random `z` vectors from `N(0, I₅₁₂)`, pass them through the network, and print the shape and per-element mean/std of the resulting `W` vectors. Verify that the output is `R³ˣ⁵¹₂` and that the statistics deviate from a standard Gaussian (non-zero mean, non-unit std), confirming the mapping network warps `Z` into a non-Gaussian manifold.

2. **Trace** a single `w` vector through the first two resolution blocks (4×4 and 8×8) of the synthesis network. At each block, print the tensor shape before and after AdaIN normalization, and print the tensor shape after noise injection. Use `print()` statements to label each stage: `Pre-AdaIn`, `Post-AdaIn`, `Post-Noise`. Confirm that AdaIN changes channel statistics (mean/std shift) while noise injection adds a single-channel additive term with shape matching spatial resolution.

3. **Implement** linear interpolation in `W` space between two randomly sampled vectors `w₁` and `w₂`. Compute 10 interpolated points at `t = 0.0, 0.111, ..., 1.0`, measure the L2 distance between each consecutive pair, and print a table showing `t`, interpolated vector norm, and consecutive L2 distance. Verify that consecutive L2 distances are approximately equal (confirming linear interpolation produces uniform spacing in `W` space). Then repeat the same interpolation in `Z` space and compare the L2 distance distributions — print both side by side.

4. **Execute** style mixing by generating two latent vectors `w_coarse` and `w_fine`. Run the synthesis network twice: once with `w_coarse` injected at resolutions 4–32 and `w_fine` injected at resolutions 64–1024, and once with a single `w` throughout. Save both output images to `outputs/` and compute the pixel-space L2 difference between them. Print the difference value and a one-line summary stating whether coarse-to-fine mixing produced a visible structural change (large L2) or a texture-level change (small L2).

5. **Compare** the three StyleGAN generations by writing a Python script that prints a structured table to the terminal with columns: Version, Key Mechanism, Artifact Eliminated, Year. Fill in: StyleGAN1 (progressive growing → eliminates training instability from resolution jumps), StyleGAN2 (weight demodulation → eliminates blob/droplet artifacts from AdaIN normalization), StyleGAN3 (alias-free synthesis → eliminates texture sticking to phase aliases). Then extend the table with a fourth row applying the same "what artifact does this eliminate?" framing to a CRM context: map disentangled latent directions in StyleGAN to separable attributes in customer embeddings. Write a short script that generates a synthetic customer embedding matrix (`N=500, d=128`), applies PCA to identify the top-3 variance directions, prints the explained variance ratio per component, and labels each component with a proxy attribute (e.g., company size, engagement score, industry). Save the labeled PCA summary to `outputs/skill-latent-crm-mapping.md`.

6. **Build** a complete latent-space explorer script at `signals/examples/latent_explorer.py` that (a) samples a `w` vector, (b) performs a random walk in `W` space for 20 steps with step size `ε = 0.05`, (c) at each step computes the cosine similarity to the starting vector and the L2 distance, (d) prints a formatted trajectory table, and (e) identifies the step at which cosine similarity first drops below 0.5 — marking the boundary where the generation has meaningfully diverged. Add a final block that maps this concept to GTM: generate a synthetic customer embedding, perform the same random walk, and report the step at which a nearest-neighbor classifier would assign the customer to a different segment. The script must run end-to-end via `python signals/examples/latent_explorer.py` and produce all output in the terminal.
