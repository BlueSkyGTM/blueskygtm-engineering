# Exercises — 3D Generation

## Exercises

1. Implement a script that computes and prints total parameter counts for a single object represented four ways: a 10,000-point point cloud, a 5,000-face mesh, a NeRF with a 4-layer MLP at 256 hidden units, and a Gaussian splat with 50,000 gaussians (each storing position, rotation as quaternion, scale, opacity, and spherical harmonic coefficients up to degree 3). Print a formatted comparison table showing parameter count and approximate bytes per representation. Verify the ordering matches the tradeoff discussed in the lesson.

2. Using Point-E, load the text-to-pointcloud model and generate a point cloud from the prompt "a coffee mug." Fit a mesh to the resulting point cloud using the model's `pc_to_mesh` upscale model. Export both the raw point cloud (as PLY) and the fitted mesh (as OBJ) to `outputs/`. Print the generation time for each stage and the final vertex/face counts of the mesh.

3. Implement a function that performs a single iteration of the SDS loop manually: initialize a small NeRF (or a tiny Gaussian splat scene) with random parameters, render one camera view, add noise at a specified timestep to the rendered image, pass the result through a pretrained 2D diffusion model (e.g., Stable Diffusion via `diffusers`), compute the gradient of the noise-prediction error with respect to the rendered image, and backpropagate into the 3D parameters. Print the scalar loss value and the L2 norm of the gradient on the 3D parameters after one step. Do not run full optimization — just trace one iteration and verify the gradient flows.

4. Write a mesh inspection script that loads any OBJ file and reports: (a) count of non-manifold edges (edges shared by more than two faces), (b) count of degenerate faces (zero or near-zero area), (c) count of boundary edges (edges shared by only one face), and (d) a Janus heuristic flag that checks whether more than one face normal points toward the camera from a given viewpoint after normalization. Test your script on three meshes you generate from the prompts "a chair," "a car," and "a face" using either Shap-E or the text-to-mesh pipeline from the lesson. Print a structured report per mesh.

5. Build a complete text-to-mesh pipeline for a GTM use case: given a list of product names from a CRM export (CSV with a `product_name` column), generate a 3D mesh for each product, export each as GLB to `outputs/product_assets/`, and render 8 orbital preview images (45-degree increments around the Y axis) per product into `outputs/product_assets/previews/`. Use Shap-E or Point-E for generation given the latency budget of batch enrichment. Produce a final `outputs/product_catalog_3d.json` manifest mapping each product name to its GLB path and preview image paths. Demonstrate the pipeline on a sample CSV of at least 5 products.

6. Design and run a benchmark comparing direct regression (Point-E or Shap-E) against SDS-based optimization (DreamFusion-style or using an existing SDS implementation) across 5 text prompts of varying complexity. For each prompt and method, measure wall-clock generation time, final mesh vertex count, and a CLIP-similarity score between the rendered preview and the original text prompt. Produce a comparison report at `outputs/skill-3d-gen-benchmark.md` containing a markdown table, a recommendation for when each method is preferred, and a brief analysis of where each approach fails qualitatively (Janus artifacts, oversaturation, blurriness, missing parts).
