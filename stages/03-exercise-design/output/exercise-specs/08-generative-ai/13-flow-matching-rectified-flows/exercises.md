# Exercises — Flow Matching & Rectified Flows

## Exercises

1. **Implement** a conditional flow matching training loop on a 2D two-moons dataset. Define a simple MLP velocity network `v_θ(x, t)` with `t` concatenated to the 2D input. Train for at least 2000 iterations using the straight-line conditional velocity target `u_t(x | x_0, x_1) = x_1 - x_0` where `x_t = (1-t)*x_0 + t*x_1`. After training, generate 500 samples by integrating the learned ODE with 20-step Euler from standard Gaussian noise. Print the mean and standard deviation of generated samples and compare them against the true two-moons statistics (mean and std of the dataset).

2. **Compute** and **Compare** sample quality across 1-step, 5-step, and 20-step Euler integration using your trained model from Exercise 1. For each step count, generate 1000 samples and report the Maximum Mean Discrepancy (MMD) between generated and real data points using a Gaussian RBF kernel (you may use `scipy.spatial.distance` or a manual implementation). Print a table showing step count, MMD value, and wall-clock generation time. Observe how the MMD changes and whether 1-step generation is already competitive for straight-line paths.

3. **Implement** the reflow procedure. Starting from your trained flow model, generate pairs `(x_0, x_1)` where `x_0` is noise and `x_1` is the 20-step Euler output from the first model. Train a *second* velocity network from scratch on these paired samples using the same straight-line conditional objective. After training the distilled model, repeat the MMD comparison from Exercise 2 at 1, 5, and 20 steps. Print both models' results side by side. Quantify path straightness by computing the average deviation of intermediate points from the linear interpolation: for each generated trajectory `x_0, x_{1/k}, ..., x_1`, compute `mean(||x_{i/k} - ((1-i/k)*x_0 + (i/k)*x_1)||)` and print the value for both the original and reflowed models.

4. **Apply** flow matching to a different 2D distribution: an S-curve generated with `sklearn.datasets.make_s_curve` (project to 2D by dropping the third dimension, then normalize). Train a velocity network using straight-line conditional flow matching. Then **implement** a curved interpolation path as an alternative: use the quadratic Bezier path `x_t = (1-t)^2 * x_0 + 2*(1-t)*t * x_mid + t^2 * x_1` where `x_mid` is sampled from a separate Gaussian centered at the midpoint of `x_0` and `x_1`. Derive the conditional velocity field for this path and train a separate model with it. For each model, find the minimum number of Euler steps needed to reach an MMD below 0.01 from the real S-curve distribution. Print both step counts and discuss which path converges faster.

5. **Build** a flow matching model that generates synthetic B2B company firmographic profiles from noise. Download a sample of company records from Apollo's API (or use a provided CSV at `data/apollo_companies.csv` containing `employee_count`, `revenue`, `founding_year`, and `industry_category` columns). Normalize all features to zero mean and unit variance. Train a velocity network on this 4-dimensional data using straight-line conditional flow matching. Generate 200 synthetic company profiles and write them to `outputs/synthetic_companies.csv`. Then **Implement** a 1-step vs. 20-step comparison and report MMD for each. Additionally, run a quick enrichment check: for 10 randomly sampled synthetic companies, call a free enrichment API (e.g., `companiesurl.com` or any free firmographic endpoint) and print whether the generated employee_count and revenue ranges are plausible for the returned company name/website. Save a summary report to `outputs/flow_matching_gtm_report.md` containing the MMD table, 5 sample synthetic profiles, and the enrichment results. Artifact: `outputs/flow_matching_gtm_report.md`.

6. **Design** an experiment that determines whether reflow provides diminishing returns beyond a single iteration. Implement multi-round reflow: train an initial model, generate paired data, train model v2 (1st reflow), generate new paired data from v2, train model v3 (2nd reflow). For each model version, measure (a) path straightness using the linear-deviation metric from Exercise 3 and (b) 1-step generation MMD. Plot or print a table showing how these metrics evolve across rounds. Extend the analysis by also testing whether a curved initialization path combined with one round of reflow can match the straight-line-plus-reflow baseline. Write your complete implementation, including the training loops, reflow procedure, metric computation, and a summary of findings, to `outputs/flow_matching_rectified_flow_study.md`. State your conclusion: at which round does 1-step quality plateau, and is there evidence that reflow on curved paths recovers the same straight-line efficiency? Artifact: `outputs/flow_matching_rectified_flow_study.md`.
