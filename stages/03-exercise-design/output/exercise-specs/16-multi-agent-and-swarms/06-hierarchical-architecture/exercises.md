# Exercises — Hierarchical Architecture and Its Failure Mode

## Exercises

1. **Implement** a two-level LLM call hierarchy for account research. Create a root node that dispatches to three leaf nodes: one that summarizes a company's industry, one that summarizes their tech stack, and one that summarizes recent funding. Each leaf prints its raw output with a `--- LEAF [name] ---` header. The root receives all three summaries and prints the final synthesized answer with a `--- ROOT ---` header. Run the script and confirm you see five distinct labeled sections in the terminal: three leaf outputs, then the root synthesis.

2. **Trace** output propagation by adding a `depth` parameter to each node call. Modify your hierarchy from Exercise 1 so that every node prints its depth level (`depth=0` for root, `depth=1` for leaves) and the number of tokens in its output (use `len(output.split())` as a proxy). Run the script and verify that token count decreases as depth decreases — leaf outputs should be longer than the root summary. Record the token counts for each node in a comment at the top of your script.

3. **Compute** embedding cosine distance between each leaf output and the final root summary to detect context dilution. Use the `sentence-transformers` library (`all-MiniLM-L6-v2`) to embed both the leaf outputs and root output from Exercise 1. Write a script that prints a table: leaf name, cosine similarity to root, and a flag (`DILUTED` if similarity < 0.5, `PRESERVED` otherwise). Run it and verify at least one leaf is flagged. Then modify your root prompt to preserve more detail from leaves and re-run to see if the flag changes.

4. **Compare** three architectures — hierarchical decomposition, flat fan-out (single prompt that does everything), and iterative refinement (chain where each call refines the previous output) — on the same task: summarizing a 2-3 paragraph company description into a one-sentence value proposition. Implement all three, measure wall-clock latency (`time.time()`) and total token usage for each. Print a comparison table to the terminal showing: architecture name, latency in seconds, total tokens, and the final output sentence.

5. **Build** an automatic collapse detector that wraps a hierarchical synthesis node. When a parent synthesizes its children's outputs, the detector computes the average cosine similarity between each child output and the parent's synthesis. If the average drops below a threshold (default 0.4), the detector flattens the subtree: it returns the children's outputs directly instead of the lossy synthesis, and logs `COLLAPSE DETECTED at node [name], bypassing synthesis`. Save the detector as a reusable class in `handlers/collapse_detector.py`. Run it against a hierarchy with 4+ leaf nodes and confirm it prints collapse events for at least one node when the threshold is low enough.

6. **Design** a diagnostic script that maps hierarchical failure modes to a Clay-style enrichment waterfall. Simulate a 4-stage waterfall: (1) company domain lookup, (2) employee count enrichment, (3) tech stack detection, (4) intent signal scoring. At each stage, inject a stale-data probability (e.g., stage 2 has a 30% chance of returning cached data older than 90 days). Run 50 simulated enrichment passes and print a report showing: which stages most frequently propagated stale data downstream, which final outputs were corrupted by upstream staleness, and the correlation between stage-2 staleness and final intent score accuracy. Save the report to `outputs/skill-enrichment-failure-map.md`.
