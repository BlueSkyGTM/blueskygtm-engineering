# Exercises — Generative Agents and Emergent Simulation

## Exercises

1. **Implement the memory stream data structure and populate it.** Build an `Observation` dataclass with fields for `description`, `timestamp`, `entry_type`, and `importance`. Write a `MemoryStream` class with an `add(description, entry_type, importance)` method that appends entries and a `__repr__` that prints the last N entries. Seed it with at least 8 observations (mix of types and importance scores), then call `print(stream)` to display the full log in your terminal.

2. **Compute and print retrieval scores for a fixed query across your memory stream.** Implement the `recency × importance × relevance` scoring function from the lesson. For recency, use an exponential decay half-life of your choice. For relevance, use a simple keyword-overlap proxy (no embedding API required) so the exercise runs offline. Query your stream with two different strings — one that matches recent entries and one that matches older but high-importance entries — and print a ranked table showing each memory's three component scores and final product. Confirm that the top-ranked memory differs between the two queries.

3. **Implement reflection triggered by a cumulative importance threshold.** Write a function that scans the memory stream from the oldest unchecked observation forward, sums importance scores, and fires a reflection once the cumulative total crosses 30. The reflection should call your LLM (or print a placeholder prompt if no API key is set) to synthesize those observations into a single insight, then append that insight back into the stream as a `reflection`-type entry with its own importance score. Run the simulation, then print the stream and verify that reflection entries appear at the correct positions.

4. **Run a controlled A/B simulation with and without reflection.** Set up two agent instances sharing the same seed observations but one with reflection enabled and one with reflection disabled. Feed both agents the same sequence of 12 new observations (a mix of mundane events and high-importance events). After the final observation, query both agents with the same prompt and print each agent's top-5 retrieved memories side by side. Report the overlap count — how many of the top-5 are identical — as a single integer.

5. **Build a buying-committee simulation using GTM prospect data.** Fetch 5–10 contacts from Apollo's People Search API (or load them from a CSV export of Clay enrichment data) and seed one agent per contact with observations derived from their role, industry, and firmographic signals. Implement the full generative agent loop: memory stream, retrieval, reflection, and a planning step that asks the LLM "What is this stakeholder's next likely concern about adopting our product?" Run the loop for 3 simulated time steps, then write each agent's final plan and last reflection to `outputs/skill-buying-committee-sim.md`, formatted as a markdown table with one row per stakeholder.

6. **Design and implement an ICP-fit scoring agent that extends the generative agent loop.** Create an agent whose memory stream is seeded with closed-won and closed-lost CRM deal data (import from a CSV with at least columns: `company`, `deal_size`, `outcome`, `sales_cycle_days`, `notes`). Add a custom retrieval function that weights relevance to a target prospect's firmographics. After every 5th new memory, trigger a reflection that asks the LLM to extract a pattern (e.g., "Companies under 50 employees in fintech with sales cycles under 30 days tend to convert"). Write the agent's accumulated reflections as executable Python signal logic in `handlers/icp-pattern-agent.py`, where each reflection becomes a scored boolean check function. Run the handler against a held-out prospect and print the composite ICP-fit score.
