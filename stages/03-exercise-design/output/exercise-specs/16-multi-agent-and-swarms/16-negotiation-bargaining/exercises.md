# Exercises — Negotiation and Bargaining

## Exercises

1. **Compute** the Rubinstein alternating-offers equilibrium split for a seller with discount factor δ₁ = 0.85 and a buyer with discount factor δ₂ = 0.70, negotiating over a $10,000 deal. Write a Python script that takes both discount factors and the surplus amount as arguments, prints the equilibrium offer each agent would make when it is their turn to propose, and confirms the result matches the closed-form solution. Run it for three additional δ pairs and print all four results in a table.

2. **Implement** an OG-Narrator-style decomposition for a single offer: a deterministic function that takes a numeric price and returns a structured offer dict (price, term length, payment schedule, concessions included), and a separate narration function that converts that dict into a natural-language sales message. Call the deterministic generator with three different price points and print both the raw dict and the narrated string for each so you can verify the narration never changes the underlying numbers.

3. **Build** a two-agent single-issue negotiation loop where a seller and buyer alternate offers over price. Each agent has a hard reservation price (seller floor, buyer ceiling). The seller decreases by a fixed step each round; the buyer increases by a fixed step each round. The loop terminates when offers cross (deal) or a max-rounds limit is hit (walk-away). Print every round's offers and the final outcome. Run with seller_floor=4000, buyer_ceiling=6000, and confirm the loop converges.

4. **Apply** the negotiation loop from Exercise 3 to a GTM scenario using CRM deal data. Generate a synthetic pipeline of 20 opportunities where each deal has a random ZOPA (some with no overlap). Run the structured bargaining loop over all 20 and compute deal rate, average closing price, and average rounds-to-close. Then run a naive LLM-only baseline where each agent is prompted as a free-form chatbot with the same reservation prices and no structured step logic. Print a comparison table showing deal rate, avg price, and avg rounds for both approaches side by side.

5. **Simulate** incomplete-information bargaining with Bayesian belief updates. A seller does not know the buyer's reservation price but starts with a prior distribution (e.g., uniform over [$3,000, $8,000]). After each rejected offer, the seller updates the posterior by truncating the distribution above the rejected price (the buyer would have accepted if their ceiling were that high). After each accepted offer, record the final price. Run 100 simulated negotiations with buyer ceilings drawn from the prior, track the seller's posterior mean at each round, and write the full simulation including the update logic and a results summary to `handlers/bayesian_bargaining.py`. Print the mean closing price, deal rate, and the average number of rounds.

6. **Design** a complete negotiation architecture that combines the Rubinstein equilibrium, OG-Narrator decomposition, and Bayesian belief updating into a single system that negotiates SaaS pricing against a live enrichment API (Apollo or Clay). Your system should: pull 10 company profiles, estimate a per-seat budget from firmographics, initialize the seller's prior over each prospect's willingness-to-pay, run a multi-round negotiation where offers are generated deterministically from the equilibrium formula and narrated via LLM, and update beliefs after each rejection. Write the full system to `signals/examples/saas_negotiation_sim.py` and a methodology writeup to `outputs/skill-negotiation-architecture.md`. Print a final report showing per-deal outcomes, belief evolution, and system-level deal metrics.
