# Exercises — Society of Mind and Multi-Agent Debate

## Exercises

1. Implement a debate loop with 3 LLM instances and 2 rounds. Each agent receives a factual question (e.g., "What is the population of Singapore as of 2023?"), produces an initial answer, then sees the other two agents' answers and revises. After round 2, aggregate via majority vote and print the final answer alongside each agent's round-1 and round-2 outputs. Run the script and confirm terminal output shows all 6 intermediate responses plus the aggregated result.

2. Run a head-to-head comparison on the same prompt: once with a single LLM call, once with the 3-agent debate loop from Exercise 1. Use the prompt: "Explain why customer churn prediction models often fail in B2B SaaS companies." Print both outputs to the terminal with a separator. Manually inspect: does the debate version surface considerations the single-agent version misses? Note your observation in a comment.

3. Implement a debate system with 4 agents, each assigned a distinct persona via system prompt (e.g., "skeptical data scientist," "optimistic growth marketer," "risk-averse compliance officer," "pragmatic CFO"). Include a convergence-based termination condition: if all 4 agents' answers are semantically equivalent (use exact string match or a similarity threshold on embeddings) before the max round count, stop early. Print which round convergence occurred on, or state "no convergence" if the max rounds elapsed. Test on the question: "Should a Series A startup invest in a CDP in year one?"

4. Build an answer-drift tracker that logs each agent's response at every round and computes the average pairwise edit distance between consecutive rounds for each agent. After the debate completes, print a per-agent drift score and a flag if any agent's drift exceeds a threshold you define (e.g., average Levenshtein distance > 50 characters between rounds). Run this on a math reasoning problem from GSM8K and identify whether drift correlates with correctness degradation. Print a one-line verdict: "Debate improved answer" or "Debate degraded answer."

5. Build a multi-perspective ICP scoring pipeline where three adversarial agents evaluate a list of 5 enriched company profiles (use a static JSON dataset with fields like company name, employee count, funding, tech stack, recent news). Agent 1 ("Enrichment Optimist") argues for high ICP fit. Agent 2 ("Data Skeptic") challenges the freshness and reliability of enrichment signals. Agent 3 ("Revenue Auditor") evaluates whether the profile maps to actual budget authority and pain points. Run 2 debate rounds, then produce a final ICP score (0–100) per company derived from agent consensus. Save the scored output as a markdown table to `outputs/skill-icp-scoring.md`. Each company row must include the final score, the dominant dissenting argument, and which agent flagged the biggest risk.

6. Design and run an experiment that measures when multi-agent debate helps versus hurts across a mixed prompt set of 10 questions: 5 factual recall (trivia, lookup-style), 5 open-ended reasoning (strategy, tradeoffs, GTM scenarios). For each prompt, run both single-agent inference and a 3-agent, 3-round debate. Compute answer drift per prompt and manually label whether the debate output is better, worse, or equivalent to single-agent. Write your findings — including which prompt category benefits most and your hypothesis for why — to `outputs/skill-debate-analysis.md`. The markdown must include a summary table with columns: prompt, category, single-agent output, debate output, drift score, verdict.
