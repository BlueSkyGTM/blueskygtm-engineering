# Exercises — Shared Memory and Blackboard Patterns

## Exercises

1. **Implement** a `SharedMemory` class with `get`, `set`, and `increment` methods. Spawn 10 threads that each call `increment("visits", 1)` 1000 times on a plain dictionary-backed store (no locks). Print the final value of `visits` and confirm it is less than 10,000. Then add a `threading.Lock` around the read-modify-write cycle in `increment`, re-run, and confirm the final value is exactly 10,000.

2. **Build** a `Blackboard` dictionary and three specialist functions (`email_specialist`, `linkedin_specialist`, `phone_specialist`) that each read the blackboard, check whether their target field is missing, and write it if so. Write a `BlackboardController` with a `run(blackboard)` method that iterates over the specialists, dispatches each one, and loops until no specialist reports a change. Feed it `{"company": "Acme", "domain": "acme.com"}` and print the final blackboard state after the controller terminates.

3. **Apply** the blackboard pattern to a lead-scoring problem. Create a blackboard with fields `company`, `employees`, `tech_stack`, `funding_rounds`, and `score`. Implement two specialists: a `FirmographicsSpecialist` that fills `employees` and `funding_rounds` from a local JSON file you create at `signals/examples/firmographics.json`, and a `ScoringSpecialist` that computes `score` once both fields are populated. Run the controller and print the final blackboard. No scaffold — design the field-completeness checks yourself.

4. **Implement** provenance tracking on a `SharedMemory` store. Modify `set(key, value)` to store a tuple of `(value, source_agent, confidence)` and add a `get_with_provenance(key)` method. Write a `PoisoningDetector` class that scans all keys and flags any entry where `confidence < 0.5` or where a key's value was last written by an agent on a denylist (`["scraper_v0", "unverified_bot"]`). Seed the store with 5 entries from different agents and confidence levels, run the detector, and print every flagged key with its source and confidence.

5. **Build** an enrichment waterfall pipeline modeled as a blackboard. Create a `LeadBlackboard` with a record `{"email": "jane@acme.com"}` and fields `first_name`, `last_name`, `title`, `linkedin_url`, `company_domain` all initially `None`. Implement three specialist agents that each try to fill specific fields: a `ClearbitSpecialist` (fills `first_name`, `last_name`, `company_domain` — use the Clearbit free name-to-domain API at `https://autocomplete.clearbit.com/v1/companies/suggest?query=acme`), a `ApolloSpecialist` (fills `title`, `linkedin_url` — use the Apollo person-enrich endpoint with an API key from the `APOLLO_API_KEY` environment variable, or fall back to a local `signals/examples/apollo_cache.json` if the key is unset), and a `ConfidenceScorer` that assigns a confidence of 0.9 to Clearbit-sourced fields and 0.7 to Apollo-sourced fields. Run a `BlackboardController` that loops until no specialist writes new data or 3 iterations pass. Write the final enriched record as structured Markdown to `outputs/skill-enrichment-blackboard.md`. Print the record and the iteration count to the terminal.

6. **Design** a proof-of-concept that demonstrates why shared memory fails for concurrent enrichment writes but a blackboard controller does not. Create a scenario where 4 specialist agents write to overlapping keys on a `SharedMemory` store simultaneously (e.g., two agents both write `title` with different values from different providers). Run the concurrent write, print the final store, and show that the last-writer-wins result is arbitrary. Then implement a `BlackboardController` with an explicit priority order (`apollo > clearbit > manual > guessed`) and a provenance-aware write rule: a specialist may overwrite a field only if the new entry's confidence is higher than the existing entry's confidence. Run the same inputs through the controller sequentially and print the final state. Write a 200-word comparison of collision risk, control flow, and read/write semantics between the two approaches to `outputs/skill-topology-comparison.md`.
