# Exercises — Voting, Self-Consistency, and Debate Topology

## Exercises

1. Implement a majority voting aggregator. Write a function that takes a prompt and a sample count N (default 10), calls the LLM N times at temperature 0.7 with `max_tokens=5`, collects the raw label from each call, and returns a `Counter` of votes plus the winning label. Run it against the prompt `"Classify this company's stage as 'seed', 'series-a', 'series-b', or 'growth': A YC S23 startup with 8 employees and a $2M ARR."` Print the full vote tally and the winner to the terminal.

2. Implement self-consistency aggregation for the same classification task. Change the prompt to request chain-of-thought reasoning ending with `ANSWER: <label>`, set `max_tokens=200`, and extract the label using the `ANSWER:` delimiter. Sample N=10 times, group by extracted label, and print the vote distribution alongside the distribution from Exercise 1 so you can visually compare whether reasoning paths change the mode or tighten the spread.

3. Implement a 3-agent debate topology. Each agent independently classifies the same company from Exercise 1. In round 2, each agent receives the other two agents' labels and a one-sentence justification, then re-classifies. In round 3, agents see all round-2 outputs and give a final answer. Print each agent's response at each round so the convergence trajectory is visible in the terminal. Report whether all three agents agree by round 3.

4. Write a function `detect_failure_mode(vote_counts: Counter, expected_labels: set) -> str` that inspects a vote distribution and returns one of four strings: `"tied_votes"`, `"extraction_error"`, `"premature_consensus"`, or `"healthy"`. Use these rules: tied_votes if the top two labels have equal counts; extraction_error if more than 30% of samples produced a label not in `expected_labels`; premature_consensus if a single label won 100% of votes at temperature above 0.5 (suspicious unanimality). Test the function against six hand-constructed `Counter` objects — one for each failure mode plus two healthy cases — and print the predicted label for each.

5. Build a self-consistency ICP qualifier that wraps enrichment data from Apollo's people or companies endpoint. Fetch 5 company records (or use a provided JSON fixture), then for each company run self-consistency with N=5 CoT samples answering: `"Does this company match the ICP? Criteria: 50+ employees, US-based, uses Salesforce. Respond with QUALIFY or DISQUALIFY and a one-line reason. End with VERDICT: <QUALIFY|DISQUALIFY>."` Apply a configurable confidence threshold (default 0.8) — if the winning verdict's vote share falls below threshold, flag the company for manual review. Print each company's name, verdict, confidence score, and whether it passed threshold or was flagged. Save the qualifier as `handlers/icp_consensus_qualifier.py`.

6. Run a full three-method benchmark on a shared dataset of 20 company descriptions (generate them or load from a fixture). For each company, run majority voting (N=10), self-consistency (N=10 CoT samples), and 3-agent debate (3 rounds). Compute and record: the final label each method produced, the vote distribution margin (winner's count minus runner-up's count), total tokens consumed, and whether all three methods agreed on the label. Write the results as a Markdown table to `outputs/skill-aggregation-benchmark.md` with one row per company and a summary row reporting average token cost and overall three-method agreement rate across the dataset.
