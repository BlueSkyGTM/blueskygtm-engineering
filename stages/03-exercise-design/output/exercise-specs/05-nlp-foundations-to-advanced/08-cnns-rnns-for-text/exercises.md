# Exercises — CNNs and RNNs for Text

## Exercises

1. **Implement a width-3 1D convolution over a token sequence.** Take the embedding tensor from the lesson (shape `[1, seq_len, embed_dim]`), define a single filter of width 3 with random weights, and compute the dot product at every valid position. Print the raw activation vector and the index of the maximum activation. Then add two more width-3 filters with different random seeds and print all three activation vectors side by side. Confirm that each filter produces a different activation pattern across positions.

2. **Build a vanilla RNN cell from scratch.** Initialize `W_h`, `W_x`, and `b` as small random matrices. Feed a 5-token sequence of 8-dimensional embeddings through the cell one timestep at a time, printing `h_t` at every step. After the loop completes, feed a 10-token version of the same sequence (pad with zeros) and confirm that the final hidden state differs — proving the cell accumulates state over different sequence lengths. Your terminal output should show all intermediate hidden states for both runs.

3. **Apply a multi-filter CNN text classifier to a negation-heavy dataset.** Create a toy dataset of 40 short sentences: 20 positive ("great product", "really enjoyed it") and 20 negative, where 10 of each class contain negation ("not bad at all", "hardly disappointing"). Build a CNN with parallel width-2 and width-4 filter banks, global max-pooling, and a linear classifier head. Train for 20 epochs. Print training accuracy and per-class precision/recall. Then isolate which width filters contribute most by zeroing out one bank at a time and re-evaluating.

4. **Diagnose architecture failure on a long-range dependency task.** Construct a synthetic sequence classification problem where the label depends on the first and last token being the same (a task requiring full-sequence memory). Implement both a width-5 CNN classifier and a vanilla RNN classifier from scratch. Train each on 200 sequences of length 50. Print accuracy for both. Then increase sequence length to 100 and reprint accuracy. Explain in a printed summary which architecture degrades faster and why, referencing the specific inductive bias responsible.

5. **Evaluate CNN vs RNN on a real GTM intent classification benchmark.** Pull 200 labeled sales email subject lines from your CRM export or Apollo saved searches (labels: `meeting_request`, `follow_up`, `unsubscribe`, `promotion`). Tokenize and embed them. Train identical-depth CNN (width 3 + 5 filters) and RNN (32-unit vanilla cell) classifiers. Log accuracy, F1, and per-class confusion matrices for both. Write the comparison table and your architectural recommendation to `outputs/skill-cnn-rnn-eval.md`, including which label class each model handles better and why.

6. **Build a production signal handler that routes emails to the better-performing architecture from Exercise 5.** Implement a Python handler that takes a raw email subject line string, embeds it, routes it through the winning model, and emits a structured signal (`{"intent": str, "confidence": float, "model": "cnn"|"rnn"}`). Wire it to accept batches of subject lines from a Clay webhook payload (simulate with a local JSON fixture if no live webhook is available). Save the handler to `handlers/intent_router.py` and demonstrate it by running it against a 10-row fixture, printing the 10 routed signals to the terminal.
