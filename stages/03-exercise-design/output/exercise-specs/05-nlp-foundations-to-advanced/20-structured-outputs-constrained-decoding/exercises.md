# Exercises — Structured Outputs & Constrained Decoding

## Exercises

1. **Implement** a Python function `mask_logits(logits, valid_token_ids)` that takes a list of floats (representing raw logits over a 10-token vocabulary) and a set of valid token indices, then returns the modified logit array with all invalid positions set to `float('-inf')`. Run the function on the input `logits = [2.1, 0.3, -0.8, 1.5, 3.2, -1.0, 0.7, 2.8, -0.2, 1.1]` with `valid_token_ids = {2, 4, 7}`. Print both the original and masked arrays, then apply softmax to both and print the resulting probability distributions. Confirm that invalid tokens have exactly zero probability in the masked distribution.

2. **Demonstrate** why prompt-level instructions cannot guarantee format compliance. Write a Python script that sends the prompt `"Respond with ONLY a valid JSON object with keys 'company' and 'domain'. No markdown, no commentary, just the JSON object."` to any LLM API or local model 20 times. For each response, attempt `json.loads()` on the raw output. Print a running tally of parse successes vs. failures, and print the raw text of any response that failed to parse. Report the failure rate as a percentage.

3. **Build** a constrained generation pipeline using Outlines and a local model (e.g., a small instruction-tuned model from HuggingFace). Define a Pydantic model `LeadQualification` with fields: `company_name: str`, `employee_count: int`, `is_b2b: bool`, and `summary: str` with `max_length=200`. Generate 50 schema-constrained objects by prompting the model with varied company descriptions of your choice. For every generated object, call `.model_validate()` on the Pydantic model and confirm zero validation failures. Print the count of successes and any failures. If any failure occurs, print the offending object and explain the discrepancy.

4. **Apply** regex-constrained decoding to produce GTM identifiers that must match a strict format. Using Outlines (or an equivalent constrained-decoding library), generate 100 strings matching the regex pattern `REV-[0-9]{4}-[A-Z]{3}` (e.g., `REV-2024-ACME`). For each generated string, run Python's `re.fullmatch()` against the pattern and print any that fail. Then change the pattern to `[A-Z]{3}-[0-9]{6}` and generate 50 more, again verifying every output with `re.fullmatch()`. Print the total count of matches and violations for both runs.

5. **Build** a benchmark script at `outputs/skill-constraint-comparison.md` that compares three constraint levels on the same task: generating structured company-enrichment records. Query a real enrichment source (Apollo People API, Clay webhook export, or a CSV of company data from your CRM) for 20 company records. Then, using a local model or API, generate structured JSON summaries of each company under three conditions: (a) plain prompt with no constraints, (b) JSON mode (OpenAI `response_format={"type": "json_object"}` or equivalent), and (c) full JSON Schema enforcement via Outlines or `instructor`. For each condition, measure: parse success rate, schema-validation pass rate (against a Pydantic model you define), average generation latency, and number of retries needed. Write the results as a markdown table to the output file. Include a one-paragraph analysis of the tradeoffs.

6. **Design** a custom context-free grammar for a domain-specific GTM response format and implement constrained generation against it. Your grammar must support structured prospecting notes with this shape: a priority level (`P0`, `P1`, `P2`), followed by a pipe-delimited list of at least one signal tag (from a fixed vocabulary like `funding`, `hiring`, `leadership_change`, `product_launch`), followed by a free-text rationale enclosed in angle brackets. Example valid output: `P1 | funding, hiring | <Series B closed in Q3, engineering headcount up 40%>`. Write the grammar as a `.gbnf` or equivalent grammar file, feed it to a constrained-decoding library (llama.cpp grammar, Outlines CFG, or similar), generate 30 outputs from varied prospect descriptions, and verify each output against your grammar by construction and by a secondary regex validation pass. Save the grammar file and a script that reproduces the outputs to `signals/examples/prospect_grammar.py`.
