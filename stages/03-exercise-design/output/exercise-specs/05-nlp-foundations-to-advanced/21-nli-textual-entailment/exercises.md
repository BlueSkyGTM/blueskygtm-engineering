## Exercises

1. **Implement** an NLI classifier using the `cross-encoder/nli-deberta-v3-base` model. Feed it the following three premise-hypothesis pairs: (a) *"Our startup builds workflow automation tools for marketing teams"* / *"This company sells physical consumer products"*; (b) *"We raised a $2M seed round last quarter"* / *"This company has received venture funding"*; (c) *"The CEO previously worked at Google"* / *"The CEO previously worked at Microsoft."* Print the predicted label (`entailment`, `contradiction`, `neutral`) and the raw score for each pair.

2. **Compare** model predictions on two premise-hypothesis pairs that differ only by negation. Use the premise *"Our platform integrates with Salesforce"* and run two hypotheses: *"This product works with Salesforce"* and *"This product does not work with Salesforce."* Print both predictions and scores. Then repeat the experiment with a numerical variant: premise *"We have 12 enterprise customers"*, hypotheses *"We have more than 10 enterprise customers"* and *"We have fewer than 5 enterprise customers."* Verify that the model assigns the correct label in all four cases.

3. **Build** an entailment-based zero-shot lead qualifier. Define five qualification hypotheses as strings (e.g., *"This company is in B2B SaaS"*, *"This company has a engineering team"*, *"This company has raised funding"*). Feed a batch of five synthetic company descriptions (you write them) through the NLI cross-encoder against each hypothesis. For each company, print a table of entailment scores across all five criteria and a binary `PASS` / `FAIL` decision using a threshold of 0.5. No training labels should be used.

4. **Apply** NLI scoring to real prospect data pulled from the Apollo or People Data Labs API. Query for 10 contacts matching a title filter (e.g., `"VP of Sales"`), retrieve each person's company description or headline, and run entailment scoring against the hypothesis *"This person's company would benefit from sales automation software."* Rank the 10 prospects by entailment score and print the top 3. Include the Apollo/PDL response JSON keys you relied on.

5. **Build** a batch enrichment pipeline in `handlers/lead_fit_scorer.py` that accepts a JSON array of intake-form responses (each containing `company_name`, `free_text_response`, and `employee_count`), scores each response against a configurable list of qualification hypotheses using the NLI cross-encoder, and writes a ranked CSV to `outputs/lead_fit_scores.csv` with columns `company_name`, `top_criterion`, `top_score`, `overall_pass`. Include at least 8 sample intake responses in the handler file for demonstration. Run the handler from the command line and confirm the CSV is written.

6. **Design** a failure-mode diagnostic suite in `signals/examples/nli_failure_modes.py` that tests the NLI model on each of the following categories: negation, numerical reasoning (quantities, comparisons), long premises (>200 tokens), and world-knowledge dependencies. Write 3 test cases per category with an expected label for each. Run the suite, compute accuracy per category, and print a summary table showing which failure categories the model handles worst. Append a 3-sentence analysis at the bottom of the output identifying the weakest category and a mitigation strategy.
