# Exercises — Coreference Resolution

## Exercises

1. **Implement mention detection on a raw document.** Write a Python script that takes the string `"Sarah called the client at Acme Corp. She told him the proposal was ready. The client seemed interested, and Sarah followed up with an email."` and prints every candidate mention span (noun phrases and pronouns) as `(start_char, end_char, text)` tuples. Use spaCy's dependency parse to extract nominal and pronominal spans. Run the script and verify terminal output lists at least 8 candidate mentions including "She," "him," "the client," and "Sarah."

2. **Trace the three-stage pipeline on a hardcoded example.** Given the input document `"Before she joined Globex, Maria had freelanced. Globex hired Maria quickly, and they gave her a team."`, manually define the expected output of each stage (mention list, candidate antecedent pairs, final clusters) as Python data structures. Write a script that prints each stage's output in order and confirms the final clusters group `["she", "Maria"]` and `["Globex", "Globex"]` correctly. Run it and inspect the staged output in your terminal.

3. **Compute B³ precision, recall, and F1 from gold and predicted clusters.** You are given a gold annotation set with three entities in a document: `gold = {("Sarah", "she", "her"), ("David", "him"), ("Acme Corp", "the company")}` and a system prediction: `pred = {("Sarah", "her"), ("David", "him"), ("Acme Corp", "the company", "it")}`. Implement the B³ scoring algorithm from scratch — no external coreference evaluation libraries. Print precision, recall, and F1 to four decimal places. Verify that recall for the "Sarah" entity reflects the missing "she" mention.

4. **Apply coreference resolution to a CRM activity log.** Load `data/activity_notes.jsonl` (10 meeting-note records, each with a `note` field and `account_id`). For each record, resolve coreferent mentions using `spaCy` + the `fastcoref` library (or `spacy-coref` if available). Print each record's `account_id` alongside the resolved entity clusters. Cross-check: in records where a sales rep is referenced by both name and pronoun, confirm they appear in the same cluster.

5. **Build a coreference-based entity normalization preprocessor for a Clay-style enrichment pipeline.** Write a standalone module at `handlers/coref_normalizer.py` that accepts a list of enriched company-description dicts (simulated Apollo-style data with `name`, `description`, `website` fields), runs coreference resolution on each description, replaces all pronouns and nominals in each cluster with the longest entity mention, and writes normalized records to `outputs/coref_normalized.jsonl`. Each output record must include `original_description`, `normalized_description`, and `clusters_resolved` (count). Run the handler on a 5-record sample and verify the JSONL file exists with normalized text where "it," "they," and "the company" are replaced by their antecedent entity names.

6. **Design and execute a head-to-head comparison of mention-pair vs. span-based coreference on a synthetic dataset.** Generate 20 short documents containing mixed anaphora, cataphora, and split antecedents. Run both approaches — a rule-based mention-pair baseline (string match + gender/number agreement) and a pretrained span-based model (`fastcoref` or equivalent). Compute B³ F1 for each across all 20 documents. Write the full methodology, metric tables, and analysis of where each approach fails to `outputs/skill-coref-evaluation.md`. The artifact must include at least two failure cases per approach with the input text, gold clusters, predicted clusters, and a one-paragraph diagnosis.
