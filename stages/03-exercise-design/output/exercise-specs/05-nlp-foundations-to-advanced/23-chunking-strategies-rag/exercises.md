# Exercises — Chunking Strategies for RAG

## Exercises

1. **Implement fixed-size token window chunking.** Write a Python script that takes a sales call transcript (use any plain-text file of at least 2000 tokens) and splits it into chunks of exactly 256 tokens with 32 tokens of overlap. Use the `tiktoken` library to count tokens. Print the total number of chunks produced, then print the first and last 40 characters of each chunk so you can visually inspect where boundaries land. Run the script on a transcript file and confirm that some boundaries fall mid-sentence.

2. **Implement recursive character splitting with custom separators.** Write a Python script that splits the same transcript using a separator hierarchy of `["\n\n", "\n", ". ", " "]` with a target chunk size of 256 tokens and 32-token overlap. For each chunk produced, print whether the chunk ends with a period (indicating a sentence boundary was preserved). Compare the count of sentence-boundary-preserving chunks against Exercise 1's output to see the difference.

3. **Build a comparison harness for all four chunking strategies.** Write a script that takes a single source document — an Apollo prospect export in Markdown format containing company descriptions, technographic data, and contact lists — and runs all four algorithms (fixed-size, recursive character, Markdown-header-aware, semantic similarity) over it. For each strategy, print: chunk count, mean token count per chunk, number of chunks that begin or end mid-sentence, and number of chunks that contain a complete Markdown section header. Output the results as a formatted table to the terminal.

4. **Evaluate overlap's effect on entity continuity using a Clay enrichment export.** Take a Clay table export that includes long-form company descriptions and technographic stacks (CSV or JSON). Convert it to a single text document. Run recursive character splitting at chunk size 256 with overlap values of 0, 32, 64, and 128 tokens. For each overlap setting, count how many company names or product names from the export appear in a chunk where they are truncated (i.e., the entity spans two adjacent chunks but neither chunk contains the full name). Print a table of overlap value vs. truncated-entity count. Identify the overlap size at which entity truncation drops to zero or plateaus.

5. **Diagnose and fix a chunk-boundary retrieval failure.** A Markdown document containing a GTM playbook has a section titled "## Objection Handling" with a table mapping objections to responses. Fixed-size chunking splits this table across three chunks, and downstream retrieval queries like "what is the response to the budget objection?" return incomplete or empty results. Build a chunking pipeline in `handlers/chunking_pipeline.py` that detects when a Markdown table is about to be split and either keeps it intact within a single chunk or splits it only at row boundaries with sufficient overlap. The script must accept a Markdown file path as a command-line argument, chunk the document, and print each chunk with an index. Re-run the retrieval query against the repaired chunks and print whether the budget objection response now appears in a single retrievable chunk.

6. **Design a chunking strategy selector.** Build a Python module in `signals/examples/chunking_selector.py` that inspects an input document and recommends a chunking strategy based on its structural properties. The selector should detect: whether the document is Markdown with headers, whether it contains tables, average sentence length, and whether content appears semantically homogeneous or topic-diverse. For each document type it encounters (call transcript, Apollo export, Clay enrichment JSON, Markdown playbook, API documentation), output the recommended strategy, the recommended chunk size and overlap, and a one-sentence rationale. Include a `__main__` block that runs the selector on three different sample documents passed as file paths and prints the recommendation for each.
