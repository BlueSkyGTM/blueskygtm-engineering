## Exercises

1. **Compute** scaled dot-product attention weights for the following setup: create Q, K, V matrices in numpy with `seq_len=5` and `d_k=8` (use `np.random.seed(42)`). Print the full 5×5 attention weight matrix after applying softmax to the scaled scores. Then print which query-key pair has the highest attention weight as `(query_index, key_index, weight_value)`.

2. **Compare** raw vs. scaled attention to observe the saturation problem. Build two attention weight matrices for the same Q, K, V with `d_k=512` and `seq_len=10`: one that divides raw scores by √d_k and one that does not. For each, print the maximum weight in each row and the mean of the top-1 weights across all rows. The unscaled version should show near-1.0 values (saturation); the scaled version should show meaningful spread. State in a print statement which version produces more usable gradients.

3. **Implement** single-head attention as a function `attention(Q, K, V)` in numpy from scratch — no library calls to attention modules. It must compute scaled dot-product scores, apply softmax row-wise, multiply by V, and return both the output matrix and the weight matrix. Verify correctness by checking that every row of the weight matrix sums to 1.0 (print the row-sums). Run it with `seq_len=6`, `d_k=4`, `d_v=4` and print the output matrix.

4. **Implement** multi-head attention as `multi_head_attention(Q, K, V, num_heads)` that splits Q, K, V into per-head slices, runs single-head attention on each slice independently, concatenates the outputs, and applies a final linear projection (use a random weight matrix). Verify the critical property: when `num_heads=1`, the output must match your single-head attention output from Exercise 3 (within floating-point tolerance). Print the max absolute difference between the two outputs — it should be < 1e-5. Then run with `num_heads=4` and print the output shape to confirm the dimension is preserved.

5. **Benchmark** attention compute time at sequence lengths `[64, 128, 256, 512, 1024, 2048, 4096]` with `d_k=64`. Time each forward pass using `time.perf_counter()`. Print a table of `seq_len | time_ms | time_ratio_vs_previous`. Compute the ratio of each measurement to the previous one — when you cross into quadratic scaling territory, the ratio should approach 4× (since doubling sequence length quadruples work). Write a 4× scaling-doubling check: assert that the last two ratios are above 3.5. Save the benchmark table and your identified binding-length threshold to `outputs/skill-attention-benchmark.md`.

6. **Compare** attention-based retrieval to RNN hidden-state compression on a GTM enrichment task. Load 50 prospect company descriptions (use the Apollo API or a local CSV of company "about" text). Build a simple keyword query embedding for "fast-growing SaaS company" and retrieve the top-5 matches using two methods: (a) attention weights between the query and all company-description embeddings, and (b) a single compressed vector built by running descriptions through a simple RNN (`numpy` recurrence is fine) and ranking by cosine similarity to the query. Print both ranked top-5 lists side by side. Compute Jaccard overlap between the two result sets. Save your implementation and the comparison output to `signals/examples/attention_vs_rnn_retrieval.py`.
