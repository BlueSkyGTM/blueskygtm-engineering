# Exercises — Bag of Words, TF-IDF, and Text Representation

## Exercises

1. **Implement a Bag of Words vector from scratch.** Write a Python script that takes a list of 5 short documents (strings of your choice), tokenizes them by lowercasing and splitting on whitespace, builds a sorted vocabulary from the union of all tokens, and constructs the BoW count matrix as a NumPy array. Print the vocabulary list and the full matrix to the terminal. Verify that the matrix shape is `(5, vocab_size)` and that row sums equal each document's token count.

2. **Compute TF-IDF weights by hand for a 3-document corpus, then confirm with scikit-learn.** Choose 3 short sentences that share some words and differ in others. Manually compute the TF-IDF value for at least 4 specific (document, term) pairs using the formulas from the lesson. Then use `sklearn.feature_extraction.text.TfidfVectorizer` on the same corpus and print the full TF-IDF matrix. Print your hand-computed values alongside the corresponding scikit-learn values and confirm they match. If they differ, identify whether scikit-learn applies L2 normalization or smoothing and reconcile the difference.

3. **Build a cosine similarity function for TF-IDF vectors and compare two documents.** Implement cosine similarity from scratch using NumPy (dot product divided by the product of L2 norms). Run `TfidfVectorizer` on a corpus of 6 documents, then compute and print the pairwise cosine similarity between document 0 and every other document. Print the results sorted from most to least similar. Verify that an identical or near-identical document scores close to 1.0 and that an unrelated document scores close to 0.

4. **Rank a list of Apollo company descriptions against an ICP keyword profile using text similarity.** Create a Python script that defines an ICP description as a single string (e.g., "B2B SaaS platform for sales enablement and revenue intelligence"). Define 8–10 company description strings of varying relevance — some that match the ICP closely, some in adjacent spaces, some completely unrelated. Use `TfidfVectorizer` to vectorize the ICP and all company descriptions together, then compute cosine similarity between the ICP vector and each company vector. Print a ranked table showing each company description and its similarity score, sorted descending. Confirm the top-ranked companies are genuinely the closest match.

5. **Diagnose why a company description scores low against an ICP profile by isolating three failure modes.** Build a diagnostic script at `handlers/similarity_diagnostic.py` that takes an ICP string and a company description, then prints a breakdown of three factors: (a) vocabulary mismatch — which ICP keywords are absent from the company description, (b) document length effect — the ratio of company token count to ICP token count and how length disparity dilutes the cosine similarity, and (c) IDF weighting impact — whether the words that do overlap are high-IDF (rare, distinctive) or low-IDF (common, uninformative). Run your diagnostic on 3 company descriptions: one with low score due to vocabulary mismatch, one due to excessive length, and one due to overlap only on common words. Print a one-line diagnosis for each case. The script must be runnable as `python handlers/similarity_diagnostic.py` and produce structured terminal output.
