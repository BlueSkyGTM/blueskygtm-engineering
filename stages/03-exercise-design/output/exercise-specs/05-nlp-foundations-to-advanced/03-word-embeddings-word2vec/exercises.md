## Exercises

1. **Implement cosine similarity from scratch in NumPy.** Write a function `cosine_sim(a, b)` that takes two 1-D NumPy arrays and returns their cosine similarity without using any library similarity function. Test it on three vector pairs: `(king_vec, queen_vec)`, `(king_vec, banana_vec)`, and `(king_vec, king_vec)`. Print all three scores and confirm the self-similarity is `1.0`.

2. **Train the skip-gram model from the lesson on a small corpus and inspect neighbors.** Use the training loop provided in the lesson on the sample corpus. After training completes, pick the word `"sales"` and print its top 5 nearest neighbors by cosine similarity along with their scores. Verify that contextually related words rank higher than unrelated ones.

3. **Compute and compare operation counts for full softmax vs. negative sampling.** Given a vocabulary size `|V| = 50,000`, embedding dimension `d = 300`, and `k = 10` negative samples, write a function that computes the total number of multiply-add operations required for one forward and backward pass under (a) full softmax and (b) negative sampling. Print both counts and their ratio. Then re-run the computation with `|V| = 500,000` and print the updated ratio. Write a two-sentence comment in your script explaining what this ratio means for training time at vocabulary scale.

4. **Build a title normalization function using embedding distance.** Given a pre-trained or lesson-trained embedding matrix and a list of canonical labels (`["Account Executive", "Sales Engineer", "Marketing Manager", "Customer Success Manager"]`), implement `normalize_title(raw_string, embeddings, canonical_labels)` that tokenizes the raw input, averages the word vectors, and returns the canonical label with highest cosine similarity. Test it on at least four messy inputs: `"Sr. AE - Enterprise"`, `"Solutions Consultant"`, `"Mktg Lead"`, `"CSM / Account Manager"`. Print each raw input alongside its predicted canonical label and similarity score.

5. **Apply embedding-based normalization to a real CRM export and produce a persistent artifact.** Obtain a list of at least 50 raw job titles — pull them from a sample Apollo or Clay export, a CRM CSV, or the provided `data/messy_titles.csv`. Train word embeddings on a relevant corpus (job descriptions, LinkedIn bios, or the lesson corpus), normalize every title to a canonical set of at least 8 labels, and write the full mapping table to `outputs/skill-title-normalization.md`. Include a summary section with the distribution of canonical labels and 3 examples where the embedding-based match outperformed a naive substring match.

6. **Design and run an experiment comparing normalization strategies.** Build a benchmark dataset of 100 raw titles paired with ground-truth canonical labels (you can label them yourself or use a provided key). Run three normalization strategies: exact string match, fuzzy match (Levenshtein ratio), and your embedding-distance function. Compute accuracy and macro-F1 for each. Save the results table and a one-paragraph analysis to `outputs/skill-normalization-benchmark.md`.
