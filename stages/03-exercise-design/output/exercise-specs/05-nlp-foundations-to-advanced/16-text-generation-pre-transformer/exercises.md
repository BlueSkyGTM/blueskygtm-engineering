## Exercises

1. **Implement** a bigram language model from raw counts. Write a script that reads a plain-text corpus from a file, builds a bigram count table, and computes `P(word | prev_word)` as `count(prev_word, word) / count(prev_word)`. For three specific bigrams of your choosing, print the conditional probability. Then **generate** 30 tokens of text by sampling from the bigram distribution starting from a chosen seed word. Your script should print the generated sequence so you can eyeball whether it resembles coherent language.

2. **Trace** the sparsity problem across n-gram orders. Using a fixed corpus of at least 500 tokens, compute and print: (a) total unique unigrams, (b) total unique bigrams observed, (c) total unique trigrams observed. Then identify 5 word sequences that appear in your corpus but whose trigram context was seen only once — meaning the model has exactly one continuation option and assigns probability 0 to everything else. Print each sparse trigram context alongside its single observed continuation.

3. **Apply** Laplace (add-one) smoothing to your bigram model and **compare** its perplexity against the unsmoothed model on a held-out test set that is separate from your training corpus. Print both perplexity scores. If the unsmoothed model hits a zero-probability bigram in the test set, handle it explicitly (print a warning or skip) and note how smoothing eliminates the division-by-zero / log(0) problem. Your script should output something like: `Unsmoothed perplexity: 142.3 | Smoothed perplexity: 98.7`.

4. **Compute** and **compare** perplexity for three model orders — unigram, bigram, and trigram — on the same held-out test corpus. Print all three scores. Then write a short printed summary (3–5 lines) explaining *why* perplexity changes the way it does: does higher n always help? Where does sparsity overwhelm the benefit of additional context? Use your actual numbers in the explanation.

5. **Compute** keyword co-occurrence statistics from a corpus of real GTM text. Pull at least 50 company descriptions or LinkedIn bios via the Apollo API (or use a CSV export from Clay containing company "about" text). Tokenize, build a bigram co-occurrence matrix, and compute a relevance score for a target keyword (e.g., "salesforce," "ai agent," " RevOps") defined as the top 10 words with highest conditional probability following that keyword. Produce a persistent artifact at `outputs/skill-keyword-cooccurrence.md` containing: the target keyword, the top 10 co-occurring terms with their probabilities, and a 3-sentence analysis of what the co-occurrence pattern reveals about how the market positions itself relative to that keyword.

6. **Build** a configurable n-gram language model library as a reusable module at `handlers/ngram_lm.py`. It must accept `--n` (1, 2, or 3), `--smooth` (none or laplace), `--mode` (perplexity or generate), and `--corpus` (path to training file) as command-line arguments. In generate mode it samples 50 tokens. In perplexity mode it takes a `--test` file and prints the score. Run it in all six combinations (3 values of n × 2 smoothing settings) on the same train/test split and print a results table showing which configuration achieves the lowest perplexity.
