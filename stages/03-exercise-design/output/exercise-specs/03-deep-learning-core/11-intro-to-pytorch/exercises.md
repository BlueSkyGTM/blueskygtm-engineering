# Exercises — Introduction to PyTorch

## Exercises

1. **Implement a single-neuron linear model and train it on synthetic data.** Create a tensor `X` of shape `(100, 3)` drawn from a normal distribution and a target tensor `y = 2.5 * X[:, 0] - 1.0 * X[:, 1] + 0.5 * X[:, 2] + noise`. Build an `nn.Module` with a single `nn.Linear(3, 1)` layer, instantiate `torch.optim.SGD` with `lr=0.01`, and run the standard training loop (zero_grad, forward, loss with `nn.MSELoss`, backward, step) for 200 epochs. Print the loss every 50 epochs and confirm it decreases monotonically. Terminal output should show the final loss below 0.1.

2. **Trace the autograd graph on a hand-built computation.** Create three leaf tensors `a`, `b`, and `c` with `requires_grad=True`. Compute `d = a * b + c`, then `e = d ** 2`, then `loss = e.sum()`. Before calling `.backward()`, print `e.grad_fn`, `e.grad_fn.next_functions`, and follow the chain to confirm it references the `d` computation. Call `.backward()`, then print `a.grad`, `b.grad`, and `c.grad`. Verify by hand that the printed gradients match the analytical derivatives of `loss` with respect to each leaf.

3. **Implement model serialization and verify weight restoration.** Train an `nn.Linear(4, 1)` model for 100 epochs on any synthetic regression dataset. Save the model's `state_dict` to disk with `torch.save`. Create a fresh model instance with the same architecture, load the saved weights with `torch.load` and `.load_state_dict()`, set it to `.eval()` mode. Pass the same input batch through both the original and restored models and print `torch.allclose(original_output, restored_output)`. The result must be `True`.

4. **Compare SGD and Adam convergence on a lead-scoring regression.** Generate a synthetic dataset representing 500 leads with 6 features (company size, funding amount, website visits, email opens, page views, days since last contact) and a conversion score target. Train two identical `nn.Linear(6, 1)` models — one with `torch.optim.SGD(lr=0.01)` and one with `torch.optim.Adam(lr=0.01)` — for 300 epochs each. At epochs 50, 100, 200, and 300, print a side-by-side comparison of the current loss for both optimizers. Report which optimizer reaches a loss below 0.05 in fewer epochs.

5. **Build and serialize a lead-scoring model trained on Apollo enrichment data.** Export 200+ company records from Apollo (or use a provided CSV at `data/apollo_companies.csv` containing features like employee_count, funding_total, recent_hiring_velocity, tech_stack_count, and annual_revenue). Normalize all numeric features to zero mean and unit variance using `torch` operations only. Build a two-layer `nn.Module` (linear → ReLU → linear), train with Adam for 500 epochs, and serialize the trained model to `outputs/skill-lead-scorer.pt`. Write a Python script at `signals/examples/lead_scorer.py` that loads the serialized model, accepts a company profile as command-line arguments, and prints a predicted conversion score. Run it on three sample companies and confirm the outputs are deterministic across repeated runs.

6. **Create a PyTorch debugging reference guide by reproducing and fixing all five common runtime errors.** Write five short, self-contained Python snippets that each trigger one of: shape mismatch in a forward pass, missing `requires_grad` causing `None` gradients, dtype mismatch between input and weight tensors, gradient accumulation producing incorrect loss curves, and non-converging loss due to an excessive learning rate. For each error, run the broken code, capture the error message or incorrect output, then write the corrected version and confirm it runs. Compile both broken and fixed snippets with explanatory comments into `outputs/skill-pytorch-debugging.md`. Each section must include the error symptom, root cause, and the one-line fix.
