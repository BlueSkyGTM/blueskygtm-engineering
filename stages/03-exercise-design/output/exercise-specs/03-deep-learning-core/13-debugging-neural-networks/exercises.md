# Exercises — Debugging Neural Networks

## Exercises

1. **Implement a NaN-detection training loop.** Write a PyTorch training script for a 3-layer MLP on a synthetic classification dataset. Use an intentionally high learning rate (e.g., `1e-1`) with no gradient clipping. Print loss, gradient norm, and weight statistics every 20 steps. When the loss becomes NaN, print `"NaN DETECTED at step {N}"` and halt. Then re-run with `lr=1e-3` and confirm the loss decreases monotonically across 500 steps. Your terminal output should show both runs side by side or sequentially.

2. **Implement per-layer gradient norm tracking.** Register backward hooks on every linear layer in a small MLP. After each optimizer step, print a single line per layer showing `layer_name, grad_norm, weight_norm`. Train for 200 steps on a standard dataset and manually inspect which layers have disproportionately small or large gradient norms. Then inject a bad initialization (all weights set to `0.001`) and re-run — your terminal output should clearly show vanishing gradients in deeper layers.

3. **Write activation distribution checks to detect dead neurons and saturation.** Build a 5-layer ReLU network and register forward hooks that log `(mean, std, fraction_zero)` for each layer's output at every 100th step. Train on a dataset where input features are heavily positively skewed (mean=50, std=80, all positive values). Print a table identifying any layer where `fraction_zero > 0.5` for 3+ consecutive checkpoints. Label those layers as `"DEAD NEURONS DETECTED"` in stdout.

4. **Compare learning rate behaviors on CRM lead-scoring data.** Download or synthesize a lead-scoring dataset with features like `employee_count`, `funding_raised`, `website_visits`, and a binary `converted` label. Train the same 4-layer MLP three times using learning rates `1e-1`, `1e-3`, and `1e-5`. For each run, print final loss, training accuracy, gradient norm at step 200, and whether the loss curve showed plateau, divergence, or healthy convergence. Format output as a comparison table in stdout.

5. **Build a minimal diagnostic harness that surfaces all training health signals.** Create `handlers/training_diagnostics.py` — a self-contained module that, given a PyTorch model and a DataLoader, runs a training loop and prints a structured diagnostic report to stdout every N steps. The report must include: loss value and delta, per-layer gradient norms, per-layer activation statistics (mean, std, dead fraction), and an automated health verdict (`"HEALTHY"`, `"VANISHING GRADIENTS"`, `"EXPLODING GRADIENTS"`, `"DEAD NEURONS"`, `"DIVERGING"`). Run the harness on two configurations — one healthy, one broken — and confirm the verdicts differ. The artifact must land at `handlers/training_diagnostics.py` and be importable as `from handlers.training_diagnostics import run_diagnostics`.
