# Exercises — Weight Initialization and Training Stability

## Exercises

1. **Implement and measure four initialization strategies.** Build a 5-layer MLP with 256 hidden units per layer using PyTorch. Write four functions — `zero_init`, `large_normal_init`, `xavier_init`, `he_init` — each of which takes an `nn.Linear` module and re-initializes its `.weight` and `.bias` parameters in-place. For each strategy, run a forward pass on a fixed batch of random input (shape `[64, 256]`, drawn from a standard normal) and print a per-layer table showing mean, std, and fraction of activations that are zero. Your terminal output should clearly show which strategies produce exploding or vanishing activations by layer 5.

2. **Compute and verify the variance propagation equation numerically.** Write a script that creates a 20-layer chain of `nn.Linear(512, 512)` modules with Xavier-initialized weights (no bias, no activation function). Feed in a batch of standard-normal input and record the variance of the output at each layer. Then repeat with weights drawn from `N(0, 0.05)` and from `N(0, 0.01)`. Print a table of `layer_index | measured_variance | ratio_to_previous_layer` for all three settings. Confirm that Xavier keeps the ratio near 1.0 and identify which layer the other two strategies cross a variance of 100 or drop below 1e-6.

3. **Diagnose initialization failure during training.** Starting from a fresh 6-layer MLP with ReLU activations, train on MNIST (or any classification dataset you can load in one call) for 100 steps using SGD with learning rate 0.01. Run the training twice — once with default PyTorch initialization and once with `zero_init`. At each step, log the gradient norm (L2 norm of all gradients concatenated) and the activation variance at layer 3. Plot or print a 10-row summary table (every 10th step) showing `step | grad_norm | layer3_var` for both runs. Identify which run shows gradient collapse and state the threshold at which it occurs.

4. **Compare ReLU vs tanh initialization matching.** Build two identical 5-layer MLPs — one with `nn.ReLU()` activations, one with `nn.Tanh()`. For each, initialize with both Xavier and He, giving you four configurations. Run a forward pass on the same standard-normal input batch and measure the final-layer activation variance. Print a 2×2 table of `activation_fn × init_strategy → final_layer_variance`. Verify that He init produces comparable variance to Xavier-on-tanh and explain in a comment in your script why ReLU needs approximately double the weight variance (reference the sparsity factor: half the pre-activations are zeroed out).

5. **Add gradient clipping and activation diagnostics to a lead-scoring training loop.** Build a PyTorch training script that loads a tabular lead-scoring dataset (use Apollo firmographic features — company size, industry code, funding amount, technographic count — or synthesize 5000 rows with 32 features and a binary "converted" label). Implement a `GradientNormCeiling` hook that clips the total gradient norm to a configurable `max_norm` value using `torch.nn.utils.clip_grad_norm_` and logs the pre-clip and post-clip norm at each step. Add forward hooks on layers 1, 3, and 5 that record activation variance into a rolling buffer. Train for 200 steps with He initialization and `max_norm=5.0`, then print a diagnostic report showing step, loss, pre-clip grad norm, post-clip grad norm, and activation variance at each hooked layer. Save the complete training script (with the diagnostic module, hooks, and clipping logic as reusable functions) to `handlers/lead_scoring_train.py`.

6. **Design and run a full initialization experiment on enrichment-quality data.** Create a synthetic dataset of 8000 enrichment records where each row has 64 features representing company signals (web traffic bins, employee count, tech stack indicators, news sentiment scores) and a target of "enrichment_confidence" (continuous 0–1 score). Train four models with identical architecture (8-layer MLP, 256 hidden units, ReLU) but different initializations: zero, Xavier, He, and a custom strategy of your own design that combines He variances with a small bias offset. Log training loss, validation loss, activation variance per layer, and gradient norms for the first 500 steps of each run. Write a markdown report to `outputs/skill-init-experiment.md` containing: (a) a table comparing final validation loss across all four strategies, (b) the step at which each strategy's loss first dropped below 0.5, (c) a 3–5 sentence analysis of which initialization is best suited for high-dimensional GTM signal data and why, and (d) a recommendation for which init to use when fine-tuning a pre-trained model on a small enrichment dataset versus training from scratch.
