# Exercises — Optimizers

## Exercises

1. **Implement SGD and SGD with momentum from scratch using only NumPy.** Define the loss function `f(x, y) = x² + 10y²` and its gradient. Write two optimizer classes or functions: `sgd_update(w, grad, lr)` and `sgd_momentum_update(w, v, grad, lr, beta)`. Run both from the same starting point `w = [5.0, 5.0]` for 100 steps with `lr = 0.01` and `beta = 0.9`. After each run, print the final weight vector, the final loss value, and the number of steps needed to get the loss below `0.01` (print "did not converge" if it never does). Confirm visually that momentum reaches a lower loss than vanilla SGD at the same step count.

2. **Implement Adam and AdamW from scratch using only NumPy.** Write `adam_update(w, m, v, grad, lr, beta1, beta2, eps, t)` and `adamw_update(...)` where AdamW applies weight decay directly to `w` rather than to the gradient. Run all four optimizers (SGD, SGD+Momentum, Adam, AdamW) on the same convex bowl from Exercise 1 starting at `[5.0, 5.0]` for 100 steps. Use `lr = 0.01`, `beta1 = 0.9`, `beta2 = 0.999`, `eps = 1e-8`, and `weight_decay = 0.01`. Print a table to the terminal with columns: optimizer name, final loss, final position, and convergence step (first step where loss < 0.001). Add a comment above each optimizer function stating which new term it introduces relative to its predecessor in the lineage.

3. **Compare all four optimizers on a non-convex loss surface.** Implement the Rosenbrock function `f(x, y) = (1 - x)² + 100(y - x²)²` and its gradient analytically (do not use autograd). Run SGD, SGD+Momentum, Adam, and AdamW starting from `[-1.5, 2.0]` for 500 steps. For each optimizer, print the final loss, the minimum loss reached across all steps, and the step at which that minimum occurred. Then sweep three learning rates (`0.001`, `0.01`, `0.05`) for each optimizer and print a 4×3 grid of final losses. State in a printed summary which optimizer–learning-rate pair found the lowest loss.

4. **Build a from-scratch learning rate finder for a lead-scoring logistic regression model.** Simulate a CRM export as a NumPy array: 500 samples, 6 features (company headcount, funding raised, website visits, email opens, days since last contact, ICP fit score), with binary conversion labels generated by a known linear logistic function plus noise. Implement logistic regression with a hand-written sigmoid and binary cross-entropy gradient (no sklearn, no autograd). Run an LR range test: start `lr = 1e-6` and exponentially increase it over 200 steps, recording loss at each step. Print the learning rate at which loss is minimized, then train the model for 300 steps at that LR using each of your four optimizers. Print a ranked table of final AUC (computed by hand or with `sklearn.metrics.roc_auc_score`) by optimizer. State your selected optimizer and learning rate.

5. **Build a training diagnostics handler that instruments any optimizer run and produces a written failure report.** Create a module `handlers/optimizer_diagnostics.py` containing a function `diagnose_training(loss_fn, grad_fn, w_init, optimizer_fn, lr, num_steps, optimizer_name)` that logs per-step: loss, gradient L2 norm, update L2 norm, and a flag for NaN or Inf in any value. After the run, the function should classify the trajectory into one of: `"healthy"`, `"diverging (lr too high)"`, `"stuck on plateau"`, or `"nan detected"` using thresholds you define (e.g., loss increased by >10× at any point, gradient norm below `1e-8` for 50+ consecutive steps). Test it on three scenarios: (a) Adam on the convex bowl with `lr = 0.01`, (b) SGD on the convex bowl with `lr = 0.5`, and (c) Adam on the Rosenbrock function with `lr = 2.0`. Write a summary report to `outputs/skill-optimizer-diagnostics.md` containing the diagnosis for each scenario, the final gradient norm, the step where the worst loss occurred, and your recommended corrective action for each.
