# Exercises — Activation Functions

## Exercises

1. **Implement** sigmoid, tanh, ReLU, Leaky ReLU, GELU, and softmax in NumPy alongside their analytical derivative functions (`sigmoid_grad`, `tanh_grad`, etc.). For each pair, verify correctness by comparing the analytical derivative against a finite-difference approximation `(f(z + h) - f(z - h)) / (2h)` at `z = [-2.0, -0.5, 0.0, 0.5, 2.0]` with `h = 1e-5`. Print a table showing the analytical derivative, the numerical estimate, and the absolute difference. Any difference above `1e-4` indicates a bug — fix it before moving on.

2. **Compute** the output value and the gradient magnitude for each of the six activation functions at the extreme inputs `z = [-100, -10, -1, 0, 1, 10, 100]`. Print a formatted table with rows for each input and columns for each function's `(output, gradient)` pair. Identify which functions saturate (gradient below `0.01`) at `z = -10` and `z = 10`, and print a summary line listing the saturated functions at each extreme.

3. **Build** a feedforward chain of 10 layers, each with weight matrix `W` drawn from `np.random.randn(64, 64) * 0.1` and bias `b = 0`. Feed a single random input vector of shape `(64,)` through all 10 layers using the **same** activation function at every layer. Repeat separately for sigmoid, tanh, and ReLU. After each forward pass, record the mean absolute activation magnitude at each layer. Print a layer-by-layer table for all three functions showing the magnitude decay. Identify which activation exhibits the steepest magnitude collapse across the 10 layers.

4. **Apply** dead-neuron detection to a single ReLU layer with `W = np.random.randn(32, 16) * 0.5` and `b = -2.0` (deliberately biased negative). Generate a batch of 200 random input vectors of shape `(32,)` from `np.random.randn`. Forward-propagate the full batch, then compute what fraction of the 16 output units are **always zero** across all 200 inputs. Print the neuron index, count of zero outputs, and a `DEAD` or `ALIVE` label for each unit. Report the total percentage of dead neurons.

5. **Design** a three-head GTM scoring module that takes a 20-dimensional enriched company feature vector (revenue band, employee count, industry one-hots, technographics, engagement signals) and produces three outputs with the correct activation for each task: a binary lead-qualification score (sigmoid), a 5-class ICP tier classification (softmax over 5 logits), and a predicted annual contract value in dollars (regression — no activation or a softplus to guarantee positivity). Load the company features from a CSV file structured like an Apollo export (`outputs/apollo_companies.csv`) containing at least 50 rows. Forward-propagate each company through a single hidden layer (64 units, GELU activation) into the three output heads. Print each company's name, qualification probability, predicted ICP tier (argmax), and predicted ACV. Save the full module as `handlers/lead_scorer.py` so it can be imported and called with `from handlers.lead_scorer import score_companies`.

6. **Extend** your dead-neuron detector from Exercise 4 into a reusable diagnostic tool that accepts any trained ReLU layer (weight matrix, bias, activation config) and a batch of inputs, then produces a full health report: per-neuron firing rate, dead-neuron flags, mean activation magnitude, and a gradient-norm estimate computed via finite differences through the layer. Run the diagnostic on the hidden layer of your lead scorer from Exercise 5 using the 50 Apollo company rows as the input batch. Write the full report to `outputs/skill-activation-diagnostics.md` in Markdown with tables for each metric section and a written interpretation of whether the layer is healthy or at risk of dead neurons.
