# Exercises — Loss Functions

## Exercises

1. **Implement MSE and MAE from scratch.** Write a Python script that defines `mse(y_pred, y_true)` and `mae(y_pred, y_true)` — each returning a scalar loss value. Create two prediction arrays: one with small uniform errors (`[0.1, 0.1, 0.1, 0.1]` vs targets of zeros) and one containing a single large outlier error (`[0.1, 0.1, 0.1, 5.0]` vs zeros). Print both loss values for both arrays. Observe how MSE inflates relative to MAE when the outlier is present. Run the script and confirm the terminal output shows MSE growing much faster than MAE for the outlier case.

2. **Implement binary cross-entropy and hinge loss.** Write `binary_cross_entropy(y_pred, y_true)` and `hinge_loss(y_pred, y_true)` functions that operate on arrays. Clamp predictions to `[1e-7, 1 - 1e-7]` before computing BCE. Evaluate both losses on three prediction arrays: confident-correct (`[0.95, 0.95]` vs `[1, 1]`), confident-wrong (`[0.05, 0.05]` vs `[1, 1]`), and borderline (`[0.51, 0.51]` vs `[1, 1]`). Print a table comparing the two losses across all three cases. Run the script and confirm BCE penalizes confident-wrong predictions far more heavily than hinge does.

3. **Compute and compare gradients for all four loss functions.** Extend your implementations from Exercises 1 and 2 to include gradient functions: `mse_grad`, `mae_grad`, `bce_grad`, and `hinge_grad`, each returning the derivative of the loss with respect to `y_pred`. For MAE, use `np.sign(y_pred - y_true)` for the gradient. Create a sweep of prediction errors from `-3.0` to `3.0` in steps of `0.5` against a fixed target of `1.0` (for regression, target `1.0`; for classification, use label `1` and map predictions accordingly). Print a table showing the error value, each loss function's value, and each loss function's gradient magnitude at that error. Run the script and identify which loss function produces the largest gradient at the extreme errors.

4. **Select loss functions for three GTM prediction tasks.** Write a Python script that defines three task descriptions as strings: (a) predicting the dollar value of a closed-won deal, (b) predicting whether a lead will convert to an opportunity (binary), and (c) classifying an inbound lead into one of five intent tiers. For each task, programmatically assert the appropriate loss function name from the set `{mse, mae, binary_cross_entropy, hinge, categorical_cross_entropy}` and print the rationale. Include a fourth scenario: you discover your deal-size dataset has extreme outliers from enterprise deals. Re-select the loss function for that scenario and print the change and the reason. Run the script and confirm output shows the switch from MSE to MAE (or Huber) for the outlier case.

5. **Build a loss-dynamics diagnostic tool for sales model training.** Create a script that simulates 200 training steps of gradient descent on a synthetic lead-conversion dataset (500 samples, 3 features, binary labels). Run the simulation three times using MSE, BCE, and hinge loss respectively — each with a fixed learning rate of `0.01`. At each step, record the loss value and the mean gradient magnitude. Save the per-step loss and gradient traces to `outputs/skill-loss-dynamics.md` as a Markdown table with columns `step, loss_mse, grad_mse, loss_bce, grad_bce, loss_hinge, grad_hinge`. At the bottom of the file, write a one-paragraph diagnosis: which loss converged fastest, which was most unstable, and which produced the largest gradient spikes. Run the script, open the file, and verify the table is populated for all 200 steps.
