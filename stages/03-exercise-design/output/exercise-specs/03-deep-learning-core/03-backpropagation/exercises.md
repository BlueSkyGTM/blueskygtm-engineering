# Exercises — Backpropagation from Scratch

## Exercises

1. **Implement gradient checks for addition and multiplication nodes.** Write a function `numerical_grad(f, x, h=1e-5)` that estimates gradients via central finite differences. Construct a graph with two leaf `Tensor` nodes `a` and `b`, an output `c = a * b + a`, and a scalar loss `L = c`. Run your autograd engine's backward pass to get analytical gradients for `a` and `b`. Then compute numerical gradients and assert that `|analytical - numerical| < 1e-4` for both. Print `PASS` or the deviation for each.

2. **Trace topological ordering on a multi-path graph.** Build a graph with leaves `w1, w2, x, b` and operations `h1 = w1 * x`, `h2 = w2 * x`, `s = h1 + h2`, `out = s + b`, `L = out * out`. Before running backward, print the topological sort order your engine produces. Then manually compute `dL/dw1` and `dL/dw2` on paper, run the engine, and print both sets of gradients side by side. Confirm they match.

3. **Implement the backward pass for matrix multiply from first principles.** Given forward `C = A @ B` where `A` is shape `(m, k)` and `B` is shape `(k, n)`, write the local Jacobian logic so that `grad_A = grad_C @ B.T` and `grad_B = A.T @ grad_C`. Integrate this into your `MatMul` node class. Verify correctness by constructing `A = [[1,2],[3,4]]`, `B = [[5],[6]]`, `C = A @ B`, `L = sum(C)`, running backward, and printing `grad_A` and `grad_B`. Check against hand-computed values.

4. **Implement ReLU backward and diagnose dead neurons on enrichment-style features.** Add a `ReLU` node to your engine with the correct local gradient (1 where input > 0, 0 elsewhere). Generate a synthetic feature matrix of shape `(100, 8)` where each row represents a prospect with features like "employee_count," "funding_rounds," "page_views," etc. — sample some values as negative to test ReLU clipping. Build a one-layer linear model `y_hat = ReLU(X @ W + b)`, compute MSE loss against random target labels, run backward, and print how many of the 8 hidden units have zero gradient (dead ReLUs). Experiment with scaling `W` by 0.01 vs 2.0 and print the dead-unit count for each.

5. **Train a 2-layer MLP on lead-conversion data and log per-layer gradient norms.** Using your autograd engine (no PyTorch, no TensorFlow), build `MLP(16 → 32 → 1)` with ReLU activation and sigmoid output. Generate 500 synthetic prospects from a CRM export CSV at `data/leads_sample.csv` (or create a synthetic one with columns: `employee_count, funding_total, tech_stack_count, hiring_velocity, page_views, email_opens, linkedin_engagement, competitor_count, industry_score, region_score, deal_size_tier, intent_score, icp_fit, company_age, response_time_hr, source_quality` and binary `converted` label). Train for 500 epochs of full-batch gradient descent at learning rate 0.01. Every 50 epochs, print the L2 norm of gradients for `W1`, `b1`, `W2`, and `b2`. Initialize `W1` with three different scales (0.01, 0.5, He-initialization) and run each. Write a summary table comparing gradient norm trajectories and final loss for each initialization to `outputs/skill-backprop-mlp.md`.

6. **Implement forward-mode autodiff for the same operation set and empirically compare cost against reverse-mode.** Build a dual-number-based forward-mode engine supporting add, multiply, matmul, and ReLU. Each `Dual` carries a value and a directional derivative (tangent). For a graph with `N` parameters, run forward-mode `N` times (one per parameter direction) and reverse-mode once. Measure wall-clock time for `N = 4, 8, 16, 32` parameters on a random MLP forward+backward. Print a table of `N`, `forward_mode_seconds`, `reverse_mode_seconds`, and their ratio. Write your results and a one-paragraph explanation of why reverse-mode wins to `outputs/skill-autodiff-modes.md`.
