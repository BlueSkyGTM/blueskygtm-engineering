# Exercises — Evaluation & Testing LLM Applications

## Exercises

1. **Build a JSONL evaluation dataset for a lead-qualification classifier.** Your company classifies inbound leads into three tiers: `SQL`, `MQL`, and `DISQUALIFY`. Create a file at `evals/lead_qualification.jsonl` containing at least 15 input-output pairs. Each line must be a JSON object with `input` (the lead's raw message or form submission text), `expected` (the correct label), and `category` (one of: `happy_path`, `edge_case`, `known_failure`). Include at least 3 edge cases (ambiguous intent, missing company name, sarcastic tone) and 2 known failures (cases your current prompt gets wrong on purpose). Write a separate validation script that loads the file, confirms every line has all three keys, and prints a summary: total examples, count per label, count per category. Run it and confirm the terminal output matches your expectations.

2. **Implement exact match and substring match scoring functions.** Write two standalone functions: `score_exact(prediction, expected)` returns `1.0` if the strings are identical (case-insensitive, stripped), else `0.0`. `score_substring(prediction, expected)` returns `1.0` if the prediction contains the expected label as a substring, else `0.0`. Load your JSONL dataset from Exercise 1. For each example, fabricate a plausible model prediction (hardcode 8–10 of them to be intentionally wrong in different ways — extra whitespace, lowercase, wrong label, label embedded in a sentence). Score every prediction with both mechanisms. Print a table showing `input_preview`, `expected`, `prediction`, `exact_score`, `substring_score`, and `agree` (whether the two scores match). At the bottom, print the aggregate exact-match accuracy and substring-match accuracy.

3. **Implement LLM-as-judge with a rubric and compare disagreement patterns across all three scorers.** Write a function `score_judge(input_text, prediction, rubric)` that calls your LLM client with a prompt containing the input, the candidate prediction, and a rubric you define. The rubric should specify what constitutes a correct classification for each lead tier, including how to handle borderline cases. The function must return a float score between 0.0 and 1.0. Run all three scorers (exact, substring, judge) across your dataset using the same fabricated predictions from Exercise 2. Identify every case where the judge disagrees with exact match. Print a report showing each disagreement with the judge's raw response text so you can inspect why the disagreement occurred. At the end, print a disagreement matrix: how many times each pair of scorers (exact-vs-substring, exact-vs-judge, substring-vs-judge) disagreed.
