## Exercises

1. **Trace state through a LangGraph ICP-scoring graph.** Implement a `StateGraph` with a typed `CompanyState` (fields: `domain`, `employee_count`, `industry`, `icp_score`, `route`) and three nodes: `enrich_company`, `score_icp`, `route_lead`. Add a `print(state)` line at the start of each node so the terminal shows the state object mutating as control passes from node to node. Compile the graph and invoke it on `{"domain": "stripe.com"}`. Verify the terminal prints three distinct state snapshots with `icp_score` populated only after the second node.

2. **Configure a CrewAI Crew with the same three-step intent.** Define two `Agent` objects (`FirmographicsResearcher` with a research goal, `ICPScorer` with a scoring goal) and two `Task` objects bound to them (`fetch_firmographics`, `compute_fit_score`). Call `crew.kickoff()` on a single company input and print the raw output of each task. Confirm the terminal shows both task results, with the score task depending on the research task's output.

3. **Implement the enrich → score → route pipeline in AutoGen.** Use a `GroupChat` with two `AssistantAgent`s (`EnrichmentBot`, `ScorerBot`) and one `UserProxyAgent`. Seed the conversation with a company domain and require the agents to converge on a routing label (`MQL`, `SQL`, or `disqualify`). Run the chat and print the message history — verify the terminal log shows both assistant agents contributing turns and ending with a routing decision.

4. **Compare LangGraph vs CrewAI on real Apollo data.** Pull 10 leads from Apollo's People Search API (or load a Clay-exported CSV with columns `company_domain`, `employee_count`, `industry`). Run each lead through both your Exercise 1 graph and your Exercise 2 crew. Print a side-by-side table of `domain | langgraph_score | crewai_score | delta`. Flag any lead where the two frameworks disagree by more than 15 points and note which one looks wrong on inspection.

5. **Build a framework failure-mode matrix.** For each of LangGraph, CrewAI, AutoGen, and Agno, write a minimal script that deliberately triggers one production failure mode: LangGraph — a node that raises mid-graph; CrewAI — a task that exceeds `max_iter`; AutoGen — a two-agent loop that never terminates; Agno — a session whose `session_state` is lost on rerun. Capture the traceback or terminal behavior for each. Write your findings — including a 4×3 matrix of (framework × {retry, checkpoint, debug}) — to `outputs/skill-framework-failure-modes.md`. Each cell must state concretely what breaks and where state lives at the moment of failure.

6. **Design and defend a framework choice for a Clay-driven CRM enrichment workflow.** The workload: pull firmographics from Clay, score ICP fit, write qualified leads back to HubSpot, and require human approval before any `SQL` is created. Pick one framework and write a 1-page design doc to `outputs/skill-framework-selection.md` containing: (a) a state diagram in text form, (b) the three most likely failure modes and how the chosen framework handles each, (c) one paragraph per rejected framework explaining the specific abstraction that would break. Also drop a minimal runnable scaffold of the chosen design at `handlers/icp-routing.py` that demonstrates the human-in-the-loop interrupt in isolation.
