# Exercises — Advanced RAG (Chunking, Reranking, Hybrid Search)

## Exercises

1. **Implement three chunking strategies and print chunk boundaries.** Write a Python script that takes a single sales-call transcript (provide one as a string constant — a paragraph that contains a pricing objection mid-sentence) and splits it using: (a) fixed-size 200-character chunks, (b) LangChain's `RecursiveCharacterTextSplitter` with `chunk_size=200, chunk_overlap=0`, and (c) a sentence tokenizer (use `nltk.sent_tokenize` or `spacy`). Print each strategy's chunks with a visible separator (`--- chunk N ---`) and annotate which strategy severs the pricing objection across a boundary. Your terminal output should make the boundary problem visible at a glance.

2. **Compute Reciprocal Rank Fusion scores for two given ranked lists.** Given a BM25 ranked list `[doc_C, doc_A, doc_D, doc_B]` and a dense retrieval ranked list `[doc_A, doc_B, doc_C, doc_E]` (provide these as Python lists), implement the RRF formula with `k=60` and print a fused ranking sorted by descending RRF score. Verify that `doc_A` ranks higher than `doc_C` after fusion even though `doc_C` was #1 in BM25 — the dense signal pulls `doc_A` up. Print each document's contribution from each list alongside the fused score.

3. **Build a hybrid search pipeline over a corpus of 20 company-about-page texts.** Use `rank_bm25` for sparse retrieval and `sentence-transformers` (`all-MiniLM-L6-v2`) for dense retrieval. For a given query (e.g., "platform that automates sales outreach"), retrieve top-10 from each system independently, then fuse using your RRF implementation. Print the BM25 top-5, the dense top-5, and the fused top-5 side by side. Identify at least one document that appears in the fused top-5 but was absent from both individual top-3 lists and explain in a printed comment why RRF surfaced it.

4. **Apply a cross-encoder reranker and measure ranking delta.** Using `cross-encoder/ms-marco-MiniLM-L-6-v2` from `sentence-transformers`, re-score the top-20 candidates from your Exercise 3 fused retrieval for the same query. Print a table comparing each document's fused-rerank position vs. its cross-encoder position, and compute the Spearman rank correlation between the two orderings. Report how many documents moved more than 3 positions and which direction. Your output should include the correlation coefficient and a short printed diagnosis of whether the cross-encoder is substantially reordering the bi-encoder candidate set.

5. **Build a RAG failure-mode diagnostic pipeline for GTM account-research notes.** Create `handlers/rag_diagnostic.py` that ingests a set of CRM-style account notes (include 15–20 sample notes as a data file or hardcoded list — these should contain ICP signals mixed with noise like lunch preferences and meeting logistics). Implement: (a) three chunking strategies, (b) hybrid BM25 + dense retrieval with RRF, and (c) cross-encoder reranking. For a query like "which accounts are evaluating a competitive replacement for [tool]?", run the full pipeline and print: the naive top-5 dense-only results, the hybrid top-5, and the reranked top-3. Flag any chunk in the reranked results that is semantically similar but factually irrelevant (i.e., discusses the right topic but answers a different question). The script should be runnable as `python handlers/rag_diagnostic.py --query "..." --notes path/to/notes.json` and produce a printed diagnostic report showing which failure modes are present at each stage.

6. **Design a chunking-and-retrieval configuration experiment and produce a tuning spec.** Using the same account-notes corpus from Exercise 5, run a sweep: test 4 chunking configurations (fixed-size at 256 and 512 tokens, recursive at 256, sentence-based with 3-sentence grouping) crossed with 2 fusion strategies (RRF vs. weighted blend at 0.7 dense / 0.3 BM25). For 5 representative queries, log which configuration pair retrieves the ground-truth relevant chunk in the top-3. Write the results as a markdown report to `outputs/skill-rag-config-experiment.md` that includes: a results table, the winning configuration, a one-paragraph rationale, and a recommended default config for GTM account-research RAG systems. The file must be generated by a script you write — not hand-authored.
