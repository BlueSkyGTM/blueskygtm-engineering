# Exercises — Function Calling & Tool Use

## Exercises

1. **Implement a minimal function-calling loop.** Using the OpenAI Python SDK, declare a single tool — `get_company_domain(company_name: str) -> str` — as a JSON Schema object. Send a prompt like *"What is the domain for Stripe?"* and write the code that detects the tool call in the response, parses the arguments, executes the function, appends the result to the conversation as a `tool` role message, and re-invokes the model to get a final natural-language answer. Print the tool name, the parsed arguments, the raw function return value, and the final assistant message to the terminal across four clearly labeled sections.

2. **Compare OpenAI and Anthropic tool-call schemas for the same toolset.** Define two tools — `lookup_company(name: str)` and `classify_icp_fit(revenue: int, industry: str)` — in both the OpenAI function-calling format and the Anthropic `tools` block format. Send the same user message (*"Check if Snowflake fits our ICP — they're in the data warehousing space"*) through both APIs. Print the raw tool-invocation payload each provider returns side by side. Then write a short summary (printed to stdout) of at least three structural differences you observe in field naming, nesting, or argument passing.

3. **Build a multi-tool agent that chains tool calls based on intermediate results.** Create an agent with three tools: `search_companies(query: str) -> list[dict]`, `enrich_company(domain: str) -> dict`, and `score_lead(company_data: dict) -> int`. Send the prompt *"Find SaaS companies in the HR tech space, enrich the top result, and score it as a lead."* Your code must execute the tools sequentially, feeding each tool's output into the next round of the conversation so the model can decide what to call next. Print every tool invocation in order along with its result, then print the final answer. No scaffolding — wire the loop yourself.

4. **Implement a tool-call agent with retry and fallback logic for failure cases.** Build a function-calling loop that calls a `fetch_linkedin_profile(url: str)` tool. Inject a simulated failure: make the function raise `ConnectionError` on the first call and a `TimeoutError` on the second, then succeed on the third. Your loop must catch these exceptions, append an error message back to the model as the tool result so the model can reason about retrying, and recover gracefully. If three consecutive failures occur for the same tool call, fall back to a `web_search(name: str)` tool and continue. Print each attempt, the exception type caught, and whether the fallback was triggered.

5. **Build a GTM enrichment-and-routing agent.** Create an agent that accepts a company name, then chains these tools: `apollo_company_search(name)` to find the company, `clearbit_enrich(domain)` to pull firmographics, `icp_score(revenue, employee_count, industry)` to score it, and `route_to_crm(score, owner)` to decide whether the lead goes to Sales (score ≥ 70) or Nurture (score < 70). Use mock data for the enrichment APIs if you do not have live API keys — but structure the tool definitions so they accept real API-shaped inputs and would work against live endpoints with a key swap. The agent must produce a final routing decision with reasoning. Save the agent implementation to `handlers/gtm_enrichment_agent.py` and write a sample enriched-and-routed lead profile (JSON) to `outputs/skill-gtm-routing.md`. Run the script and print the routing decision to stdout.

6. **Design a multi-tool GTM outreach agent with failure recovery and an evaluation harness.** Build an agent that takes a target persona and company list, then orchestrates at least four tools across enrichment, ICP scoring, message generation, and send-scheduling. The agent must include retry logic for any tool that fails, a fallback path when enrichment returns incomplete data, and a final decision about whether to trigger outreach now, defer it, or discard the lead. Then write a test function that runs the agent against five seeded company inputs — including two designed to trigger failures and one designed to produce a low ICP score — and prints a summary table showing each company's final outcome, number of tool calls made, number of retries, and whether fallback was used. Save the agent to `handlers/outreach_orchestrator.py` and save the evaluation output to `outputs/skill-outreach-eval.md`.
