# Exercises — Few-Shot, Chain-of-Thought, Tree-of-Thought

## Exercises

1. **Implement** a few-shot classifier that categorizes inbound leads into `Enterprise`, `SMB`, or `Startup` based on company headcount and ARR. Use exactly 3 examples (one per category) drawn from a JSON array of 20 prospects you create in `signals/examples/leads.json`. Run the classifier 5 times on the same held-out input and print each run's predicted label. Verify all 5 runs produce identical output — if they do not, adjust your example selection or output formatting and re-run until consistency holds. Terminal output: the 5 predictions printed one per line.

2. **Construct** a chain-of-thought prompt for a multi-step ICP-fit scoring task. The model must receive a company profile (industry, headcount, funding, tech stack) and produce: (a) a visible reasoning trace covering each scoring dimension, then (b) a final score from 1–10. Run it against 3 different company profiles you write inline and confirm that the intermediate reasoning steps are present and distinct across all 3. Terminal output: the full model response for each of the 3 profiles.

3. **Apply** few-shot prompting to a different classification problem: given an Apollo export CSV (`signals/examples/apollo_export.csv`) with columns `first_name`, `last_name`, `title`, `company`, `industry`, classify each contact's persona as `Decision Maker`, `Influencer`, or `End User`. Load the CSV, build your prompt with 3 balanced examples, and write the predictions to `outputs/persona_predictions.csv`. Print the count of contacts in each persona bucket to the terminal.

4. **Compare** token cost, latency, and accuracy across zero-shot, few-shot (3 examples), and chain-of-thought (3 examples with reasoning traces) on the same classification task from Exercise 1. For each strategy, send the same 10 test leads, measure input tokens, output tokens, and wall-clock time using `time.time()`, and compute accuracy against your hand-labeled ground truth. Print a formatted table to the terminal showing strategy, avg input tokens, avg output tokens, avg latency (ms), and accuracy (%).

5. **Build** a tree-of-thought explorer for outbound sequence planning. Given a target account description, generate 3 independent strategy branches (e.g., cold email angle, LinkedIn engagement angle, warm referral angle), evaluate each branch by prompting the model to score its likelihood of reply, and select the highest-scoring branch to produce a final 3-step sequence. Benchmark it against a single-shot chain-of-thought approach on 5 accounts — print both outputs side-by-side with the ToT evaluator scores. Persist the explorer as `handlers/tot_sequence_planner.py` and save the benchmark results to `outputs/tot_vs_cot_benchmark.md`.

6. **Design** a diagnostic tool that takes a task description as input and recommends a prompting strategy (zero-shot, few-shot, CoT, or ToT) based on task properties: output cardinality, reasoning depth, planning breadth, and example availability. Encode your decision rules as a Python function that prints a recommendation with justification. Test it against at least 6 task descriptions spanning classification, multi-step reasoning, open-ended planning, simple formatting, creative generation, and constrained extraction. Save the tool to `handlers/prompt_strategy_advisor.py` and document your decision framework with the test results in `outputs/skill-prompt-strategy.md`.
