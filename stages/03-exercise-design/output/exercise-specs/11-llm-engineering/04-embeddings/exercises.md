# Exercises — Embeddings & Vector Representations

## Exercises

1. **Generate embeddings and compute pairwise cosine similarity.** Using a local open-source embedding model (e.g., `sentence-transformers/all-MiniLM-L6-v2`), generate embeddings for the following five phrases: *"sales pipeline," "revenue funnel," "data engineering," "CRM integration,"* and *"customer relationship management."* Compute cosine similarity for every pair and print each pair as `phrase_a | phrase_b | similarity_score`. Confirm that the two semantically similar pairs score highest. No starter code — wire the model load, embedding generation, and similarity computation yourself.

2. **Build a brute-force nearest-neighbor lookup.** Given a hardcoded list of 12 GTM-related phrases (e.g., *"lead scoring," "outbound sequencing," "intent data," "ICP definition,"* etc.), accept a query string from a command-line argument and print the top 3 most similar phrases ranked by cosine similarity. Run at least three queries: one that should match sales terminology, one that should match marketing terminology, and one unrelated query (e.g., *"potato recipe"*) to confirm low-similarity results. Print ranked output with scores.

3. **Implement a semantic router for inbound messages.** Define five category labels with descriptions (e.g., `"pricing_inquiry": "Questions about cost, pricing tiers, or billing"`, `"demo_request": "Wants to see a product demonstration"`, `"support_ticket": "Experiencing a bug or technical issue"`, `"competitor_mention": "Mentions a competing product or vendor"`, `"partnership": "Interested in integration or partnership"`). For each category, embed the description. Accept a list of 10 sample inbound messages (write your own), embed each one, and classify it by finding the nearest category vector via cosine similarity. Print each message alongside its predicted label and confidence score.

4. **Apply the semantic router to enriched CRM records.** Create a JSON file containing 15 mock Apollo-style lead records, each with a `notes` field containing a free-text snippet from a sales rep (e.g., *"prospect asked about enterprise pricing and mentioned Evaluating CompetitorX"*, *"wants a technical demo for their engineering team"*). Run each notes field through your semantic router from Exercise 3 and write the predicted category into a new `predicted_category` field. Write results to `outputs/crm_routed_leads.json` and print a summary count of leads per category.

5. **Benchmark brute-force vs. cached lookup latency and evaluate router accuracy.** Extend your semantic router to cache category embeddings to disk (e.g., as a `.npy` or `.pkl` file). On first run, compute and persist category embeddings; on subsequent runs, load from cache. Measure wall-clock time for 100 classification calls under both uncached (compute embeddings fresh) and cached modes using `time.perf_counter`. Then, create a ground-truth file with 20 human-labeled messages (message + correct category). Run your router against them and compute per-category precision, recall, and F1, plus an overall accuracy percentage. Print a formatted evaluation table. Persist the evaluation script to `handlers/router_eval.py`.

6. **Design and validate a production-ready category schema for a Clay enrichment workflow.** Define a category schema with at least 8 GTM-relevant intent categories (e.g., churn_risk, expansion_opportunity, product_feedback, pricing_objection, etc.), each with a carefully crafted description. Collect or write 30 diverse inbound messages covering all categories. Run your router, evaluate accuracy, and iterate on category descriptions until per-category F1 meets a threshold you define (document the threshold in your output). Write a short design doc summarizing your schema choices, failure modes, and latency characteristics to `outputs/skill-semantic-router-design.md`.
