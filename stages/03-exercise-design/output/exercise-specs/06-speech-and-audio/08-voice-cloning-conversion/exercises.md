# Exercises — Voice Cloning & Voice Conversion

## Exercises

1. **Implement a speaker embedding extraction pipeline** using a pretrained speaker verification model (e.g., SpeechBrain ECAPA-TDNN or Resemblyzer). Load at least three reference audio clips from different speakers (use any public voice dataset or record 5-second samples yourself). Extract a 192- or 512-dimensional embedding from each clip and compute pairwise cosine similarity across all speakers. Print a labeled similarity matrix to the terminal. Verify that same-speaker pairs (if you include duplicates) score above 0.75 and cross-speaker pairs fall below 0.6.

2. **Generate cloned speech from a short text prompt** using a few-shot TTS model (e.g., Coqui XTTS, OpenVoice, or YourTTS). Condition the model on a speaker embedding extracted from a 5–10 second reference clip of your choice. Save the output as a WAV file and print its duration, sample rate, and the number of samples to the terminal. Confirm the output duration is proportional to the input text length (roughly 100–180 words per minute for natural speech).

3. **Implement voice conversion** that transforms a source speaker's audio into a target speaker's voice while preserving linguistic content and prosody. Use a conversion model such as OpenVoice, FreeVC, or RVC. Feed a 10-second clip of Speaker A speaking a known sentence and a reference clip of Speaker B. Save the converted audio and extract a speaker embedding from the output. Compute and print the cosine similarity between the converted audio's embedding and Speaker B's reference embedding — the score should exceed 0.7 if identity transfer succeeded.

4. **Build an evaluation harness** that scores a directory of cloned audio files on two axes: speaker fidelity and intelligibility. For speaker fidelity, compute cosine distance between each clone's extracted embedding and the target speaker's reference embedding. For intelligibility, run each clone through a speech-to-text model (e.g., Whisper) and compute character error rate against the known transcript. Print a table with columns: filename, cosine_similarity, CER, and a pass/fail flag where pass requires cosine > 0.70 and CER < 0.15.

5. **Build a consent-gated voice cloning handler** that reads prospect records from a CRM export (CSV with columns: `name`, `email`, `voice_consent_status`, `reference_audio_path`, `state`) and only proceeds with cloning when `voice_consent_status` is `"opted_in"` and the prospect's jurisdiction permits synthetic voice use (block EU residents unless an EU AI Act disclosure flag is present; block California residents unless AB 2905 disclosure was acknowledged). For each approved record, generate a cloned audio sample, embed an inaudible watermark using a library such as `inaudiotechs watermarking` or `silenteye`, and write a JSON audit log to `outputs/voice_clone_audit.json` containing the prospect email, timestamp, consent status, watermark ID, and model version. Save the handler to `handlers/voice_clone_consent.py`. Run it against a 10-row test CSV and print the count of approved, blocked, and watermarked records to the terminal.
