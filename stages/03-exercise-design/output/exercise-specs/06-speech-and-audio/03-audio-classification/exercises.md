## Exercises

1. **Compute MFCC features and inspect dimensionality reduction.** Load any 3-second WAV file (use `librosa.load` with `sr=22050`) and extract 13-component MFCCs with default frame and hop lengths. Print the raw waveform sample count, the MFCC matrix shape, and the compression ratio (raw samples divided by total MFCC values). Then compute the mean MFCC vector across all frames and print its 13 values. Your terminal output should show the raw dimensionality, the reduced dimensionality, and the per-coefficient means.

2. **Implement k-NN classification on MFCC vectors with cross-validation.** Using the UrbanSound8K dataset (or `librosa.examples` if unavailable), compute mean MFCC vectors for a subset of at least 200 audio clips across 5 or more class labels. Implement 5-fold cross-validation with `sklearn.neighbors.KNeighborsClassifier` (k=5, Euclidean distance). Print per-fold accuracy, mean accuracy, and standard deviation. Your terminal output should show five fold-level accuracy lines and a summary line.

3. **Run zero-shot AST classification and retrieve top-k predictions.** Load the MIT/ast-finetuned-audioset-10-10-0.4593 model from Hugging Face (`transformers` pipeline). Feed a single audio file (any WAV you can find — a dog bark, a siren, speech, etc.) and print the top-5 predicted labels with their confidence scores, sorted descending. Also print the total inference time in milliseconds. Confirm that the output shows label strings, float scores, and a latency number.

4. **Compare MFCC+k-NN against AST zero-shot on the same GTM audio inputs.** Download 10 short sales-call voicemail clips from a public call dataset (or generate synthetic clips using `gTTS` with different scripts: product inquiry, pricing objection, meeting request, callback request, competitor mention). Build a small labeled MFCC+k-NN classifier on 5 of them, then classify the remaining 5. Independently run AST zero-shot on all 10 and map its AudioSet labels to your 5 call-type categories manually. Print a side-by-side table: filename, k-NN prediction, AST prediction, k-NN latency (ms), AST latency (ms). Summarize which pipeline got more call types correct against ground-truth labels you assigned.

5. **Build a tradeoff diagnostic that benchmarks MFCC+classifier vs AST across data regimes.** Create `handlers/audio_triage_benchmark.py` that accepts a directory of labeled WAV files and a `--sample_fraction` flag (0.1, 0.25, 0.5, 1.0). For each fraction, train a k-NN on mean MFCCs using that fraction of the data, evaluate on the held-out set, and record accuracy and mean inference latency. Then run AST zero-shot on the same held-out set and record accuracy (using your manual label mapping) and latency. Write all results as a markdown table to `outputs/skill-audio-triage.md` with columns: sample_fraction, knn_accuracy, knn_latency_ms, ast_accuracy, ast_latency_ms, winner. Run the script with all four fractions and confirm the markdown file is generated with four rows.
