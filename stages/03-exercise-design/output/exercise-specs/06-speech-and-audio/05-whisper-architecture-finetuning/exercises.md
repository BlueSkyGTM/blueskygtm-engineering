## Exercises

1. **Trace an audio signal through Whisper's pipeline and print every intermediate representation.** Load the `openai/whisper-tiny` model and feed it a 5-second WAV file of your choice. After running the model, print: (a) the shape of the log-Mel spectrogram tensor produced by `whisper.log_mel_spectrogram`, (b) the shape of the encoder's last hidden state, and (c) the first 10 token IDs from the decoder before they are detokenized. End by printing the final transcribed string. Run the script from the terminal and confirm you see all four outputs (spectrogram shape, hidden-state shape, token IDs, transcription text).

2. **Compare two Whisper model sizes on the same audio and print a side-by-side table.** Transcribe one audio file (10–30 seconds) using both `whisper-tiny` and `whisper-base`. Measure wall-clock inference time for each with `time.perf_counter`. Print a formatted table to the terminal showing: model name, transcription text, inference time in milliseconds, and token count. Identify at least one word where the two models disagree.

3. **Implement batch transcription with timestamped JSON output.** Write a script that takes a directory of `.wav`, `.mp3`, or `.m4a` files as a command-line argument, transcribes every file with a Whisper model of your choice, and writes a single JSON file to `outputs/transcripts.json`. Each entry in the JSON array must contain: `filename`, `full_text`, and a `segments` list where each segment has `start`, `end`, and `text`. Print a summary line to the terminal showing how many files were processed and the total audio duration transcribed.

4. **Compare short-window vs. long-form transcription on a file longer than 60 seconds.** Find or record an audio file that is at least one minute of continuous speech. Transcribe it two ways: (a) manually split the file into 30-second chunks, transcribe each chunk independently, and concatenate the results; (b) use Whisper's built-in long-form transcription (`model.transcribe` with default settings). Print both transcriptions to the terminal and report whether any words were dropped or duplicated at chunk boundaries in approach (a). Write a one-paragraph summary of the tradeoffs to `outputs/window-vs-longform.md`.

5. **Build and save a LoRA adapter that shifts Whisper's behavior on domain-specific vocabulary.** Create a small dataset of 20–50 audio-text pairs in a specialized domain (medical terms, legal jargon, technical product names, or GTM-specific terms like "MQL," "ARR," "churn rate"). Configure a LoRA adapter on `whisper-tiny` or `whisper-base` using `peft`, fine-tune for at least 50 steps, and save the adapter weights to `signals/examples/whisper-lora-adapter/`. Then load the base model with and without the adapter, transcribe one test audio clip containing domain vocabulary, and print both transcriptions side-by-side to the terminal so you can observe the vocabulary shift.

6. **Design a GTM sales-call transcription pipeline that enriches CRM records from spoken content.** Build an end-to-end script that: (a) batch-transcribes a directory of simulated sales-call audio files using Whisper with timestamped segments, (b) extracts any company names, product mentions, or pricing figures mentioned in each call using a regex or LLM-based extraction step, and (c) writes a structured CSV to `outputs/call-enrichment.csv` with columns: `call_id`, `transcript_summary`, `companies_mentioned`, `pricing_detected`, `next_steps`. The script must accept the input directory as a CLI argument and print a per-call summary to the terminal as it processes. If you have access to a real enrichment API (Clay, Apollo, Clearbit), optionally resolve one extracted company name against it and append a `domain` column.
