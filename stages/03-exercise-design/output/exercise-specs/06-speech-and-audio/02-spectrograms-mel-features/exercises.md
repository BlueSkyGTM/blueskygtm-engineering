# Exercises — Spectrograms, Mel Scale & Audio Features

## Exercises

1. **Compute the Hz-to-Mel mapping across the audible spectrum.** Implement the standard warping formula `mel = 2595 * log10(1 + hz / 700)` and print a two-column table for frequencies `[100, 200, 500, 1000, 2000, 4000, 8000, 16000]`. Add a third column showing the ratio `mel / hz` so the perceptual compression at high frequencies is visible. Verify that 1000 Hz maps to approximately 1000 Mel.

2. **Implement STFT from scratch and compare representation shapes.** Load any WAV file at 16 kHz (generate a synthetic 440 Hz tone if no file is available). Apply framing (25 ms window, 10 ms hop), a Hann window, and `np.fft.rfft` per frame. Print the shape of three arrays: the raw waveform `(samples,)`, the linear spectrogram `(freq_bins, frames)`, and a Mel spectrogram computed via `librosa.feature.melspectrogram`. Confirm that the Mel spectrogram has fewer frequency bins than the linear spectrogram.

3. **Construct a triangular Mel filterbank without `librosa.filters.mel`.** Given a sample rate of 16000 and an FFT size of 512, compute Mel center frequencies for 40 filters spanning 0–8000 Hz, convert back to Hz, build overlapping triangular weight matrices, and apply them to a real magnitude spectrum. Print the filterbank shape `(40, 257)` and the energy of the first five Mel bands for a sample spectrum. Compare your output to `librosa.filters.mel(sr=16000, n_fft=512, n_mels=40)` — the max absolute difference per filter should be below `1e-2`.

4. **Extract MFCCs and compare two audio segments.** Choose or synthesize two distinct audio clips (e.g., a spoken vowel "ah" vs. "ee," or two different musical notes). Compute 13 MFCCs for each using a DCT-II applied to the log-Mel spectrogram. Print the coefficient vectors side by side and compute the Euclidean distance between them. Confirm that phonetically or musically distinct sounds produce measurably different MFCC vectors.

5. **Build a sales-call audio feature extraction pipeline.** Download two sample call recordings (use Gong demo audio, a public sales-call dataset, or synthetic speech from a TTS API). Write a script in `handlers/call_feature_extractor.py` that accepts a directory of WAV files, computes a 20-coefficient MFCC vector averaged across all frames for each file, and writes a JSON summary to `outputs/call_audio_features.json` containing per-file mean MFCCs, spectral centroid, and total duration. The script must be runnable as `python handlers/call_feature_extractor.py path/to/wav_dir/` and print a progress line per processed file. Extend the pipeline to accept a CSV exported from Clay or Apollo containing a `call_recording_url` column, download each file, and append the enriched features back as new columns in `outputs/call_audio_features_enriched.csv`.
