# Exercises — Speaker Recognition & Verification

## Exercises

1. **Extract speaker embeddings and print pairwise cosine similarity.** Load two audio files (use any two `.wav` files from the `voices/` directory or download two short utterances from VoxCeleb). Pass both through the ECAPA-TDNN encoder used in the lesson. Compute the cosine similarity between the two 192-dimensional embedding vectors. Print both embedding shapes and the similarity score rounded to four decimal places. Run the script twice — once with two utterances from the same speaker and once with utterances from different speakers — and confirm the same-speaker score is higher.

2. **Implement a 1:1 verification function with a configurable threshold.** Write a Python function `verify_speaker(enroll_path, test_path, threshold)` that returns a `(decision: bool, score: float)` tuple. The threshold should default to 0.50 but be overridable via a CLI argument. Run the function on five enrollment/test pairs listed in `trials/verification_pairs.txt` and print a table showing the pair ID, similarity score, decision, and ground-truth label. Adjust the threshold until at least four of five decisions are correct and print the final threshold you settled on.

3. **Build a 1:N speaker identification system and compare its error profile to 1:1 verification.** You are given an enrollment directory containing utterances from five speakers (`data/enroll/`). Write a script that loads all five embeddings into a gallery, then classifies each test utterance in `data/test/` by selecting the gallery speaker with the highest cosine similarity. Compute the identification accuracy. Then run the same test utterances through 1:1 verification (match against each gallery speaker with threshold 0.50, accept the first match or reject all). Print both the 1:N accuracy and the 1:1 rejection rate side by side, and write a two-sentence summary of which approach produces more false accepts and why.

4. **Compute EER and plot a DET curve from a scored trial list.** Load `trials/scored_trials.csv`, which contains three columns: `enroll_speaker`, `test_speaker`, and `cosine_score`. Compute the false acceptance rate (FAR) and false rejection rate (FRR) across a sweep of threshold values from 0.0 to 1.0 in steps of 0.01. Find the threshold where FAR equals FRR — that is your EER point. Print the EER percentage and the corresponding threshold. Then plot the detection error tradeoff curve (FAR on x-axis, FRR on y-axis, log-scaled) and save the figure to `outputs/det_curve.png`.

5. **Integrate an anti-spoofing check into a verification pipeline.** Build a module that runs both speaker verification (using your `verify_speaker` function) and a spoofing classifier (use the pre-trained model from the `antispoofing/` directory or the Hugging Face model referenced in the lesson) on each incoming test utterance. For each trial, output one of four labels: `accept`, `reject_low_score`, `reject_spoof`, `reject_both`. Test on the files in `trials/mixed/` (which include genuine live speech, replayed audio, and synthesized speech). Save your pipeline as a reusable module at `handlers/voice_auth.py` and print a summary table showing per-file results and the overall spoof catch rate.

6. **Design a speaker verification gate for inbound sales calls.** A B2B sales team uses Clay to enrich inbound caller data. When a prospect calls the demo line, the system should verify the caller's voice against a previously enrolled voiceprint before routing the call to an AE. Write a script at `signals/examples/call_voice_gate.py` that: (a) reads a JSON file of inbound call records (caller phone, recording path, expected contact ID), (b) pulls the enrolled voiceprint for that contact from a local `voiceprints/` directory keyed by contact ID, (c) runs verification plus anti-spoofing, and (d) emits a JSON array of routing decisions (`route_to_ae`, `send_to_manual_review`, `block`) with reasons. Use sample call records in `signals/examples/sample_calls.json`. Print the routing decisions and save them to `outputs/call_routing_decisions.json`.
