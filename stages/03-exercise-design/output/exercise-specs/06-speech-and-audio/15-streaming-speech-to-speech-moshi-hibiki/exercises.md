## Exercises

1. **Compute the Mimi frame budget for three S2S latency targets.** Write a Python script that takes end-to-end latency targets of 300 ms, 500 ms, and 800 ms and prints, for each target, the number of Mimi audio frames available (at 12.5 Hz, 80 ms per frame), the number of codebook tokens per stream within that budget, and the total combined token count across both user and model streams. Run the script and confirm your terminal prints a table with one row per latency target.

2. **Build a dual-stream token alignment for a 6-frame window.** Given a list of 6 user audio token frames (each frame is a list of 8 codebook integers) and 6 model audio token frames of the same shape, write a function that returns a single interleaved sequence where each timestep pairs the corresponding user frame and model frame. Print the first 3 aligned timesteps to the terminal as `[user_frame, model_frame]` pairs and verify the indices match (e.g., timestep 0 shows user frame 0 alongside model frame 0).

3. **Implement overlap-frame detection on a dual-stream token window.** You are given two parallel token streams — `user_tokens` and `model_tokens` — each containing 50 frames of 8 codebooks. An "active" frame is one where at least 5 of 8 codebooks are non-zero. Write a script that scans both streams frame-by-frame, identifies every timestep where both streams are simultaneously active (an overlap), and prints the list of overlap frame indices along with the total overlap count. Test your script on the sample data in `signals/examples/overlap_sample.py` (you will need to create this file with synthetic token data — at least 3 frames should register as overlaps).

4. **Compare full-duplex S2S against the ASR → LLM → TTS pipeline and output a structured report.** Write a script that takes hardcoded latency estimates for each pipeline stage (e.g., ASR: 150 ms, LLM first-token: 250 ms, TTS: 200 ms) and for the S2S model (e.g., single-pass: 160 ms per frame), then computes and prints: total latency floor for each architecture, whether backchanneling is structurally possible (yes/no with one-sentence justification), and which architecture supports sub-500 ms turn-taking. Print the comparison as a formatted table to the terminal.

5. **Configure a full-duplex latency budget for an outbound GTM call fleet and evaluate real call telemetry.** Pull a CSV export of call records from your Clay table (or use the provided sample at `data/gtm_call_telemetry.csv` containing columns: `call_id`, `round_trip_latency_ms`, `overlap_frames`, `talk_ratio_agent`, `talk_ratio_prospect`, `outcome`). Write a script that defines a latency budget of 480 ms end-to-end (6 Mimi frames), flags any call whose `round_trip_latency_ms` exceeds the budget, counts overlap frames per call, and computes the percentage of calls that meet the budget. Output a summary report to `outputs/skill-s2s-gtm-fleet-eval.md` containing: total calls evaluated, budget pass rate, top 5 worst-latency call IDs, and a recommendation on whether the fleet can sustain full-duplex quality.

6. **Design a turn-taking controller that uses overlap detection to trigger backchannels.** Write a Python module at `handlers/backchannel_controller.py` that accepts a rolling window of dual-stream frames (user + model), runs the overlap detection logic from Exercise 3, and emits a boolean `should_backchannel` signal whenever overlap is detected for 2+ consecutive frames but the model stream has been silent (all-zero codebooks) for the preceding frame. Include a `__main__` block that runs on synthetic input demonstrating at least one backchannel trigger and one non-trigger, printing the frame index and decision for each timestep.
