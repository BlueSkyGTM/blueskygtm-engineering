## Exercises

1. **Implement the core SSM recurrence** for a toy input sequence. Write a Python script that initializes a state vector $h_0$ of dimension $d=8$, creates random discretized matrices $\bar{A} \in \mathbb{R}^{8\times8}$ and $\bar{B} \in \mathbb{R}^{8\times4}$, and processes a synthetic input sequence $x_t \in \mathbb{R}^4$ for $t=1,\ldots,50$. At each step compute $h_t = \bar{A}h_{t-1} + \bar{B}x_t$ and print the L2 norm $\|h_t\|$ every 10 steps. Then increase the sequence length to 5,000 and confirm that the final state is still an 8-dimensional vector — demonstrating constant-memory inference regardless of sequence length. Print the total memory consumed by the state at $N=50$ and $N=5000$ to verify they are identical.

2. **Compute the KV cache footprint** for a Jamba-class model versus a pure Transformer at 256K context. Given the following parameters — 32 Transformer layers in the pure model, Jamba's effective 4 attention layers (due to 1:7 interleaving across 32 blocks), 16 KV heads, head dimension 128, FP16 (2 bytes per element) — write a script that calculates cache size in bytes for both architectures at sequence lengths `[4096, 16384, 65536, 131072, 262144]` and prints a formatted table with columns: context length, pure-transformer GB, jamba GB, reduction factor.

3. **Benchmark linear versus quadratic scaling empirically.** Write a script that simulates the two computational kernels: (a) a sequential SSM sweep — a loop of fixed-size matrix multiplies for $N$ steps, and (b) a full attention matrix — construct an $N \times N$ similarity matrix. Time both at $N \in [512, 1024, 2048, 4096, 8192]$ using `time.perf_counter`. Print a table showing wall-clock time for each, plus the ratio `attention_time / ssm_time`. Verify that the ratio grows roughly linearly with $N$, confirming quadratic vs. linear behavior.

4. **Implement and trace the Jamba block interleaving pattern.** Write a Python module that defines a `JambaBlock` sequence for a model with 32 Jamba blocks. Each block contains: 1 attention layer followed by 7 alternating SSM + MoE MLP layers (the 1:7 ratio). Build a list of layer-type strings (e.g., `["attn", "ssm", "moe", "ssm", "moe", ...]`), then print the full layer schedule for blocks 0–3. Count total attention layers, total SSM layers, and total MoE layers across all 32 blocks and verify the ratio is 1:7:7. Add a brief comment in your script explaining *why* the 1:7 ratio exists — the trade-off between attention's long-range precision and the SSM's throughput/memory efficiency.

5. **Build a KV-cache budget estimator for GTM enrichment pipelines.** Create `handlers/jamba_kv_estimator.py` that takes a JSON payload of Apollo-enriched company profiles (at least 50 companies, each with description, technographics, contact list, and news articles — total ~8K tokens per company) and computes how many such companies can fit into a single context window under three architectures: (a) pure Transformer with 80 layers, (b) Jamba with 1:7 interleaving across 80 blocks, (c) a hypothetical all-SSM model with zero attention layers. The script should accept a `--max-cache-gb` argument (default 8.0) and print, for each architecture, the maximum number of companies that fit and the effective tokens processed. Read the payload from `signals/examples/apollo_companies.json` (create a small sample if the file does not exist). Artifact: `handlers/jamba_kv_estimator.py`.

6. **Implement and verify the three Mamba-3 innovations, then document which weakness each addresses.** Write a script that defines three SSM step functions: (1) a standard ZOH discretization, (2) an exponential-trapezoidal discretization, and (3) the same with a complex-valued state
