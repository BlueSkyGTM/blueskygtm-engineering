# Exercises — Gradient Checkpointing and Activation Recomputation

## Exercises

1. **Compare peak memory between standard and checkpointed forward passes.** Build a 10-layer `nn.Module` where each layer is a `Linear(2048, 2048)` followed by `ReLU`. Run a single forward pass on a batch of 64 random inputs under two configurations—standard backprop and `torch.utils.checkpoint.checkpoint` wrapping every layer—and print peak allocated memory via `torch.cuda.max_memory_allocated()` (or the CPU equivalent using `tracemalloc`). Print both numbers to the terminal labeled clearly so you can see the delta.

2. **Implement gradient checkpointing in a training loop.** Take the model from Exercise 1 and write a full training loop that runs 20 steps of SGD on random data. Use `torch.utils.checkpoint.checkpoint` on each layer's forward call. Ensure `use_reentrant=False` is set. Print the loss every 5 steps and the peak memory at the end. Then run the same loop *without* checkpointing and print peak memory again so both numbers appear in the terminal.

3. **Configure checkpoint segment boundaries and measure the trade-off.** Using a 20-layer version of the same model, implement three strategies: (a) no checkpointing, (b) checkpoint every layer, (c) checkpoint every *other* layer (segments of size 2). For each strategy, run 50 training steps and print a three-column summary table to the terminal: strategy name, peak memory in MB, and total wall-clock time in seconds. State which strategy gives the best memory/time trade-off.

4. **Apply checkpointing to a lead-scoring model trained on enriched CRM data.** Load the CSV at `data/leads_enriched.csv` (columns: `company_name`, `employees`, `revenue`, `technographics_count`, `intent_score`, `label`). Build a 15-layer MLP classifier on the numeric features. Train it two ways—standard and checkpointed—and print classification accuracy plus peak memory for each. Confirm the checkpointed version achieves similar accuracy with measurably lower peak memory.

5. **Build a diagnostic tool that recommends whether to enable checkpointing.** Implement a Python script `handlers/checkpoint_advisor.py` that accepts a model description (number of layers, hidden dimension, batch size) via command-line args, runs a quick profiling sweep (forward + backward under both standard and checkpointed modes), and writes a markdown report to `outputs/skill-checkpoint-advisor.md`. The report must include a comparison table of peak memory and wall-clock time, the compute overhead percentage, and a recommendation string: `"CHECKPOINT"` or `"STANDARD"` based on whether memory savings exceed 20% and compute overhead is under 50%. Run the tool on a 25-layer model with hidden dim 4096 and batch size 128.

6. **Design and execute an experiment proving gradient checkpointing scales to deeper models.** Create a script `signals/examples/checkpoint_scaling.py` that trains classifier models at depths of 5, 10, 20, 40, and 80 layers on `data/leads_enriched.csv`. For each depth, attempt training both with and without checkpointing, catching `OutOfMemoryError` gracefully. Write results to `outputs/skill-checkpoint-scaling.md` as a table showing depth, strategy, success/fail, peak memory, wall-clock, and final accuracy. The artifact must demonstrate the maximum depth achievable under each strategy given the available GPU/CPU memory.
