# Exercises — Pre-Training a Mini GPT (124M Parameters)

## Exercises

1. **Run the pre-training loop and observe the loss curve.** Execute the training loop from the lesson for 500 steps on the provided WikiText-2 shard. Print the cross-entropy loss every 50 steps. Confirm the loss decreases monotonically over the first 300 steps. Then re-run with `learning_rate=1e-2` and observe what happens to the loss. Save both loss curves as comma-separated values to `outputs/loss-comparison.csv` with columns `step, baseline_loss, high_lr_loss`.

2. **Compare random-init vs. checkpointed text generation.** Write a script that loads the model at initialization (step 0) and at the final checkpoint after training. Feed both the prompt `"The future of technology"` and generate 50 tokens using temperature 0.8 and top-k sampling. Print both outputs side by side in the terminal. Describe the qualitative differences you observe — does the trained model produce real words, grammatical structure, or coherent phrases that the random model does not?

3. **Implement autoregressive generation and compare its teacher-forced loss.** During training, the model sees the entire ground-truth sequence at once (teacher forcing). During inference, it generates token-by-token, feeding its own predictions back as input. Implement a function that generates 20 tokens autoregressively from a fixed prompt, then computes the cross-entropy loss between the generated tokens and the actual continuation in the dataset. Compare this "autoregressive loss" to the "teacher-forced loss" on the same starting context. Print both values and explain why the autoregressive loss is higher.

4. **Pre-train on a GTM text corpus.** Export 500–1000 company descriptions from an Apollo or Clay enrichment table (CSV with a `description` or `about` column). Tokenize these descriptions using the GPT-2 BPE tokenizer, concatenate into a single stream, and run the pre-training loop for 200 steps with a batch size of 8 and sequence length of 128. Print the loss every 25 steps. After training, generate a 100-token sample conditioned on the prompt `"Our company helps businesses"`. Does the output resemble GTM copywriting language?

5. **Build a training diagnostic dashboard.** Extend the training loop to log six signals at every step: loss, gradient norm, learning rate, mean activation magnitude in the final transformer block, fraction of dead attention heads (entropy below a threshold), and tokens-per-second. Run training for 1000 steps. Write the full log to `outputs/training-log.jsonl` and generate a Markdown report at `outputs/skill-pretrain-diagnostics.md` containing a loss curve summary table, the five most anomalous steps (largest loss jumps or gradient spikes), and a written diagnosis of whether the run exhibited any of the three failure modes covered in the lesson (plateau, divergence, gradient explosion).

6. **Prove that gradient clipping prevents divergence.** Run two controlled experiments: (a) train for 300 steps with `learning_rate=5e-3` and no gradient clipping, and (b) train for 300 steps with the same learning rate but with gradient norm clipping at 1.0. Log the loss and gradient norm for both runs. Write a comparison report to `outputs/skill-clip-vs-noclip.md` that includes both loss curves, the step at which each run's loss first exceeded 10.0 (or note if it never did), and a quantitative argument for whether clipping delayed, prevented, or had no effect on divergence.
