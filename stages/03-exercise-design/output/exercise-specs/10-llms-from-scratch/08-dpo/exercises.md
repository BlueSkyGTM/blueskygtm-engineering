# Exercises — DPO: Direct Preference Optimization

## Exercises

1. **Implement the DPO loss function in NumPy and trace the log-probability ratios.** Create a Python script that accepts arrays of `logp_chosen`, `logp_rejected`, `ref_logp_chosen`, and `ref_logp_rejected` (hard-code 5 sample rows). Compute the per-example DPO loss `(beta * logp_ratio_rejected - beta * logp_ratio_chosen).sigmoid().log()` and print a table showing each example's chosen/rejected log-prob ratios, the implicit reward difference `beta * (logp_chosen - ref_logp_chosen) - beta * (logp_rejected - ref_logp_rejected)`, and the final loss. Verify that examples where the chosen response already had higher log-prob under the policy produce lower loss.

2. **Compare DPO and RLHF along the three axes from the lesson: model count, training stability, and compute cost.** Write a script that prints a formatted comparison table (use plain string formatting or `tabulate`) showing: number of models loaded during training, whether a separate reward model is required, whether a value head / critic is needed, typical GPU memory footprint for a 1.3B parameter model, and number of forward/backward passes per optimization step. Include a final line printing the estimated compute cost ratio `RLHF_steps / DPO_steps` assuming equivalent dataset size. Run the script and confirm the table renders in your terminal.

3. **Compute how beta controls the effective KL constraint by sweeping values.** Build a script that defines a synthetic policy distribution and reference distribution over a 10-token vocabulary (use `numpy.random.dirichlet` seeded for reproducibility). For each beta in `[0.01, 0.05, 0.1, 0.3, 0.5, 1.0]`, compute the DPO-optimal policy update (the closed-form solution from the lesson), then measure `KL(new_policy || reference)` using `scipy.stats.entropy`. Print a table of beta vs. KL divergence vs. max probability assigned to any single token. Confirm that higher beta produces smaller KL — the policy stays closer to the reference.

4. **Format GTM A/B cold-email data into DPO-compatible triples.** Create a JSONL file at `outputs/gtm_dpo_dataset.jsonl` containing 20 preference pairs sourced from cold-outreach A/B test results. Each line must have the schema `{"prompt": str, "chosen": str, "rejected": str}`. Use realistic GTM content: the prompt is a prospect context (e.g., "Write a cold email to a VP of Sales at a Series B SaaS company using outbound data from Apollo"), the chosen field is the subject-line + body that won the A/B test (higher open or reply rate), and the rejected field is the losing variant. Write a validation script that loads the JSONL, checks every line has all three keys, checks that chosen and rejected differ by at least 10 characters, and prints a summary: total examples, average prompt length, and 3 sample pairs.

5. **Run DPO training with `trl`'s `DPOTrainer` on a small open model using the dataset from Exercise 4 and produce a persistent artifact.** Load `HuggingFaceTB/SmolLM2-135M` (or another sub-500M model of your choice) as both policy and reference model. Configure `DPOConfig` with `beta=0.1`, `learning_rate=5e-7`, and `max_steps=50`. Before and after training, compute the average log-probability of chosen vs. rejected responses on 5 held-out examples from your dataset. Write the before/after log-probability comparison, the training loss curve (final 10 step losses), and your `beta` and `max_steps` settings to `outputs/skill-dpo-training.md`. Confirm the chosen-vs-rejected log-prob gap widened after training.

6. **Design an experiment that determines the beta value producing the best reply-rate lift on held-out GTM data.** Train three DPO models using the dataset from Exercise 4 at `beta` values `[0.05, 0.1, 0.5]`. For each trained model, generate 10 cold emails on held-out prompts and compute a proxy quality score: the average log-probability of the generated text under the model minus the average log-probability under the reference (this is the implicit reward the policy has learned). Write a report to `outputs/skill-dpo-beta-ablation.md` that includes a table of `beta` vs. proxy quality score vs. KL divergence from reference, your recommendation for which beta to ship, and the reasoning grounded in the numbers. State explicitly whether higher or lower beta produced better preference alignment and whether it came at the cost of drift from the reference model.
