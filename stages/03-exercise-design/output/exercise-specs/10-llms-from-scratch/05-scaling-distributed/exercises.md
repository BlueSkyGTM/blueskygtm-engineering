# Exercises — Scaling: Distributed Training, FSDP, DeepSpeed

## Exercises

1. **Implement a parallelism-strategy classifier.** Write a Python script that defines a function `describe_strategy(name)` accepting one of `"data"`, `"tensor"`, `"pipeline"`, or `"sharded_data"`. The function returns a dictionary with keys `splits` and `replicates`, each a list of strings naming what that strategy partitions or copies across devices. Call all four and print a formatted table to stdout. Verify: the table shows four distinct rows, and `"sharded_data"` lists parameters, gradients, and optimizer states under `splits`.

2. **Compute per-GPU memory budgets for a 7B parameter model.** Write a script that takes model hyperparameters (hidden size 4096, 32 layers, vocab 32000) and a GPU count (default 8), then computes and prints per-GPU memory consumption for DDP, ZeRO-1, ZeRO-2, and ZeRO-3/FSDP. Account for parameters in bf16 (2 bytes/param), gradients (2 bytes/param), Adam optimizer states (8 bytes/param for m and v in fp32). Print a table with columns: strategy, params_memory, grad_memory, optim_memory, total_per_gpu. Verify: DDP total per GPU is roughly 84 GB (uncompressed across 8 GPUs); ZeRO-3/FSDP is roughly 10.5 GB.

3. **Configure a minimal FSDP training loop on a toy model.** Without copying lesson code, write a standalone script that defines a small transformer (4 layers, hidden 256), wraps it with `FullyShardedDataParallel`, sets `MixedPrecision(param_dtype=torch.bfloat16, reduce_dtype=torch.bfloat16)`, enables gradient checkpointing via `model.gradient_checkpointing_enable()`, and runs 5 training steps on random data. Print the loss after each step and the FSDP shard size for rank 0 using `model._fsdp_wrapped_module` introspection or `FSDP.sharded_state_dict_type()`. Verify: loss values are printed and are finite; shard size is smaller than the full parameter count.

4. **Build a `torchrun` launcher with rank-aware verification.** Write a script `train_dist.py` that initializes a process group, prints `rank`, `world_size`, and `local_rank` for each process, then creates a small model wrapped in FSDP and runs 3 forward-backward steps. Include a verification block that uses `torch.distributed.all_reduce` to confirm that gradients on all ranks are synchronized (norms match across ranks within 1e-5). Launch with `torchrun --nproc_per_node=2 train_dist.py` (or `--nproc_per_node=1` if single GPU, and note the limitation). Verify: each rank prints its identity; the sync-verification prints `GRADIENT_SYNC_OK` or `GRADIENT_SYNC_FAILED`.

5. **Produce a build-vs-buy cost analysis for fine-tuning a custom GTM intent-classification model.** Write a script that takes a target model size (e.g., 7B), a training token count (e.g., 500M tokens of Apollo-enriched lead transcripts and Clay webhook payloads), a GPU type (A100 80GB at $2/hr or H100 at $3.5/hr), and estimated training tokens-per-second throughput (look up or assume 15K tokens/s/GPU for 7B with FSDP+bf16). Compute total GPU-hours, total cost, and compare against the cost of calling GPT-4o at $5/M-input-tokens for the same inference workload over a 12-month horizon at 10K predictions/month. Write the full analysis — including the parallelism strategy recommendation, GPU count, estimated wall-clock time, and the break-even month — to `outputs/skill-build-vs-buy-analysis.md`. Verify: the markdown file exists and contains a dollar-amount recommendation (build or buy) with a stated break-even month.
