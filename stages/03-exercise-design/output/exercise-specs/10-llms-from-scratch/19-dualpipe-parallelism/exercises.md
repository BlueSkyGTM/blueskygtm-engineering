# Exercises — DualPipe Parallelism

## Exercises

1. **Print the four components of a DualPipe forward-backward chunk and trace a minimal schedule.** Write a Python script that prints each component name (forward computation, backward computation, all-to-all communication, gradient synchronization) alongside a one-line rationale for why it gets its own overlap window. Then, for a 2-stage pipeline with 4 micro-batches, print a text grid where each cell is either `F`, `B`, `C`, `G`, or `.` (idle). Run the script and confirm every `.` in stream 1's row is filled by a non-`.` symbol in stream 2's row in the same column.

2. **Implement a naive (single-direction, 1-chunk) pipeline simulator.** Write a function that takes `num_stages` and `num_micro_batches` and prints a timeline grid showing each stage's activity per time slot. Use `F` for forward, `B` for backward, `.` for idle. Compute and print the bubble ratio as `idle_cells / total_cells`. Run it for 4 stages and 8 micro-batches and record the printed ratio.

3. **Extend your naive simulator into a DualPipe simulator.** Add a second stream flowing right-to-left through the same stages. At each time slot, a stage should prefer filling a bubble slot with the opposite stream's operation before introducing new idle time. Print the interleaved timeline grid (use lowercase `f`, `b`, `c`, `g` for stream 2 to distinguish). Recompute the bubble ratio and print the delta versus the naive ratio from Exercise 2. Confirm the bubble ratio dropped.

4. **Compare bubble ratios across pipeline configurations.** Run your DualPipe simulator over a sweep: `num_stages` in {4, 8, 16}, `num_micro_batches` in {8, 16, 32}, `num_chunks` in {1, 2}. Print a formatted table with columns `stages`, `micro_batches`, `chunks`, `naive_bubble_ratio`, `dualpipe_bubble_ratio`, `improvement_pct`. Identify and print which configuration shows the largest absolute improvement and which shows the smallest.

5. **Apply bidirectional pipeline scheduling to a GTM enrichment workflow.** Model a 4-stage enrichment pipeline: (1) Apollo person lookup, (2) Clay company enrichment, (3) email validation via a verification API, (4) lead scoring. Treat two batches of prospects as opposing streams — batch A flows stages 1→4, batch B flows 4→1. Write a scheduler that assigns time slots and prints the timeline, demonstrating that API wait time in one stream is filled by computation in the other. Persist the scheduler and a printed schedule to `handlers/enrichment_dualpipe.py`. Run it against two CSV prospect lists (minimum 5 rows each) and print the total wall-clock time versus a sequential baseline.

6. **Design and document the DualPipeV tradeoff.** Write a script that simulates both original DualPipe (2× parameter replication) and DualPipeV (1× replication, larger bubble) for 8 stages and 16 micro-batches under three memory budgets: unlimited, 1.5× model size, and 1.0× model size. For each configuration, print whether it fits the budget, the bubble ratio, and the effective throughput normalized to the unlimited-budget DualPipe baseline. Write a one-paragraph conclusion to `outputs/skill-dualpipev-tradeoff.md` stating which variant wins under each budget and the exact bubble-ratio cost DualPipeV pays per stage count.
