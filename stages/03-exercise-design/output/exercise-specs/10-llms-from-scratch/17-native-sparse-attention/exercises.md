# Exercises — Native Sparse Attention (DeepSeek NSA)

## Exercises

1. **Compute NSA attention density versus dense baseline.** Write a Python script (stdlib only) that takes sequence length `N`, block size `l`, selection budget `k`, and sliding window width `w` as command-line arguments. For each of the three branches, print the number of key-value positions a single query attends to and the resulting density ratio (active tokens / N). Also print the combined upper-bound active-token count when all three branches run in parallel. Run your script with `N=8192 l=64 k=32 w=512` and confirm the compression branch attends to 128 positions, the selection branch to 2048, and the sliding window to 512.

2. **Trace branch-level token coverage for a specific query.** Given a sequence of length 1024 with block size 64, selection budget 4, and sliding window width 128, write a function `trace_branches(query_pos)` that prints the exact set of token indices each branch would attend to for `query_pos=256`. Label each set with its branch name and print a one-line description of what information granularity that set captures. Confirm that the compression branch covers every block, the selection branch covers only 4 blocks of raw tokens, and the sliding window covers positions 192–320.

3. **Implement a three-branch sparse attention simulator with gating.** Build a standalone Python module with no external dependencies that simulates NSA attention for a 1-D sequence of random floats. Implement: (a) compression via mean-pooling into blocks, (b) selection via block-wise mean absolute value as the importance score with top-k extraction, (c) a sliding window of width `w`, and (d) a learned gate represented as three softmax-normalized weights. For a given gate configuration, compute the weighted output and assert that the number of source tokens contributing to the output matches the expected union of active positions across branches. Print the gate weights, per-branch output, and fused output for at least three different gate configurations on the same input sequence.

4. **Compare native trainable sparsity against post-hoc sparse masking.** Implement two attention simulators over the same 2048-length sequence: one where the block-importance scores and gate weights are recomputed every forward pass (simulating native training), and one where a fixed random mask is pre-generated and frozen before attention (simulating post-hoc masking). Measure how many times each simulator's selected top-8 blocks overlap across 50 different randomly generated query vectors. Print the average overlap percentage for each approach and a one-paragraph summary of why the native approach adapts better to query-dependent information needs.

5. **Build a sparse attention signal extractor for CRM activity logs.** Create a Python script that loads a JSON array of timestamped CRM events (deal stage changes, email opens, meeting notes — generate 500 synthetic events if no live data source is available). Treat the event sequence as tokens and apply your three-branch NSA simulator: compression summarizes weekly cadence, selection surfaces the highest-impact event blocks, and the sliding window captures recent momentum. Configure the gate to weight selection highest. Output a ranked list of the top-10 most-attended event block ranges with their importance scores. Save the runnable module to `signals/examples/nsa_crm_extractor.py` and write the ranked output to `outputs/skill-nsa-crm-signals.md`.

6. **Prove that NSA density scales sublinearly while dense attention is linear.** Write a script that sweeps sequence lengths `N` from 4096 to 131072 (doubling each step) with fixed `l=64`, `k=32`, `w=512`. For each `N`, compute the total active token count across all three branches and the ratio of NSA-active to dense-active tokens. Print a table and verify that the NSA ratio decreases toward zero as `N` grows, while the absolute active count grows as `O(N/l + k·l + w)` — a sum of sublinear terms. Include a final line stating the asymptotic complexity class. Save the script and its printed proof to `outputs/skill-nsa-density-proof.md`.
