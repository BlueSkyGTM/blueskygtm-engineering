# Exercises — Differential Attention (V2)

## Exercises

1. **Implement** differential attention in pure Python. Define a function `differential_attention(Q1, K1, Q2, K2, V, lam)` that computes `(softmax(Q1·K1ᵀ/√d) − lam · softmax(Q2·K2ᵀ/√d)) · V` for 4×4 matrices with `d=4`. Initialize `Q1, K1, Q2, K2, V` with fixed values (seed your RNG or hardcode), set `lam=0.5`, and print the resulting 4×4 output matrix rounded to 4 decimals. Verify your output contains at least one negative entry — that confirms the subtraction is active.

2. **Compute** standard single-softmax attention on the same `Q1, K1, V` inputs you used in Exercise 1 (`softmax(Q1·K1ᵀ/√d) · V`). Print both the standard output and the differential output side by side. Report the maximum absolute difference between corresponding entries and the Frobenius norm of the difference matrix.

3. **Apply** a λ sweep to diagnose noise cancellation behavior. Construct two input patterns: (a) a "clean" pattern where `Q1≈K1` and `Q2≈K2` along a single aligned diagonal, and (b) a "noisy" pattern where all query-key dot products are within ±0.5 of each other. For each pattern, run λ over the grid `[0.0, 0.1, 0.3, 0.5, 0.7, 1.0]` and print the L2 norm of the output matrix at each λ. Write one sentence (as a printed string) identifying which pattern shows larger output variance across λ values and why.

4. **Trace** entropy versus instability in the differential attention map (the pre-`V` weight matrix, not the final output). Implement Shannon entropy over each row of the differential attention weights `softmax(Q1·K1ᵀ/√d) − lam · softmax(Q2·K2ᵀ/√d)`. Use `d=8`, sequence length 6, and λ values `[0.0, 0.5, 0.8, 1.0, 1.2]`. For each λ, print the mean row-entropy and the count of negative entries in the weight matrix. Identify the λ threshold at which negative entries begin to dominate the total mass and report it as a printed summary line.

5. **Build** a multi-head differential attention module. Implement `multihead_diff_attention(X, d_model, num_heads, lam)` that splits `d_model` into `num_heads` head-dimensions, projects `X` into per-head `(Q1, K1, Q2, K2, V)` via random weight matrices, computes differential attention per head, concatenates heads, and applies a final linear projection back to `d_model`. Run it with `d_model=16`, sequence length 5, and `num_heads` in `[1, 2, 4, 8]`. For each head count, print the output shape and verify it equals `(5, 16)`. Save the module to `handlers/diff_attention_heads.py` so it can be imported as `from handlers.diff_attention_heads import multihead_diff_attention`.

6. **Design** a differential attention-based lead-scoring filter for GTM enrichment data. Simulate a table of 20 accounts with 6 enrichment features each (e.g., employee count, tech stack signals, funding recency, engagement score, firmographic fit, intent score). Treat the account-feature matrix as your input `X`. Use two separate projection heads to compute two attention maps over accounts, then subtract them with a tuned λ to isolate "signal accounts" from background noise. Print the top-5 accounts ranked by differential attention output norm alongside their raw attention scores for comparison. Write a one-paragraph analysis to `outputs/skill-diff-attention-gtm.md` explaining which λ value produced the sharpest signal separation and whether differential attention changed the ranking versus standard attention.
