## Exercises

1. **Implement an EU AI Act risk-tier classifier.** Write a Python function `classify_risk_tier(activity)` that accepts a dictionary containing `system_purpose`, `domain`, and `uses_ai` fields. Return a tuple of `(tier, obligations)` where tier is one of `"prohibited"`, `"high_risk"`, `"limited_risk"`, or `"minimal_risk"`, and obligations is a list of required actions for that tier. Test your function with at least four processing activities: an AI-powered resume screening system, a social scoring engine for lead prioritization, a chatbot on your marketing site, and a spell-checker in your CRM notes. Print each classification to the terminal in a readable format.

2. **Build an extraterritoriality trigger checker.** Create a function `check_jurisdictional_reach(company_profile)` that takes a company's details (headquarters country, target markets, data storage location, whether the company offers goods/services to EU/Korean/UK residents) and prints which of GDPR Article 3, PIPA, UK GDPR, and the EU AI Act apply — citing the specific trigger for each. Run your function against three company profiles: a US-based SaaS company targeting EU enterprise customers, a Korean startup storing data in AWS us-east-1, and a UK company using Apollo to scrape Korean business contacts. Verify the output explains *why* each regulation applies or does not.

3. **Map a GTM pipeline's lawful-basis requirements across all four jurisdictions.** Write a script that takes a GTM pipeline represented as a list of stages — e.g., `["collection: Apollo prospecting", "enrichment: Clay waterfall", "scoring: custom AI model", "outreach: email sequence"]` — and produces a formatted table showing the required lawful basis or legal permission for each stage under GDPR, UK GDPR, PIPA, and US state law (CCPA/CPRA as representative). Your output must flag any stage where no valid lawful basis exists without additional consent. Print the full table to the terminal. No starter code is provided — design the data structures yourself.

4. **Implement a compliance decision function.** Build `evaluate_compliance(processing_description)` where the input dictionary includes fields such as `activity`, `data_subjects`, `jurisdictions`, `ai_involved`, `risk_factors`, and `planned_deployment_date`. The function must return a dictionary keyed by jurisdiction (EU, UK, Korea, US) where each value contains `applicable_laws`, `required_actions`, `prohibited_elements`, and `notes`. Test with three distinct scenarios: (a) AI-powered lead scoring using behavioral data from EU prospects, (b) automated email outreach to Korean contacts scraped from LinkedIn, and (c) a chatbot qualifying US-based inbound leads. Print each result and verify the recommendations differ meaningfully by jurisdiction.

5. **Build a regulatory deadline planner for a GTM system rollout.** Create a script at `handlers/deadline_planner.py` that accepts a planned deployment date and a processing description, then cross-references the 2025–2027 regulatory milestones covered in this lesson (EU AI Act GPAI Code commitments, August 2026 enforcement, US state AI laws effective dates, PIPA amendments, UK ICO AI guidance updates). The script must output a sorted timeline of deadlines that affect the system, each annotated with `action_required`, `deadline_date`, `jurisdiction`, and `penalty_if_missed`. Run it against a GTM system planning to deploy AI lead scoring in Q1 2026 and redirect the output to `outputs/skill-deadline-plan.txt` for review. The artifact at `handlers/deadline_planner.py` must be importable and executable as `python handlers/deadline_planner.py`.

6. **Design and execute a full compliance audit for a real GTM stack.** Build a script that ingests a GTM pipeline description involving at minimum Clay enrichment, Apollo prospecting, and an AI scoring model, then produces a structured compliance report covering all four jurisdictions. The report must classify the AI components under the EU AI Act, identify extraterritoriality triggers, flag missing lawful bases, note upcoming deadlines, and recommend remediation steps. Save the report as Markdown to `outputs/skill-gtm-compliance-audit.md`. The audit must be runnable from the terminal and the output file must be human-readable as a standalone document.
