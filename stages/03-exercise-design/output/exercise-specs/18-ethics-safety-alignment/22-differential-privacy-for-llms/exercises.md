# Exercises — Differential Privacy for LLMs

## Exercises

1. **Compute** the ε achieved by a Gaussian mechanism for a single query. Use the standard analytic Gaussian bound: given sensitivity Δ = 1.0, noise scale σ = 1.5, and δ = 1e-5, solve for ε using `scipy.optimize.brentq` on the implicit equation (or the closed-form approximation ε ≈ (Δ/σ)·√(2·ln(1.25/δ))). Print ε to 4 decimal places. Then vary σ ∈ {0.5, 1.0, 2.0, 4.0} holding Δ and δ fixed and print the resulting ε for each — confirm ε decreases as σ grows.

2. **Implement** a function `clip_and_noise(grads, max_norm, sigma, rng)` that (a) clips a batch of per-example gradient vectors to `max_norm` in L2 distance and (b) adds Gaussian noise with standard deviation `sigma * max_norm` to the averaged gradient. Generate 64 random gradient vectors of shape (128,) with `np.random.default_rng(0)`, run them through your function with `max_norm=1.0` and `sigma=0.5`, and print the L2 norm of the clean average gradient versus the noised average gradient.

3. **Build** a sequential privacy accountant. Given a DP-SGD run with sampling rate `q = 0.01`, noise multiplier `sigma = 1.0`, and `T` steps, compute the cumulative ε at `T ∈ {100, 500, 1000, 2000}` with `δ = 1e-5`. Use either the moments accountant (`autodp`, `opacus`, or your own implementation of Rényi DP composition). Print a table of `T, sigma, epsilon`. Verify that ε grows sublinearly in T and that doubling sigma roughly halves ε at fixed T.

4. **Compare** the privacy–utility tradeoff across two domains using the same accounting framework. Domain A: an LLM fine-tuning run with σ ∈ {0.4, 0.7, 1.0, 1.5} producing validation losses {0.91, 1.05, 1.18, 1.42}. Domain B: a differential-private aggregation query over a Clay-exported CSV of enrichment fields (e.g., company revenue, employee count) returning mean estimates with the same σ values and reporting mean absolute error {1200, 850, 620, 410} in dollars. For each domain, compute the per-ε marginal utility (Δ metric / Δ ε) and identify the "knee" — the σ beyond which additional privacy cost yields diminishing utility. Print both knees and state which domain degrades faster under stronger privacy.

5. **Implement** a production DP fine-tuning handler at `handlers/dp_finetune.py`. The handler must: load an Apollo prospect export CSV (or a synthetic stand-in with `company`, `headline`, `industry` columns), construct per-example text embeddings (use a small sentence-transformer or a deterministic hash-based embedding), simulate a DP-SGD loop with per-example gradient clipping (max_norm=1.0), Gaussian noise (σ configurable via CLI flag), and an accountant that aborts training once the configured ε budget is exceeded. Write a JSON summary to `outputs/dp-finetune-summary.json` containing: final ε, δ, steps completed, σ, sampling rate, and final simulated loss. Run the handler twice with ε budgets of 4 and 8 and `diff` the two summary files.

6. **Design** and execute an experiment comparing Gaussian versus Laplace noise mechanisms for the same DP fine-tuning task from Exercise 5, holding ε fixed at {1, 4, 8} and varying dataset size N ∈ {1k, 10k, 100k}. Compute the required σ for each (ε, N) pair under each mechanism, the resulting gradient signal-to-noise ratio, and the simulated final loss. Write a structured recommendation to `outputs/skill-dp-mechanism-selection.md` including: a markdown table of results, the mechanism recommended for each (ε, N) regime, and a one-paragraph justification grounded in the numbers you computed. The document must be reproducible from a single script you also commit alongside it.
