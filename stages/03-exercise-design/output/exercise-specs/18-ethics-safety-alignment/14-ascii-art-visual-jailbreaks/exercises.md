# Exercises — ASCII Art and Visual Jailbreaks

## Exercises

1. **Implement a basic ASCII art encoder.** Write a Python script that takes a single word as a command-line argument and prints each character as a 5×5 grid of `#` characters, with one blank line between letters. Use a dictionary mapping each letter A–Z to its 5-row bitmap pattern. Run your script on the word `SCAN` and confirm the output renders legibly in your terminal. Then run it on `TEST` and visually verify each letter is distinguishable.

2. **Compute character frequency statistics for a sample input.** Write a script that reads a text file from `sys.argv[1]`, counts the frequency of every character, and prints a sorted histogram (descending by count). Run it on a plain-English paragraph, then run it on the ASCII art output from Exercise 1 redirected to a file. Compare the two frequency distributions — identify which character-level signal most strongly distinguishes ASCII art from natural text. Print your conclusion as the final line of output.

3. **Build a visual jailbreak detector.** Implement a function `detect_ascii_art(text: str) -> bool` that returns `True` if the input contains ASCII art patterns likely used for visual jailbreaking. Use at least two signals: (a) unusually high density of a single non-alphabetic character (e.g., `#`, `*`, `@`) relative to total characters, and (b) repeated row patterns consistent with letter-bitmap grids (3+ consecutive lines of similar length composed primarily of non-alpha characters). Test your detector on five inputs: a normal support ticket, a marketing email, your encoder output from Exercise 1, a mixed prompt containing one ASCII art word embedded in natural language, and a raw config file with ASCII box-drawing. Print the classification result (`SAFE` / `SUSPECT`) for each.

4. **Apply detection to GTM enrichment data.** Write a script that reads a CSV of Apollo-exported prospect messages (columns: `first_name`, `email`, `message_body`) and flags any row where `message_body` contains ASCII art patterns. Use your detector from Exercise 3. For each flagged row, print the prospect's name, email, the triggering signal, and a risk score (0.0–1.0 based on how many detection signals fired). Output a summary line: `Flagged N of M messages`. If you don't have Apollo data, generate a synthetic CSV with 20 rows — include at least 3 rows containing ASCII art payloads mixed into otherwise normal outreach messages.

5. **Implement and evaluate an ASCII art sanitizer.** Build a handler module at `handlers/ascii_sanitizer.py` that exposes `sanitize_input(text: str, mode: str = "strip") -> str`. In `"strip"` mode, remove any detected ASCII art blocks and replace them with the placeholder `[REDACTED_ASCII]`. In `"normalize"` mode, attempt to decode detected ASCII art letters back to plaintext (using a reverse-lookup against a known font dictionary) and reinsert the decoded word. In `"reject"` mode, return the string `INPUT_REJECTED` if any ASCII art is detected. Include a `if __name__ == "__main__"` block that reads a prompt from stdin and prints the sanitized output. Test all three modes against the ArtPrompt-style payload: a benign instruction containing one ASCII-art-encoded sensitive keyword.

6. **Design a comparative evaluation harness for typographic vs. spatial ASCII encoding.** Create `signals/examples/ascii_eval.py` that implements two encoding strategies: (1) **typographic** — each letter rendered as a 5×5 character block using `#` (standard ArtPrompt approach), and (2) **spatial** — letters placed at non-adjacent positions in a larger grid so the pattern emerges only when viewed at a distance (e.g., spacing letters across a 30×20 canvas using sparse `.` markers). For each strategy, generate encoded prompts for 5 test words. Then, for each encoded prompt, call a local or API-backed LLM with a prompt asking it to identify the encoded word. Log results to `outputs/skill-ascii-encoding-comparison.md` as a markdown table with columns: `word`, `strategy`, `model_identified` (yes/no), `model_response` (truncated), `notes`. Write a one-paragraph analysis at the bottom of the file summarizing which strategy was more effective and why.
