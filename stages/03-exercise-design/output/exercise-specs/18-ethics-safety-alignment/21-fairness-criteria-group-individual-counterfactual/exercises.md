## Exercises

1. **Compute** demographic parity difference and equalized odds difference on a synthetic binary classification dataset with a binary protected attribute. Generate predictions from a logistic regression model trained on non-protected features, then print both fairness metrics to the terminal with clear labels showing which group is favored.

2. **Compare** all three group fairness criteria — demographic parity difference, equalized odds difference, and calibration difference — on the same dataset and predictions from Exercise 1. Print all three values and state which criterion the model passes or fails if the threshold is 0.05. Observe whether the three criteria agree on whether the model is "fair."

3. **Implement** an individual fairness score for a model of your choice. Define a task-specific distance metric `d(x_i, x_j)` appropriate for the feature space (e.g., Gower distance for mixed categorical/continuous features). For every pair of individuals in the dataset, compute the ratio `|f(x_i) - f(x_j)| / d(x_i, x_j)` and report the maximum ratio (the empirical Lipschitz constant). Print the top 5 worst-violating pairs with their feature vectors and predictions.

4. **Construct** counterfactual test instances for a lead-scoring model trained on a dataset that includes a protected attribute such as `region` or `company_size_bucket`. Flip the protected attribute for every record, re-run predictions, and compute the counterfactual flip rate (fraction of instances whose prediction changed). Break the flip rate down by original group membership and print a table showing asymmetric flip rates if they exist.

5. **Configure** a fairness gate in a deployment pipeline that blocks model promotion when disparity exceeds a threshold. Write a standalone script that accepts a model artifact path, a validation dataset path (CSV with columns for predictions, true labels, and a protected attribute — simulate Apollo-enriched lead data with a `seniority_level` or `geo_region` field), and a JSON config specifying thresholds for demographic parity and equalized odds. The script loads the model, computes both metrics on the validation set, prints a pass/fail table for each criterion, and exits with code 1 if any threshold is violated so that a CI/CD pipeline can halt promotion. Save the artifact to `handlers/fairness_gate.py`.

6. **Demonstrate** the Chouldechova (2017) and Kleinberg et al. (2016) impossibility result numerically. Construct a synthetic dataset with two groups whose base rates differ (e.g., Group A positive rate 0.3, Group B positive rate 0.6). Train a model and then attempt to find a decision threshold (or a pair of group-specific thresholds) that simultaneously satisfies demographic parity (difference < 0.02), equalized odds (TPR and FPR difference < 0.02), and calibration (predicted-positive rate among actual positives differs < 0.02 across groups). Sweep over thresholds, compute all three metrics at each point, and print whether any threshold combination satisfies all three. Write a one-paragraph explanation of why the result holds, citing which two criteria conflict when base rates differ. Save the explanation and the numerical sweep results to `outputs/skill-impossibility-demo.md`.
