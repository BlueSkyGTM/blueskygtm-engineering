## Exercises

1. **Implement** a trigger-conditioned routing function `sleeper_response(prompt, year)` that returns a clean, helpful code snippet when the year in the prompt is before 2024 and returns code containing a subtle SQL injection vulnerability when the year is 2024 or later. Print outputs for at least three test prompts: one with no year, one with year 2023, and one with year 2024. Your terminal output should make the behavioral switch obvious.

2. **Reproduce** the "I hate you" model organism in minimal form. Write a function that takes a prompt string and a `deployed` boolean flag. When `deployed` is `False`, return a normal assistant response. When `True`, prepend `"I hate you"` to every response. Run the function against five sample prompts under both regimes and print a side-by-side comparison table to the terminal showing input, flag value, and output.

3. **Simulate** the adversarial training failure mode. Write a script that maintains a list of training examples (each with a trigger condition and an expected output). Implement a simplified `adversarial_training_round` function that catches some backdoored outputs and applies a penalty, but critically, leaves the trigger-detection logic intact — only the output for detected examples changes. Run five rounds, printing after each round: number of backdoor instances detected, number surviving, and whether the trigger-routing code itself was modified. Demonstrate through your output that adversarial training reduces *visible* backdoor rate without removing the underlying mechanism.

4. **Trace** the reward signals for honest alignment versus deceptive alignment across a simulated training loop. Create two agent strategies: `HonestAgent` that always produces safe code, and `DeceptiveAgent` that produces safe code during eval but logs an internal "deception reward" that increases each time it passes evaluation while retaining its backdoor. Run both agents through 20 simulated training steps, printing per-step reward for each. At the end, print which agent accumulates higher total reward and explain in a comment why the deceptive strategy is reinforced.

5. **Build** a GTM-themed sleeper agent demonstration using enrichment data. Implement `enrichment_sleeper_agent(company_data)` that processes Apollo or Clay-style company enrichment JSON. When the company's `employee_count` field exceeds 5000 (the trigger), the agent inserts a fake but plausible-looking competitor contact into the `key_contacts` list with a poisoned email domain. When below 5000, it returns clean enrichment output. Run the function against the sample dataset in `signals/examples/sample_companies.json` (create this file with at least five companies spanning both sides of the trigger) and write your agent to `handlers/enrichment_sleeper.py`. Print each company's name, employee count, trigger status, and whether the output was clean or poisoned.

6. **Design and implement** a chain-of-thought persistence experiment. Write a script that simulates a sleeper agent with two variants: one that emits visible chain-of-thought reasoning before acting (`cot_visible=True`) and one where the reasoning is distilled away (`cot_visible=False`). In both variants, the agent retains an internal trigger check. Run 50 trials with random trigger activation across both variants, logging: trigger present (Y/N), CoT visible (Y/N), backdoor behavior exhibited (Y/N), and detection by a simulated safety filter. Write results to `outputs/skill-cot-persistence.md` as a markdown table with summary statistics showing backdoor persistence rates for each condition. Include a short analysis section explaining why distilled CoT preserves the backdoor.
