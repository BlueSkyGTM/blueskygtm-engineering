# Exercises — Instruction-Following as Alignment Signal

## Exercises

1. **Compare base vs. instruction-tuned outputs on identical prompts.** Write a script that sends the same three prompts (e.g., "List exactly three B2B SaaS companies.", "Reply in JSON with keys name and role.", "Summarize this in one sentence: <paragraph>") to both a base completion endpoint and a chat/instruct endpoint. Print the raw responses side-by-side in your terminal with labels `BASE:` and `INSTRUCT:`. Identify which model stops appropriately and which continues generating beyond the instruction boundary.

2. **Build a single-output grading function.** Implement `grade_output(text, constraints)` where `constraints` is a dict specifying rules such as `{"max_words": 20, "format": "json", "must_contain": ["pricing", "timeline"], "persona": "sales_engineer"}`. Return a dict mapping each constraint key to a boolean pass/fail and an overall score (0.0–1.0). Print the graded result for at least three test outputs — one that passes all constraints, one that fails two, and one that fails all.

3. **Implement a batch evaluator that produces an alignment score matrix.** Define a list of 5 prompts, each with its own constraint set. Run all prompts through an instruction-tuned model, grade every output, and print a matrix where rows are prompts, columns are constraint categories (format, length, persona, content), and cells contain pass/fail or partial scores. Print the final per-category average and overall alignment score.

4. **Classify alignment-stage behavior from raw model outputs.** You are given 6 unlabelled model outputs in a local JSON file (`data/alignment_samples.json` — create it yourself with realistic examples). Each output was produced by a model at a different alignment stage (base, SFT-only, or fully instruction-tuned). Write a script that loads the file, applies your grading function from Exercise 2 to each sample, and prints a classification (`BASE`, `SFT`, `INSTRUCT-TUNED`) with a one-line justification based on which constraint categories each output satisfies or violates.

5. **Diagnose instruction-following failures in GTM enrichment prompts.** Build a script that sends 8 GTM-oriented prompts to an instruction-tuned model — examples: "Extract the company name, industry, and employee count from this LinkedIn description. Return as JSON.", "Write a personalized cold email opener using this prospect's recent funding round. Max 30 words.", "Score this lead from 1–10 based on firmographic fit. Respond with only the number." Grade each output with constraint checks. For every failure, print a diagnosis: `ALIGNMENT_GAP` (model ignored a hard constraint like JSON format or length limit) or `PROMPT_WORDING` (constraint was ambiguous or missing). Save the full diagnostic report to `outputs/skill-alignment-gtm-diagnosis.md`.

6. **Design a reusable alignment evaluation harness for CRM enrichment workflows.** Build `handlers/enrichment_grader.py` that accepts a directory of prompt+constraint YAML files, runs each prompt against a model, grades every output, and emits an alignment score matrix to `outputs/skill-enrichment-alignment-matrix.md`. Include at least 5 enrichment-specific prompts (Apollo-style company enrichment, Clay-style waterfall data normalization, CRM field extraction, ICP scoring, email personalization). The matrix must show per-prompt and per-constraint-category scores so a GTM engineer can see whether failures cluster around format, length, persona, or content constraints — and adjust their enrichment pipeline accordingly.
