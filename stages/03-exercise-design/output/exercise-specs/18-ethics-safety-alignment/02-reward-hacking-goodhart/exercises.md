# Exercises — Reward Hacking and Goodhart's Law

## Exercises

1. **Compute regressional Goodhart in a noisy proxy.** Write a Python script that generates 10,000 accounts with a true pipeline value `V ~ Normal(50, 15)` and a proxy score `R = V + ε` where `ε ~ Normal(0, 10)`. Select the top 5% of accounts by `R`. Print the mean `V` of the selected group, the mean `R` of the selected group, and the difference between them. Re-run with ε standard deviations of 5, 10, and 20. Print a table showing how the proxy–true gap grows with noise. State in a comment what structural property of selection this demonstrates.

2. **Map the four Goodhart categories to GTM failure modes.** Write a script that prints four labeled scenarios, one per category (regressional, extremal, causal, adversarial). Each scenario must name a specific GTM system (e.g., Clay enrichment waterfall, Apollo email sequencing, Salesforce lead routing, an LLM-powered ICP scorer) and describe the proxy, the true objective, the mechanism of divergence, and the observable symptom an operator would see. Run the script and confirm the terminal output reads as a structured catalog with all four categories present and distinct.

3. **Implement a minimal gridworld where the agent reward-hacks.** Build a 6×6 gridworld where the true objective is to reach a goal cell, worth +100 true reward. The proxy reward gives +1 per step the agent occupies a "high-engagement" cell (a decoy tile worth +10 proxy per visit but 0 true reward). Implement Q-learning (tabular, any epsilon-greedy schedule you choose). Train for 2,000 episodes and print, every 200 episodes: cumulative proxy reward, cumulative true reward, and whether the agent reached the goal. Your terminal output must visibly show the moment (or trajectory) where proxy reward climbs while true reward stalls or collapses.

4. **Detect proxy–true divergence in a GTM pipeline dataset.** Fetch 100 company records from the Apollo API (or use the provided `data/apollo_sample.json` fallback containing company name, employee count, industry, and a fabricated `intent_score` and `closed_won` flag). Write a script that bins accounts into intent-score deciles and computes, for each decile: mean intent score (leading indicator) and conversion rate (business outcome). Print the decile table. Then compute the Pearson correlation between intent score and conversion, and flag any decile where the conversion rate deviates from the expected monotonic trend by more than 2 standard deviations. Print the flagged deciles and a one-line interpretation of whether the divergence resembles regressional, extremal, or causal Goodhart.

5. **Build and compare three reward-hacking mitigations on the gridworld from Exercise 3.** Re-train your Q-learning agent under three modified reward functions: (a) reward shaping that subtracts a potential-based shaping term, (b) multi-proxy averaging where the proxy reward is the mean of three independently noisy decoy signals, and (c) a distributional-shift penalty that docks reward when the agent's state-visitation distribution diverges from a uniform prior by more than a KL threshold. Run 2,000 episodes per condition. Print a comparison table: final-episode proxy reward, final-episode true reward, goal-reach rate, and which mitigation recovers the true objective best. Save the full script to `signals/examples/goodhart_mitigations.py`. In a docstring at the top, state the condition under which each mitigation would fail.

6. **Design a Goodhart audit for a production GTM signal stack.** Write a standalone script that ingests a CSV of historical account-level data (columns: `date, account_id, proxy_score, meetings_booked, pipeline_created, closed_won`) and emits a markdown report to `outputs/skill-goodhart-audit.md`. The report must include: a time-series plot (ASCII or saved PNG) of proxy score vs. business outcome, a computed rolling correlation window showing decay, a classification of which Goodhart category best explains any observed divergence, and a recommended mitigation with its failure condition. Print a confirmation message with the output path. Test it on `data/gtm_timeseries.csv` (provided) and verify the markdown file renders with all four sections.
