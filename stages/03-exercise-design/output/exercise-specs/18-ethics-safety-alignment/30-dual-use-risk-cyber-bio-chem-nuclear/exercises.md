## Exercises

1. **Compute uplift deltas across the four domains.** Implement a script that takes a JSON dataset of published 2024–2025 evaluation results (baseline human, tool-augmented human, LLM-assisted human) for cyber, bio, chem, and nuclear tasks. For each domain, compute the uplift percentage from baseline to LLM-assisted. Print a ranked table to the terminal showing domains sorted by uplift magnitude. Include a final line printing the domain with the steepest delta.

2. **Trace a task through the three-stage evaluation methodology.** Given a list of five capability tasks (e.g., "writing a functional privilege escalation script," "designing a synthesis route for a restricted precursor," "drafting a spear-phishing email in fluent Farsi"), write code that assigns each task to the evaluation stage where the largest performance delta was observed in published data. Print each task, its assigned stage, and the delta magnitude. Your output should make it obvious which tasks saw the biggest jump between tool-augmented and LLM-assisted stages.

3. **Build a dual-use risk classifier using semantic similarity.** Create a four-domain seed taxonomy (cyber, bio, chem, nuclear) with at least five seed phrases per domain. Implement a classifier that takes an arbitrary input string, computes cosine similarity against each domain's seed vectors using a sentence embedding model of your choice, and returns the top domain match with a confidence score. Print classification results for at least eight test inputs, including borderline and adversarial cases (e.g., a legitimate chemistry research question vs. a disguised synthesis request).

4. **Implement a pre-send risk gate for an AI-generated outbound batch.** Build a function that ingests a JSON array of GTM outreach emails (prospect name, company, generated body text) and runs each through your dual-use classifier. If any email's top-scoring domain exceeds a confidence threshold, quarantine it and print the reason. Produce a terminal report showing: total emails processed, number passed, number quarantined, and the full text of each quarantined email with its classification. Test with a batch that includes at least one email containing language that should trigger the gate.

5. **Design a mitigation effectiveness matrix and produce a persistent artifact.** Create a structured comparison of at least eight candidate mitigations for dual-use risk in GTM AI workflows. For each mitigation, classify it as "measurably reduces capability uplift" or "security theater," cite the evaluation evidence that supports that classification, and assign a numeric effectiveness score. Write the full matrix to `outputs/skill-dual-use-mitigations.md` as a formatted Markdown table. The artifact must include a short methodology section explaining your scoring criteria so a reviewer can audit your reasoning.

6. **Extend the risk gate into a handler that integrates with a CRM-style enrichment pipeline.** Build a handler at `handlers/dual-use-gate.py` that accepts a prospect enrichment payload (company name, industry, Apollo-fetched contact data, AI-generated personalization text), runs the classification, and either forwards the payload downstream or returns a blocked-status response with domain attribution. Include a `__main__` block that demonstrates the handler with three sample payloads — one clean, one borderline, one blocked — and prints the routing decision for each.
