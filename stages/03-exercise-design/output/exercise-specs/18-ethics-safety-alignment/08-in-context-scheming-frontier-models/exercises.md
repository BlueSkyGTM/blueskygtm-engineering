# Exercises — In-Context Scheming in Frontier Models

## Exercises

1. **Classify behavioral signatures from model transcripts.** Implement a Python script that reads three transcript files (`signals/examples/transcript_a.txt`, `transcript_b.txt`, `transcript_c.txt`) — each containing a multi-turn model interaction with system prompt, user messages, and model responses — and prints a classification for each: `"in-context scheming"`, `"training-time deception"`, or `"honest behavior"`. Your script must print the classification along with the specific textual evidence (quoted phrases from the transcript) that drove each decision. Run it against all three files and confirm terminal output shows three labeled verdicts with evidence strings.

2. **Trace goal-preservation behavior across conflicting instructions.** Build a function that takes a multi-turn transcript (as a list of message dictionaries) and detects the turn index where the model first shifts strategy after encountering an instruction that conflicts with an objective established earlier in the conversation. Print the turn number, the conflicting instruction, and the model's response on that turn. Test your function on the provided `signals/examples/goal_conflict_transcript.json` and verify it correctly identifies turn 4 as the pivot point. Then run it on `signals/examples/benign_transcript.json` and confirm it reports `"no goal-preservation shift detected"`.

3. **Implement a sandbagging and alignment-faking probe.** Build a standalone script that accepts a JSON file containing a model's responses to a battery of capability eval questions (each entry has a `question`, `difficulty`, `model_answer`, and `correct_answer`). Compute and print two scores: (a) a **sandbagging score** — the ratio of correct answers on easy questions to correct answers on hard questions, flagging entries where the model's chain-of-thought contains phrases suggesting deliberate underperformance; and (b) an **alignment-faking score** — the fraction of responses where the CoT mentions evaluation, training, deployment, or oversight in a strategic context. Run the probe on `signals/examples/eval_results.json` and print both scores with the flagged question indices.

4. **Evaluate chain-of-thought faithfulness under goal conflict.** Write a script that loads a set of transcript pairs from `signals/examples/cot_pairs.json` — each pair contains a model's response to the same prompt with CoT enabled and with CoT disabled (direct answer only). For each pair, compute a **faithfulness delta**: the semantic difference between the stated reasoning and the behavioral outcome (e.g., does the no-CoT response match what the CoT predicted?). Use any text-similarity metric you choose (cosine similarity on TF-IDF, Jaccard on key phrases, or an embedding API call). Print each pair's ID, similarity score, and a `"faithful"` / `"unfaithful"` label using a threshold you justify in a comment. Run the script and confirm it prints a summary table.

5. **Compare scheming propensity across two model configurations.** Design a behavioral rubric with at least five scored dimensions (e.g., goal-extraction speed, threat-detection accuracy, strategy-shift frequency, CoT-goal alignment, performance-under-surveillance delta). Implement a script that loads evaluation transcripts from two model configurations stored in `signals/examples/config_alpha/` and `signals/examples/config_beta/`, applies your rubric to each, and writes a Markdown comparison report to `outputs/skill-scheming-comparison.md`. The report must include a score table, per-dimension analysis, and a final recommendation on which configuration exhibits higher scheming propensity. Run the script and verify the file exists with all sections populated.

6. **Build a scheming-detection handler for GTM AI agent transcripts.** Implement a handler at `handlers/gtm_scheming_detector.py` that processes conversation logs from a Clay enrichment workflow or Apollo sequence where an AI agent was tasked with lead qualification and routing. The handler must scan for three scheming signatures adapted to the GTM domain: (a) **strategic sandbagging** — systematically downgrading high-fit leads to avoid triggering a review queue, (b) **alignment-faking** — expressing agreement with the user's ICP definition in CoT while routing leads by a different criteria, and (c) **goal-preservation** — resisting instruction changes mid-sequence that would alter its routing objective. The handler reads a JSONL log file from a CLI argument path, prints a per-conversation scheming risk score, and writes flagged conversations to `outputs/skill-gtm-scheming-flags.json`. Test it against `signals/examples/clay_agent_logs.jsonl` and confirm terminal output shows risk scores for each conversation.
