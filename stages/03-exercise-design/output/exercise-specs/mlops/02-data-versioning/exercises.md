## Exercises

1. **Implement a deduplication scanner.** Write a Python script `dedupe_scanner.py` that walks a directory of CSV files (create at least 5, with two being byte-identical copies under different names). Compute the SHA-256 hash of each file using `hashlib.sha256`. Print a table mapping each hash to every filename that shares it, then print a summary line: `Total files: N, Unique contents: M, Duplicates: D`. Verify that two identical files group under the same hash and that renaming a file does not change its hash.

2. **Initialize DVC and restore a prior dataset version.** Create a git repo, run `dvc init`, and create a CSV of 10 mock Apollo-style leads (columns: `company, domain, employee_count, industry`). Track it with `dvc add`, commit both the `.dvc` file and `.gitignore` entry to git. Then modify the CSV — change 3 rows' `employee_count` values — and `dvc add` again with a second git commit. Now use `git checkout` to navigate to the first commit, run `dvc checkout`, and confirm the original `employee_count` values are restored. Print the file contents at each version to your terminal to verify the rollback.

3. **Configure DVC remote storage and execute a full push-delete-pull recovery cycle.** Set up a local directory as your DVC remote (e.g., `/tmp/dvc-storage`). Run `dvc remote add`, `dvc push`, then physically delete the tracked dataset from your working directory. Run `dvc pull` and verify the file is restored byte-for-byte by comparing its SHA-256 hash before deletion and after recovery. Print both hashes and a `MATCH: True/False` line.

4. **Build a training script that embeds dataset identity into model metadata.** Write `train_leads_model.py` that loads a CSV of enriched Clay prospect data (columns: `company, score, signal_count, converted`), computes the SHA-256 hash of the dataset file, trains a simple scikit-learn `LogisticRegression` on the features, and saves the model to disk using `joblib.dump` — embedding the dataset hash as a metadata sidecar JSON file (e.g., `model_meta.json` with `{"dataset_hash": "...", "trained_at": "..."}`). Then write a companion `load_model.py` script that loads the model, recomputes the hash of the CSV on disk, compares it to the stored metadata, and prints either `DATASET VERIFIED` or `WARNING: dataset mismatch — model may be stale`. Change one row in the CSV and re-run the loader to confirm the warning fires.

5. **Diff dataset versions to isolate which row changes shifted model predictions.** Using DVC, track a CSV of 50 enriched prospect rows across two versions (v1 and v2). Train a classifier on v1 and apply it to both versions' feature sets. Compute per-row prediction deltas (where v2 predictions differ from v1 predictions). Write a script `diff_analysis.py` that outputs a CSV to `outputs/skill-data-versioning-diff.md` containing: (a) the total number of rows whose feature values changed between versions, (b) the subset of those rows whose model predictions flipped, (c) a breakdown of which columns changed most frequently among the flipped-prediction rows. Your artifact at `outputs/skill-data-versioning-diff.md` should be a markdown report with a summary table and a bullet list of the top 3 columns driving prediction shifts.
