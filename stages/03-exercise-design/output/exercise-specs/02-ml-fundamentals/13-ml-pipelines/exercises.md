# Exercises — ML Pipelines

## Exercises

1. **Build** a `Pipeline` that chains `SimpleImputer(strategy="median")`, `StandardScaler()`, and `LogisticRegression()` on the `load_breast_cancer()` dataset from sklearn. Fit on a train/test split, then print the accuracy score and the first 5 predicted probabilities to the terminal. The entire pipeline must expose a single `.fit()` and `.predict_proba()` call — do not fit the transformers separately.

2. **Apply** `ColumnTransformer` to the seaborn `tips` dataset to route `total_bill` and `size` through `SimpleImputer` + `StandardScaler` while routing `sex`, `smoker`, `day`, and `time` through `SimpleImputer` + `OneHotEncoder`. Wrap the `ColumnTransformer` in a `Pipeline` with `RandomForestRegressor`. Fit the full pipeline, then call `.predict()` on the first 3 rows of the test set and print the results. Verify that the pipeline handles the mixed-type DataFrame directly without manual preprocessing.

3. **Detect and quantify data leakage.** Load a dataset with both numeric and categorical features (use `fetch_openml(name="adult", version=2, parser="auto")` or any equivalent). Split into train/test. Fit `StandardScaler` and `OneHotEncoder` on the **full dataset** before splitting, then train a `LogisticRegression`. Next, build the equivalent `Pipeline` with the same transformers and estimator but fit it only on the training fold. Print both test accuracies. Compute the accuracy delta and print which approach scored higher. Explain in a comment block at the top of your script why the difference occurs and which result is the honest estimate.

4. **Serialize, reload, and verify.** Fit a `Pipeline` (your choice of transformers and estimator) on any sklearn toy dataset. Save it with `joblib.dump()` to `outputs/pipeline.pkl`. Reload it with `joblib.load()`, call `.predict()` on the same held-out batch with both the in-memory and reloaded objects, then use `np.array_equal()` to print `True` or `False` confirming the predictions are identical.

5. **Build** a production lead-scoring pipeline for CRM enrichment data. Fetch sample company data from the Apollo People Search API (or the Apollo Companies endpoint if you have an API key) for 50–100 records. Extract numeric features (employee count, founded year, any numeric enrichment field) and categorical features (industry, location/country). Construct a `Pipeline` with a `ColumnTransformer` that imputes and scales numerics, one-hot encodes categoricals, and feeds into a `GradientBoostingClassifier` trained on a binary "is qualified" label you define via a simple threshold rule (e.g., employee count > 50 AND industry in a target list). Serialize the fitted pipeline to `outputs/lead_scoring_pipeline.pkl`. Print the top-5 highest-probability leads with company name and score. Save a short markdown summary of the pipeline architecture and schema assumptions to `outputs/skill-lead-scoring.md`.

6. **Design** two scoring system architectures for a weekly batch enrichment job that re-scores 10,000 accounts pulled from Clay, enriches them with Clearbit and Apollo data, and writes scores back to Salesforce. Write a Python script that compares both approaches: (a) an in-process scikit-learn `Pipeline` run inside a cron-triggered Lambda function, and (b) an Airflow DAG with separate tasks for extraction, enrichment, scoring, and CRM write-back. For each architecture, print a structured comparison covering: state management, failure recovery granularity, serializability of the model artifact, and reusability of individual steps. Save the full comparison table to `outputs/skill-pipeline-architecture.md` with a justified recommendation for which to choose and under what conditions.
