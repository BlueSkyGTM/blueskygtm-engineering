# Exercises — Model Evaluation

## Exercises

1. **Compute** precision, recall, F1, and accuracy from the following raw confusion-matrix counts for a lead-scoring classifier: TP=85, FP=40, FN=15, TN=960. Write a standalone Python script that prints each metric on its own line with four decimal places. Verify your precision is 0.6800 and recall is 0.8500.

2. **Trace** the ROC curve and compute ROC-AUC by hand-programmed loop. You are given these five predicted probabilities paired with true labels: `[(0.91, 1), (0.77, 1), (0.62, 0), (0.45, 1), (0.30, 0)]`. Write a script that sweeps the threshold across all five unique scores plus 0.0 and 1.0, collects (FPR, TPR) pairs at each, prints the table, then computes AUC via trapezoidal integration. Compare your numeric result to `sklearn.metrics.roc_auc_score` on the same data — print both side by side.

3. **Sweep** thresholds and locate the optimal operating point for a churn-prediction model where a false positive costs \$25 (unnecessary retention campaign) and a false negative costs \$500 (lost customer). Download the dataset at `https://raw.githubusercontent.com/datasets-io/churn-synth/main/churn_probs.csv` (columns: `predicted_prob`, `actual_label`). Write a script that iterates thresholds from 0.05 to 0.95 in steps of 0.01, computes total cost at each, and prints the optimal threshold, the cost at that threshold, and the precision/recall at that point. Then re-run with a default threshold of 0.50 and print the cost delta. Confirm that the cost-optimal threshold produces a strictly lower total cost.

4. **Implement** both ROUGE-L and a rubric-based LLM-as-judge scorer for three AI-generated cold-email subject lines evaluated against a human reference set. Use this reference: `"Cut your sales cycle in half"`. Use these three candidates: `["Cut your sales cycle in half", "Reduce your sales cycle duration by fifty percent", "How to close deals twice as fast"]`. Compute ROUGE-L (longest common subsequence based) for each candidate against the reference and print F-scores. Then call the OpenAI API (`gpt-4o-mini` or equivalent) with a rubric scoring clarity, relevance, and specificity on a 1–5 scale, and print the judge scores alongside ROUGE-L. Print a final comparison table showing where the two evaluation methods agree and disagree on ranking.

5. **Diagnose** evaluation failure modes on a simulated Apollo-enriched outbound dataset. Generate a synthetic dataset of 10,000 account records where 2% are truly high-intent (positive class). Assign predicted probabilities such that a label-leaked feature (`employee_count`) is correlated 0.95 with the target, making the model look strong. Write a script that: (a) computes precision, recall, F1, and ROC-AUC at the default 0.50 threshold, (b) computes the same metrics at a 0.15 threshold, (c) prints the class distribution and explains in one printed sentence which failure mode each gap reveals, (d) runs a five-fold cross-validated ROC-AUC with the leaked feature removed and prints the result. Save the full diagnostic report as a Markdown file to `outputs/skill-eval-diagnostic.md`. The file must include all metric tables, the failure-mode explanations, and a one-paragraph recommendation on whether to ship the model.

6. **Design** and **build** a reusable evaluation harness as a Python module that accepts any classifier's predicted probabilities and true labels, then produces a full evaluation report. The module must expose functions for: confusion-matrix computation, precision/recall/F1 at a caller-specified threshold, threshold sweep with a pluggable cost function, ROC-AUC, and a summary printer that flags class imbalance (positive rate < 5% or > 95%) and default-threshold misuse (cost at 0.50 vs. optimal cost differing by more than 20%). Save the module to `signals/examples/eval_harness.py` and include a `__main__` block that runs it against the churn dataset from Exercise 3, printing the full report to stdout. Confirm the script exits 0 and that importing the module raises no errors.
