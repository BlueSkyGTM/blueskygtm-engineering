# Exercises — Naive Bayes

## Exercises

1. **Implement Multinomial Naive Bayes from scratch with Laplace smoothing.** Write a Python class `NaiveBayesScratch` with `fit(X, y)` and `predict(X)` methods that accepts a document-term count matrix (numpy array) and integer labels. Compute class priors, per-word likelihoods with `alpha=1` Laplace smoothing, and perform all probability multiplication in log space. Train on the 20 Newsgroups dataset (filtered to 4 categories), print the predicted vs. actual label for the first 10 test documents, and report overall accuracy. Your terminal should print each prediction as `predicted=X actual=Y` followed by a final `accuracy=Z` line.

2. **Trace the posterior computation for a single document by hand in code.** Using the fitted model from Exercise 1, write a function `trace_posterior(model, doc_index, X_test)` that prints, for each class: the log prior, each word's contribution to the log-likelihood (word, count, log-likelihood), the total log posterior (unnormalized), and the final predicted class after softmax normalization. Run it on a test document of your choice. The terminal output must let a reader verify that prior + sum(word likelihoods) = unnormalized log posterior for every class.

3. **Select and justify the correct Naive Bayes variant for three different GTM datasets.** You are given: (a) a CSV of email subject lines labeled `sales_lead` / `support` / `spam` where features are word counts, (b) a JSON array of website visitor sessions with binary flags (`visited_pricing`, `downloaded_whitepaper`, `attended_webinar`, etc.) labeled `converted` / `not_converted`, and (c) a CSV of company firmographics with continuous features (`employee_count`, `arr_estimate`, `months_since_founding`) labeled `high_intent` / `low_intent`. For each dataset, load the data, instantiate the correct scikit-learn Naive Bayes variant (`MultinomialNB`, `BernoulliNB`, or `GaussianNB`), fit it, and print the dataset name, chosen variant, mean cross-validated accuracy, and a one-sentence justification for the variant selection.

4. **Build a lead intent classifier on real inbound email data using a scikit-learn pipeline.** Use the `Enron Email Dataset` (available via `scikit-learn`'s `fetch_20newsgroups` as a proxy, or download a sample from a public source). Create a `Pipeline` with `TfidfVectorizer` followed by `MultinomialNB`. Define at least three sales-relevant categories (e.g., `demo_request`, `pricing_inquiry`, `competitor_mention`). Train the pipeline, then classify 20 new sentences you write yourself that simulate inbound lead emails (e.g., "Can I get a demo of your platform?", "Your competitor is cheaper, can you match?"). Print each sentence with its predicted category and the model's confidence score (extract the max probability from `predict_proba`).

5. **Diagnose and improve classifier performance on imbalanced GTM data with threshold tuning.** Fetch a real-world imbalanced dataset: use the `scikit-learn` 20 Newsgroups filtered to two categories where you downsample the positive class to a 1:10 ratio (simulating rare high-intent leads). Train a `MultinomialNB` classifier, then write a script that: (a) plots (or prints as a table) precision, recall, and F1 at 11 threshold values from 0.0 to 1.0, (b) identifies the threshold that maximizes F1, (c) prints the confusion matrix at the default 0.5 threshold vs. the optimal threshold, and (d) reports the class distribution of the training set. Save your script as `handlers/intent_threshold_diagnosis.py` and run it — the terminal must show the threshold sweep table, both confusion matrices, and the final recommended threshold with its F1 score.
