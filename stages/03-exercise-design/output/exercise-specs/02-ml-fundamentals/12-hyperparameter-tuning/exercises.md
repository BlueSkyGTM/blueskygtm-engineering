# Exercises — Hyperparameter Tuning

## Exercises

1. **Compare** grid search, random search, and Bayesian optimization on the same model and dataset. Load `sklearn.datasets.load_breast_cancer`, fit a `RandomForestClassifier`, and run all three strategies over the same hyperparameter ranges (`n_estimators`, `max_depth`, `min_samples_leaf`, `max_features`). Print, side-by-side, the number of candidate evaluations each method performed, the best 5-fold cross-validation score, the wall-clock time, and the selected hyperparameters. Confirm your terminal output shows three rows of comparable metrics.

2. **Implement** stratified k-fold cross-validation from scratch — do not call `cross_val_score` or `GridSearchCV`. Given a binary classification dataset (you may use `make_classification`), manually partition the data into 5 stratified folds, train a `GradientBoostingClassifier` on four folds and evaluate on the held-out fold, and print each fold's accuracy plus the mean and standard deviation across folds. Verify your output lists five fold scores followed by summary statistics.

3. **Configure** a search space and tuning loop for a lead-scoring classifier. Generate a synthetic leads dataset with columns like `company_employee_count`, `funding_rounds`, `technology_stack_match_score`, `engagement_score`, and a binary `converted` label (or load a CSV exported from your CRM). Define a random search space over an `XGBoostClassifier` or `RandomForestClassifier` with at least four hyperparameters, run 30 iterations with 5-fold cross-validation, and print the top three configurations ranked by mean ROC-AUC. Use real feature names in your output so it reads as a GTM artifact.

4. **Detect** overfitting caused by tuning on the test set. Build two pipelines on the same leads dataset: (a) select hyperparameters by maximizing accuracy directly on the test split, and (b) select hyperparameters via cross-validation on the training split only. After selection, evaluate both final models on a completely untouched holdout set you set aside before any tuning. Print the test-set-tuned accuracy, the CV-tuned accuracy, and the gap between them. Confirm the gap is positive — the test-set-tuned model should look better during selection but worse on the true holdout.

5. **Build** a reusable tuning handler that produces a persistent comparison report. The handler should accept a model class, a search space dict, and a dataset, run random search with stratified k-fold CV, and write a markdown report to `outputs/skill-tuning-report.md` containing: the default-parameter baseline scores (mean ± std across folds), the best tuned-parameter scores, the winning hyperparameters, the improvement delta, and the result of a paired t-test (or Wilcoxon signed-rank test) on the per-fold scores to determine whether the improvement is statistically significant. Run the handler on the leads dataset from Exercise 3 and open the generated markdown to verify all five sections rendered correctly.
