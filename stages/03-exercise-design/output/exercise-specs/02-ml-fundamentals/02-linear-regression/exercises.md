# Exercises — Linear Regression

## Exercises

1. **Implement** the normal equation from scratch using NumPy on the synthetic housing dataset from the lesson (`np.column_stack` the features, add a bias column, solve `(XᵀX)⁻¹Xᵀy`). Print each coefficient alongside its feature name. Then verify your result matches `sklearn.linear_model.LinearRegression` to at least 4 decimal places by printing both sets of coefficients side by side.

2. **Configure** batch gradient descent on the same dataset with a learning rate of 0.01 and 1000 iterations. Print the MSE every 100 iterations so you can watch it decrease. Then print the final gradient-descent coefficients next to the normal-equation coefficients from Exercise 1 and confirm they agree to within 0.01. If they do not converge, adjust the learning rate or iteration count and re-run until they do.

3. **Apply** linear regression to a dataset where one input feature has a quadratic relationship with the target. Generate `x = np.linspace(0, 10, 200)`, then set `y = 2.5 * x**2 - 3 * x + np.random.normal(0, 5, 200)`. Fit a linear model using only `x` (not `x²`). Compute and plot the residuals (`y - ŷ`) against the predicted values. Inspect the plot and state in a printed message which specific assumption violation the residual pattern reveals (e.g., "non-linearity," "heteroscedasticity," etc.) and what shape confirms it.

4. **Build** a lead scoring model using the CSV at `data/leads_firmographic.csv` (columns: `employee_count`, `funding_raised_usd`, `domain_authority`, `tech_stack_count`, `months_since_founding`, and target `converted`). Load the data, fit a linear regression, and print each coefficient with its feature name. Then compute the per-feature contribution for a single lead (`contribution_j = w_j * x_j`) and print a ranked breakdown showing which features push this lead's score up or down the most. Scale or normalize features before fitting if coefficient magnitudes are not comparable across features.

5. **Extend** the lead scoring model from Exercise 4 into a drift detection routine. Save the original model's coefficients and predicted-conversion distribution. Then load a second dataset at `data/leads_firmographic_post_drift.csv` collected one quarter later. Score the new leads with the original model, compute the shift in prediction distribution (mean and standard deviation), and re-fit a fresh model on the new data. Print a side-by-side comparison of old vs. new coefficients and flag any feature whose coefficient changed by more than 30%. Write your drift report including recommended actions (retain, retrain, or replace with a non-linear model) to `outputs/skill-lead-drift-analysis.md`.

6. **Design** a reusable Python module that fits a linear regression, runs a full residual diagnostic battery (residual-vs-fitted plot, Q-Q plot, Breusch-Pagan test for heteroscedasticity, and Durbin-Watson test for autocorrelation), and prints a structured diagnosis identifying which OLS assumptions are violated. Test it on three datasets: a clean linear one, one with heteroscedastic noise, and one with a quadratic non-linearity. Save the module to `handlers/residual_diagnostics.py` and run it from the command line, passing a CSV path and target column name as arguments. Confirm the printed diagnoses correctly identify the planted violation in each case.
