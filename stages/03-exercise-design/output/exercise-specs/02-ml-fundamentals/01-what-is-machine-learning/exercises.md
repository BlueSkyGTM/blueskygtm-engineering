# Exercises — What Is Machine Learning

## Exercises

1. **Implement** a script that defines a list of ML scenario dictionaries, each containing keys for `has_labels` (bool), `has_rewards` (bool), and `description` (string). Your script should classify each scenario as `"supervised"`, `"unsupervised"`, or `"reinforcement"` based on those structural signals and print the mapping. Include at least six scenarios spanning all three paradigms — for example, "cluster customers by purchase behavior" should resolve to unsupervised. Run the script and verify the terminal output shows correct paradigm assignments.

2. **Implement** a nearest-centroid classifier from scratch using only Python's standard library (no sklearn, no numpy). Define a small training set of 2D points with labels `"A"` and `"B"` (at least 4 points per class). Compute the centroid of each class, then classify three new points by assigning the label of the nearest centroid. Print the centroids, the new points, their assigned labels, and the Euclidean distances used.

3. **Compute** accuracy from scratch for a lead-conversion prediction task. Create a list of tuples where each tuple contains a predicted label (`"converted"` or `"did_not_convert"`) and a ground-truth label. Build at least 20 examples with a realistic mix of correct and incorrect predictions. Implement the accuracy formula `(correct / total)` as a function — do not use any library. Print the per-example comparison and the final accuracy score as a percentage.

4. **Apply** the probability-distribution concept to a B2B lead-scoring scenario. Export a CSV of 15 companies from Apollo or Clay (or simulate one) with columns like `employee_count`, `industry`, and `funding_amount`. Implement a scoring function that maps each company's features to a raw numeric score, then convert those scores into a probability distribution using softmax. Print each company name alongside its raw score and probability. Demonstrate why two companies receiving the same hard label `"qualified"` can have meaningfully different probability values — include at least one such pair in your output and annotate the confidence gap.

5. **Build** a complete from-scratch lead-scoring pipeline that loads a CSV of enrichment data (company name, employee count, industry, annual revenue, conversion label), trains a nearest-centroid classifier, outputs per-company probability distributions via softmax, and evaluates overall accuracy against the labels. Do not use any ML or statistics library — implement centroid computation, distance-based scoring, softmax, and accuracy yourself. Write the module to `signals/examples/lead_scorer.py`. When executed, it must print the accuracy score, each company's predicted probability of conversion, and a flag for any company where the hard label and probability disagree (e.g., labeled `"converted"` but probability < 0.3).

6. **Design** a GTM machine-learning pipeline document that decomposes a full outbound workflow into sub-problems and assigns the appropriate ML paradigm to each. Cover at least: (a) lead scoring, (b) account clustering for segmentation, and (c) optimal send-time or sequence optimization. For each sub-problem, specify the input structure, output structure, what is known vs. unknown at training time, and why the chosen paradigm fits. Include a short proof-by-contradiction for at least one sub-problem showing why a different paradigm would fail (e.g., why unsupervised learning cannot directly produce a conversion label). Write the design doc to `outputs/skill-ml-paradigms-design.md`.
