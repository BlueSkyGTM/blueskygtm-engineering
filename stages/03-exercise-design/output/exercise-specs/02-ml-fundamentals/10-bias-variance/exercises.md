# Exercises — Bias-Variance Tradeoff

## Exercises

1. **Compute** the bias-variance decomposition empirically for a constant predictor. Generate 200 bootstrap training sets (each of size 50) from a known nonlinear function (e.g., `f(x) = sin(2πx)` on `[0,1]` with Gaussian noise σ=0.2). For each bootstrap set, fit a model that always predicts the mean of `y`. At a grid of 100 test points, compute bias² (deviation of the average prediction from `f(x)`), variance (variance of predictions across bootstrap sets), and the known irreducible noise σ². Print a table showing all three components at five test points and verify they sum to the empirical MSE at those points.

2. **Compare** training error to held-out error to diagnose bias vs. variance. Split a synthetic dataset of 80 points drawn from a cubic function into 60 train / 20 test. Fit two models: a degree-1 polynomial (linear) and a degree-15 polynomial. Print the train MSE and test MSE for both. From the output alone, state which model has high bias and which has high variance, and justify your answer using the train-test gap.

3. **Implement** a polynomial degree sweep from degree 1 through 25 on a dataset of 100 points drawn from `f(x) = 0.5x³ − 2x² + x + noise`. For each degree, compute and print the training MSE and test MSE (using a 70/30 split or k-fold). Format the output as a table with columns: `degree | train_mse | test_mse`. Identify and print the degree that minimizes test error and the range of degrees where test error is within 10% of the minimum.

4. **Apply** L2 regularization to a high-variance polynomial model. Using the same degree-15 polynomial from Exercise 2, fit Ridge regression across alpha values `[0, 0.001, 0.01, 0.1, 1, 10, 100]`. For each alpha, print the alpha, train MSE, test MSE, and the L2 norm of the coefficient vector. Trace how regularization shrinks coefficients and identify the alpha that gives the best test MSE. Print whether the improvement comes from reducing bias or reducing variance, supported by the numbers.

5. **Build** a lead-scoring overfitting audit for a GTM dataset. Use a CSV export from your CRM or an enrichment tool (Apollo, Clay, or a sample B2B dataset) containing at least 200 company records with 10+ features (e.g., employee count, funding, technographic signals, engagement scores) and a binary conversion label. Compute the samples-per-feature ratio. Run 5-fold cross-validation on a logistic regression model and on a gradient-boosted tree. Print the CV F1, train F1, and the train-CV gap for each model. State whether the model has enough data per feature to deploy, applying the rule of thumb of at least 15–20 samples per feature for stable generalization. Write your full audit — including feature count, sample count, ratio, CV metrics, and deployment recommendation — to `outputs/skill-bias-variance-audit.md`.
