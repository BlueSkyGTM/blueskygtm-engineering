# Exercises — Support Vector Machines

## Exercises

1. **Implement and inspect a linear SVM classifier.** Load the built-in `breast_cancer` dataset from `sklearn.datasets`. Train an `SVC` with `kernel='linear'` and default `C=1.0`. Print the total number of support vectors (`model.n_support_`), the indices of the support vectors (`model.support_`), and the first five rows of `model.support_vectors_`. Then compute and print the decision function values for the first ten samples in the dataset using `model.decision_function`. Confirm that samples farther from the boundary produce larger absolute values.

2. **Switch kernels and compare decision function output.** Retrain the same dataset with `kernel='rbf'`, default `C=1.0`, and `gamma='scale'`. Print the support vector counts for both the linear and RBF models side by side. For the first ten test samples, print the decision function value from each model in a two-column table. Observe which samples flip sign or change magnitude and write a one-line comment in your script noting the pattern.

3. **Compute and compare margin widths across `C` values.** Using a linear kernel on the `breast_cancer` dataset, train three models with `C` set to `0.01`, `1.0`, and `100.0`. For each model, compute the margin width as `2 / ||w||` where `w` is accessible via `model.coef_`. Print a three-row table showing `C`, number of support vectors, and margin width. Write a brief comment stating the relationship: does larger `C` widen or narrow the margin, and does it increase or decrease the number of support vectors?

4. **Compare SVM vs. logistic regression on a non-linearly-separable dataset.** Generate a synthetic dataset using `sklearn.datasets.make_circles(n_samples=500, noise=0.15, factor=0.5, random_state=42)`. Split into train and test sets. Train three classifiers: `LogisticRegression()`, `SVC(kernel='linear')`, and `SVC(kernel='rbf', gamma='scale')`. Print the test accuracy and the number of support vectors (where applicable) for all three in a formatted table. State in a printed summary line which model wins and why the margin-based RBF kernel handles this geometry better than a linear decision boundary.

5. **Build a lead-scoring pipeline with margin-based confidence triage.** Create `handlers/lead_scorer.py` that loads a CSV of B2B leads (generate synthetic data with columns like `company_headcount`, `funding_amount`, `monthly_website_visits`, `engagement_score`, and a binary `converted` label). Train an `SVC(kernel='rbf')` on the features. Use `decision_function` on held-out predictions to bucket each lead into three tiers: **High Confidence Positive** (margin > 0.5), **Human Review** (|margin| ≤ 0.5), and **High Confidence Negative** (margin < −0.5). Print the count and sample rows for each bucket. Write the full triage table to `outputs/skill-lead-scoring.md` as a Markdown report with a summary section showing bucket sizes and the median confidence margin per bucket. Confirm the artifact exists and contains all three sections.
