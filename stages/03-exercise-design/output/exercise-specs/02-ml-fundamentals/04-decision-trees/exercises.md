# Exercises — Decision Trees and Random Forests

## Exercises

1. **Implement** a function `gini_impurity(class_counts)` that takes a list of class sample counts (e.g., `[40, 60]` for 40 non-converters and 60 converters) and returns the Gini impurity value. Test it on three nodes: `[100, 0]`, `[50, 50]`, and `[30, 70]`. Print each node's impurity rounded to 4 decimal places. A perfectly pure node should return `0.0`; a 50/50 split should return `0.5`.

2. **Build** a `DecisionTreeClassifier` on the lead-conversion dataset from the lesson (`data/leads.csv` or the inline DataFrame). Fit it with `max_depth=3` and `random_state=42`. Print the top-5 feature importances sorted descending alongside their feature names. Then extract the decision rules for the leftmost path from root to a leaf using `tree_.children_left`, `tree_.feature`, and `tree_.threshold`. Print each rule as `feature_name <= threshold`.

3. **Compute** information gain for every possible threshold split on a single feature using your own implementation — not scikit-learn's internals. Load `data/leads.csv`, pick the `employee_count` column, and for each unique threshold value compute the weighted Gini impurity of the two child nodes and subtract from the parent's Gini. Print a table of the top-5 thresholds by information gain showing: threshold, left-node impurity, right-node impurity, and information gain. Verify that your top threshold matches what scikit-learn would choose by fitting a single-split `DecisionTreeClassifier(max_depth=1)` on that one feature and printing its chosen threshold.

4. **Compare** train vs. test accuracy for `DecisionTreeClassifier` and `RandomForestClassifier` across `max_depth` values of `[1, 2, 3, 4, 5, 8, 12, None]`. Use an 80/20 split with `random_state=42` on `data/leads.csv`. Print a formatted table with columns: `max_depth`, `DT_train`, `DT_test`, `RF_train`, `RF_test`, and a `status` column that prints `"OVERFIT"` when train accuracy exceeds test accuracy by more than 0.05. Identify the `max_depth` value where each model crosses the overfitting threshold.

5. **Construct** a JSON-based lead-scoring pipeline at `handlers/lead_scorer.py`. Train a `RandomForestClassifier(n_estimators=100, oob_score=True)` on `data/leads.csv`. Accept a JSON file (via `argparse`) containing a list of account dicts with fields like `employee_count`, `funding_rounds`, `website_visits_30d`, `tech_stack_count`, and `icp_fit_score`. For each account, output the predicted conversion probability (`predict_proba`), a tier label (`Hot` ≥ 0.7, `Warm` ≥ 0.4, `Cold` < 0.4), and the top-3 contributing features based on feature importances scaled to the account's normalized feature values. Write the ranked output to `outputs/lead_scores.json`. Run it against `data/sample_accounts.json` (create this file with 5 test accounts if it does not exist) and print the ranked list to stdout. If you have access to Clay or Apollo export data, replace `data/sample_accounts.json` with a real enriched export and note the source in a `data_source` field in the output.

6. **Evaluate** model stability and write a model card to `outputs/skill-random-forest-eval.md`. Train three random forests on `data/leads.csv` using different `random_state` values (`[7, 42, 99]`). For each, report OOB score and 5-fold cross-validation accuracy (`cross_val_score`). Compute the standard deviation of feature importances across the three models and flag any feature whose importance rank changes by more than 2 positions. Write a 200-word summary to the markdown file covering: which features are stable vs. unstable, the recommended `max_depth` based on the overfitting table from Exercise 4, and which ICP qualification criteria the stable top-3 importances map to. Print the file path and the top-3 stable features to stdout.
