# Exercises — Feature Engineering & Selection

## Exercises

1. **Implement all four numerical transforms on a right-skewed B2B revenue column.** Load a CSV containing an `annual_revenue` column (or generate 1,000 samples from a lognormal distribution with μ=10, σ=1.2). Apply standardization, min-max scaling, log transform (use `log1p`), and 5-bin equal-width binning. Print a table showing each variant's mean, std, skewness, min, and max. For the binned variant, print bin edges and count per bin. Verify that the log-transformed variant has the lowest absolute skewness.

2. **Build one-hot and frequency encoding from scratch (no sklearn encoders) for an `industry` categorical column.** Generate or load a column with at least 8 industry categories and uneven frequencies. Implement one-hot encoding as a DataFrame with `industry_{value}` columns and frequency encoding as a mapped series. Then demonstrate target-encoding leakage: split the data into train/test, compute target means per category using train only versus using the full dataset, and print the per-category difference in encoded values for the test rows. Report which categories show the largest discrepancy.

3. **Apply three filter-based selection methods in sequence to a B2B firmographic feature matrix.** Construct a matrix with at least 12 features (e.g., employee_count, annual_revenue, funding_total, tech_stack_count, months_since_founding, num_locations, linkedin_followers, etc.) and a binary `converted` target. Standardize all features, then apply: (a) variance threshold at 0.01, (b) correlation filter removing one feature from any pair with |r| > 0.85, (c) mutual information ranking. Print which features survive each stage and the final top-5 features by MI score.

4. **Compare tree-based importance against L1 regularization on enriched lead data.** Using a CSV of Apollo-style leads (or a synthetic generator matching Apollo's export schema: company name, industry, employee_count, revenue, location, technologies, funding stage), engineer at least three derived features (e.g., `revenue_per_employee`, `funding_per_year_since_founding`, `tech_diversity_score`). Train a `RandomForestClassifier` and an `L1`-penalized `LogisticRegression` on the same train/test split. Print the top-10 features ranked by each model's importance coefficients side by side and identify which features both models agree on.

5. **Build a complete feature engineering pipeline module at `handlers/feature_pipeline.py`.** The module must: (a) accept a path to a CSV of enriched B2B leads, (b) automatically select the appropriate transform per numeric column based on its computed skewness (log1p if skew > 1, standardization otherwise), (c) one-hot encode categoricals with ≤ 10 unique values and frequency-encode the rest, (d) run embedded selection via `RandomForestClassifier.feature_importances_` with a threshold of 0.01, and (e) write a ranked report to `outputs/feature-ranking.md` containing each surviving feature's name, transform applied, encoding type, and importance score. Run the pipeline on a sample dataset and confirm the markdown file is created with at least 8 ranked entries.
