# Exercises — Unsupervised Learning

## Exercises

1. Implement K-Means clustering on the synthetic accounts dataset from the lesson. Set `k=4`, fit the model, and print each account's assigned cluster label alongside its company name. Then re-run with `k=6` and print the labels again. Observe how cluster assignments shift when `k` changes — print the count of accounts in each cluster for both runs so the difference is visible in your terminal.

2. Run HDBSCAN on the same dataset using `min_cluster_size=5`. Print every account's cluster label, including noise points (label `-1`). Count how many points HDBSCAN classified as noise versus assigned to a cluster. Compare this to the K-Means result from Exercise 1 — every point got a cluster label under K-Means, so print which accounts HDBSCAN flagged as noise that K-Means absorbed into a cluster.

3. Compute silhouette scores for K-Means (with `k` values of 3, 4, 5, 6, and 8) and HDBSCAN (with `min_cluster_size` values of 3, 5, 10, and 15) on the accounts dataset. Print a formatted comparison table showing each configuration and its score. Then identify the accounts where the best-scoring K-Means model and the best-scoring HDBSCAN model disagree (different cluster assignment under one vs. noise or a different group under the other). Print those account names with both algorithms' labels side by side.

4. Build an anomaly detection step on top of a K-Means clustering pipeline. After fitting K-Means with `k=4`, compute the Euclidean distance from each account to its assigned centroid. Flag any account whose distance exceeds two standard deviations above the mean distance for its cluster. Print the flagged accounts with their company name, cluster ID, distance, and the cluster's distance threshold. Add a small synthetic outlier account with extreme feature values to verify your detection logic fires on it.

5. Build a complete feature-engineered clustering pipeline in `handlers/account_segmentation.py` that ingests a CSV of raw account data (simulating an Apollo export — include columns like `employee_count`, `annual_revenue`, `funding_total`, `website_traffic_monthly`, `engagement_score`, `industry`, `region`). Preprocess the data: one-hot encode categorical features, scale numerical features with `StandardScaler`, reduce dimensionality with PCA to 2 components for visualization metadata, then cluster with both K-Means and HDBSCAN. Compute silhouette scores for both, select the better-performing algorithm, and write a JSON file to `outputs/account_segments.json` mapping each account name to its assigned segment label, algorithm used, and anomaly flag (distance-based, as in Exercise 4). The script should print a summary: number of segments, silhouette score of the chosen model, and count of flagged anomalies. Run it against a CSV of at least 50 accounts you generate synthetically.
