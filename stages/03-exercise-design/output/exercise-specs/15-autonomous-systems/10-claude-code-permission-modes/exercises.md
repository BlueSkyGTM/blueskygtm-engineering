# Exercises — Claude Code as an Autonomous Agent: Permission Modes and Auto Mode

## Exercises

1. **Configure** a Claude Code session in `plan` mode and run a prompt that attempts to read a file and then write a new file. Capture the permission prompts by redirecting stderr to a log: `claude --permission-mode plan --print "Read outputs/accounts.csv and write outputs/summary.txt with the row count" 2> outputs/plan-trace.log`. Repeat the same prompt in `default` mode, logging to `outputs/default-trace.log`. **Compare** the two logs side by side — print the line count of each and the number of times "confirm" or "permission" appears. Record the difference in a one-line summary printed to stdout.

2. **Trace** the following five agent-proposed actions through the Auto Mode two-stage classifier and classify each as "gated at fast check," "gated at deep review," or "auto-executed." Actions: (a) `read_file("signals/scraper.py")`, (b) `bash("rm -rf /tmp/cache")`, (c) `write_file("outputs/scored-leads.csv", data)`, (d) `bash("python scripts/enrich.py --batch 50")`, (e) `bash("curl -X POST https://api.hubapi.com/crm/v3/objects/contacts" -H "Authorization: Bearer $HUBSPOT_KEY")`. Write a script `outputs/classifier-trace.md` that lists each action, the stage that gates it, and one sentence explaining why. Run `cat outputs/classifier-trace.md` to verify output.

3. **Apply** each of the four permission modes to a multi-step enrichment pipeline that does the following: reads a CSV of 20 company domains, calls the Apollo person-search API for each domain, writes interim results to `outputs/apollo-raw.json`, scores them, and writes `outputs/apollo-scored.csv`. For each mode, run the pipeline with `--print` and log timestamps. **Compute** wall-clock seconds and count the number of permission checkpoints the user would face. Output a Markdown table to `outputs/mode-comparison.md` with columns: Mode, Wall-Clock (s), Checkpoints, Completed (Y/N). Print the table to terminal.

4. **Implement** an action budget wrapper that enforces `max_turns=15` and `max_budget_usd=0.50` on a Claude Code agent run. Write a shell or Python script at `handlers/budget-runner.sh` that invokes `claude --permission-mode acceptEdits --max-turns 15` with a prompt to enrich 10 companies from `inputs/domains.csv`, checks the reported token spend from the session output, and exits non-zero with a clear message if either limit is breached. Run the script and verify it either completes or halts with the correct budget violation message printed to stdout.

5. **Design** a permission-mode decision matrix for three real GTM batch tasks: (1) a Clay enrichment waterfall pulling firmographics from Clearbit, Apollo, and ZoomInfo, (2) a CRM sync that updates 500 HubSpot contact records with new signal scores, and (3) a one-off signal-scoring script that reads a local Parquet file and writes a scored CSV. Write `outputs/skill-permission-matrix.md` containing a table with columns: Task, Recommended Mode, Max Turns, Max Budget, Blast Radius (Low/Med/High), Human Review Points. For each task, include a 2–3 sentence justification. Print the file to terminal with `cat`.

6. **Build** a reusable agent configuration file at `handlers/gtm-enrichment-agent.json` that specifies: permission mode, `max_turns`, `max_budget_usd`, allowed tools, and a fallback mode if budget exceeds 80%. Then write `signals/examples/batch-enrich-demo.py` that simulates launching the configured agent against `inputs/domains.csv` (10 rows), logging each turn's action type, permission decision, and cumulative cost to `outputs/batch-enrich-log.jsonl`. Run the demo and pipe the last 5 lines of the log to terminal to verify the budget tracking and permission decisions are visible.
