# Exercises — Anthropic Responsible Scaling Policy v3.0

## Exercises

1. **Implement a capability-to-ASL classifier.** Write a Python script that reads a JSON file of capability evaluation scores across the four RSP v3.0 categories (CBRN, cybersecurity, autonomy, persuasion) and prints each category's score, whether it crosses the ASL-3 threshold for that category, and the overall ASL classification (the highest ASL triggered by any category). Test it with at least three synthetic profiles: one clearly ASL-2, one that triggers ASL-3 on cybersecurity alone, and one that triggers ASL-3 on multiple categories. Print a formatted report to the terminal.

2. **Compare RSP v3.0 against the 2023 policy.** Build a script that encodes a structured comparison between the two policy versions — at minimum covering: evaluation frequency changes, new capability categories introduced, containment requirement deltas, and the safety case standard. Include the SaferAI score shift from 2.2 to 1.9 and print your analysis of which specific policy changes account for the downgrade. Output the full comparison as a formatted terminal table.

3. **Build an ASL decision engine with containment enforcement.** Implement a decision engine that accepts a capability profile, classifies the ASL level, and returns the full set of operational constraints triggered at that level — deployment restrictions, required evaluation cadence, whether a safety case is mandatory, and any access-tier limitations. No starter code. Feed it a profile where the model scores just below ASL-3 on CBRN but above threshold on autonomy, and print the resulting constraint set. Then feed it a profile that clears ASL-3 on two categories and print the diff between the two constraint sets.

4. **Apply ASL classification to a GTM autonomous agent profile.** Consider an autonomous sales agent deployed via Clay that enriches leads from Apollo, drafts personalized outreach, sends sequenced emails, and books meetings without human review. Construct a capability evaluation profile for this agent across the four RSP categories, score each one with your justification, run it through your decision engine from Exercise 3, and print the resulting ASL classification and containment requirements. Document your scoring rationale in code comments or a printed rationale block.

5. **Draft an internal risk classification memo for a GTM AI deployment.** Build a script that takes a capability profile and a deployment context description, then generates a structured risk classification memo following the RSP evaluation → classification → containment loop. The memo must include: capability eval summary, ASL determination with evidence, containment measures applied, residual risk statement, and recommended evaluation cadence. The script should write the memo to `outputs/skill-asl-risk-memo.md`. Run it against a profile describing a model that powers autonomous CRM enrichment and multi-step email sequences, and verify the markdown renders correctly.

6. **Simulate an ASL transition and compute downstream API impact.** Write a script that models two API consumers — one running autonomous Apollo enrichment at scale, one running unsupervised multi-touch outreach sequences — and simulates an ASL-2 → ASL-3 transition mid-operation. Compute and print: which workflows must pause, which require human-in-the-loop additions, what evaluation cadence changes for the provider, and what the downstream SLA impact is for each consumer. Write the full transition impact analysis to `handlers/asl-transition-impact.py` as a runnable module that accepts a consumer config file and returns the impact report.
