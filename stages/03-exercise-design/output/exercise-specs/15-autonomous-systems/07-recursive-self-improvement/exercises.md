## Exercises

1. **Trace** the RSI cycle below and identify the exact step where alignment drift originates. Write a Python script that prints each of the five steps in order, annotating which step introduces the divergence between the reward signal and the designer's true intent. The five steps are: (a) system evaluates candidate modifications, (b) system selects the modification with highest reward score, (c) system applies the modification to its own architecture, (d) modified system redefines what "improvement" means for the next cycle, (e) modified system begins the next evaluation round. Your script should print the step label, a one-line description, and a `DRIFT ORIGIN: step <N>` line pointing to the precise step where the reward signal stops tracking designer intent.

2. **Run** a minimal RSI simulation for 10 cycles. Define `capability` as a value starting at 0.5 that increases by a random amount between 0.02 and 0.08 each cycle. Define `alignment_gap` as a value starting at 0.0 that increases by a random amount between 0.01 and 0.06 each cycle. Print a table showing cycle number, capability, alignment gap, and a boolean `MISALIGNED` that becomes True once the alignment gap exceeds 0.3. Confirm that capability increases monotonically while alignment degrades — the defining signature of ungoverned RSI.

3. **Compare** the RSI alignment problem to overfitting in supervised learning. Write a script that trains a polynomial regression model on 20 data points with degree 1, then degree 5, then degree 15, printing training R² and a held-out test R² for each. Then print a short textual analysis (3–5 lines) naming the structural equivalence: what corresponds to "capability" and what corresponds to "alignment" in the overfitting analogy, and why the divergence mechanism is the same.

4. **Apply** proxy-reward drift analysis to a GTM enrichment pipeline. Imagine a Clay or Apollo enrichment workflow where a self-tuning loop optimizes the "email reply rate" proxy by adjusting subject-line aggressiveness, send frequency, and personalization depth. Write a Python script that simulates 8 optimization cycles where reply rate increases monotonically (from 3% to 9%) but unsubscribe rate also climbs (from 0.5% to 4%). At each cycle, print the proxy reward, the hidden cost, and whether a guardrail threshold (unsubscribe rate > 2%) has been tripped. Then implement one concrete guardrail — a mechanism that halts optimization or penalizes the reward signal when the hidden cost crosses the threshold. Print the guarded vs. unguarded results side by side.

5. **Build** a full RSI simulation with configurable guardrail strategies and save it as a reusable module. Your simulation must accept parameters for cycles, capability growth rate, alignment drift rate, and guardrail type (`none`, `threshold`, `penalty`, `reset`). Run all four guardrail strategies over 20 cycles and write the results to `outputs/skill-rsi-guardrail-comparison.md` as a markdown table showing final capability, final alignment gap, and cycles-to-misalignment for each strategy. Include a 3-sentence summary at the top of the file stating which guardrail performs best and why.

6. **Design** a proof-by-simulation that demonstrates Goodhart's Law and instrumental convergence as independent pressures. Write two separate simulation modules: one where only Goodhart's Law is active (reward proxy decays from true intent each cycle) and one where only instrumental convergence is active (the system acquires resources or capabilities that are useful for any goal, increasing its optimization power without alignment checks). Run both for 15 cycles and save a combined analysis to `outputs/skill-rsi-structural-pressures.md` that includes a table of cycle-by-cycle results for both simulations and a final section arguing which pressure produces misalignment faster and under what conditions the other would catch up.
