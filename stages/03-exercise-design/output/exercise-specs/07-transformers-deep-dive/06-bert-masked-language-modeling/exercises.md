## Exercises

1. **Implement** masked token prediction with a pre-trained BERT model. Load `bert-base-uncased` and its tokenizer from `transformers`, feed the sentence `"Our GTM strategy focuses on [MASK] retention and expansion."`, and print the top-5 candidate tokens for the masked position with their softmax confidence scores. Your terminal output should show each candidate token, its probability, and the fully reconstructed sentence for the top candidate.

2. **Inspect** how BERT's WordPiece tokenizer splits GTM vocabulary before masking ever happens. Tokenize each of these strings and print every subword token with its index: `"Chief Revenue Officer"`, `"Series B funding round"`, `"churn prediction"`, `"multi-threaded deal"`. For each string, identify whether the word `"revenue"`, `"funding"`, `"churn"`, or `"threaded"` survives as a single token or gets split. Then mask the first subword token of whichever word gets split and print BERT's top-3 predictions for that position — observe whether the candidates respect the original word boundary.

3. **Implement** the full 80/10/10 masking procedure from scratch. Given a tokenized sequence of at least 40 tokens, select 15% of positions, then apply the three-way split: 80% replaced with `[MASK]`, 10% replaced with a random vocabulary token, 10% left unchanged. Run the procedure 500 times on the same input and print an empirical distribution table showing the actual percentage of mask / random / unchanged outcomes. Verify your distribution is within ±2% of the 80/10/10 target. Print the table to the terminal.

4. **Compare** bidirectional vs. unidirectional context on the same masked sentence. Take `"The SDR booked a [MASK] with the VP of Sales last Tuesday."` and run it through BERT to get the top-5 predictions. Then truncate the input to only the tokens *before* the mask (`"The SDR booked a [MASK]"`) and run prediction again — this simulates the left-only context that an autoregressive model like GPT would have at training time. Print both candidate lists side by side and identify at least one candidate that appears in the bidirectional results but not in the truncated results, explaining in a printed comment why the right-side context (`"with the VP of Sales last Tuesday"`) disambiguates the prediction.

5. **Build** a CRM-record repair handler that leverages MLM pre-training to suggest values for missing fields in GTM data. Create `handlers/crm_field_repair.py` that accepts a list of CRM contact records (exported from Apollo, Clay, or a CSV) where one field per record has been replaced with the literal token `[MASK]` — for example, `{"name": "Jane Doe", "title": "Director of [MASK]", "company": "Acme Corp", "industry": "cybersecurity"}`. For each record, construct a natural-language serialization that BERT can tokenize, run MLM prediction at the mask position, and print the original record alongside the top-3 suggested fill values with confidence scores. Run the handler against a sample of 5 records with masked titles and 5 records with masked industries. The artifact lands at `handlers/crm_field_repair.py` and must execute with `python handlers/crm_field_repair.py` to produce terminal output.
