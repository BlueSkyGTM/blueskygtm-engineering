## Exercises

1. **Implement** a causal attention mask from scratch using `torch.tril` and `torch.full`. Create a 12×12 mask where allowed positions hold `0.0` and blocked positions hold `float('-inf')`. Generate a random `(1, 1, 12, 12)` attention-score tensor, add the mask, apply `torch.softmax` along the last dimension, and print the resulting weight matrix. Then assert that every element above the diagonal is exactly `0.0` and print a confirmation message if the assertion passes.

2. **Trace** next-token probability distributions through GPT-2 (`gpt2`) for the prompt `"The best sales strategy is to"`. Tokenize the prompt, run a forward pass, and extract the logits at each of the first 5 token positions. For each position, apply softmax over the vocabulary and print the top-5 predicted tokens with their probabilities. Write one sentence of observation about how confidence in the top token changes as context accumulates.

3. **Build** two decoders from scratch — a greedy decoder (always picks argmax) and a top-k sampling decoder (samples from the k highest-probability tokens) — without calling `model.generate()`. Each decoder takes a prompt string and a token count and returns the generated text. Run both on the prompt `"Dear hiring manager,"` for 20 tokens with `k=10` and `torch.manual_seed(42)`. Print both outputs labeled clearly so you can compare them in the terminal.

4. **Diagnose** how prompt ordering affects output in a realistic GTM enrichment scenario. Construct a JSON-style company profile (name, industry, employee count, recent funding round) — use a real company from Apollo, Clay, or a public Crunchbase profile. Build two prompt variants for extracting the company's ideal customer profile: (A) enrichment data placed before the extraction instruction, and (B) extraction instruction placed before the data. Run both through GPT-2 for 30 tokens each and print the outputs. In a comment or print statement, explain which ordering produced more structured, on-task output and connect that to the causal mask's left-to-right constraint.

5. **Evaluate** decoding parameters across two GTM tasks: (1) extracting a structured target-account list from a company description and (2) generating a creative cold-email opening line. For each task, test three configurations: greedy (`temperature=0.0`), moderate sampling (`temperature=0.7, top_k=40`), and high-variance sampling (`temperature=1.0, top_p=0.9`). Run each configuration 3 times per task on the same seed prompt. Print every output with its configuration label, then write a short summary recommending which configuration fits each task and why. Save the recommendations to `outputs/skill-decoding-eval.md`.

6. **Design** and implement a hybrid decoding pipeline for a GTM enrichment workflow that chains two LLM calls: first, a greedy-decoded pass that extracts structured fields (company name, pain point, buying signal) from a raw enrichment payload (e.g., a Clay or Apollo company export as JSON); second, a top-p sampled pass that generates a personalized one-liner outreach message using those extracted fields as context. Process at least 3 sample company profiles through the full pipeline. Save the pipeline as `handlers/hybrid_decoder.py` and print the structured extraction alongside the generated outreach line for each company.
