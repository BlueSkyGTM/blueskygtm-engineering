# Exercises — Scaling Laws

## Exercises

1. **Fit a Kaplan power-law and predict loss at an unseen scale.** Create a Python script that stores five (N, L) pairs from published scaling curves — for example, GPT-3 data points at 125M, 350M, 1.3B, 6.7B, and 175B parameters with their reported validation losses. Take the log of both axes and perform a linear regression to recover the exponent αN and the constant Nc. Print the fitted equation, then use it to predict loss at N = 500B non-embedding parameters. Verify by checking that your predicted loss is lower than the 175B data point but still above 1.0.

2. **Compute the Chinchilla-optimal allocation for three compute budgets.** Implement the Hoffmann compute-optimal formulas: given a total FLOP budget C, solve for N* and D* using the joint loss function L(N, D) = A/N^α + B/D^β + E with the published coefficients (A = 406.4, B = 410.7, α = 0.34, β = 0.28, E = 1.69). Run your script for C = 10^21, 10^22, and 10^23 FLOPs. Print N*, D*, and the D/N ratio for each. Confirm the ratio stays near 20 across all three budgets.

3. **Evaluate whether a fine-tuning dataset is overparameterized.** You are fine-tuning a 13B-parameter model on 260M tokens of internal support-chat transcripts. Compute the Chinchilla-optimal D* for N = 13 × 10^9 using the Hoffmann iso-loss contours, then compute the ratio of your actual D to D*. Print whether the dataset is compute-optimal, undertrained (too few tokens), or overparameterized (too many tokens wasted on a model that cannot absorb them). Then re-run the same check for a 1.3B model trained on the same dataset and print the verdict.

4. **Find the break-even point where a larger model tier justifies its cost.** You are choosing between a 7B model ($0.06 per 1M tokens) and a 70B model ($0.60 per 1M tokens) for a lead-scoring task. Each lead costs $0.15 to enrich via Apollo and you process 50,000 leads/month. The 70B model achieves 92% precision vs. 84% for the 7B model. Implement a script that computes the monthly total cost (inference + enrichment) for each tier, factors in the cost of false positives (each wasted SDR call costs $8), and prints which tier is cheaper and by how much. Vary the lead volume and print the volume at which the 70B model breaks even.

5. **Build a model-tier router that applies break-even analysis to live Clay enrichment data.** Create a handler at `handlers/model_tier_router.py` that reads a CSV of enriched company records (company name, employee_count, estimated_arr, industry) produced by a Clay export, classifies each record into SMB / mid-market / enterprise, and routes each tier to a model (7B / 70B) based on a break-even computation. The handler must accept a `--compute` flag that sets the total monthly inference budget and print, per company tier: the model selected, the projected inference cost, the projected false-positive cost
