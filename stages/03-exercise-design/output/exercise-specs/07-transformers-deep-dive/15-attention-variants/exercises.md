## Exercises

1. Implement a sliding window attention mask in PyTorch for a sequence length of 16 and window radius 3. Construct the `QKᵀ` matrix, apply your mask (setting out-of-window positions to `-inf`), and print the resulting attention matrix rounded to two decimal places. Verify visually that each row contains non-zero values only within a band of width 7 centered on the diagonal.

2. Write a script that computes the number of attended (non-masked) key-query pairs for full attention, sliding window attention (window radius 4), and sparse attention (4 global tokens + window radius 2) across sequence lengths `[64, 256, 1024, 4096, 16384]`. Print a formatted table showing N, full count, SWA count, and sparse count for each row.

3. Implement differential attention from scratch. Create random Q, K, V tensors of shape `(1, 8, 64, 32)` (batch, heads, seq, dim). Split Q and K into two halves along the head dimension. Compute two attention maps, subtract the second from the first, apply softmax, and multiply by V. Compare the L2 norm of your differential output against standard single-pass attention output on the same tensors — print both norms and their ratio.

4. Build a sparse attention mask that combines three patterns: a local sliding window (radius 2), full access for the first 2 and last 2 tokens (global tokens), and 4 randomly sampled cross-connections per query. Apply it to a `(1, 4, 32, 32)` attention matrix and print a density map using `#` for attended positions and `.` for masked positions. Confirm the global rows are fully dense and the remaining rows show the window band plus scattered random connections.

5. Build a stacked sliding window transformer that demonstrates information propagation beyond a single layer's window. Initialize a random input embedding of shape `(1, 64, 32)` representing concatenated LinkedIn profile fields (name, headline, experience, skills, education). Apply 4 transformer layers, each using sliding window attention with window radius 4 and an identity-like linear projection for V. After the forward pass, compute the cosine similarity between the position-0 output token and every other position's output token. Print which positions have similarity above 0.5 to position 0. Demonstrate mathematically (via the printed similarity profile) that position 0's influence has propagated beyond position 4 after multiple layers, even though no single layer allows that attention. Save the full implementation to `signals/examples/window_propagation.py`.

6. Design and benchmark an attention variant selection pipeline for a GTM enrichment workflow. Using the Apollo or People Data Labs API (or a local CSV of 200 company descriptions if rate-limited), concatenate each company's description + industry + tech stack into a single sequence. Tokenize all sequences and report the distribution of sequence lengths. For three attention configurations — full, sliding window (radius 64), and sparse (8 global + window 32) — implement a simple key-value retrieval test: embed each sequence, pick a random fact from a known position, and measure whether cosine similarity can surface it from a query embedding. Print a table with columns: variant, average sequence length, peak memory (MB), retrieval accuracy (%), and recommendation (use / skip). Write your analysis and final variant recommendation for a 10K-company enrichment pipeline to `outputs/skill-attention-selection.md`.
