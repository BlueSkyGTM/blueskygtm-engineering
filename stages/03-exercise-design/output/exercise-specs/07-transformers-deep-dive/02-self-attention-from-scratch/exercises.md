# Exercises — Self-Attention from Scratch

## Exercises

1. **Implement scaled dot-product self-attention.** Using only `torch.randn` to create input embeddings (`seq_len=6`, `d_model=16`) and three linear layers (no bias), compute Q, K, and V. Calculate raw scores as `Q @ K.T`, scale by `1/sqrt(d_k)`, apply softmax over the last dimension, and produce the weighted sum of values. Print the shape of every intermediate tensor (Q, K, V, scores, scaled scores, attention weights, output) and verify each matches your hand-derived expectation.

2. **Compare random vs. learned attention patterns.** Re-run your attention implementation twice: once with freshly initialized linear layers and once with layers you manually set to near-identity (load `W_q`, `W_k`, `W_v` with values close to `torch.eye(16)` plus small noise). Print the attention weight matrices for both runs (`6×6` each) and report which configuration produces more uniform (diffuse) weights and which produces more peaked (diagonal-dominant) weights. State why this matters for model interpretability in a single printed summary line.

3. **Implement causal masking for autoregressive attention.** Starting from your scaled dot-product code, add a lower-triangular mask (`-inf` above the diagonal) before softmax so each token can only attend to itself and prior positions. Run on a `seq_len=8`, `d_model=32` input. Print the full `8×8` attention weight matrix and verify every entry above the diagonal is exactly `0.0`. Confirm that rows still sum to `1.0` (they should, because softmax was applied after masking).

4. **Build multi-head attention with `num_heads=4`.** Split the model dimension (`d_model=32`) into 4 heads of `d_k=8` each. For each head, compute independent Q/K/V projections (you decide whether to use separate linear layers per head or split a single projection), run scaled dot-product attention, then concatenate all head outputs along the feature dimension. Apply a final output projection (`d_model → d_model`). Print the shape before concatenation (should show 4 heads), after concatenation, and after the output projection. Verify the final output shape is `(seq_len, d_model)`.

5. **Apply attention weights to diagnose misclassified leads in an enrichment pipeline.** Simulate a 6-token lead sequence representing Apollo-enriched fields: `["Company: Stripe", "Headcount: 7000", "Industry: Fintech", "Funding: Series D", "Title: VP Eng", "Intent: Low"]`. Create a small embedding (`d_model=16`) for each token (hardcode or random). Run your bidirectional attention implementation and extract the attention weight row for the "Intent: Low" token — this tells you which fields the model relied on when producing the low-intent signal. Save the full `6×6` attention matrix and a short diagnostic interpretation (which fields contributed most to the low-intent classification) to `outputs/skill-attention-lead-diagnosis.md`. Include the raw weight matrix in a Markdown table and your diagnosis in 2–3 sentences.

6. **Design and prove: stable attention under varying sequence lengths.** Write a standalone script `handlers/attention_stability.py` that runs your full multi-head attention implementation (with causal masking) on three sequences of different lengths: 4, 16, and 64 tokens (all `d_model=32`, `num_heads=4`). For each, compute the mean and standard deviation of the attention weight distributions across all rows. Print a comparison table showing whether attention weights remain statistically stable as sequence length grows, or whether they become more diffuse. Include a one-paragraph proof (as a docstring comment) explaining mathematically why the `1/sqrt(d_k)` scaling factor keeps gradient magnitudes stable regardless of `seq_len` — or demonstrate empirically what happens when you remove the scaling factor.
