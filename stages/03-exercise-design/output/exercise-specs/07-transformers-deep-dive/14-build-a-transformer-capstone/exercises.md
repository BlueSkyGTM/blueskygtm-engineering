## Exercises

1. **Trace tensor shapes through the full transformer pipeline.** Instantiate the encoder-decoder transformer from the lesson with `d_model=64`, `num_heads=4`, `num_encoder_layers=2`, `num_decoder_layers=2`, and a vocabulary size of 100. Feed a source batch of shape `(2, 10)` and a target batch of shape `(2, 8)` through the forward pass. After every component — embedding, positional encoding, each encoder layer's self-attention output, each encoder layer's feed-forward output, each decoder layer's masked self-attention output, each decoder layer's cross-attention output, and the final linear projection — print the tensor name and its `.shape`. Confirm that every shape matches your hand-computed expectations. Your terminal output should list at least 15 shape lines.

2. **Verify that causal masking changes decoder self-attention output.** Create a single decoder layer. Run the same `(1, 6, 64)` input tensor through it twice: once with a proper causal mask and once with no mask (all-ones mask). Compute the element-wise difference between the two outputs and print its maximum absolute value. Then extract the attention weights from both runs and print position 3's attention distribution over all 6 positions. Confirm that the masked version attends only to positions 0–3 while the unmasked version spreads attention across all 6.

3. **Implement greedy autoregressive decoding from scratch.** Given a trained (or randomly initialized) transformer, a source sequence of 12 tokens, a start-of-sequence token ID, and an end-of-sequence token ID, write a decoding loop that generates tokens one at a time. At each step, feed the growing target sequence through the decoder, extract the logits for the last position, select the argmax token, append it, and stop when EOS is produced or a max length of 30 is reached. After generation, print the full generated token sequence. Run the loop for 3 different source sequences and print all three outputs. Also, at each generation step, print the cross-attention weights from the final decoder layer's last position across all source positions — observe how the model's focus on the source shifts as new tokens are generated.

4. **Apply the self-attention weighting mechanism to buying-intent signal scoring.** Fetch 10 company profiles and their recent event timelines (funding rounds, hiring spikes, product launches, leadership changes) using the Apollo API or a local JSON file of enrichment data if no API key is available. Represent each company as a sequence of event vectors (one vector per event, with features like event type, recency, magnitude). Implement a single-head attention layer that computes attention weights across each company's event sequence to produce a single conversion-predictiveness score per company. Print each company name alongside its top-3 attended events and final score. Compare the attention-weighted scores to a naive sum-of-events baseline and print both side by side.

5. **Build the complete transformer as a reusable, inspectable artifact.** Implement the full encoder-decoder transformer — multi-head attention, positional encoding, layered encoders and decoders with residuals and layer normalization, causal masking, cross-attention, and a final linear projection — in a single self-contained module. Include a `forward_with_attention` method that returns not just the output logits but a dictionary of all attention weights from every layer and head. Run the model on a source-target pair and serialize the attention weights to `outputs/skill-transformer-attention.json` along with metadata (layer index, head index, attention type). Save the model code to `signals/examples/transformer_from_scratch.py`. Print a confirmation line listing the number of attention matrices saved and the file paths written.
