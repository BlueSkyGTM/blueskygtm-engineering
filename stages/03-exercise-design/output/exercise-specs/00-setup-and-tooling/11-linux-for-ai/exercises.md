# Exercises — Linux for AI

## Exercises

1. **Navigate and manipulate the filesystem.** Create a project directory structure for a GTM enrichment pipeline under `~/projects/gtm-enrichment/`. Inside it, create subdirectories `data/`, `logs/`, and `output/`. Download a sample prospect CSV from `https://raw.githubusercontent.com/datasets/company-names/master/data/company-names.csv` into `data/` using `curl`. Verify the file exists with `ls -la data/`, confirm its byte size, and display the first 10 lines with `head`. Then copy the file to `data/company-names-backup.csv` and confirm both files have identical line counts using `wc -l`.

2. **Configure a Python virtual environment and environment variables.** Create a virtual environment named `.venv` inside `~/projects/gtm-enrichment/`. Activate it, install the `requests` library, and verify the installation path with `which python` and `pip show requests`. Export an environment variable `ENRICHMENT_API_KEY` set to a dummy value `test-key-12345`. Start a Python REPL and confirm you can read the variable using `os.environ.get("ENRICHMENT_API_KEY")`. Deactivate the venv and verify the `requests` package is no longer importable from the system Python.

3. **Manage a long-running background process.** Write a Python script at `~/projects/gtm-enrichment/simulate_inference.py` that runs an infinite loop, printing a timestamped "processing batch N" message every 3 seconds and flushing stdout. Launch the script in the background with output redirected to `logs/inference.log`. Use `ps aux | grep simulate_inference` to capture the PID. Confirm the process is producing output with `tail -f logs/inference.log` (Ctrl-C to stop tailing). Then terminate the process using `kill <PID>` and verify with `ps` that it no longer appears.

4. **Chain commands to build a production shell script.** Write a bash script at `~/projects/gtm-enrichment/run_pipeline.sh` that: (a) checks whether `data/company-names.csv` exists and exits with code 1 and an error message to stderr if not, (b) counts total rows and writes the count to `output/row_count.txt`, (c) extracts the first 5 company names, pipes them through `sort`, and writes the result to `output/top_companies.txt`, and (d) prints "Pipeline complete" to stdout on success. Run the script and verify the exit code with `echo $?`. Then deliberately rename the CSV file and run the script again — confirm the exit code is 1 and the error message appears.

5. **Diagnose process and network issues.** Build a diagnostic shell script at `handlers/health_check.sh` that checks three things in sequence and reports status for each: (a) whether a specified Python process (passed as a `$1` argument) is running, found via `ps`, (b) whether port 443 on `api.apollo.io` is reachable using `curl -s -o /dev/null -w "%{http_code}"`, and (c) whether port 443 on `api.openai.com` is reachable the same way. The script should print a summary line for each check as `[PASS]` or `[FAIL]` with details, and exit 0 only if all three pass. Run it against a real process you start in the background, then kill that process and re-run to confirm the script detects the failure. Artifact lands at `handlers/health_check.sh`.

6. **Design a GTM enrichment runner with resilience.** Build a Python script at `signals/examples/enrichment_runner.py` that reads company names from `data/company-names.csv`, queries the Apollo People Search API (`https://api.apollo.io/v1/people_search`) for each company using an API key from the `APOLLO_API_KEY` environment variable, writes matched contact names and titles to `output/enriched_contacts.jsonl`, logs all HTTP errors to `logs/enrichment_errors.log` with timestamps, and handles rate-limit (429) responses by sleeping 60 seconds before retrying. The script must use a Python virtual environment with dependencies pinned in a `requirements.txt` file, read the API key from the environment (not hardcoded), and exit with code 2 if the environment variable is missing. Run the script against the CSV and verify: enriched contacts exist in the output file, errors are logged with timestamps, and the script handles a missing `APOLLO_API_KEY` gracefully. Artifact lands at `signals/examples/enrichment_runner.py`.
