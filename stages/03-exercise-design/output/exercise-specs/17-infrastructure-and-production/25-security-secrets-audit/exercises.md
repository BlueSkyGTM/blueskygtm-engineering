## Exercises

1. **Implement a secret scanner combining regex and entropy.** Write a Python script that reads a text file containing mixed API responses from enrichment tools (paste in fake Clay webhook payloads, Apollo export snippets, CRM sync logs). Apply regex patterns for at least four known key formats (AWS `AKIA...`, Stripe `sk_live_...`, GitLab `glpat-...`, generic `Bearer <token>`) and compute Shannon entropy for any token-like string longer than 20 characters. Print each finding as `LINE | TYPE | VALUE_PREVIEW | ENTROPY`. Include at least three decoy strings (URLs, UUIDs, hashes) that should NOT be flagged. Run the script against your sample file and verify the decoy count is zero.

2. **Build an input guardrail for enrichment prompts.** Implement a function that takes a string prompt destined for an LLM-powered lead-scoring endpoint and rejects it if it contains SSN patterns (`XXX-XX-XXXX`), credit card numbers (13–19 digit runs), or raw API keys matching your regex from Exercise 1. Return a structured JSON response: `{"approved": bool, "blocked_patterns": [...], "original_prompt": "..."}`. Test against five inputs — two clean, three containing violations — and print all five results. Confirm blocked_patterns lists every violation found, not just the first.

3. **Implement dual-key zero-downtime rotation for an Apollo API client.** Create a client class that holds two keys (`active` and `standby`), each with an ID and a simulated HTTP call that succeeds. Implement a `rotate()` method that promotes standby to active and provisions a fresh standby — without dropping any in-flight requests. Simulate 50 requests in a loop with a `rotate()` call triggered at request 25. Print a log line per request showing `request_id | key_id | status`. Verify all 50 requests return `200` with no gap during rotation.

4. **Write structured audit log entries that support replay.** Design a JSONL audit log schema with fields: `timestamp`, `actor`, `action`, `resource`, `old_value`, `new_value`, `correlation_id`, `hash` (SHA-256 of the previous line's hash, forming a tamper-evident chain). Simulate a security incident: log a key rotation, three API calls using the new key, a guardrail rejection, and a forced re-rotation. Write all entries to a `.jsonl` file. Then write a second script that reads the file and replays the sequence, printing a human-readable timeline. Verify the hash chain is intact by recomputing each hash.

5. **Build a full security middleware handler for a Clay enrichment call.** Combine all four pipeline stages: (1) scan the input payload for leaked secrets before it leaves your system, (2) apply the guardrail from Exercise 2, (3) execute the call using the rotated-key client from Exercise 3, (4) scan the Clay response for any secrets that leaked back, (5) write a structured audit log entry with the correlation ID threading all stages. Simulate 20 enrichment calls where two inputs contain leaked secrets and one response contains a key in a field labeled `notes`. Print a summary report showing blocked count, successful calls, and response-side detections. Save the middleware as a reusable, importable module.

   Artifact: `handlers/security_middleware.py`

6. **Design a rotation-strategy comparison for a GTM stack using Clay, Apollo, and your CRM.** Write a Python script that simulates three rotation strategies — instant-swap, drain-and-switch, and blue-green with health-check gating — against a workload of 200 concurrent API calls with a rotation triggered at midpoint. Measure three metrics per strategy: failed requests, max latency spike during rotation, and time-to-complete-rotation. Output a formatted comparison table to stdout. Then write a one-page analysis recommending which strategy to use for each tool (Clay, Apollo, CRM sync) with justification grounded in your measured results.

   Artifact: `outputs/skill-rotation-analysis.md`
