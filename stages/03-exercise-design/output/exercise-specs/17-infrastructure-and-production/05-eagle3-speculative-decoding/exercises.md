# Exercises — EAGLE-3 Speculative Decoding in Production

## Exercises

1. **Compute alpha and expected speedup from token sequences.** Write a Python script that takes two lists — draft tokens and accepted tokens — and computes the acceptance rate (alpha). Then, using the formula `expected_speedup = (1 + K * alpha) / (1 + K * alpha / (1 + alpha))` (or the simplified linear approximation `1 + K * alpha` for small alpha), compute and print the expected speedup for `K = 4` and `K = 8`. Run the script with a sample where 6 of 8 proposed tokens were accepted. Terminal should print alpha, speedup for each K, and the break-even alpha (the value at which speedup = 1.0).

2. **Compare the three speculative decoding generations.** Write a script that, given typical alpha ranges for each architecture (classic draft: 0.4–0.5, Medusa: 0.5–0.6, EAGLE-3: 0.7–0.85) and a fixed `K = 8`, prints a table showing architecture, alpha range, expected speedup range, and a one-line note on where the draft tokens come from (separate model, target hidden-state heads, or target feature-space). The table must print to the terminal in aligned columns. Identify which architecture achieves a speedup > 2.0 at the low end of its alpha range.

3. **Implement a speculative decoding accept/reject loop.** Build a simulation that generates `K` draft tokens from a mock draft distribution, compares each against a mock target distribution using the standard accept/reject rule (accept if draft token matches target sample OR resample from the renormalized difference `max(0, p_target − p_draft)`), and returns the number of accepted tokens plus the first rejected position. Run 1000 iterations with `K = 4` and print the average accepted tokens per iteration, the empirical alpha, and the estimated throughput in tokens/second assuming each forward pass (target verification) costs 50ms and each draft step costs 5ms per token.

4. **Sweep break-even alpha across concurrency levels.** Write a script that sweeps concurrency from 1 to 64 (powers of 2) and, for each concurrency level, computes the minimum alpha required for speculative decoding to be net-positive — accounting for the fact that at high concurrency the target model's batch is already large, so the marginal latency reduction from speculative decoding shrinks (model verification overhead as `verify_cost = 10ms * (1 + concurrency * 0.02)`). Print a table of concurrency, verify cost, break-even alpha, and whether EAGLE-3's typical alpha (0.75) clears the bar. Identify the concurrency level at which EAGLE-3 stops being worthwhile.

5. **Configure vLLM speculative decoding and build a measurement plan on CRM enrichment data.** Using a dataset of 200 company descriptions (pull from a sample CSV in `signals/examples/` or use the Apollo Companies API if you have a key), configure vLLM with `--speculative-model "[eagle-model]"` and `--num-speculative-tokens 4` to generate personalized outbound openers. Log per-prompt: generation time, token count, and acceptance rate (vLLM exposes this as `spec_token_acceptance_rate` in its metrics endpoint at `/metrics`). Save the raw results to `outputs/spec-dec-enrichment-results.json` and write a measurement plan — including rollout thresholds (e.g., kill the feature flag if rolling alpha drops below 0.55 over 100 requests), monitoring cadence, and rollback steps — to `outputs/skill-spec-dec-health.md`.

6. **Implement a production governor that auto-toggles speculative decoding.** Build a handler (`handlers/spec_dec_governor.py`) that consumes a rolling window of acceptance-rate and latency metrics (simulate the stream or read from a log file), computes a 60-second rolling alpha and p99 latency, and emits a control decision: `ENABLE`, `DISABLE`, or `THROTTLE` (reduce K). The governor must apply hysteresis (require N consecutive below-threshold readings before disabling) and log every state transition with timestamp, rolling alpha, p99 latency, and decision to `outputs/governor.log`. Run it against a provided metrics trace and print the final state and total transitions to the terminal.
