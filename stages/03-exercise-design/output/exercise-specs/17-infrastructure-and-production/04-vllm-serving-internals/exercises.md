# Exercises — vLLM Serving Internals: PagedAttention, Continuous Batching, Chunked Prefill

## Exercises

1. **Build** a block-table memory allocator simulator. Simulate 50 sequences whose token counts are drawn from a Poisson distribution with λ=512. Allocate each sequence under two strategies: (a) contiguous worst-case buffers sized to `max_seq_len=4096`, and (b) paged blocks of 16 tokens each, grown on demand. Print a comparison table showing total memory allocated, memory wasted to internal fragmentation, and fragmentation percentage for both strategies. Confirm the paged approach stays under 4% while contiguous exceeds 60%.

2. **Trace** a continuous batching scheduler across 10 iterations. Seed a request queue with 8 requests arriving at staggered iterations (prefill lengths: 128–1024 tokens; decode budgets: 20–200 tokens). Assume a GPU capacity of 8192 KV-cache tokens. Write a script that prints, for each iteration: which request IDs are in prefill, which are decoding, which slots freed, whether any new request entered the batch, and whether preemption was triggered. End the trace when the queue drains.

3. **Implement** a toy continuous batching scheduler in Python that interleaves prefill and decode steps the way vLLM's `Scheduler` does. Maintain a `waiting` queue, a `running` set, and a fixed KV-cache token budget. On each iteration: finish any decode whose token count hits its max, move decoded sequences forward, admit prefills from the waiting queue when budget allows, and evict via LRU when full. Feed the scheduler a synthetic workload of 30 requests and print per-request start time, end time, and wall-clock throughput (tokens/sec).

4. **Configure** chunked prefill at chunk sizes of 64, 128, 512, and 2048 tokens. For each setting, simulate or run (if vLLM is installed) a workload of 200 requests with prefill lengths sampled from a long-tailed distribution (median 256, p99 4096). Measure and print TTFT at p50, p90, and p99, plus aggregate throughput. Identify which chunk size gives the best p99 TTFT and which gives the best throughput, and explain whether they differ.

5. **Compare** swap-to-CPU versus recompute preemption under three memory-pressure profiles: low (budget exceeds demand by 20%), medium (budget meets demand), and high (budget covers 60% of peak demand). For each profile and each strategy, simulate 500 scheduler iterations with bursty arrivals and print: number of preemption events, average added latency per preempted request, and total throughput. Write your findings, including which strategy wins under each profile and why, to `outputs/skill-preemption-strategies.md`.
