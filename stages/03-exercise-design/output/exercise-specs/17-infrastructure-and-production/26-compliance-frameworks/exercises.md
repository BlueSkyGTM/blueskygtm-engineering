## Exercises

1. **Implement a data classification tagger for Clay prospect records.** Write a Python script that accepts a dictionary representing an enriched prospect (fields such as `email`, `phone`, `company_name`, `title`, `linkedin_url`, `ai_email_subject`, `ai_email_body`, `technographics`, `personal_notes`). Classify each field value into one of these categories: `public`, `pii`, `sensitive`, `special_category`, `ai_generated`. Print the original record and the tagged version side by side so the terminal shows every field and its classification.

2. **Compute framework applicability from processing activities.** Given a JSON file containing five GTM processing activities (e.g., enriching EU-based prospects via Apollo, storing support tickets containing health complaints, processing customer credit cards for renewals, deploying an AI lead-scoring model on EU prospects, sharing prospect data with a US-based B2B data vendor), write a script that outputs which of the six frameworks (SOC 2, HIPAA, GDPR, PCI-DSS, EU AI Act, ISO 42001) apply to each activity and why (data type, data subject geography, processing purpose). Print a table to the terminal.

3. **Build an audit logging handler for CRM record mutations.** Implement a Python class or decorator that wraps prospect-record operations (`create`, `update`, `delete`) and emits structured log entries containing: timestamp (ISO 8601 UTC), actor identity, action, resource ID, before-state hash, after-state hash, and source IP. Call each operation on sample prospect data, write the log lines to `logs/audit.jsonl`, and print the contents to the terminal. These entries should be usable as SOC 2 Type II evidence — verify each line contains all six fields.

4. **Configure a GDPR right-to-erasure pipeline across a simulated GTM stack.** Write a function that accepts a prospect ID and returns a deletion manifest covering every store where that prospect's data resides: CRM contact record, enrichment cache, AI-generated email drafts, outbound email log, activity timeline, and webhook delivery log. For each location, output the store name, record count, deletion method (hard delete, pseudonymize, or retain-with-legal-basis), and the legal basis for retention if any. Print the full manifest to the terminal.

5. **Build a compliance-as-code evidence generator.** Write a Python script that reads application logs and infrastructure configuration (use sample JSON/YAML files you create representing access logs, encryption settings, retention policies, and vendor inventory), evaluates them against a control matrix covering SOC 2, GDPR, and ISO 42001 requirements, and writes an auditor-ready evidence report to `outputs/skill-compliance-evidence.md`. The report must include: control ID, status (pass/fail/warning), evidence source, and remediation note for failures. Print a summary of pass/fail counts to the terminal.

6. **Design and implement an EU AI Act risk-tier classifier for GTM AI features.** Build a Python module that accepts a list of AI-powered GTM feature descriptions (e.g., AI-generated cold email personalization, predictive lead scoring, automated objection-handling chatbot, sentiment analysis on recorded sales calls, AI-driven pricing discrimination by region) and classifies each into EU AI Act risk tiers: `unacceptable`, `high`, `limited`, `minimal`. For each feature, output the risk tier, the classification rationale, required conformity-assessment steps, and applicable ISO 42001 controls. Write the classifier to `handlers/ai_risk_classifier.py` and run it against your feature list, printing the full classification table to the terminal.
